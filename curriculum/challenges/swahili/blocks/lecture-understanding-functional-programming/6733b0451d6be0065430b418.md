# --interactive--

Funkcionalno programiranje se tiče pisanja koda koji je čist, predvidljiv i lakši za testiranje i debagovanje. Dva ključna koncepta koje ćemo danas istražiti su čiste funkcije i izbegavanje nuspojava.

Počinjemo sa čistim funkcijama.

Čista funkcija je funkcija koja, dato isti ulaz, uvek će vratiti isti izlaz, i ne modifikuje ništa van sebe.

Evo primera čiste funkcije:

```js
function add(a, b) {
  return a + b;
}
```

Ova funkcija je čista jer uvek vraća isti rezultat za iste ulaze i ne menja ništa izvan sebe. Bez obzira koliko puta pozovete `add(2, 3)`, uvek će vratiti `5` i neće uticati na nijedan drugi deo vašeg programa.

Sada, hajde da uporedimo ovo sa nečistom funkcijom:

```js
let total = 0;
function addToTotal(value) {
  total += value;
  return total;
}
```

Ova funkcija je nečista jer modifikuje varijablu izvan sebe (`total`). Rezultat od `addToTotal(5)` će biti drugačiji zavisno od toga šta je bio `total` pre nego što je funkcija pozvana. Ova nepredvidivost može učiniti naš kod teškim za razumevanje i debagovanje.

Ovo nas dovodi do našeg sledećeg važnog koncepta, a to su nuspojave.

Sporedni efekat je bilo kakva promena koja nastaje u stanju programa kada se pozove funkcija. Ovo može uključivati modifikovanje globalne varijable, pisanje u fajl ili izvršavanje API poziva.

Čiste funkcije nemaju nus-efekte, ali nečiste funkcije imaju ove.

Izbegavanje nuspojava je ključni princip funkcionalnog programiranja. Kada minimiziramo nuspojave, naš kod postaje predvidljiviji i lakši za testiranje.

Dok pišemo kod, naš glavni cilj bi trebalo da bude da funkcije rade samo jednu stvar i da je to rade veoma dobro bez menjanja drugih delova našeg programa.

Evo primera funkcije sa sporednim efektom:

TEKST:

U funkcionalnom programiranju, možda bismo ovo prepisali kao:

:::interactive_editor

```js
function greet(name) {
  return `Hello, ${name}!`;
}
console.log(greet("Alice"));
```

:::

Sada `greet` je čista funkcija koja jednostavno vraća niz znakova, i mi ruku sa bočnom efektom (logovanje u konzolu) odvojeno.

Funkcionalno programiranje nas podstiče da sastavljamo naše programe korišćenjem čistih funkcija tako što razbijemo naš kod na male, predvidive delove. I time, možemo izgraditi programe koji su lakši za razumevanje, testiranje i održavanje.
