# --interactive--

Kuriranje je tehnika gde transformišemo funkciju koja prima više argumenata u sekvencu funkcija, od kojih svaka prima samo jedan argument.

Počnimo sa osnovnim primerom. Zamislite da imamo funkciju koja sabira dva broja:

:::interactive_editor

```js
function add(a, b) {
  return a + b;
}

console.log(add(3, 4)); // 7
```

:::

Ovo je funkcija koja prima dva argumenta i vraća njihov zbir. Sada, hajde da vidimo kako možemo "curry" ovu funkciju:

:::interactive_editor

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

console.log(curriedAdd(3)(4)); // 7
```

:::

U ovom kodu konvertovanom u kuriju, umesto da uzme dva argumenta odjednom, imamo funkciju koja uzima prvi argument i vraća drugu funkciju. Ova vraćena funkcija zatim uzima drugi argument i izvršava sabiranje. Pozivamo ga kao `curriedAdd(3)(4)`, gde svaki par zagrada predstavlja poziv funkcije.

Ali zašto bismo hteli da ovo uradimo?

Kuriranje nam omogućava da lako kreiramo neke specijalne funkcije. Na primer, mogli bismo kreirati funkciju koja uvek dodaje pet bilo kom broju:

:::interactive_editor

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

const addFive = curriedAdd(5);
console.log(addFive(10)); // 15
console.log(addFive(20)); // 25
```

:::

Ovde, `addFive` je funkcija koja je uvek spremna da doda pet bilo kom broju koji joj damo. Ovo je jednostavan primer delimične primene (partial application), gde fiksiramo određeni broj argumenata funkciji, proizvodeći drugu funkciju koja prima manje argumenata.

Iako su se naši primeri fokusirali na funkcije sa dva argumenta, kuriranje može biti primenjeno na funkcije sa bilo kojim brojem argumenata.

Dok nastavite svoj put u JavaScriptu, verovatno ćete naići na mnogo situacija gde currying može učiniti vaš kod čistijim i izražajnijim.

## --text--

Koja od sledećih je primer kurirane funkcije u JavaScript-u?
## --answers--

```js
function dodaj(a, b, c) {
  return a + b + c;
}
```
### --feedback--

Ovo je regularna funkcija koja uzima sve parametre odjednom.
---

```js
function add(a) {
  return function (b) {
    return function (c) {
      return a + b + c;
    };
  };
}
```

---

```js
function add(a, b, c) {
  return a() + b() + c();
}
```

### --feedback--

Ova funkcija prima sve parametre odjednom i poziva ih kao funkcije, što ne prati definiciju kurirane funkcije.
---

```js
function add(a) {
  function inner(b) {
    return a + b;
  }
}
```

### --feedback--

Unutrašnja funkcija se nikada ne vraća, pa će ``add(2)`` vratiti ``undefined``, a ``add(2)(3)`` će baciti grešku.
