# --description--

Pogledajmo neke od fundamentalnih komandi koje možete koristiti za unos i prikaz podataka u SQL-u.

Prvo svega, da biste videli podatke, morate ih uneti. Zato ćemo početi sa unosima.

Primeri koje ćemo pokriti u ovoj lekciji zasnovani su na ovoj tabeli zapisa o psima:

```sql
CREATE TABLE dogs(
  id SERIAL,
  name VARCHAR(100),
  age INTEGER
);
```

Tabela će čuvati imena i godine raznih pasa.

Ako pretpostavimo da radimo sa PostgreSQL-om, možemo koristiti tip podataka `SERIAL` za `id`. Pošto je `id` kreiran pomoću `SERIAL`-a, njegova vrednost će biti `INTEGER`, počevši od `1`, i automatski se povećavati kada se unese novi zapis. Zbog toga nećete morati da prosleđujete vrednost za njega kada unesete zapis.

Počnimo sa unosom zapisa u našu tabelu `dogs` pomoću naredbe `INSERT INTO`. Postoji mnogo načina za to. Prvi bi bio unos jednog reda i navođenje kolona vrednosti koje će biti unete.

Pišete `INSERT INTO`, praćen imenom tabele (u ovom slučaju, `dogs`), zatim kolone u zagradi, pa `VALUES`, i njihove odgovarajuće vrednosti u zagradi.

Ovo je primer: unosimo zapis o psu sa `name`-om `'Gino'`, čija je `age` `3`. Zapamtite da će `id` biti automatski dodeljen:

```sql
INSERT INTO dogs (name, age)
VALUES ('Gino', 3);
```

Ovo je obično najbezbedniji izbor jer eksplicitno navodite kolone, tako da će vrednosti biti dodeljene tim kolonama po redu.

Primećujte da `'Gino'` mora biti u jednostru trimu jer je tip `VARCHAR`.

Druga alternativa je unos zapisa bez navođenja kolona, kao što je ovo:

```sql
INSERT INTO dogs
VALUES ('Gino', 3);
```

Ova sintaksa je validan SQL, ali je sklona greškama.

Pošto se vrednosti dodeljuju na osnovu reda kolona, PostgreSQL će pokušati da unese `'Gino'` u kolonu `id`, koja očekuje ceo broj. To rezultira greškom kao što je `invalid input syntax for type integer`.

Zbog ovog razloga, sigurnije je eksplicitno navesti imena kolona prilikom unosa podataka.

Takođe možete uneti više zapisa istom SQL komandom tako što ćete ih razdvojiti zarezom.

Ovde unosimo dva zapisa o psima, jedan za `'Gino'` i drugi za `'Nora'`.

```sql
INSERT INTO dogs (name, age)
VALUES
  ('Gino', 3),
  ('Nora', 2);
```

I baš kao što možete da unosite zapise, možete upitovati informacije iz baze podataka. U kontekstu baza podataka, upit (query) je zahtev za podatke.

U SQL-u, naredba `SELECT` se koristi za upitovanje podataka iz jedne ili više tabela. Možete prilagoditi upit da biste dobili tačne informacije koje su vam potrebne.

Ako želite da upitate sve informacije iz tabele `dogs`, samo treba da napišete `SELECT`, praćen zvezdikom (`*`), zatim `FROM`, i naziv tabele, `dogs`:

```sql
SELECT *
FROM dogs;
```

U ovoj komandi, zvezdica je karakter divljokartira koji predstavlja "sve kolone".

Dobićete sve podatke svih zapisa u tabeli:

```sql
 id | name | age 
----+------+-----
  1 | Gino |   3
  2 | Nora |   2
```

Da upitate specifične kolone, možete napisati imena tih kolona u komandu, odmah nakon `SELECT`, razdvojena zarezom. U ovom primeru, upitujemo kolone `name` i `age` iz tabele `dogs`:

```sql
SELECT name, age
FROM dogs;
```

Rezultat će uključivati samo kolone `name` i `age`:

```sql
 name | age 
------+-----
 Gino |   3
 Nora |   2
```

Ponekad možda ćete morati da upitate podatke na osnovu specifičnog uslova. Na primer, da biste dobili sve pse koji su mlađi od 3 godine, možete koristiti ključnu reč `WHERE` i operator poređenja manji od (`<`):

```sql
SELECT *
FROM dogs
WHERE age < 3;
```

Rezultat će uključivati samo pse čija je starost manja od 3:

```sql
 id | name | age 
----+------+-----
  2 | Nora |   2
```

Ako pokušavate samo da pronađete `age` za `'Gino'`, možete koristiti operator poređenja jednakost (`=`):

```sql
SELECT age
FROM dogs
WHERE name = 'Gino';
```

I evo rezultata:

```sql
 age 
-----
  3
```

Ovo su uobičajeni i jednostavni načini za unos i prikaz podataka u SQL-u, ali postoji mnogo različitih opcija koje možete izabrati, a neke od njih su naprednije. Naučićete o njima u narednim lekcijama.
