# --description--

U SQL-u, primarni ključevi i strani ključevi se koriste za uspostavljanje odnosa između tabela.

Počinjemo sa primarnim ključevima.

Primarni ključ je kolona ili skup kolona koji jedinstveno identifikuje svaki zapis (red) u tabeli. Ovaj ograničavač osigurava da nijedan zapis u tabeli neće imati istu vrednost za primarni ključ.

Ovo je razlog zašto tabela može imati samo jedan primarni ključ.

Vrednosti u koloni primarnog ključa ne mogu biti ni ``NULL``, pa će uvek imati važeću vrednost.

U PostgreSQL-u, da učinite kolonu primarnim ključem vaše tabele, samo morate dodati `PRIMARY KEY` nakon tipa podataka:

```sql
column_name data_type PRIMARY KEY
```

Ovo je primer `students` tabele:

```sql
CREATE TABLE students (
  student_id SERIAL PRIMARY KEY,
  name VARCHAR(100)
);
```

Svaki student će imati svoj `student_id`, i ovaj ID će biti primarni ključ tabele. Tip `SERIAL` je koristan ovde jer osigurava da će `student_id` uvek imati jedinstvenu vrednost.

Kompozitni primarni ključ služi kada tabela nema jednu jedinstvenu kolonu za identifikaciju reda. U ovom slučaju, možete koristiti kombinaciju dve ili više kolona kao primarni ključ koji zajedno čine jedinstvenost.

Da biste to uradili, označite koje kolone predstavljaju složeni primarni ključ kada kreirate tabelu na ovaj način:

```sql
CREATE TABLE table_name (
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ...
  PRIMARY KEY (column1, column2)
);
```

Zamislite da imate tabelu sa `student_id` i `course_id` kolonama koje vam govori šta su studenti upisani na razne predmete.

Svaki student će biti upisan na nekoliko predmeta, a svaki predmet će imati više studenata upisano na njega. Dakle, nijedna kolona nije jedinstvena – ali nikada nećete imati istog studenta upisanog na isti predmet više od jednom. Zato možete koristiti kombinaciju dve kolone kao primarni ključ:

```sql
CREATE TABLE course_enrollments (
  student_id INT,
  course_id INT,
  ...
  PRIMARY KEY (student_id, course_id),
);
```

Ključni primarni ključevi su neophodni za osiguravanje jedinstvenosti i integriteta podataka, za optimizovanje preuzimanja podataka i za uspostavljanje odnosa sa drugim tabelama u relacionoj bazi podataka.

To nas vodi ka sledećem tipu ključa koji ćemo pokriti u ovoj lekciji: strani ključevi.

Strani ključ je kolona (ili skup kolona) u tabeli koja referencira primarni ključ druge tabele. Tabela može imati više stranih ključa.

U ovom primeru, imamo dve tabele, `customers`, i `orders`:

```sql
CREATE TABLE customers (
  customer_id SERIAL PRIMARY KEY,
  first_name VARCHAR(100) NOT NULL,
  ...
);

CREATE TABLE orders (
  order_id SERIAL PRIMARY KEY,
  customer_id INTEGER,
  ...
  FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

Primarni ključ tabele `customers` je `customer_id` i primarni ključ tabele `orders` je `order_id`.

Ali primetite da postoji i kolona `customer_id` u tabeli `orders`.

Zašto skladištimo informacije o klijentima u tabelu ``orders``?

Radimo ovo da bismo kreirali vezu između tabela `customers` i `orders`. Svaka porudžbina će čuvati ID klijenta koji ju je podneo.

Radi stvaranja veze između tabela, učinićemo kolonu `customer_id` u tabeli `orders` stranim ključem koji referencira kolonu `customer_id` u tabeli `customers`:

```sql
FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
```

Vrednosti u koloni stranog ključa moraju ili odgovarati vrednostima primarnog ključa u referenciranoj tabeli ili biti `NULL` (ako strani ključ dozvoljava `NULL`s).

Zašto je ovo korisno?

Osiguravanjem da se ove kolone u obe tabele tačno poklapaju, strani i primarni ključevi sprečavaju kreiranje "siroćene" zapise, zapisa koji se odnose na nepostojeći zapis u drugoj tabeli.

Primarni i strani ključevi su fundamentalni za relacione baze podataka. Oni pokreću „relacioni“ aspekt ovih baza podataka, omogućavajući vam da modelujete podatke iz stvarnog sveta na tačan način.
