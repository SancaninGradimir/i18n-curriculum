# --interactive--

Ako napišete izraz sa nekoliko operatora u JavaScriptu, kako JavaScript odlučuje koji će prvo evaluirati? Ovde ulazi u igru prioritet operatera. Istražićemo prioritet operatera u detalju sa primerima koda, kao i šta se dešava kada operateri imaju isti prioritet.

Prioritet operatora određuje redosled kojim se operacije procenjuju u izrazu. Operatori sa višim prioritetom procenjuju se pre onih sa nižim prioritetom. Zamislite operativni prioritet kao u matematici, gde deljenje i množenje se dešavaju pre sabiranja i oduzimanja.

Bez praćenja ovog pravila, osnovne jednačine bi vam dale pogrešan odgovor. JavaScript radi na isti način. Ima svoja pravila o tome koji operateri dolaze prvi, drugi i tako dalje. Na primer, pogledajte izraz ispod:

:::interactive_editor

```js
const result = 2 + 3 * 4;

console.log(result); // 14
```

:::

Ako bi JavaScript evaluirao ovaj izraz od levi ka desni, očekivali biste `2 + 3 = 5`, pa `5 * 4 = 20`. Ali zato što množenje ima viši prioritet od sabiranja, JavaScript evaluira deo `3 * 4` prvi, što rezultira sa `2 + 12 = 14`.

Ponekad želite da neki delovi vašeg izraza prođu prvi, bez obzira na pravila prioriteta. Možete koristiti zagrade, `()`, za ovo. Sve što je unutar zagrada procenjuje se prvo, bez obzira na šta. Hajde da učinimo da deo `2 + 3` iz prethodnog primera prođe prvi:

:::interactive_editor

```js
const result = (2 + 3) * 4;

console.log(result); // 20
```

:::

U gornjem primeru, zagrade prisiljavaju JavaScript da prvo evaluira `2 + 3`, a zatim rezultat pomnoži sa `4`. Ovo vam daje `20` umesto `14`.

Operater deljenja ima veći prioritet od sabiranja ili oduzimanja takođe:

:::interactive_editor

```js
const result = 2 + 6 / 3;

console.log(result); // 4
```

:::

Ako bi JavaScript evaluirao ovaj izraz od levi ka desni, mogli biste očekivati `2 + 6 = 8`, pa zatim `8 / 3 = 2.67`. Ali pošto deljenje ima viši prioritet od sabiranja, JavaScript prvo evaluira deljenje: `6 / 3 = 2`, a zatim dodaje `2 + 2`, dajući rezultat `4`.

Dakle, u množenju i deljenju, te operacije će se uvek izvršiti pre sabiranja i oduzimanja osim ako ne koristite zagrade da promenite redosled. Šta se dešava ako operatori imaju isti prioritet? JavaScript koristi asocijativnost da utvrdi redosled evaluacije tih operacija.

Asocijativnost je ono što govori JavaScriptu da li treba da proceni operatore od leve ka desnoj ili od desne ka levoj strani. Za većinu operatera kao što su sabiranje i množenje, asocijativnost je od leve ka desnoj strani. Stoga, JavaScript ih obrađuje od najlevije strane izraza ka desno:

:::interactive_editor

```js
const result = 10 - 2 + 3;

console.log(result); // 11
```

:::

Prvo, `10 - 2 = 8`, zatim `8 + 3 = 11`. JavaScript se u ovom slučaju kreće od leve ka desnoj strani. Neki operateri, kao što je dodela (`=`), su asocijativni od desna ka levo. To znači da se desna strana izraza prvo procenjuje:

:::interactive_editor

```js
let a, b;
a = b = 5;

console.log(a); // 5
console.log(b); // 5
console.log(a + b); // 10
```

:::

U gornjem primeru, JavaScript prvo dodeljuje `5` na `b`, a zatim dodeljuje `b` (koji je sada `5`) na `a`.

Operator eksponenta je takođe asocijativan od desna ka levo:

:::interactive_editor

```js
const result = 2 ** 3 ** 2;

console.log(result); // 512
```

:::

Prvo, JavaScript evaluira `3 ** 2`, što je jednako `9`, zatim, on evaluira `2 ** 9`, što je jednako `512`. Da operater eksponenta ima asocijativnost od levi ka desni, JavaScript bi izračunao `2 ** 3` prvo da dobije `8`, zatim `8 ** 2` da dobije `64`.
