# --interactive--

Asertacije za predgled i asertacije za pregled pozadi omogućavaju vam da poklapate specifične obrasce na osnovu prisustva ili nedostatka okolnih obrazaca. Postoje četiri varijacije ovih asertacija.

Prvo je pozitivna provera predskoka (positive lookahead assertion). Ova tvrdnja će poklopiti obrazac kada ga prati drugi obrazac.

Da biste konstruisali pozitivan predgled, morate početi sa obrascem koji želite da pokupite. Zatim, koristite zagrade da obuhvatite obrazac koji želite da koristite kao uslov. Nakon otvarajuće zagrade, koristite ``?=`` da definišete taj obrazac kao pozitivan predgled.

Ovaj obrazac će odgovarati reči `free` ako je praćen sa `code`:

```js
const regex = /free(?=code)/i;
```

Neka testiramo ponašanje našeg obrasca:

:::interactive_editor

```js
const regex = /free(?=code)/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

Primećujte kako samo niz gde je `free` odmah praćen sa `code` prolazi test.

Ali šta ako želite da poklapate prisustvo `free` kada ga NE prati `code`? Možete pretvoriti svoj pozitivan predgled u negativan predgled kako biste inverzovali ponašanje. Da biste to uradili, promenite vaš `?=` u `?!`:

```js
const regex = /free(?!code)/i;
```

Neka ovo testiramo na našim istim nizovima:

:::interactive_editor

```js
const regex = /free(?!code)/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

Kako je i očekivano, rezultati su obrnuti. Jedina žica koja ne uspeva je prva žica, gde `free` je odmah praćen sa `code`.

Asertacije za pregled u prošlost funkcionišu slično asertacijama za pregled u budućnost, osim što ne uslovno poklapaju na osnovu sledećeg obrasca, već uslovno na osnovu prethodnog obrasca. Pogledajmo pozitivan pregled u prošlost.

Pozitivan pogled unazad je označen sa `?<=` umesto `?=`. Neka naš regularni izraz poklopi se sa `code` kada ga prethodi `free`:

```js
const regex = /(?<=free)code/i;
```

Kao i sa našim pozitivnim predgledom, naš pozitivan pregled iza odgovara prvom nizu jer `code` je odmah prethodeн od strane `free`:

:::interactive_editor

```js
const regex = /(?<=free)code/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

Da bismo poklopili `code` kada ga NE prethodi `free`, možemo koristiti negativan pogled unazad. Negativni pogled unazad je definisan zamjenom `?<=` sa `?<!`:

```js
const regex = /(?<!free)code/i;
```

TEKST:

Ovo bi odgovaralo bilo kom pojavljivanju `code` koje NIJE neposredno prethodno od `free`:

:::interactive_editor

```js
const regex = /(?<!free)code/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

Zapamtite da `Regex.prototype.test` samo potvrđuje da li niz odgovara regularnom izrazu. Koristićemo naš negativni pogled iza sa `String.prototype.match` kako bismo videli kako tvrdnje utiču na to:

```js
const regex = /(?<!free)code/i;
console.log("freeCodeCamp".match(regex)); // null
console.log("free code camp".match(regex)); // ['code', index: 5, input: 'free code camp', groups: undefined]
console.log(
  "I need someone for free to write code for me".match(regex)
); // ['code', index: 33, input: 'I need someone for free to write code for me', groups: undefined]
```

Primećujete kako, iako naš regex koristi *lookbehind* da proveri prisustvo `free`, onaj ne odgovara `free`. Jedini tekst uključen u podudarnost je `code`.

Pogledi napred i pogledi unazad su neverovatno korisni za uslovno uklapanje teksta bez uticaja na vraćenu vrednost vašeg poklapanja.
