# --interactive--

Modifikatori, često nazvani "flagovi", modifikuju ponašanje regularnog izraza. Podsetimo se našeg primera iz ranije lekcije:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // false
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Ako se setite, nizovi sa malim i velikim slovima `freeCodeCamp` nisu uspeli da poklope obrazac. Ovo je zato što su redovne izraze po podrazumevanju osetljivi na veličinu slova.

Ali šta ako bismo mogli rečiti regularnom izrazu da bude neosetljiv na veličinu slova? Pa, postoji modifikator za to. Flag `i` čini da regex ignoriše veličinu slova. Kako ga možemo koristiti? Flagi idu nakon zatvarajućeg naslovnog kosog crte u regularnom izrazu:

```js
const regex = /freeCodeCamp/i;
```

Primećujte promenu u regularnom izrazu na prvoj liniji. Sada možemo proveriti kako ovo menja stvari:

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Budući da je naš regularni izraz sada neosetljiv na veličinu slova, stringovi koji su potpuno u malim i potpuno u velikim slovima "prošli" su test. Ovo takođe može da radi za string sa nasumičnom mešavinom velikih i malih slova:

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("dO yOu LoVe fReEcOdEcAmP?")); // true
```

:::

Postoji prilično mnogo drugih zastavica koje možete koristiti. Zastava ``g``, ili globalni modifikator, omogućava vašem regulacionom izrazu da pronađe obrazac više od jednom.

Pogledaćemo kako to utiče na naš kod. Primeći ćete da smo zadržali ``i`` zastavicu – regularni izraz može koristiti više zastavica (koliko god je potrebno) kako bi postigao željeno ponašanje:

```js
const regex = /freeCodeCamp/gi;
```

Sačekajte trenutak... šta je ovo? Čini se da globalni modifikator uzrokuje da neke od naših nizova koji bi trebalo da prolaze zaista ne prolaze:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Zašto? Pa, globalni modifikator čini vaš regex izraz stanjivim. To znači da prati gde je prethodno pronašao obrazac. Dakle, kada pronađe prvi `freeCodeCamp` niz, pamti da je pronašao podudarnost koja počinje na indeksu `0`.

Zatim ga testiramo protiv `freeCodeCamp is great`, ali ne počinje na indeksu `0`. Regularni izraz „zna“ da je pronašao podudarnost na indeksu `0` već, pa čak i ako je ovo drugačiji niz znakova, počinje od krajnjeg indeksa podudarnosti.

`freeCodeCamp` je `12` karaktera dugačak, pa podudaranje na `0` se završava na indeksu `11`. Podudaranje će se nastaviti na indeksu `12`. I pošto `is great` ne odgovara `freeCodeCamp`, vraća `false`.

Zatim, jer ne uspe da pronađe podudarnost, "gubi" svoj status i započinje sledeću podudarnost nazad od `0`.

Ako preuredimo naše logove tako da je niz sa podudarnošću na `0` odmah praćen nizom koji ima podudarnost koja je kasnije od indeksa `11`:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("I loooooooove freeCodeCamp")); // true
```

:::

Kada je redovna izraz globalan, dobija novu svojstvo nazvano `lastIndex`. Uzimajući naš prethodni kod, hajde da vidimo kako ovo svojstvo radi:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.lastIndex); // 0
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.lastIndex); // 19
console.log(regex.test("freecodecamp")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("free")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("code")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("camp")); // false
```

:::

Na ovom primeru možete videti kako se stanje regularnog izraza menja sa svakim pozivom testa, koristeći `lastIndex` za praćenje njegovih prethodnih podudaranja.

Global zastavica je odlična kada vam treba da dobijete više podudaranja iz jednog stringa. Ali ako testirate više stringova sa istim regularnim izrazom, najbolje je da ostavite zastavicu `g` isključenu.

Pre nego što naučite o sledećem flag-u, morate naučiti o ankarama. Ankra krezik (caret) (``^``), na početku regularnog izraza, kaže "poklapa se sa početkom niza":

```js
const start = /^freecodecamp/i;
```

Znak dolara (`Znak dolara () jangkor, na kraju regularnog izraza, znači "poklapanje kraja niza":

```js
const end = /freecodecamp$/i;
```

Odvojite trenutak da uporedite rezultate sa desne strane:

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
console.log(start.test("freecodecamp is great")); // true
console.log(end.test("freecodecamp is great")); // false
console.log(start.test("i love freecodecamp")); // false
console.log(end.test("i love freecodecamp")); // true
console.log(start.test("have met freecodecamp's founder")); // false
console.log(end.test("have met freecodecamp's founder")); // false
```

:::

Pogledajte kako početni ankar odgovara samo na početku niza, a krajnji ankar odgovara samo na kraju niza? Ali šta je sa podudarnim unosima kroz više linija? Pogledajmo to:

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // false
console.log(end.test(string)); // false
```

:::

Iako je ``freecodecamp`` tamo na sopstvenoj liniji, ne prolazi oba testa. Ovo je zato što podrazumevano ankeri traže početak i kraj celog niza karaktera.

Ali možete naterati regex da obrađuje više linija sa `m` zastavicom, ili modifikatorom za više linija. Dodajmo to našim regularnim izrazima da vidimo šta dobijemo:

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // true
console.log(end.test(string)); // true
```

:::

Sada oba odgovaraju! Zato što je ``freecodecamp`` potpuno na sopstvenoj liniji, početni anker odgovara početku te linije, a krajnji anker odgovara kraju te linije.

Konačno, imate `d` flag, ili modifikator indeksa. Zapamtite da je `i` flag za neosetljivost na veličinu slova, pa je modifikator indeksa zahtevao drugi flag.

Flaga `d` proširuje informacije koje dobijate u objektu podudaranja. Dodajmo je našem regularnom izrazu:

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

I rezultat je:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

Naš objekat za podudarnost dobija novu svojstvo `indices`! Ovo svojstvo je niz od dve brojeve, prvi predstavlja indeks u originalnom nizu gde podudarnost počinje, a drugi predstavlja indeks nakon što je podudarnost završila. Ovaj niz takođe ima dodatno svojstvo `groups`, koje služi i za nazvane grupe snimanja.

Postoje još nekoliko zastavica koje bi trebalo da znate da su vam dostupne, ali su manje uobičajene u tipičnom kodu.

Prvi je unicode modifikator, ili zastavica ``u``. Ovo proširuje funkcionalnost regularnog izraza kako bi mu omogućilo da odgovara specijalnim unicode karakterima.

Naučićete više o klasama karaktera u budućoj lekciji, ali vam `u` zastavica daje pristup posebnim klasama kao što su `Extended_Pictographic` za podudaranje sa većinom emodžija:

:::interactive_editor

```js
const regex = /🍎/u;

const str = "I have an apple 🍎";
console.log(regex.test(str)); // true
```

:::

Postoji takođe zastavica ``v``, koja dodatno proširuje funkcionalnost podudarnosti Unicode-a.

Drugi je lepljivi modifikator, ili zastavica `y`. Lepljivi modifikator se veoma slično ponaša globalnom modifikatoru, ali sa nekoliko izuzetaka.

Najveće je da globalni regularni izraz će početi od `lastIndex` i pretraživati ostatak celog niza za još jedno poklapanje, ali lepljivi regularni izraz će vratiti `null` i resetovati `lastIndex` na `0` ako nema odmah poklapanja na prethodnom `lastIndex`.

I poslednji je modifikator za jednu liniju, ili zastavica `s`. Zapamtite da modifikator za više linija omogućava da se sidra početka i kraja poklope sa početkom i krajem linije, umesto celog niza karaktera.

Modifikator za jednu liniju omogućava karaktera wildkarda, predstavljenog tačkom (`.`) u regex-u, da odgovara prekidima linija – efektivno tretirajući niz kao jednu liniju teksta.

Postoji prilično mnogo ovih modifikatora, ali su zastavice `i` i `g` one koje ćete najčešće koristiti, i najvažnije za pamćenje.
