# --interactive--

Redovni izrazi, ili regex, su funkcija koju podržavaju mnogi različiti programski jezici.

Regularni izraz je posebna sintaksa za kreiranje "šablona", koji zatim možete koristiti za proveru naspram niza, ekstrakciju teksta i više.

Pogledajmo osnovni regularni izraz:

```js
const regex = /freeCodeCamp/;
```

Primećujete kako u JavaScriptu definišete regularni izraz kreiranjem vašeg obrasca između dve kose crte (`/`). Pokušajte da ne pomešate ovo sa komentarom, gde tekst dolazi nakon obe kose crte.

Ovaj određeni regularni izraz će poklopiti tekst `freeCodeCamp`, sa velikim slovima `C`s, bilo gde u nizu karaktera. Ali kako to zapravo učiniti?

To nas dovodi do naše prve metode – metode ``test()``. Metoda ``test()`` postoji na objektima ``RegExp``, koji su objekti koji predstavljaju regularni izraz (kao što je onaj koji smo upravo definisali).

Metoda `test()` prima string, koji je string za testiranje podudaranja sa regularnim izrazom. Na primer, hajde da pokušamo da testiramo string `e`:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test);
```

:::

Možete videti da smo pozvali metodu `test()` na našem novom regexu, i prosledili niz `e` kao argument.

Metoda `test()` je vratila `false` jer string `e` ne odgovara obrascu `freeCodeCamp`. Iako obrazac `freeCodeCamp` uključuje slovo `e`, to je suprotan pravcu od načina na koji redovni izrazi rade.

Pogledajmo još nekoliko primera. Odvojite trenutak za razmišljanje o ovim:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp"));
console.log(regex.test("freeCodeCamp is great"));
console.log(regex.test("I love freeCodeCamp"));
console.log(regex.test("freecodecamp"));
console.log(regex.test("FREECODECAMP"));
console.log(regex.test("free"));
console.log(regex.test("code"));
console.log(regex.test("camp"));
```

:::

Primećujete kako prve tri niza vraćaju `true`. Svi ovi nizovi sadrže tekst, `freeCodeCamp`, tačno, negde u nizu.

Linije 5 i 6 vraćaju `false`. Iako sadrže tekst `freecodecamp`, slučaj ne odgovara. Regularni izrazi su podrazumevano osetljivi na veličinu slova.

Konačno, iako poslednja tri sadrže deo obrasca, stringovi ne sadrže ceo obrazac.

Метода `test()` vraća `boolean`, što ukazuje da li string odgovara regularnom izrazu uopšte.

Ali šta ako želite više informacija od toga? Pa, nizovi imaju metod `match()`. Ovaj metod prima regularni izraz, iako možete proslediti i niz koji će biti konstruisan u regularni izraz.

`match()` vraća niz podudarnosti za string. Šta je niz podudarnosti? Pa, hajde da pogledamo:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

Ako ovo pokrenemo, vraća nam niz! Ali to je niza čudnog izgleda. Ima neke dodatne svojstva:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

Svojstvo `groups` bi pokazivalo bilo koje uhvaćene grupe. Naučićete šta to znači u budućoj lekciji.

Svojstvo ``index`` vam govori na kom karakteru u nizu je pronađen podudarnost. U našem slučaju, pronađeno je na početku niza.

Svojstvo `input` ukazuje vam string na kojem je metoda `match()` pozvana.

Hajde da probamo još nekoliko puta i vidimo kako se rezultat menja:

```js
const regex = /freeCodeCamp/;
console.log("freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
console.log("freeCodeCamp is great".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
console.log("I love freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
console.log("freecodecamp".match(regex)); // null
console.log("FREECODECAMP".match(regex)); // null
console.log("free".match(regex)); // null
```

Već znamo da prve tri niza bi trebalo da proizvedu podudarnost, pa hajde da pogledamo te:

```js
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
// ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
```

Je li ovo ono što ste očekivali? Možete videti kako su se `input` i `index` promenili zavisno od priložene žice, i lokacije podudaranja u žici.

Ostale tri linije, koje ne odgovaraju, vraćaju `null` umesto niza.

Sada kada možemo da testiramo i uparimo nizove sa našim regularnim izrazom, šta ako želimo da zamenimo sadržaj niza? Možda je neko napisao ``freecodecamp`` u malim slovima, i želimo da im automatski popravimo veličinu slova.

Prvo, moramo ažurirati naš regex da odgovara malonakonskom obliku `freecodecamp` i kreirati naš testni string:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
```

Sada, stringovi imaju metod `replace()` koji prima dva argumenta: izraz regulisanje za podudarnost (ili string, ako vam nisu potrebne sve funkcije regex-a), i string kojim ćete zameniti podudarnost (ili funkciju koja se izvršava za svako podudarnost).

Dakle, ako bismo želeli da zamenimo naš `freecodecamp` ispravnim kapitalizacijom:

:::interactive_editor

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced);
```

:::

Možete videti da `replace()` vraća ažurirani niz sa odgovarajućim obrascem `replaced`.

Regularni izrazi, i sve metode povezane s njima, mogu delovati složeno i opterećujuće. Ali dobićete priliku da ih dalje istražite u ovom sledećem setu lekcija.
