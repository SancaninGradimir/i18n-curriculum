# --interactive--

Grupa za hvatanje omogućava vam da "uhvatite" deo poklopljenog niza kako biste ga koristili kako god je potrebno. Grupe za hvatanje su definisane zagradama koje sadrže obrazac koji treba uhvatiti, bez vodećih karaktera kao što je *lookahead*.

Zapaciraćemo `code` iz našeg `freeCodeCamp` regularnog izraza. Za to, obuhvatićemo `code` u zagrade i definisati ga kao grupu za hvatanje:

```js
const regex = /free(code)camp/i;
```

Da bismo potvrdili ponašanje, možemo ga testirati na stringu `freecodecamp`:

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log(regex.test("freecodecamp")); // true
```

:::

Ali ovo zapravo ne koristi našu uhvaćenu grupu. Umesto toga, hajde da pogledamo rezultat korišćenja `match`:

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".match(regex));
// [
//   'freecodecamp',
//   'code', <--
//   index: 0,
//   input: 'freecodecamp',
//   groups: undefined
// ]
```

Ovde možemo videti da naš ``match`` niz ima drugi element, koji je deo niza koji je uhvatio naša grupa za hvatanje.

Primećujete kako grupa za hvatanje odgovara tačnom obrascu `code`, gde bi klasa karakaktera odgovarala jednom karakteru iz liste `c`, `o`, `d`, i `e`.

Ali kako ovo zapravo možemo koristiti? Pa, grupe za hvatanje se često koriste prilikom zamene sadržaja niza. Neka postavimo neki kod za to. Preći ćemo `freecodecamp` u `paidcodeworld`:

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".replace(regex, "paidcodeworld"));
```

:::

Ovo radi samostalno, ali šta ako ne znamo koliko `o`' je u `code`? Ako nam je potreban kvantifikator za jedno ili više `o`s:

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paidcodeworld"));
```

:::

Dobijamo `paidcodeworld` kao naš rezultat. Želimo da sačuvamo broj `o`', pa moramo ponovo koristiti ono što je uhvaćeno regularnim izrazom.

Ovde nam je potrebna povratna referenca. Umesto da hardkodiramo deo našeg zamenjivačkog niza '`code`', možemo direktno referencirati uhapšenu grupu.

U pozivu `replace`, postižete referencu nazad korišćenjem znaka dolara (`U pozivu `replace`, postižete referencu nazad korišćenjem znaka dolara () praćenog brojem grupe za hvatanje koju želite da koristite. U našem slučaju, to bi bilo `$1`, pošto je `code` uhvaćeno u prvoj grupi za hvatanje:

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world")); // paidcoooooooodeworld
```

:::

Sada smo uspešno sačuvali nepoznat broj `o` karaktera prilikom konverzije `freecodecamp` u `paidcodeworld`. Ali povratne reference nisu ograničene samo na poziv za zamenu. Zapravo ih možete koristiti direktno u regularnom izrazu.

To bi vam omogućilo da kasnije u regularnom izrazu pronađete prethodno uhvaćeni obrazac.

Recimo da želimo da poklopimo ``freecodecamp`` dva puta, sa istim brojem od ``o``, ali bilo gde u stringu.

Prvo, moramo ih razdvojiti našim wildkart karakterom, i dopustiti da bilo koji broj znakova odgovara tom wildkartu:

```js
const regex = /free(co+de)camp.*free(co+de)camp/i;
```

Ovaj trenutni izraz neće osigurati da je broj `o` karaktera isti, međutim. Da bismo to postigli, moramo zameniti drugu grupu hvatanja referencom na prvu.

Unutar regularnog izraza, backreferenca se označava povratnim kosom crtom praćenim brojem grupe za hvatanje (capture group):

:::interactive_editor

```js
const regex = /free(co+de)camp.*free\1camp/i;
console.log(regex.test("freecooooodecamp is great i love freecooooodecamp")); // true
console.log(regex.test("freecooooodecamp is great i love freecodecamp")); // false
```

:::

I i sa tim, možemo videti da string sa tačnim brojem `o` odgovara, dok string sa dva različita broja `o` ne odgovara.

Ova sintaksa je odlična, ali može brzo da postane zbunjujuća kada se referencirate na više grupa snimanja. Srećom, umesto korišćenja brojeva, možete dati imime svojim grupama.

Da biste definisali nazvani grupni hvatač, dodajete znak pitanja (`?`) praćen imenom koje je zatvoreno između manjeg i većeg od znaka na početak grupe. Nazovićmo našu grupnu hvatač `code`:

```js
const regex = /free(?<code>co+de)camp.*free\1camp/i;
```

Sada možemo ažurirati našu povratnu referencu u regularnom izrazu da se odnosi na ovu grupu. Imenovana povratna referenca počinje kosom crtom praćenu slovom `k` u JavaScriptu. Zatim dodajete ime, ponovo zatvoreno između znakova manjeg od (<) i većeg od (>) (`<`) i (`>`). Pogledajmo to:

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
```

Sada ako proverimo naš poziv `test()`, možemo videti da i dalje prosleđujemo:

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
console.log(regex.test("freecooooodecamp is freecooooodecamp")); // true
```

:::

Da bismo koristili našu nazvanu grupu za hvatanje u pozivu ``replace()``, umetnuli bismo znak dolara u niz, praćen imenom zatvorenim manjim i većim nego znakovima:

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp/i;
console.log("freecooooodecamp".replace(regex, "paid$<code>camp")); // paidcooooodecamp
```

:::

Na kraju, ponekad želite kreirati grupu znakova, ali ne trebate uhvaćeni rezultat.

Recimo da želimo da poklopimo ili `freecodecamp` ili `freecandycamp`. Možete kreirati dva obrasca odvojena OR tvrdnjom:

```js
const regex = /freecodecamp|freecandycamp/i;
```

Ali ovo može postati prilično dugačko za regularne izraze veće skale. Umesto toga, možete kreirati neuhvatljivu grupu oko karaktera koje želite da spojite (OR):

```js
const regex = /free(?:code|candy)camp/i;
```

Non-capturing grupa ne skladišti podudarnost `code|candy` zasebno u memoriji. Ali može biti korisna za kreiranje alternativnih obrazaca bez žrtvovanja čitljivosti ili performansi.
