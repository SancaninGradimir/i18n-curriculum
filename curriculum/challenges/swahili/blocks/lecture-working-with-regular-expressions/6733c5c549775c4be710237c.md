# --description--

Naučićemo kako da pronađemo ili zamenimo sve pojavljivanje obrasca u nizu karaktera.

Prethodno ste naučili o metodama `replace()` i `match()`, kao i o globalnom modifikatoru `g`. Sada možete kombinovati te znanje da obradite sve obrasce u nizu karaktera. Podsetimo se našeg prvobitnog koda za podudarnost:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

I naš rezultat objekat podudarnosti:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

Ali šta ako imamo niz karaktera sa više pojavljivanja `freecodecamp` koja treba da se pronađu? Pogledajmo kako se `match()` ponaša u tom slučaju. Dodaćemo i naš stari primer sa `replace()`, samo radi poređenja:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

I rezultat je ovo:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
// freeCodeCamp is the best we love freecodecamp
```

Oh, ne! `match()` je vratio samo prvo podudarnost, a `replace()` je zamenio samo prvo podudarnost. To je zato što, po podrazumevanju, `match()` i `replace()` operišu samo na prvom pojavljivanju obrasca.

Srećom, možete ovo izbeći korišćenjem globalnog modifikatora na vašem regularnom izrazu. Dodajmo ga našem:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

I potvrđujemo rezultat:

```js
// [ 'freecodecamp', 'freecodecamp' ]
// freeCodeCamp is the best we love freeCodeCamp
```

To je uspelo! Naš poziv `replace()` zamenio je sve male slova nizove `freecodecamp`, a naša metoda `match()` podudarila se sa oba.

Ono što je zanimljivo ovde jeste da kada koristite globalni modifikator sa `match()`, gubite dodatne informacije o grupama za hvatanje (capture groups) i indeksima niza karaktera koje bi došle u niz podudarnosti
