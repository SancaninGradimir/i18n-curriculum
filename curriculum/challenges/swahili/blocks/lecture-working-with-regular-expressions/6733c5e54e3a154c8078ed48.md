# --interactive--

Naučićemo o kvantifikatorima u regulisanim izrazima.

Razmotrite scenarij u kojem želite da poklopite identifikacioni kod od četiri cifre. Znate da možete koristiti karakter klasu ``\d``, pa biste to mogli napisati četiri puta. A kako biste izbegli bilo kakve suvišne karaktere, uključili biste i početni i krajnji anker:

```js
const regex = /^\d\d\d\d$/;
```

I ovo radi – poklapaće se sa četiri numerička karaktera. Ali umesto da pišete istu klasu više puta, možete mu dati kvantifikator.

Kvantifikatori su definisani uglastim zagradama koje sadrže jedan ili dva broja. Koristićemo kvantifikator u našem obrascu:

```js
const regex = /^\d{4}$/;
```

Primećujete kako naš kvantifikator sadrži samo broj `4`. Ova sintaksa znači "poklapanje prethodnog karaktera tačno četiri puta". Pogledaćemo kako to funkcioniše:

:::interactive_editor

```js
const regex = /^\d{4}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("123456")); // false
console.log(regex.test("1234567")); // false
```

:::

Šablon odgovara samo stringu sa tačno četiri cifre, jer smo koristili sidra i naš kvantifikator dopušta samo tačno četiri cifre. Ali možda identifikacioni kod treba da ima samo minimum od četiri cifre.

Da biste omogućili četiri ili više cifara, dodajte zarez nakon broja u vašem kvantifikatoru:

```js
const regex = /^\d{4,}$/;
```

Sada, naša sintaksa omogućava obrascu da odgovara četiri ili više cifara. Testirajmo ga:

:::interactive_editor

```js
const regex = /^\d{4,}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // true
```

:::

Sedmoredni identifikator je prilično dugačak. Ovi identifikatori bi trebalo da imaju maksimum od 6 cifara i minimum od 4 cifre. Da biste ovo postigli, možete dodati drugi broj vašem kvantifikatoru nakon zareza:

```js
const regex = /^\d{4,6}$/;
```

Sada naš obrazac više ne odgovara sedmorednom identifikatoru, jer je veći od našeg šestostranog maksimuma.

:::interactive_editor

```js
const regex = /^\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Napomena da ne možete koristiti ovu sintaksu samo za postavljanje maksimuma – uvek morate postaviti minimum. Ali ako postavite minimum na `1`, možete efikasno postići isti rezultat.

Primili smo ažurirane zahteve od naših korisnika. Identiifikatori sada opcionalno mogu početi slovom. Već znamo karakter klasu za ovo, pa hajde da to dodamo u naš regularni izraz:

```js
const regex = /^[a-zA-Z]\d{4,6}$/;
```

Ali sada nametamo prisustvo slova. Kako ga možemo učiniti opcionim?

Možete koristiti sintaksu kvantifikatora sa `0` kao minimumom i `1` kao maksimumom:

```js
const regex = /^[a-zA-Z]{0,1}\d{4,6}$/;
```

Ali zapravo postoji poseban skraćeni kvantifikator za jedan opcionalni karakter – znak pitanja (`?`). Zamenimo naš kvantifikator sa znakom pitanja:

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
```

Trebalo bi da validiramo rezultat:

:::interactive_editor

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // false
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Naš obrazac sada dozvoljava jedno opcionalno slovo, praćeno sa četiri do šest cifara.

Nažalost, upravo smo shvatili da smo pogrešno pročitali zahteve. Moramo dozvoliti bilo koji broj slova pre brojeva. Možemo koristiti naš kvantifikator sa minimalnim vrednošću `0` i bez definisanog maksimuma:

```js
const regex = /^[a-zA-Z]{0,}\d{4,6}$/;
```

Ali naš obrazac ponovo postaje dugačak. Srećom, postoji još jedan skraćeni način za "poklapanje prethodnog karaktera nula ili više puta" – znak asteriksa (`*`). Zamenimo naš kvantifikator tim u obrascu i testirajmo ga:

:::interactive_editor

```js
const regex = /^[a-zA-Z]*\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Sada uspešno poklapamo bilo koji identifikator sa nula ili više slova praćenih od četiri do šest brojeva. Ali ispostavlja se da ovo ruši naš sistem – zapravo moramo zahtevati barem jedno slovo.

Opet, mogli bismo koristiti kvantifikator sa minimumom od jedan i bez definisanog maksimuma, ili bismo mogli koristiti još jednu posebnu sintaksu – znak plus (+) (`+`):

:::interactive_editor

```js
const regex = /^[a-zA-Z]+\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Sada identifikatori koji ne počinju sa barem jednom slovom propadaju, bez obzira na to koliko brojeva postoji.

Možete koristiti kvantifikatore da značajno poboljšate sažetost i čitljivost vaših regulacionih izraza.
