# --interactive--

Funkcija `minmax()` definiše opseg veličine mrežnog traka (grid track), navodeći koliko prostora red ili kolona mogu zauzeti.

Zapamtite da možete postaviti veličinu trake sa jedinicama kao što su `px` (pikseli), `rem`, ili čak `em`, i sa frakcionim jedinicama (`fr`).

Funkcija ``minmax()`` ide korak dalje omogućavanjem postavljanja minimalne i maksimalne veličine za mrežni trag.

Evo sintakse funkcije `minmax()`:

```css
minmax(min, max)
```

`min` je minimalna veličina mrežnog traka, koju se može podesiti korišćenjem piksela, procenta ili `auto`. A `max` je maksimalna veličina mrežnog traka koju možete podesiti istim jedinicama.

Dve vrednosti rade zajedno na ovaj način:

- Vrednost `min` osigurava da mreža staza nikada neće smanjiti ispod zadate veličine.

- Vrednost `max` ograničava koliko veliki može da raste mrežni trag.

Veličina mrežnog traka dinamički se prilagođava između vrednosti ``min`` i ``max`` na osnovu sadržaja i veličine kontejnera.

Pogledajmo praktičan primer:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div>
    <h2>Item 1</h2>
  </div>
  <div>
    <h2>Item 2</h2>
  </div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-columns: minmax(150px, 300px) 1fr;
  gap: 20px;
}

.grid-container > div {
  background: crimson;
  padding: 20px;
  text-align: center;
}
```

:::

Šta se ovde dešava?

Prva kolona, `minmax(150px, 300px)`, uvek će biti najmanje `150px` i najviše `300px`, zavisno od dostupnog prostora.

S druge strane, druga kolona, `1fr`, zauzimaće sav raspoloživi preostali prostor u kontejneru mreže pošto nema dodatnih kolona sa kojima bi delile prostor.

Prednost funkcije ``minmax()`` u odnosu na fiksne veličine i čak jedinice ``fr`` jeste što je fleksibilnija, što je čini idealnom za prilagodljivost i reaktivnost.

## --text--

Šta minimalna i maksimalna vrednost u funkciji ``minmax()`` kontroliše?
## --answers--

Minimalna vrednost omogućava smanjenje, a maksimalna vrednost postavlja fiksne dimenzije.
### --feedback--

Minimalna vrednost sprečava smanjenje ispod navedene veličine, a maksimalna vrednost ograničava rast.
---

The min value ensures the grid track won't shrink below a set size, and the max value limits how large it can grow.

---

The min value limits growth, and the max value determines the smallest size.

### --feedback--

Maksimalna vrednost ograničava rast, a minimalna vrednost osigurava minimalnu veličinu, ne obrnuto.
---

Both values set fixed sizes for the grid track.

### --feedback--

Minimalne i maksimalne vrednosti definišu opseg, a ne fiksne veličine.
