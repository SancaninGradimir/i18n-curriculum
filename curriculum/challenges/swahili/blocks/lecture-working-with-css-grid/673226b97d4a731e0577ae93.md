# --interactive--

Svojstvo `grid-template-areas` vam omogućava da dizajnirate vizuelni raspored mreže korišćenjem imenovane oznake.

Zatim dodeljujete oznake specifičnim mrežnim elementima koristeći svojstvo `grid-area`. Dakle, drugim rečima, te nazvane oznake takođe se nazivaju "ime mrežnog područja".

Ovo je osnovna sintaksa svojstva `grid-template-areas`:

```css
grid-template-areas:
 'header header header'
 'left-sidebar main right-sidebar'
 'footer footer footer';
```

Evo tačaka koje treba zapamtiti iz osnovne sintakse:

- Vrednosti kao što su `header` i `main` su imena mrežnih područja.

- Svaka vrednost razdvojena razmakom unutar niza odgovara koloni.

- Svaki niz predstavlja red u mrežici.

Dakle, u sintaksi, imamo kontejner mreže 3 na 3.

Nakon definisanja šablona, morate zatim koristiti svojstvo ``grid-area`` i naznačene oznake (ili oblasti) kao vrednosti da kažete CSS-u da određeni element pripada toj oblasti mreže.

Svojstvo `grid-area` povezuje element mreže sa imenovanom regijom koju definišete u `grid-template-areas`.

Popularan način demonstriranja mogućnosti svojstva ``grid-template-areas`` je kreiranje klasičnog Holy Grail rasporeda.

Ako ne znate šta je, Holy Grail raspored je obrazac web dizajna sa zaglavljem, podnožjem, dve bočne trake i glavnim sadržajnim područjem. Osigurava da glavni sadržaj ima prioritet, dok se bočne trake i ostali odeljci prilagođavaju responzivno unutar rasporeda.

Postoje mnoge rešenja za implementaciju 'svetog grala' rasporeda, ali korišćenje ``grid-template-areas`` i svojstvo ``grid-area`` je najjednostavniji način da ga kreirate.

Here's an example for the holy grail layout:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div class="header">
    <h2>Header</h2>
  </div>
  <div class="sidebar-left">
    <h2>Left Sidebar</h2>
  </div>
  <div class="main"><h2>Main Content</h2></div>
  <div class="sidebar-right">
    <h2>Right Sidebar</h2>
  </div>
  <div class="footer">
    <h2>Footer</h2>
  </div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-areas:
    'header header header'
    'sidebar-left main sidebar-right'
    'footer footer footer';
  grid-gap: 10px;
  background-color: #2196F3;
  padding: 10px;
}

.header {
  grid-area: header;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.sidebar-left {
  grid-area: sidebar-left;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.main {
  grid-area: main;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.sidebar-right {
  grid-area: sidebar-right;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.footer {
  grid-area: footer;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}
```

:::

Molimo vas da napomenete da se svojstva `grid-template-areas` i `grid-area` mogu koristiti nezavisno jedno od drugog.

Svojstvo ``grid-template-areas`` se koristi isključivo za definisanje vizuelnog rasporeda mapiranjem imenovanih mrežnih područja unutar mrežnog kontejnera.

S druge strane, svojstvo `grid-area` se koristi za pozicioniranje pojedinačnih elemenata mreže, bilo specifikovanjem njihovih rednih i kolumnih pozicija ili referenciranjem imenovanih područja definisanih sa svojstvom `grid-template-areas`.
