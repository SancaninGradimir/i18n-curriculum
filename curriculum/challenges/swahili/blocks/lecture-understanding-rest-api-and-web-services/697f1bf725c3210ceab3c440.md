# --description--

Veb servisi i REST API-ji pokreću većinu aplikacija i veb sajtova koje koristite svakodnevno. Omogućavaju različitim aplikacijama da komuniciraju međusobno i dele podatke preko mreže, najčešće interneta.

API (Application Programming Interface) je opšti način na koji softver komunicira sa drugim softverom. Web usluga je specifičan tip API-ja koji koristi veb tehnologije, kao što su HTTP, za razmenu podataka preko mreže, obično interneta. Zbog toga je svaka web usluga API – ali nije svaki API web usluga. REST API-ji su uobičajen tip web usluge koji prate određeni set pravila o tome kako se podaci zahtevaju i vraćaju.

Ovaj čas će se fokusirati na to šta su veb servisi na visokom nivou, a sledeći će ući u više detalja o tome kako REST API-ji funkcionišu.

Web uslugu možete smatrati mostom između softvera i softvera. Nema korisničkog interfejsa i nije namenjen za direktnu upotrebu. Umesto toga, fokusiran je isključivo na razmenu podataka između aplikacija.

Da bi se bilo koji sistem smatrao web uslugom, mora imati sledeće karakteristike:

* **Pokreće se na serveru**: web usluga mora biti hostovana online kao veb sajt, ali umesto stranica koje korisnici mogu da vide, ona izlaže podatke ili neku funkcionalnost
    
* **Komunikacija mašina sa mašinama**: web usluga nije napravljena da je ljudi direktno vide, već za mobilne i veb aplikacije da je konzumiraju, tako da mogu izložiti neke podatke ili funkcionalnost koju korisnik na kraju može da vidi
    
* **Nezavisna od jezika**: web usluga se može napisati na bilo kom jeziku kao što su Java ili C#, i svaka aplikacija bi trebalo da može sa njom da interaguje, čak i ako je napisana na drugom jeziku
    
* **Nezavisna od platforme**: ne bitno na kojoj platformi mašine rade. Na primer, web usluga koja radi na Linuksu ne bi trebalo da spreči uređaj koji radi sa operativnim sistemom Windows da se sa njom komunikuje
    
* **Koristi standardne veb protokole**: web usluga bi obično trebalo da koristi HTTP/HTTPS za prenos podataka preko interneta, FTP za prenos fajlova i SMTP za e-poštu

**REST** i **SOAP** su dva od najpoznatijih tipova web servisa, iako postoje i drugi pristupi, kao što je **XML-RPC**.

**REST**, što stoji za **Transfer reprezentativnog stanja**, je daleko najčešće korišćen danas. Većina modernih aplikacija se oslanja na RESTful veb usluge jer su jednostavne, fleksibilne i prirodno rade sa webom.

REST API-ji izlažu resurse, koji su obično predstavljeni putem URL putanja kao što su ``/users`` ili ``/users/20``. Kada klijent zahteva resurs, server vraća predstavljanje tog resursa. Ovo predstavljanje je često u JSON formatu, ali može biti i u formatima kao što su XML ili čak običan tekst.

Klijenti mogu da navedu željeni format korišćenjem HTTP zaglavlja, kao što je zaglavlje `Accept`. Na primer, klijent može zatražiti JSON odgovor postavljanjem odgovarajuće vrednosti `Accept`.

```bash
Accept: application/json
```

I možete da navedete da želite XML na ovaj način:

```bash
Accept: application/xml
```

Resurs koji dobijete nazad može izgledati ovako ako je u JSON formatu:

```json
{
  "id": 20,
  "name": "John Doe",
  "email": "john@example.com",
  "role": "admin",
  "createdAt": "2025-09-22T10:30:00Z"
}
```

I ovo, ako je XML:

```xml
<user>
  <id>20</id>
  <name>John Doe</name>
  <email>john@example.com</email>
  <role>admin</role>
  <createdAt>2025-09-22T10:30:00Z</createdAt>
</user>
```

SOAP znači **Simple Object Access Protocol**. Koristi samo XML format podataka, a najčešće je u sistemima na nivou preduzeća koji zahtevaju strože standarde i bezbednost.

Evo kako bi SOAP podaci mogli da izgledaju:

```xml
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/">
   <soapenv:Body>
      <getUser>
         <userId>20</userId>
      </getUser>
   </soapenv:Body>
</soapenv:Envelope>
```

XML-RPC koristi XML za enkardiranje svojih zahteva i odgovora. To ga čini laganim, ali manje fleksibilnim od REST-a.

Što se tiče UDDI, nije usluga razmene podataka, već direktorijum zasnovan na XML-u gde preduzeća mogu navesti, otkriti i opisati svoje dostupne veb usluge.
