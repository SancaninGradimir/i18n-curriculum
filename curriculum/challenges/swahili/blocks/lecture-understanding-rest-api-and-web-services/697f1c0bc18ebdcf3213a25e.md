# --description--

**REST** je skraćenica za *representational state transfer*. Zamislite ga kao skup smernica koje opisuju kako se veb klijenti i serveri treba da komuniciraju međusobno preko interneta.

U ovoj lekciji, naučićete šta je REST, koji su mu ključni koncepti i kako funkcioniše.

REST definiše način na koji različiti sistemi međusobno komuniciraju korišćenjem standardnih veb tehnologija. Sa REST-om, klijenti kao što su veb pregledači, mobilne aplikacije i API-ji komuniciraju sa serverom koristeći uobičajene HTTP metode poput `GET`, `POST`, `PUT`, `PATCH` i `DELETE`, o kojima smo razgovarali u prethodnoj lekciji.

Neki osnovni koncepti REST-a uključuju **bezdržljivost stanja (statelessness)**, **resurse**, **HTTP metode** i **kodove odgovora ili status kodove**.

**Bezdržljivost stanja (Statelessness)** znači da je svaki zahtev koji klijent pošalje serveru obrađen nezavisno. Server ne pamti prethodne zahteve. Zbog toga, svaki zahtev mora da sadrži sve informacije potrebne za njegovu obradu. Na primer, ako je korisnik autentifikovan, njegov token za autentikaciju mora biti poslat sa svakim zahtevom.

U REST-u, sve se tretira kao resurs. Resurs je deo podataka koji se može pristupiti ili modifikovati. Svaki resurs je identifikovan putem URL-a, kao što su `/products` za sve proizvode, ili `/products/1` za određeni proizvod.

Niže je pregled HTTP metoda, status kodova i korak-po-korak primer kako funkcioniše REST arhitektura.

## HTTP Metode

HTTP metode definišu akciju koja se izvodi na resursu:

* `GET`: Dohvati resurs
    
* `POST`: Kreiraj novi resurs
    
* `PUT / PATCH`: Ažuriraj postojeći resurs
    
* `DELETE`: Ukloni resurs
    

## HTTP Status Kodovi

Nakon obrade zahteva, server odgovara **status kodom** koji ukazuje rezultat:

* `200` **(OK)**: Zahtev je uspešan
    
* `201` **(Created)**: Kreiran je novi resurs
    
* `400` **(Bad Request)**: Klijent je poslao nevažeće podatke
    
* `404` **(Not Found)**: Zahtevani resurs ne postoji
    
* `500` **(Internal Server Error)**: Došlo je do greške na serveru
    

## Kako REST funkcioniše: Korak-po-korak primer

### 1. Klijent šalje zahtev

* Klijent šalje HTTP zahtev koristeći metodu poput `GET` ili `POST`
    
* Zahtev uključuje URL koji identifikuje resurs
    
* Može takođe da uključuje zaglavlja (headers), metapodatke i telo zahteva (request body)
    

### 2. Server prima zahtev

* Server validira format zahteva i autentifikaciju (ako je potrebno)
    
* Identifikuje ciljani resurs iz URL-a
    
* Određuje traženu akciju na osnovu HTTP metode
    
* Proverava dozvole i kontrolu pristupa
    

### 3. Server izvršava akciju

* `GET`: Dohvati resurs
    
* `POST`: Kreiraj novi resurs
    
* `PUT / PATCH`: Ažuriraj resurs
    
* `DELETE`: Ukloni resurs
    
* Izvršena je bilo koja druga neophodna logika
    

### 4. Server gradi odgovor

* Server bira odgovarajući status kod (na primer, `200`, `201` ili `404`)
    
* Formatira podatke odgovora, obično kao **JSON**
    
* Dodaje relevantne zaglavlja poput `Content-Type` ili `Cache-Control`
    

### 5. Server šalje odgovor

* Odgovor uključuje status kod, zaglavlja i opcionalno telo (body)
    
* Nije sačuvano stanje klijenta na serveru (bezdržljivost stanja)
    

### 6. Klijent obrađuje odgovor

* Klijent proverava status kod kako bi utvrdio uspeh ili neuspeh
    
* Obrađuje sve podatke koje je server vratio
