# --interactive--

U JavaScriptu, postoji nekoliko načina za proveru da li objekat ima određeno svojstvo. Razumevanje ovih metoda je važno za efikasno rad sa objektima, posebno kada obrađujete podatke iz eksternih izvora ili kada morate osigurati da određena svojstva postoje pre nego što ih koristite.

Istražićemo neke uobičajene pristupe: `hasOwnProperty()` metodu, `Object.hasOwn()` metodu, `in` operatoru, i proveravanje naspram `undefined`.

Počinjemo sa metodom ``hasOwnProperty()``. Ovaj metod vraća boolean vrednost koja ukazuje da li objekat poseduje navedeno svojstvo kao svoje svojstvo. Evo primera:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

U ovom primeru, imamo objekat nazvan `person` sa dve svojstva: `name` i `age`. Da bismo proverili da li je `name` svojstvo u objektu `person`, koristimo metodu `hasOwnProperty()`. Pošto je `name` svojstvo, on će vratiti `true`. Ali kada koristimo `hasOwnProperty()` da bismo proverili da li je `job` svojstvo, on će vratiti `false` jer ne postoji u objektu.

`Object.hasOwn()` je moderan, preporučeni način za proveru da li objekat poseduje svojstvo kao svoje (nije nasleđeno). Razmislite o tome kao o nadograđenoj, sigurnijoj verziji `hasOwnProperty()`. Sintaksa je `Object.hasOwn(object, propertyName)` — prosleđujete objekat kao prvi argument, a ime svojstva kao drugi.

Evo osnovnog primera:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

U ovom primeru, `Object.hasOwn(person, "name")` vraća `true` jer `name` postoji direktno na objektu `person`. `Object.hasOwn(person, "job")` vraća `false` jer `job` nikada nije dodato objekte.

Veoma je važno razumeti da `Object.hasOwn()` proverava samo da li svojstvo **postoji** — nije mu stalo do vrednosti svojstva. To znači da i dalje vraća `true` čak i kada je vrednost `0`, `false`, `null` ili `undefined`:

:::interactive_editor

```js
const user = {
  username: "coder123",
  score: 0,
  isActive: false,
  nickname: null
};

// Object.hasOwn() correctly reports these all exist
console.log(Object.hasOwn(user, "score"));    // true  (value is 0, but property exists)
console.log(Object.hasOwn(user, "isActive")); // true  (value is false, but property exists)
console.log(Object.hasOwn(user, "nickname")); // true  (value is null, but property exists)
console.log(Object.hasOwn(user, "email"));   // false (property was never added)

// Danger! Using if() directly gives wrong results for falsy values
if (user.score) {
  console.log("Has score"); // This will NOT print even though score exists!
}

// Safe! Object.hasOwn() gives correct result
if (Object.hasOwn(user, "score")) {
  console.log("Has score:", user.score); // Has score: 0
}
```

:::

Drugi način za proveru postojanja svojstva u objektu je korišćenje operatera `in`. Kao što operater `hasOwnProperty()`, operater `in` će vratiti `true` ako svojstvo postoji na objektu. Evo kako ga možete koristiti:

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};
console.log("name" in person);  // true
```

:::

U ovom primeru, `"name" in person` vraća `true` jer `name` je svojstvo `person`.

Treći metod podrazumeva proveru da li je svojstvo `undefined`. Ovaj pristup može biti koristan, ali ima neke ograničenja. Evo primera:

:::interactive_editor

```js
const car = {
  brand: "Toyota",
  model: "Corolla",
  year: 2020
};

console.log(car.brand !== undefined); // true
console.log(car.color !== undefined); // false
```

:::

U ovom kodu, proveravamo da li `car.brand` i `car.color` nisu `undefined`. Ovo radi jer pristup nepostojećoj svojstvu na objektu vraća `undefined`. Međutim, ova metoda može dati lažne negativne rezultate ako svojstvo eksplicitno ima vrednost `undefined`.

U praksi, izbor između ovih metoda često zavisi od specifičnih zahteva vašeg koda. Razumevanje razlika između njih pomoći će vam da napravite pravi izbor u različitim scenarijima.
