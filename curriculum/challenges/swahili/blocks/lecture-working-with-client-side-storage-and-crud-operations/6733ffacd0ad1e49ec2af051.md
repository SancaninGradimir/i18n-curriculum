# --description--

Kao što ste naučili u prethodnim časovima, kolačići su jednostavni podaci koje sajtovi mogu sačuvati na uređaju korisnika.

Međutim, zapravo možete sačuvati složenije strukture podataka u kolačićima (cookies). Jedan uobičajen metod je korišćenje JSON-a za skladištenje objekata ili nizova. Evo primera:

```js
const userData = {
  name: "John Doe",
  age: 30,
  role: "admin"
};

document.cookie = "userInfo=" + JSON.stringify(userData) + "; path=/";
```

U ovom primeru, kreiramo objekat sa podacima korisnika, konvertujući ga u JSON string, a zatim ga skladištimo u kolačić (cookie). Kada želimo da dohvatimo ove podatke, možemo da parsiramo JSON string u objekat koristeći `JSON.parse()`.

Sada, možda se pitate za aspekt "HTTP zaglavlja" naše teme. Obično, kolačići postavlja server koristeći HTTP zaglavlja.

Na primer, server može poslati zaglavlje ovako:

```http
Set-Cookie: username=John Doe; expires=Thu, 31 Dec 2024 6:00:00 IST; path=/
```

Ovo zaglavlje govori pretraživaču da postavi kolačić.

Takođe možemo postaviti kolačiće direktno u pretraživaču koristeći JavaScript. Ovo je korisno za skladištenje podataka koji ne moraju biti odmah poslati na server.

Molimo vas da napomenete da kolačići imaju ograničenje veličine od oko 4KB, i skladištenje previše podataka u njima može usporiti vašu veb aplikaciju.

Čuvanje velikih količina podataka u kolačićima može povećati mrežni saobraćaj jer se kolačići šalju sa svakim HTTP zahtevom.
