# --description--

Web Storage API pruža mehanizam za pretraživače da skladište parove ključ-vrednost direktno unutar pretraživača, omogućavajući programerima da sačuvaju informacije koje se mogu koristiti tokom različitih ponovnih učitavanja stranica i sesija.

Dva glavna komponenta u Web Storage API su ``localStorage`` i ``sessionStorage``.

`localStorage` je deo Web Storage API-ja koji omogućava podacima da ostanu trajni čak i nakon zatvaranja prozora pretraživača ili osveženja stranice. Ovi podaci ostaju dostupni sve dok ih aplikacija ili korisnik eksplicitno ne uklone.

`sessionStorage` je drugi deo Web Storage API-ja koji skladišti podatke tokom sesije stranice, što znači da su podaci dostupni sve dok je otvoren pregledački kartica ili prozor. Međutim, za razliku od `localStorage`, podaci u `sessionStorage` se brišu kada se kartica ili prozor zatvore. Vi ćete saznati više o `sessionStorage` u sledećoj lekciji.

Uobičajene upotrebe za `localStorage` uključuju skladištenje korisničkih podešavanja, kao što su teme ili preferencije jezika, pamćenje podataka iz formulara tokom sesija pretraživača i keširanje malih delova informacija radi poboljšanja performansi veb aplikacija.

Keširanje se odnosi na skladištenje često pristupačnih podataka u privremenoj lokaciji za skladištenje, poznato kao keš, tako da se naknadni zahtevi za te podatke mogu servirati brže bez potrebe za ponovnim izračunavanjem ili preuzimanjem sa sporijeg izvora podataka, kao što je baza podataka ili eksterni server.

Neki uobičajeni metodi `localStorage` uključuju metode `setItem`, `getItem`, `removeItem` i `clear`.

Evo primera korišćenja metode `setItem()` koja skladišti par ključ-vrednost u `localStorage`.

```js
localStorage.setItem('username', 'JaneDoe');
```

Zatim, ako želimo da dohvatimo tu vrednost određenog ključa iz `localStorage`, možemo koristiti metodu `getItem()`.

```js
let username = localStorage.getItem('username');
console.log(username); // JaneDoe
```

Da biste uklonili stavku iz ``localStorage`` koristeći njen ključ, možete koristiti metodu ``removeItem()``.

```js
localStorage.removeItem('username');
```

Da biste obrisali sve podatke u `localStorage`, možete koristiti metodu `clear()`.

```js
localStorage.clear();
```

Sada, hajde da pogledamo primer gde koristimo `localStorage` za skladištenje preferirane teme korisnika.

```js
// Store the user's theme preference
localStorage.setItem('theme', 'dark');

// Retrieve the stored theme preference
const userTheme = localStorage.getItem('theme');
console.log(userTheme); // 'dark'

// Remove the theme preference
localStorage.removeItem('theme');

// Clear all localStorage data
localStorage.clear();
```

U ovom primeru:

- Prvo skladištimo izbor teme (`dark`) za korisnika.

- Zatim dohvatamo tu temu i ispisujemo je na konzolu.

- Konačno, demonstriramo kako ukloniti određeni stavak ili obrisati sve pohranjene podatke.

`localStorage` je veoma korisno za skladištenje malih delova podataka koji moraju da ostanu sačuvani između sesija, ali je važno napomenuti da `localStorage` ne bi trebalo koristiti za skladištenje osetljivih informacija, kao što su lozinke, jer može predstavljati sigurnosne rizike.

Kako se `localStorage` razlikuje od `sessionStorage`?
## --answers--

`localStorage` podaci se brišu kada se kartica zatvori, `sessionStorage` je trajno.
