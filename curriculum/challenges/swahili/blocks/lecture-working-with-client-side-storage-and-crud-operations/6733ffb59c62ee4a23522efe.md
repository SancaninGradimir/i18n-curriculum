# --interactive--

`IndexedDB` je alat za skladištenje strukturiranih podataka u pretraživaču. Ovo je ugrađeno u moderne veb pretraživače, omogućavajući veb aplikacijama da efikasno skladište i dohvate JavaScript objekte.

Za razliku od drugih mehanizama skladištenja kao što su `localStorage`, koji je ograničen na skladištenje stringova, `IndexedDB` može da skladišti JavaScript objekte, fajlove i gotovo svaki drugi tip podataka. Ovo olakšava web aplikacijama koje moraju raditi sa velikim i kompleksnim strukturalima podataka.

Sada, pogledajmo kako `IndexedDB` radi.

The first step is to open a database. Here's an example:

:::interactive_editor

```js
let request = indexedDB.open("Sample DB", 1);

request.onerror = function(event) {
  console.log("Error opening database");
};

request.onsuccess = function(event) {
  let db = event.target.result;
  console.log("Database opened successfully");
};
```

:::

U ovom kodu, otvaramo bazu podataka pod nazivom `"Sample DB"` sa verzijom 1. Pružamo dve funkcije povratne poziva: jednu za rukovanje greškama, i drugu za slučaj kada je baza podataka uspešno otvorena. Objekat `db` koji dobijemo u funkciji povratne poziva za uspeh je ono što ćemo koristiti za interakciju sa bazom podataka.

Ako proverite interfejs aplikacije alata za razvoj pretraživača (dev tools), videćete da je vaš `Sample DB` u sekciji `IndexedDb` otvoren. Kada otvorite bazu podataka, možete početi da radite sa skladištima objekata.

Objekti skladišta u IndexedDB-u su slični tabelama u tradicionalnim bazama podataka. Sadrže stvarne podatke koje želite da sačuvate. Evo kako kreirati objekat skladište:

```js
let request = indexedDB.open("Sample DB", 1);

request.onupgradeneeded = function(event) {
  let db = event.target.result;
  let objectStore = db.createObjectStore("customers", { keyPath: "id" });
};
```

Ovaj kod kreira objekat skladištenje nazvan `"customers"` sa `"id"` kao njegovom putanjom ključa. Putanja ključa je poput primarnog ključa u tradicionalnoj bazi podataka – koristi se za jedinstveno identifikovanje svakog zapisa.

Da bismo dodali podatke u naše skladište objekata, uradili bismo nešto slično ovome. ``db`` u ovom primeru predstavlja bazu podataka instancu ``IndexedDB``.

```js
let transaction = db.transaction(["customers"], "readwrite");
let objectStore = transaction.objectStore("customers");
let request = objectStore.add({ id: 1, name: "John Doe", email: "john@example.com" });

request.onerror = function(event) {
  console.log("Error adding data");
};

request.onsuccess = function(event) {
  console.log("Data added successfully");
};
```

Ovaj kod dodaje novog klijenta u naše objekatno skladište ``"customers"``. Pokrećemo transakciju (što je način na koji grupišemo operacije baze podataka), dobijamo referencu na naše objekatno skladište i zatim dodajemo podatke.

Dohvatanje podataka funkcioniše na sličan način. Pokrećemo transakciju, dobijemo naš objekatni skladište (object store), i zatim koristimo metode kao što je `get` za dohvaćanje podataka:

```js
let transaction = db.transaction(["customers"]);
let objectStore = transaction.objectStore("customers");
let request = objectStore.get(1);

request.onerror = function(event) {
  console.log("Error retrieving data");
};

request.onsuccess = function(event) {
  console.log("Customer:", request.result);
};
```

Ovaj kod dohvaća klijenta sa `id` od `1` iz našeg skladišta objekata `"customers"`.

Jedna od ključnih karakteristika ``IndexedDB`` je što je asinhkronan.

To znači da kada interagujete sa ``IndexedDB``, operacije ne blokiraju glavni niti veb aplikacije. Ovo osigurava da vaša veb aplikacija ostaje reaktivna čak i pri rukovanju velikim količinama podataka.

Iako `IndexedDB` pruža moćne mogućnosti, ima strmiju krivu učenja u poređenju sa drugim jednostavnijim opcijama za skladišni API koje mogu biti izazovne za početnike. Međutim, za aplikacije koje moraju da obrađuju velike količine strukturiranih podataka na strani klijenta, `IndexedDB` nudi neuporedivne mogućnosti.
