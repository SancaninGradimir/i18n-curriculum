# --description--

U prethodnoj lekciji, naučili smo o radu sa `localStorage` i kratko nas je predstavio za `sessionStorage`.

Zapamtite, da je `sessionStorage` kada se podaci obrišu čim korisnik zatvori karticu ili prozor u kojem web aplikacija radi. Idealno je za situacije u kojima podaci moraju da ostanu samo tokom trajanja jedne sesije, kao što je održavanje podataka forme tokom navigacije ili skladištenje privremenih informacija o stanju tokom procesa plaćanja.

Slično kao `localStorage`, `sessionStorage` koristi parove ključ-vrednost za skladištenje i dohvaćanje podataka. Metode korišćene sa `sessionStorage` su takođe iste kao `localStorage`, a jedina stvarna razlika je koliko dugo se podaci skladište.

Evo nekoliko primera rada sa različitim metodama:

- `sessionStorage.setItem()`: Skladišti par ključ-vrednost u `sessionStorage`.

```js
sessionStorage.setItem('cart', '3 items');
```

- `sessionStorage.getItem()`: Dohvata vrednost dato ključa iz `sessionStorage`.

```js
let cart = sessionStorage.getItem('cart');
console.log(cart); // Outputs: '3 items'
```

- `sessionStorage.removeItem()`: Uklanja određeni stav iz `sessionStorage` koristeći njegov ključ.

```js
sessionStorage.removeItem('cart');
```

- `sessionStorage.clear()`: Briše sve podatke sačuvanije u `sessionStorage`.

```js
sessionStorage.clear();
```

Pogledaćemo primer gde skladištimo podatke u `sessionStorage` koji traje samo dok je otvoren kartica ili prozor pretraživača:

```js
// Store data in sessionStorage
sessionStorage.setItem('currentUser', 'JohnDoe');

// Retrieve the stored data
const user = sessionStorage.getItem('currentUser');
console.log(user); // 'JohnDoe'

// Remove a specific key from sessionStorage
sessionStorage.removeItem('currentUser');

// Clear all sessionStorage data
sessionStorage.clear();
```

U ovom primeru, mi:

1. Sačuvajte ime trenutnog korisnika (`JohnDoe`) u `sessionStorage`.

2. Povući i prikazati ga.

3. Uklonite stavku povezanu sa ključem `currentUser`.

4. Obrišti sve podatke `sessionStorage`.

Ključna razlika od ``localStorage`` je u tome što čim korisnik zatvori karticu, sve sačuvane sesijske podatke će biti izgubljene.

`sessionStorage` je posebno korisno u scenarijima kao što su:

- Čuvanje privremenih podataka, kao što su unosi formulara tokom procesa sa više stranica.

- Čuvanje privremenih selekcija ili preferencija koje nemaju potrebu za trajanjem između sesija.

- Održavanje stanja u aplikaciji jedne stranice koje nije potrebno pametiti nakon zatvaranja kartice.

`sessionStorage` osigurava da se kada korisnik napusti stranicu, podaci sesije obrišu, što je odlično za scenarije gde ne želite zadržati informacije izvan trenutne sesije.
