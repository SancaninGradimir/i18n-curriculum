# --interactive--

Operator opcionalnog lančeg (`?.`) je koristan alat u JavaScriptu koji vam omogućava siguran pristup svojstvima objekata ili pozivanje metoda bez brige da li postoje. To je kao sigurnosna mreža za rad sa objektima koji možda imaju nedostajuće delove.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.name); // "Alice"
console.log(person.job); // undefined
```

:::

U ovom primeru, `person.name` postoji, pa loguje `Alice`. Ali `person.job` ne postoji, pa nam daje `undefined`.

Sada, recimo da želimo pristup svojstvu objekta koje možda ne postoji:

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.address.street); // This will throw an error!
```

Ovaj primer će baciti `Uncaught TypeError`. Pošto je `person.address` jednako `undefined`, ne možemo pristupiti svojstvu `street`. Ovde nam je koristan operater opcionalnog lančeg. Evo primera korišćenja operatera opcionalnog lančeg:

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user?.profile?.address?.street); // "123 Main St"
console.log(user?.profile?.phone?.number);   // undefined
```

:::

Koristeći operater opcionalnog lančeg, kažemo JavaScriptu da nastavi sa operacijom samo ako objekat (ili vrednost pre `?.`) postoji i nije `null` niti `undefined`.

Ako je vrednost pre ``?.`` jeste ``null`` ili ``undefined``, JavaScript vraća ``undefined`` umesto da pokuša nastaviti sa operacijom i baci grešku.

Zapamtite, operator opcionalnog lančegovanja je najkorisniji kada niste sigurni da li postoji svojstvo ili metoda. Pomaže u sprečavanju grešaka i čini vaš kod robusnijim.
