# --interactive--

Hoisting je ponašanje u JavaScriptu koje često zbunjuje početnike, ali razumevanje može pomoći da izbegnete suptilne greške u vašem kodu. Jednostavnim rečima, hoisting je podrazumevano ponašanje JavaScripta koje premesta deklaracije na vrh njihovih odgovarajućih opsega tokom faze kompajliranja pre nego što se kod izvrši.

Da biste razumeli *hoisting*, važno je znati da JavaScript radi u dve faze: faza kompajliranja i faza izvršavanja. Tokom faze kompajliranja, JavaScript motor prolazi kroz vaš kod i uspostavlja memorijski prostor za varijable i funkcije. Ovde ulazi u igru *hoisting*.

Počnimo sa podizanjem promenljivih (hoisting), kada deklarišete promenljivu koristeći ključnu reč `var`, JavaScript podiže deklaraciju na vrh svog opsega. Međutim, važno je napomenuti da se podiže samo deklaracija, a ne inicijalizacija. To znači da možete koristiti promenljivu u svom kodu pre nego što je deklarišete, ali će njena vrednost biti `undefined` sve dok joj zapravo ne dodelite vrednost.

:::interactive_editor

```js
console.log(x); // undefined
var x = 5;
console.log(x); // 5
```

:::

U ovom kodu, iako koristimo `x` pre deklarisanja ga, ne dobijamo grešku; umesto toga dobijamo `undefined`. To je zato što JavaScript podiže deklaraciju `var x` na vrh svog opsega, ali ne i inicijalizaciju `x = 5`. Kao da se kod prepisuje ovako:

:::interactive_editor

```js
var x;
console.log(x); // undefined
x = 5;
console.log(x); // 5
```

:::

Funkcionalno podizanje (Function hoisting) funkcioniše malo drugačije. Kada deklariš funkciju koristeći sintaksu deklaracije `function`, podizaju se i ime funkcije i telo funkcije. To znači da možete pozvati funkciju pre nego što ste je deklarisali u svom kodu. Evo primera funkcionalnog podizanja (hoisting) funkcija:

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello(){
  console.log("Hello, World!");
}
```

:::

U ovom slučaju, možemo pozvati `sayHello()` pre njegove deklaracije jer je cela funkcija podignuta na vrh svog opsega.

Važno je napomenuti da se *hoisting* drugačije ponaša sa `let` i `const` deklaracijama uvedene u ES6.

```js
console.log(y); // Throws a ReferenceError
let y = 10;
```

Ove deklaracije su podignute, ali nisu inicijalizovane i ne možete pristupiti njima pre stvarne deklaracije u vašem kodu. Ovo se često naziva vremenska mrtva zona.

Razumevanje "hoistinga" može vam pomoći da napišete čistiji, predvidljiviji kod. Međutim, oslanjanje na "hoisting" može učiniti vaš kod teže čitljivim i održivim. Kao najbolja praksa, preporučuje se da deklarišete vaše varijable na vrhu njihovog opsega, a funkcije pre nego što ih koristite, bez obzira na "hoisting". Ovo čini ponašanje vašeg koda eksplicitnijim i lakšim za razumevanje ne samo drugima, već i vama u budućnosti.
