# --description--

U prethodnoj lekciji ste naučili kako da kreirate novi repozitorijum na GitHubu. Ali šta ako imate postojeći projekat lokalno koji želite dodati na GitHub? U ovoj lekciji ćete naučiti kako da poslujete (push) postojeće lokalne repozitorijume na GitHub i povlačite (pull) promene sa udaljenih repozitorijuma.

Počnite kreiranjem novog repozitorijuma na GitHubu bez fajlova. Kada je sve ispravno urađeno, trebalo bi da budete preusmereni na stranicu koja sadrži URL potreban za kloniranje vašeg repozitorijuma.

Sada navigirajte do svog terminala i izaberite postojeći projekat koji želite dodati na GitHub. U ovoj lekciji, biram lokalni projekat pod nazivom `super-awesome-game` sa sledećim fajlovima unutra:

```sh
jessicawilkins super-awesome-game >> ls
README.md  index.html script.js  styles.css
```

Da biste postavili kontrolu verzija za ovaj projekat koristeći Git, moraćete da pokrenete komandu ``git init`` u direktorijumu projekta. Ovo će inicijalizovati prazno Git repozitorijum kako bi Git mogao da počne praćenje promena za ovaj projekat. Kada je sve pravilno urađeno, trebalo bi da vidite sličan izlaz u terminalu:

```sh
jessicawilkins super-awesome-game >> git init
Initialized empty Git repository in /Users/jessicawilkins/workspace/freeCodeCamp/super-awesome-game/.git/
```

Kada inicijalizujete prazan Git repozitorijum za projekat, dodati će se novi ``.git`` skriveni direktorijum. Da biste videli dodavanje direktorijuma, možete pokrenuti komandu ``ls -a`` koja navodi sve fajlove i direktorijume, uključujući skrivene:

```sh
jessicawilkins super-awesome-game >> ls -a
.          ..         .git       README.md  index.html script.js  styles.css
```

Važno je ne brišati taj `.git` direktorijum, inače će cela vaša Git istorija biti izgubljena.

Sada što je prazno Git repozitorijum inicijalizovano, možete pokrenuti komandu ``git status``. Ova komanda se koristi za prikaz trenutnog stanja vašeg radnog direktorijuma – ovu komandu ćete mnogo koristiti u svom radnom procesu. Trebalo bi da vidite izlaz sličan ovom:

```sh
jessicawilkins super-awesome-game >> git status
On branch main

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
	README.md
	index.html
	script.js
	styles.css

nothing added to commit but untracked files present (use "git add" to track)
```

`On branch main` znači da ste na grani `main`. Grana u Git-u je zasebno radno okruženje gde možete napraviti promene. Grana `main` često će predstavljati glavnu ili produkcionu granu u stvarnoj aplikaciji. Timovi programera kreiraće više grana za nove funkcije i popravke grešaka, a zatim te promene spajati nazad u granu `main`. Vi ćete saznati više o tom procesu u budućoj lekciji.

`Untracked files` znači da je Git detektovao fajlove koje trenutno ne prati.

Postoje pet stanja u kojima fajl može biti:

1. "Neprograđeni": Ovo znači da je fajl nov za repozitorijum, i da ga Git ranije nije "video".
2. "Izmenjen": Ovaj fajl je postojao u prethodnom komitu, i ima promene koje nisu komitovane.
3. "Ignorisano": Verovatno nećete videti ignorisane fajlove u Git-u, ali vaš IDE može imati indikator za njih. Ignorisani fajlovi su isključeni iz operacija Git-a, obično zato što su uključeni u fajl `.gitignore`.
4. "Izbrisano": Izbrisan fajl je suprotnost nepraćenom fajlu – to je fajl koji je ranije postojao, i koji je uklonjen.
5. "Preimenovan": Preimenovani fajl je fajl čiji sadržaj nije promenjen, ali je ime ili lokacija fajla modifikovana. U nekim slučajevima, fajl se može smatrati preimenovanim čak i ako ima malu količinu promena.

Kada imate neobjavljene promene u vašem repozitorijumu (repo), prvo ćete morati da ih "staging-ujete". "Staging zona" je specijalna datoteka u vašem direktorijumu `.git` koja sadrži informacije koje će ići u vaš sledeći komit. Komit je efektivno snimak trenutnog stanja vašeg repozitorijuma.

Ako želite da pripremate po jedan fajl, onda možete koristiti sledeću komandu:

```sh
git add name-of-file
```

Ako želite da pripremite sve ne-pripremljene promene, onda možete koristiti `git add .` Tačka (`.`) je alias za trenutni direktorijum u kojem se nalazite. Neki programeri će upozoriti na korišćenje `git add .` jer biste mogli da pripremite fajlove koje niste nameravali. Ako se to dogodi, možete ih "unstage-ovati" koristeći sledeću komandu:

```sh
git reset name-of-file
```

Međutim, sve dok nadzirate koje fajlove se dodaju koristeći `git status` pre nego što ih pripremite (stage), bi trebalo da budete u redu.

Kada staged-ujete svoje fajlove, moraćeš ih da komituješ. Možete komitovati svoje promene pokretanjem komande ``git commit``. Ovo će otvoriti vaš omiljeni editor koji ste podesili u Git konfiguraciji. Kada je editor otvoren, možete uneti detaljnu poruku o svojim promenama. Takođe možete odlučiti da unesete kraću poruku korišćenjem komande ``git commit -m``:

```sh
git commit -m "short message goes here"
```

Ako planirate da koristite zastavicu `-m`, i dalje ćete želeti da pružite opisnu poruku kako biste dali kontekst za promene. Evo nekoliko primera dobrih poruka koje možete koristiti:

```sh
git commit -m "feat(api): implement JWT-based authentication"
git commit -m "fix(auth): correct token expiration time"
git commit -m "refactor: simplify login flow logic"
```

`feat(api)` i `fix(auth)` prate stil Conventional Commits koji se koristi za pružanje konteksta o vrstama promena u tom commitu.

Nakon što ste sačuvali svoj rad i ponovo pokrenete ``git status``, trebalo bi da vidite sledeći izlaz u terminalu:

```sh
On branch main
nothing to commit, working tree clean
```

Da biste videli sve vaše prethodne komite za ovaj projekat, možete pokrenuti komandu ``git log``. Ovo će listati sve prethodne komite sa korisnim informacijama kao što su autor, datum commita, poruka commita i hash commita. Commit hash je dugi niz znakova koji služi kao jedinstveni identifikator za komit.

Kada završite sa komitovanjem svog rada, sada možete da potaknete svoje promene na GitHub. Prvo ćete morati da postavite udaljenu konekciju do vašeg udaljenog repozitorijuma (repo). Naučili ste o ovome u prethodnoj lekciji, ali evo podsetnika korišćenjem SSH-a:

```sh
git remote add origin git@github.com:your-github-username/name-of-repo.git
```

Poslednji korak je da vaše promene pošaljete na GitHub koristeći ovu komandu:

```sh
git push -u origin main
```

Zastavica (flag) `-u` je skraćeni oblik za `--set-upstream`. Povezuje vašu lokalnu granu `main` sa udaljenom granom `main` na `origin`. Morate ga koristiti samo jednom po novoj grani. Nakon toga, možete jednostavno pokrenuti `git push` bez navođenja imena grane.

Kada je pravilno urađeno, trebalo bi da vidite sličan rezultat u terminalu:

```sh
Enumerating objects: 3, done.
Counting objects: 100% (3/3), done.
Delta compression using up to 8 threads
Compressing objects: 100% (2/2), done.
Writing objects: 100% (3/3), 253 bytes | 253.00 KiB/s, done.
Total 3 (delta 0), reused 0 (delta 0), pack-reused 0 (from 0)
To https://github.com/jdwilkin4/super-awesome-game.git
 * [new branch]      main -> main
branch 'main' set up to track 'origin/main'.
```

Takođe biste trebali da vidite nove promene na vašoj GitHub repozitorijum stranici kada osvježite stranicu.

Ako sarađujete sa drugima na projektu i morate da povučete nove promene iz repozitorijuma, možete koristiti komandu ``git pull``. Vi ćete saznati više o radu u timu i saradnji u budućoj lekciji.
