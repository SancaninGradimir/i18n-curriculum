# --description--

Ranije ste naučili kako da commiteš promene i pošalješ ih na svoj udaljeni repozitorijum, ali je sve to bilo na tvojoj podrazumevanoj grani ``main``. To može da funkcioniše za manji projekat, ali kako doprinosite drugim open source projektima, ili kako vaš projekat raste, grane su neophodne.

Na primer, ako vorkujete repozitorijum i napravite komitove na granu ``main`` vašeg forka, možete završiti sa promenama u svom forku koje originalni repozitorijum nije prihvatio. To može učiniti izuzetno teškim ažurirati vašu granu sa promenama iz forka koje su bile prihvaćene. Ili, kada radite na sopstvenom projektu, prebacivanje između različitih zadataka postaje mnogo teže ako ne koristite grane. Umesto toga, morate ručno pratiti koje promene se odnose na koju funkciju ili popravak.

Ali šta tačno je grana? Razmotrite svoju granu ``main`` kao auto-put, pri čemu je svaki commit automobil koji ulazi na auto-put. Možda odlučite da želite da pogledate nešto, poput kreiranja nove funkcije, pa vam je potrebno izaći sa auto-puta na neko vreme da biste otišli da vidite.

Grana je u suštini odstupanje od vaše glavne istorije, gde možete slobodno eksperimentisati sa promenama za novu funkciju ili popravak greške. Ako ste zadovoljni svojim promenama, možete se vratiti na glavni tok merdžujući granu u `main`. Ako niste zadovoljni svojim promenama, možete obrisati granu i vratiti se na `main` bez prihvatanja promena.

Pre nego što kreirate novu granu, pregledavate svoje grane sa komandom ``git branch``. Izlaz bi mogao izgledati ovako:

```sh
* main
```

Trenutno imamo samo jednu granu, naš podrazumevani `main` grana. Asterisk (`*`) znači da je to grana koju trenutno imate "izvučenu". Vi ćete više o tome saznati uskoro.

Možete kreirati novu granu sa komandom ``git branch``, sa argumentom za specifikovanje imena nove grane:

```sh
git branch feature
```

Ovo kreira novu granu pod nazivom `feature`. Pogledajmo ponovo naše grane sa `git branch`:

```sh
  feature
* main
```

Sada možemo videti našu novu granu, ali i dalje smo na grani `main`. Moramo da pređemo na našu novu granu tako što ćemo je "izvući". Kada izvučete novu granu, kažete Git-u da učita fajlove u tačnom stanju u kojem su bili u tom trenutku.

Pogledajmo našu novu granu sa:

```sh
git checkout feature
```

ili

```sh
git switch feature
```

Ispustite `git branch` ponovo da vidite:

```sh
* feature
  main
```

Primećujte da je znak asteriksa sada prešao na granu koju trenutno imate "izabranu".
 
Često kada kreirate novu granu, želite ćete je odmah aktivirati. Umesto da kreirate granu, a zatim je aktivirate koristeći dve komande, možete kreirati i prebaciti se na granu u jednoj komandi sa:

```sh
git checkout -b feature
```

ili

```sh
git switch -c feature
```

Tako sada smo kreirali i prebacili se na novu granu. Ova grana je klon `main` grane, ima sav isti kod koji je `main` imao u trenutku kada ste kreirali granu. Sada možemo dodati bilo koji kod na ovu granu bez uticaja na `main` granu. Suštinski smo napustili autoput da bismo kreirali funkciju. Ako proverite vašu `git status`, videli biste nešto slično ovome:

```sh
On branch feature
nothing to commit, working tree clean
```

Kreiraj novi `feature.md` fajl pokretanjem ovoga:

```sh
echo "This is our new feature" > feature.md
```

A sada će status izgledati ovako:

```sh
On branch feature
Untracked files:
  (use "git add <file>..." to include in what will be committed)
	feature.md

nothing added to commit but untracked files present (use "git add" to track)
```

Sada je vreme za komitovanje vaše nove funkcije. Prvo, dodajte fajl u stagišing:

```sh
git add feature.md
```

Zatim, sačuvajte promene:

```sh
git commit -m "my new feature"
```

Konačno, gurnite vašu `feature` granu u vaš repozitorijum:

```sh
git push -u origin feature
```

Flag `-u` povezuje vaš lokalni `feature` grana sa udaljenom granom, tako da se budući push-evi mogu izvršiti samo pomoću `git push`.

Zatim ćemo naučiti kako da napravimo pull request za našu novu funkcionalnost!
