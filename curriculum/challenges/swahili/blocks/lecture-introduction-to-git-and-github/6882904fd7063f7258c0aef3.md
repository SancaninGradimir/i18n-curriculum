# --description--

Pre nego što možete da pokrenete svoj repozitorijum na GitHub, moraćete da budete autentifikovani. I SSH i GPG ključevi koriste javno-privatni par. To znači da čuvate privatni ključ na svojoj lokalnoj mašini, koji koristite za autentifikaciju sebe. Delite javni ključ sa ljudima ili uslugama koje moraju da validiraju vašu autorizaciju. Drugim rečima, privatni ključ je potreban za izvršavanje akcije, a javni ključ se koristi za verifikaciju te akcije.

Ključevi GPG-a, ili Gnu Privacy Guarda, obično se koriste za potpisivanje fajlova ili commit-ova. Neko zatim može koristiti vaš javni GPG ključ da verifikuje da li je potpis fajla iz vašeg ključa i da li sadržaj fajla nije modifikovan ili narušen. Ključevi SSH-a, ili Secure SHElla, obično se koriste za autentifikaciju udaljene veze ka serveru - putem alata `ssh`. Međutim, kako ćete naučiti u ovoj lekciji, možete koristiti i SSH ključ za potpisivanje commit-ova.

Da biste generisali GPG ključ, morate pokrenuti:

```sh
gpg --full-generate-key
```

Za SSH ključ, pokovaćeš:

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Uverite se da navedete vaš pravi e-mail.

Ove dve komande bi trebalo da zahtevaju frazu za reči radi obezbeđivanja ključa. Ovo je opcionalno, ali se snažno preporučuje.

Za SSH ključeve, dobićete dva fajla – jedan za vaš privatni ključ, a drugi za javni. Imena fajlova javnog ključa će se završavati sa `.pub`, tako da slučajno ne podelite svoj privatni ključ. Ovi fajlovi će biti u `~/.ssh`.

Ključevi GPG, međutim, biće u skrivenom i šifrovanom direktorijumu. Umesto toga, da biste naveli svoje javne ključeve, pokovaćete:

```sh
gpg --list-secret-keys --keyid-format=long
```

Zatim, da biste dobili javni ključ, koristite:

```sh
gpg --armor --export "<key id>"
```

Ali šta možete da uradite sa ovim novim ključevima? Pa, pričali smo o potpisivanju vaših commit-ova, i to je nešto za šta možete koristiti bilo koji od ovih ključeva.

Da biste potpisali svoje commit-ove sa vašim GPG ključem, moraćete da učitate svoj javni ključ, a ne privatni ključ, na svoj GitHub nalog. Zatim, uzmite kratki ID koji ste dobili prilikom spravljanja ključeva ranije, i pokrenite ovu komandu da ga postavite kao vaš git signing ključ:

```sh
git config --global user.signingkey <your_gpg_key_id>
```

Zatim, možete proslediti flag `-S` vašoj komandi `git commit` da potpišete određeni commit – biće vam potrebno da unesete vašu lozinku (passphrase). Alternativno, ako želite automatski potpisati svaki commit, možete postaviti autosign konfiguraciju na `true`:

```sh
git config --global commit.gpgsign true
```

Da biste potpisivali sa SSH ključem, što je relativno nova funkcija na GitHubu, moraćete da počnete postavljanjem ključa na svoj GitHub nalog. Zatim ćete morati da podesite režim potpisivanja za Git da koristi SSH:

```sh
git config --global gpg.format ssh
```

Zatim, da postavite ključ za potpisivanje, ćete proslediti putanju do fajla umesto ID-a:

```sh
git config --global user.signingkey <path_to_your_ssh_keys>
```

Konačno, možete omogućiti automatsko potpisivanje kao što biste to uradili za GPG ključ.

Sa podešenjem bilo kog od ovih pristupa, sada ste spremni da imate verifikovane commit-ove koji će dobiti posebnu oznaku na GitHubu.
