# --description--

Bubbleovanje događaja, ili propagacija, odnosi se na to kako događaj "penje" do roditeljskih objekata kada se aktivira. Na primer, razmotrite ovaj kod:

```html
<p>
  <span>Click me~!</span>
</p>
```

Element `p` ovde bi se smatrao roditeljem elementa `span`.

Kada kliknete na element `span`, kao što ste upućeni, element `span` postaje cilj događaja `click`. Taj događaj, međutim, takođe se širi do roditelja – element `p` može primiti i konzumirati taj događaj po potrebi.

Ali šta ovo zapravo znači? Pa, mogli biste priložiti slušalicu događaja na element `p`:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  console.log(event.target);
});
```

Zatim, kada kliknete na element `span`, videćete tekst `Click me~!` logovan u konzolu.

Događaj se propagira na roditeljski element ``p``, koji ga konzumira u slušaocu događaja kako bi prikazao cilj događaja.

Primetite kako je cilj i dalje element `span`. To je zato što je element `span` primio početni klik.

Da bismo bili sigurni kako sve radi, hajde da proširimo naš kod:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
});
```

Da biste dobili ideju kako se događaj širi, evo šta ćete videti u konzoli nakon klika na element `span`:

```html
"Span listener: "
<span>Click me~!</span>
"P listener: "
<span>Click me~!</span>
```

Sada hajde da vidimo šta se dešava kada sprečite propagaciju događaja sa `stopPropagation()`. Pozvaćemo ga u slušaoc događaja našeg `span`:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
  event.stopPropagation();
});
```

A zatim kliknite na naš `span` ponovo:

```html
"Span listener: "
<span>Click me~!</span>
```

Ovog puta, ne vidimo našu `p` slušalicu koja se aktivira. Događaj nikada ne izaziva za element `p`, jer smo mu rekli da zaustavi propagaciju dok se obrađuje za element deteta `span`.

Delegacija događaja se može smatrati suprotnim. To je proces uzimanja uhvaćenog događaja i delegiranja ga drugom elementu.

Vraćajući se našem kodu, ukrasit ćemo ga tako da klikatanje na ``span`` element promeni boju u crvenu:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {});
span.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

Ali šta ako imate dvadeset `span` elemenata? Ili možda koristite JavaScript da kreirate još `span` elemenata na letu?

Umesto da kačite slušalicu događaja na svaki pojedinačni `span` element, zapravo možete koristiti slušalicu na `p` element za sve njih. Drugim rečima, možete delegirati rukovanje klikovima na `span` roditeljski `p` element.

Naš kod sada bi mogao izgledati otprilike ovako:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

Primećujete kako više ne imamo nijednog slušaoca vezanog za element `span` uopšte. Ispravno ste delegirali rukovanje događajima na element `p`. Ali da li radi?

Hajde da generišemo nekoliko dodatnih `span` elemenata i vidimo:

```html
<p>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
</p>
```

Sada, svaki put kada kliknemo na `span`, tekst tog elementa će postati crven.

I samo tako, sa jednim slušaocem događaja ispravno smo dozvolili da se `click` događaj propagira od elemenata `span` do roditeljskog `p`, i delegirali logiku za taj `click` događaj elementu `p`.

Propagacija događaja i delegacija mogu biti složena tema, posebno kada uđete u elemente sa mnogo ugnježdenih struktura kao što su tabele. Preporučuje se da istražite ovo dalje i eksperimentišete sa nekim od koda koji smo ovde napisali.
