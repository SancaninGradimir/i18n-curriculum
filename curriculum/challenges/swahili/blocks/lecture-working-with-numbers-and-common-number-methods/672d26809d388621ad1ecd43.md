# --interactive--

U JavaScriptu, ``NaN`` označava "Not a Number". To je posebna vrednost koja predstavlja nepredstavljiv ili nedefinisani numerički rezultat. ``NaN`` je svojstvo globalnog objekta, i takođe se smatra tipom broja u JavaScriptu, što na prvi pogled može delovati kontraintuitivno.

`NaN` je tipično rezultat operacija koje bi trebalo da vrate broj, ali ne mogu proizvesti smislenu numeričku vrednost. Na primer:

:::interactive_editor

```js
let result = 0 / 0;
console.log(result); // NaN
```

:::

U ovom slučaju, deljenje nule sa nulom je matematički nedefinisano, pa JavaScript vraća `NaN`. Jedno neobično svojstvo od `NaN` je da nije jednako ničemu, uključujući i sebe:

:::interactive_editor

```js
console.log(NaN === NaN); // false
```

:::

Ovo jedinstveno ponašanje čini teškim proveru da li je vrednost `NaN` korišćenjem standardnih operatora za poređenje. Da bi se rešio ovo pitanje, JavaScript pruža funkciju `isNaN()`. Svojstvo funkcije `isNaN()` koristi se za određivanje da li je vrednost `NaN` ili ne. Međutim, važno je razumeti kako radi `isNaN()`, jer ponekad može proizvesti neočekivane rezultate. Evo kako se `isNaN()` ponaša:

:::interactive_editor

```js
console.log(isNaN(NaN));       // true
console.log(isNaN(undefined)); // true
console.log(isNaN({}));        // true

console.log(isNaN(true));      // false
console.log(isNaN(null));      // false
console.log(isNaN(37));        // false

console.log(isNaN("37"));      // false: "37" is converted to 37
console.log(isNaN("37.37"));   // false: "37.37" is converted to 37.37
console.log(isNaN(""));        // false: empty string is converted to 0
console.log(isNaN(" "));       // false: string with a space is converted to 0

console.log(isNaN("blabla"));  // true: "blabla" is not a number
```

:::

Kao što vidite, `isNaN()` prvo pokušava da konvertuje parametar u broj. Ako se ne može konvertovati, vraća `true`. Ovo ponašanje može dovesti do nekih iznenađujućih rezultata, posebno kada se radi sa nizovima koji mogu biti prisiljeni u brojeve.

Zbog ovih potencijalnih neusklađenosti, ES6 (šesta edicija JavaScript-a, objavljena 2015.) uveo je `Number.isNaN()`. Ova metoda ne pokušava da konvertuje parametar u broj pre testiranja. Vraća samo `true` ako je vrednost tačno `NaN`:

:::interactive_editor

```js
console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN(Number.NaN)); // true
console.log(Number.isNaN(0 / 0));      // true

console.log(Number.isNaN("NaN"));      // false
console.log(Number.isNaN(undefined));  // false
console.log(Number.isNaN({}));         // false
console.log(Number.isNaN("blabla"));   // false
```

:::

`Number.isNaN()` nudi pouzdaniji način za proveru vrednosti `NaN`, posebno u slučajevima gde koercija tipa može dovesti do neočekivanih rezultata sa globalnom funkcijom `isNaN()`. U praksi, kada se radi sa numeričkim operacijama ili unosima korisnika koji bi trebalo da budu brojevi, često je neophodno proveriti `NaN` kako bi se greške ili neočekivani unosi adekvatno obradili. Na primer:

:::interactive_editor

```js
let a = 0;
let b = 0;
let result = a / b;

if (Number.isNaN(result)) {
  result = "Error: Division resulted in NaN";
}

console.log(result); // "Error: Division resulted in NaN"
```

:::

U ovom primeru, koristimo `Number.isNaN()` da uhvatimo slučajeve gde operacija deljenja rezultira u `NaN`, što nam omogućava odgovarajuću obradu ovog scenarija. Razumevanje `NaN` i kako pravilno proveriti ga ključno je za pisanje robusnog JavaScript koda, posebno kada se radi o matematičkim operacijama ili parsiranju unosa korisnika.
