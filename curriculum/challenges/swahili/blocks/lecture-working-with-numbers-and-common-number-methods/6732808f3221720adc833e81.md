# --interactive--

`parseFloat()` i `parseInt()` su dva esencijalna metoda u JavaScript-u za konvertovanje stringova u brojeve. Ovi metodi su posebno korisni prilikom rukovanja unosom korisnika ili obrade podataka koji dolaze u formatu niza, ali treba da se tretiraju kao numeričke vrednosti.

Počinjemo sa `parseFloat()`. Ova metoda parsira string argument i vraća broj sa plutajućom zarezom. Dizajnirana je da izvadi broj sa početka stringa, čak i ako string kasnije sadrži nenumeričke karaktere. Zapamtite da su float brojevi sa decimalnim tačkama. Evo kako radi `parseFloat()`:

:::interactive_editor

```js
console.log(parseFloat("3.14"));     // 3.14
console.log(parseFloat("3.14 abc")); // 3.14
console.log(parseFloat("3.14.5"));   // 3.14
console.log(parseFloat("abc 3.14")); // NaN
```

:::

Kao što možete da vidite, `parseFloat()` počinje parsiranje od početka niza i nastavlja sve dok ne naiđe na karakter koji ne može biti deo broja sa decimalnim delom. Ako ne može da pronađe validan broj na početku niza, vraća `NaN` (Not a Number).

`parseInt()`, s druge strane, parsira string argument i vraća ceo broj. Kao `parseFloat()`, počinje od početka stringa, ali zaustavlja se na prvom karakteru koji nije cifra. Evo kako funkcioniše `parseInt()`:

:::interactive_editor

```js
console.log(parseInt("42"));       // 42
console.log(parseInt("42px"));     // 42
console.log(parseInt("3.14"));     // 3
console.log(parseInt("abc123"));   // NaN
```

:::

`parseInt()` zaustavlja parsiranje na prvoj cifri koja nije cifra koju naiđe. Za brojeve sa decimalnim delom, vraća samo ceo deo. Ako ne može da pronađe validan ceo broj na početku niza, vraća `NaN`.

Oba metoda imaju neko primetno ponašanje. Ignorišu vodeće razmake:

:::interactive_editor

```js
console.log(parseFloat("  3.14"));  // 3.14
console.log(parseInt("  42"));      // 42
```

:::

Oni obrađuju znake plusa i minusa na početku niza:

:::interactive_editor

```js
console.log(parseFloat("+3.14"));  // 3.14
console.log(parseInt("-42"));      // -42
```

:::

Vredi napomenuti da su iako ove metode moćne, imaju neka ograničenja. Na primer, ne obrađuju sve formate brojeva, kao što je naučna notacija, direktno. Za složenije potrebe za parsiranjem, možda ćete morati da koristite dodatne tehnike ili biblioteke.

Zaključno, `parseFloat()` i `parseInt()` su dragoceni alati za konverziju stringova u brojeve u JavaScriptu. Razumevanje kako funkcionišu i njihovih specifičnih ponašanja omogućava vam da efikasnije rukujete numeričkim podacima u svojim aplikacijama, posebno kada se radi o unosu korisnika ili eksternim izvorima podataka.
