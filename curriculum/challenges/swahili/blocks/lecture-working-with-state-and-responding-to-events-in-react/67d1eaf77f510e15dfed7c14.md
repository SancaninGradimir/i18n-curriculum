# --description--

Stanje je jedan od najvažnijih fundamentalnih aspekata u Reactu i drugim front-end okvirima. Liči na mozak komponente, što znači da čuva informacije koje se mogu menjati tokom vremena i kontroliše kako komponente funkcionišu i izgledaju.

Pogledajmo kakvo je stanje i kako vam `useState` hook omogućava da radite sa njim.

Stanje predstavlja dinamičke podatke u vašem React komponentu, kao što je vrednost iz korisničkog unosa, podaci dohvaćeni sa API-ja ili stavka u listi zadataka.

Kada god se stanje promeni, React ponovo renderuje komponentu bez osvežavanja stranice kako bi odražavao te promene u korisničkom interfejsu. Ova reaktivnost čini vašu aplikaciju interaktivnom.

``useState`` hook je funkcija koja vam omogućava da deklarišete varijable stanja u funkcionalnim komponentama.

Pre hookova, mogli ste koristiti stanje samo u klasnim komponentama. Ali sa uvođenjem hookova od Reacta 16.8, možete koristiti stanje u funkcionalnim komponentama korišćenjem hook-a ``useState``.

TEKST:

Da biste koristili `useState` kuku, morate ga uvesti iz React-a:

```js
import { useState } from "react";
```

Takođe možete uvesti sam React i dobiti pristup hook-u `useState` kao svojstvu:

```js
import React from "react";
```

Evo kako možete deklarisati varijablu stanja kada uvezete `useState`:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

Evo kako možete deklarisati varijablu stanja kada uvezite React:

```js
const [stateVariable, setStateFunction] = React.useState(initialValue);
```

U varijabli stanja imate sledeće:

- `stateVariable` drži trenutnu vrednost stanja
- `setStateFunction` (funkcija za postavljanje) ažurira varijablu stanja
- `initialValue` postavlja početno stanje

Napomena da je stanje u React komponenti privatno i izolovan za svaku instancu komponente. To znači da, ako renderujete istu komponentu dva puta, stanje jedne komponente ne utiče na drugu. To takođe znači da, ako želite da podelite stanje između komponenti, moraćete da podignete (lift) to stanje do zajedničkog roditelja i prosledite ga kao propse.

Još jedna stvar je da se hookovi moraju pozvati na najvišem nivou komponente, odmah pre ključne reči ``return``, kako bi stanje i efekti ostali konzistentni tokom renderovanja. To znači da ne možete koristiti stanje unutar petlji, uslova ili ugnježdene funkcije.

Evo primera upravljanja stanjem sa pomoću `useState` hooka u `Counter` komponenti:

```jsx
// Importing the useState hook
import { useState } from "react";

function Counter() {
  const initialValue = 0;

  // The state variable and setter function
  const [count, setCount] = useState(initialValue);

  return (
    <div>
      {/* Display current state value */}
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

U kodu iznad, imamo hook `useState` uvozen od React-a. U komponenti `Counter`, `count` predstavlja trenutno stanje dok je `setCount` funkcija za postavljanje odgovorna za ažuriranje stanja. Trenutna vrednost stanja je `0`. Izjava `return` sadrži brojač i dva dugmeta za smanjenje i povećanje broja sa `1`.

Možete upravljati više delova stanja pozivanjem kuka ``useState`` više puta. Ovo je posebno važno kada imate nepovezane varijable stanja:

```jsx
function UserProfile() {
  const [isOnline, setIsOnline] = useState(false);
  const [notifications, setNotifications] = useState(0);

  // The rest of the component logic
}
```

Takođe možete pozvati kuku `useState` više puta prilikom upravljanja više stanja koja se ažuriraju zasebno, kao što su polja formulara:

```jsx
function SignUpForm() {
  const [name, setName] = useState("");
  const [username, setUsername] = useState("");
  const [email, setEmail] = useState("");

  // The rest of the component logic
}
```

Ali u ovom slučaju, najbolje je kombinovati stanja jer su sva deo istog obrasca:

```jsx
function SignUpForm() {
  const [formData, setFormData] = useState({
    name: "",
    username: "",
    email: "",
  });

  // The rest of the component logic
}
```

To je šta je stanje i kako možete koristiti kuka ``useState``.
