# --description--

Ažuriranje objekata u stanju u Reactu može biti komplikovano ako ste navikli na direktno menjanje vrednosti svojstava objekta.

React tretira stanje kao nepromenljivo, što znači da ga ne biste trebali direktno modifikovati.

Pogledaćemo šta se dešava ako pokušate direktno da promenite objekte u React stanju, a zatim ćemo zaroniti u ispravan način za to.

Recimo da imate objekat u stanju vaše komponente koji predstavlja profil korisnika, i želite da korisnik ažurira svoju starosnu godine:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({
    name: "John Doe",
    age: 31,
    city: "LA",
  });

  // Change user age directly
  const handleAgeChange = (e) => {
    user.age = e.target.value;
    console.log(user);
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" value={user.age} onChange={handleAgeChange} />
    </div>
  );
}

export default Profile;
```

Ovaj kod neće raditi jer direktno modifikujete svojstvo `user.age`.

Iako će `console.log(user)` prikazati novu starost u konzoli, React neće ponovo renderovati komponentu da bi je prikazao u korisničkom interfejsu jer niste koristili funkciju za postavljanje (setter funkciju), `setUser`.

Da biste direktno ažurirali objekat u stanju, morate koristiti funkciju za postavljanje kako biste kreirali novi objekat sa ažuriranom vrednošću. Primer:

```js
const handleAgeChange = (e) => {
  setUser({
    age: e.target.value,
  });
};
```

To radi. Ali ako pogledate stranicu sada, uzrast korisnika se ažurira, ali vrednosti za ime i grad su izgubljene.

Ovo je zato što novi objekat koji ste prosledili setter funkciji sadržao je samo par ključ/vrednost za `age`.

Da biste sprečili da se to dogodi, prvo možete kopirati postojeći objekat, a zatim ažurirati samo svojstvo koje želite da ažurirate, u ovom slučaju, ``age``.

Da biste ovo uradili, možete proslediti posebnu funkciju nazvanu funkcija za ažuriranje vašoj funkciji za postavljanje (setter function), ``setUser``. Funkcija za ažuriranje prima stanje čekanja kao argument, ovde nazvano ``prevUser``, i treba da vrati sledeće stanje:

```js
const handleAgeChange = (e) => {
  setUser((prevUser) => {
    const updatedUser = { ...prevUser, age: e.target.value };
    console.log('Previous State:', prevUser);
    console.log('Updated State:', updatedUser);
    return updatedUser;
  });
};
```

Kao što vidite, kreiramo novi objekat korisnika nazvan `updatedUser` korišćenjem spread sintakse za kopiranje čekajućeg objekta korisnika, `...prevUser`. Zatim ažuriramo godine na osnovu unosa forme i vraćamo `updatedUser` na dnu funkcije kao sledeće stanje.

Sada vaš projekat radi kako je očekivano, a ažuriranja unosa starosti ne utiču na ime korisnika niti na naziv grada. Takođe možete videti prethodno i ažurirano stanje u konzoli.

Ovo je idealan način za ažuriranje objekta u stanju, posebno kada ne ažurirate sva svojstva.

Da biste ažurirali preostala svojstva `name` i `city`, možete ih napisati kao zasežne funkcije za podešavanje i povezati ih na njihove odgovarajuće ulaze:

```js
const handleNameChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    name: e.target.value,
  }));
};

const handleCityChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    city: e.target.value,
  }));
};
```

Ili ih možete kombinovati u jednu funkciju za postavljanje (setter) na ovaj način:

```js
const handleChange = (e) => {
  const { name, value } = e.target;
  setUser((prevUser) => ({
    ...prevUser,
    [name]: value,
  }));
};
```

Da bi ovo funkcionisalo, svako polje za unos mora imati atribut `name`.

Evo celog koda sada:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```
