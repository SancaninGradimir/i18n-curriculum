# --description--

U Reactu, renderovanje je proces kojim se komponente pojavljuju u korisničkom interfejsu (UI), obično u pretraživaču.

React uzima sav vaš JavaScript, JSX i CSS kod, utvrđuje kako bi trebalo da izgleda, a zatim ga prikazuje u korisničkom interfejsu.

Proces renderovanja sastoji se od tri faze: pokretač (trigger), renderovanje (render) i komitovanje (commit). Pogledajmo ih detaljnije.

Faza okidača se dešava kada React detektuje da se nešto promenilo i da je korisnički interfejs možda potreban za ažuriranje. Ova promena je često uzrokovana ažuriranjem stanja ili propova.

Na primer, primetiti da je vreme za večeru može vas navesti da odete u kuhinju da počnete sa kuvarstvom.

U primeru `Counter` ispod, klikatanje na dugme za povećanje ili smanjenje pokreće React da prikaže novu vrednost:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

Kada se okidač desi, React ulazi u fazu renderovanja. Ovde, React ponovo procenjuje vaše komponente i utvrđuje šta da prikaže.

Da bi to uradio, React koristi laganu kopiju "pravog" DOM-a. Ovo se naziva virtuelni DOM. Sa virtuelnim DOM-om, React može brzo proveriti šta treba da se promeni u komponenti.

Zamislite ovu fazu kao tačku gde ste u kuhinji, skupili sastojke i kuvate svoju večeru.

Za `Counter` komponentu, faza rendera je tačka gde React ponovo pokreće funkcije sa novom vrednošću `count`. React ponovno izračunava kako bi trebalo da izgleda deo komponente `<h1>{count}</h1>` na osnovu ažurirane vrednosti `count`, ali nećete videti nikakve promene na ekranu sve dok ne dođe sledeća faza – commit.

Faza commita je mesto gde React uzima pripremljene promene iz virtuelnog DOM-a i primenjuje ih na pravi DOM. Drugim rečima, ovo je faza gde vidite konačan rezultat na ekranu.

Da bi se ovo dogodilo, React upoređuje virtuelni DOM sa stvarnim DOM-om, identifikuje samo delove koji zahtevaju ažuriranje, i primenjuje te promene na stvarni DOM kako bi ažurirao korisnički interfejs.

Možete smatrati ovu fazu tačkom gde poslužujete hranu koju ste kuvali, čime je činite vidljivom baš kao što to React radi kada izvrši ažuriranja na stvarni DOM.

Što se tiče `Counter` komponente, faza commita je tačka u kojoj se nova vrednost `count` primenjuje na element `h1`, i možete videti promenu na stranici.

Ova tri procesa su izuzetno brza jer React minimizira direktnu manipulaciju DOM-om tako što prvo izračunava promene u virtuelnom DOM-u, a zatim ažurira samo delove koji treba da se promene u stvarnom DOM-u.
