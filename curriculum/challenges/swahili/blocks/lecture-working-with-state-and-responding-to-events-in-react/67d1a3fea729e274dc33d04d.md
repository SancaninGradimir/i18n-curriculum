# --description--

Обрада догађаја је кључни део сваког интерактивног веб сајта.

React pruža moćan i dosledan način za rukovanje događajima putem svog Sintetičkog sistema događaja (Synthetic Event System), koji je omotač oko naivnih događaja kao što su `click`, `keydown` i `submit`, o kojima ste naučili u ranijim lekcijama.

Ovaj omotač za više pregledača osigurava da događaji rade isto u svim pregledačima, pa nema neslaganja.

Pogledaćemo kako događaji rade u Reactu kako biste počeli da ih koristite u svojim projektima.

U Reactu, ručkici za događaje rade na sličan način kao i na izvornim pretraživačkim događajima, ali sa nekoliko izmena.

Umesto korišćenja malonakonskih imena atributa događaja kao što su ``onclick`` i ``onsubmit``, React koristi camelCase, kao što su ``onClick`` i ``onSubmit``.

Pored toga, umesto korišćenja stringova za specifikovanje vrste događaja, React očekuje funkciju za rukovanje događajem.

Funkcija rukovatelja događajem se prosleđuje elementu kao prop, i tip događaja poput `onClick` ili `onSubmit` koristi se kao atribut u JSX-u.

Evo podsetnika kako raditi sa klik događajem u običnom HTML-u:

```html
<button onclick="alert('Button clicked!')">Click Me</button>
```

I evo kako da uradite isto u Reactu:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

U ovom primeru, `handleClick` loguje poruku na konzolu kada korisnik klikne na dugme.

Napomena da vam ne trebaju zagrade nakon `handleClick` u atributu `onClick`, jer prosleđujete referencu na funkciju, a ne pozivate je.

U Reactu, funkcije za rukovanje događajima obično počinju sa prefiksom `handle` da bi ukazale da su odgovorne za rukovanje događajima, kao što su `handleClick` ili `handleSubmit`.

Kada korisnička akcija pokrene događaj, React prosleđuje objekat Sintetičkog Događaja vašem rukovaocu. Ovaj objekat se ponaša veoma slično nativnom objektu događaja u čistom JavaScript-u, pružajući svojstva kao što su `type`, `target` i `currentTarget`.

Možete proslediti ``event`` kao parametar funkciji rukovatelju i logovati ga u konzolu da biste pogledali njegovu strukturu:

```js
function handleClick(event) {
  console.log(event);
}
```

Da biste sprečili podrazumevane ponašanja poput osvežavanja pregledača tokom događaja `onSubmit`, na primer, možete pozvati metodu `preventDefault()`:

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

Takođe možete zaustaviti događaj od toga da se propagira do roditeljskih elemenata pozivanjem `event.stopPropagation()`.

Ponekad, dok obrađujete posebne slučajeve kao što su funkcije za brisanje i uređivanje, možda želite da prosledite dodatne podatke hendleru događaja. To možete učiniti tako što ćete hendlera umotati u inline strelicastu funkciju:

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

U redu je koristiti ugrađene hendlere događaja u Reactu jer React efikasno upravlja ponovnim renderovanjem i izbegava probleme sa performansama.

U čistom JavaScriptu, inline rukovači događajem mogu dovesti do problema sa performansama kreiranjem novih funkcija pri svakom renderovanju, jer ne postoji virtuelni DOM da optimizuje proces.
