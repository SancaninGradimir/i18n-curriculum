# --description--

U ovoj lekciji ćete naučiti kako da radite sa tipovima funkcija. Počnimo sa anotacijama tipa parametara.

Evo primera osnovne funkcije:

```ts
function circleArea(radius) {
  return Math.PI * radius * radius;
}
```

Trenutno, TypeScript bi prikazao poruku: `Parametar 'radius' implicitno ima tip 'any'`. Tip `any` ukazuje da vrednost može imati bilo koji tip. Vi ćete više naučiti o ovom tipu u budućoj lekciji.

Da biste se rešili greške i eksplicitno definisali tip parametra, možete ga definisati ovako:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}
```

Ako pokušate da pozovete funkciju `circleArea` sa različitim tipom koji nije broj, TypeScript će prikazati grešku ovako:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Argument of type 'string' is not assignable to parameter of type 'number'
circleArea("3");
```

Ako pokušate da pružite više argumenata nego što funkcija očekuje, TypeScript će prikazati grešku:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 2.
circleArea(3, 4);
```

Ako pružite premalo argumenata, TypeScript će takođe prikazati grešku:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 0.
// An argument for 'radius' was not provided.
circleArea();
```

Možete učiniti parametre opcionalnim koristeći simbol `?` pored parametra, ovako:

```ts
function area(radius: number, height?: number): number {
  const baseArea = Math.PI * radius * radius;

  if (height !== undefined) {
    const sideArea = 2 * Math.PI * radius * height;
    return baseArea + sideArea;
  }

  return baseArea;
}
```

Imajte na umu da opcioni parametri uvek moraju dolaziti nakon obaveznih parametara. Postavljanje obaveznog parametra nakon opcionog rezultiraće greškom TypeScript-a.

Uslovna provera unutar funkcije postoji u slučaju da argument `height` nije dostavljen. 

Ako želite da tipizujete svoje povratne vrednosti, možete dodati anotacije tipa za povratnu vrednost ovako:

```ts
function circleArea(radius: number): number {
  return Math.PI * radius * radius;
}
```

Anotacija tipa za povratnu vrednost dolazi nakon liste parametara. Slično anotacijama tipa promenljivih, većinom vremena TypeScript će biti dovoljno pametan da izvede (inferira) tip povratne vrednosti. Zato koristite eksplicitnu anotaciju tipa za povratnu vrednost kada želite da učinite nameru funkcije jasnijom ili sprečite neželjene promene povratne vrednosti.

Sledeći primer na koji ćemo pogledati baviće se asinhronim funkcijama i obećanjima (promises). Ako vaša funkcija treba da vrati obećanje, možete koristiti tip `Promise` ovako:

```ts
async function getCityTemperature(city: string): Promise<number> {
  const response = await fetch(`https://api.example.com/weather?city=${city}`);
  const data = await response.json(); 
  return data.temperature;
}
```

U ovoj situaciji, ako pokušate da uklonite tip za povratnu vrednost, TypeScript bi mogao da izvede povratnu vrednost kao `Promise<any>`. Zato bi ovo bila dobra situacija za eksplicitno tipizovanje povratne vrednosti.

Do ovog trenutka, pogledali smo samo primere funkcija sa imenom. Ali kako tipovi rade sa anonimnim funkcijama?

Evo primera korišćenja anonimne funkcije u povratnoj funkciji (callback) prosleđenoj `forEach`-u:

```ts
const languages = ["JavaScript", "Python", "TypeScript"];

languages.forEach((lang) => {
  console.log(`${lang.toUpperCase()} has ${lang.length} characters`);
});
```

Parametar `lang` nema anotaciju tipa. Umesto toga, TypeScript koristi kontekstualno tipizovanje (contextual typing), što znači da izvede tip iz konteksta u kojem se pojavljuje.
