# --description--

JavaScript se smatra dinamički tipiziranim jezikom. To znači da varijable mogu primiti bilo koje vrednosti u vreme izvršavanja – deklarisanje varijable kao broja ne sprečava vas od dodeljivanja joj vrednost tipa string kasnije, a funkcionim parametrima se može proslediti bilo koja vrednost.

Izazov jezika dinamičkog tipiziranja je što nedostatak sigurnosti tipa može uvesti greške ako niste pažljivi. TypeScript proširuje jezik JavaScript da bi uključio statičko tipiziranje, što pomaže u hvatanju tih grešaka čak i pre nego što deployujete svoj kod. Ali kako to radi?

Razmotrite ovu JavaScript funkciju:

```js
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Naša funkcija `getRandomValue` prima niz i vraća nasumičnu vrednost iz tog niza. Ali pošto JavaScript ne validira tipove, ništa vas ne sprečava da pozovete funkciju sa brojem:

```js
console.log(getRandomValue(10));
```

Izlaz konzole za trenutni primer će vratiti ``undefined`` jer je očekivao niz umesto broja.

Ali sa TypeScript-om možete definisati tip za niz parametar:

```js
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Ova definicija tipa govori TypeScriptu da argument niza mora biti niz stringova. Zatim kada pozovete `getRandomValue` i prosledite mu broj, dobijate drugačiji tip greške nazvan greška kompajlera.

Većina JavaScript okruženja za izvršavanje, kao što su pretraživač ili Node.js, ne može pokrenuti TypeScript na izvorni način. Umesto toga, prvo morate kompajlirati, ili prevesti, TypeScript kod u običan JavaScript. To možete učiniti pomoću ugrađenog kompajlera koji dolazi sa TypeScript jezikom. Kada pokrenete kompajler, TypeScript će evaluirati vaš kod i baciti grešku za bilo kakve probleme gde tipovi ne odgovaraju – kao što je prosleđivanje broja u funkciju koja očekuje niz.

Naučićete više o tome kako ove vrste funkcionišu u sledećim lekcijama. Ali upravo ova dodatna sigurnost čini TypeScript privlačnim jezikom za mnoge programere i organizacije.
