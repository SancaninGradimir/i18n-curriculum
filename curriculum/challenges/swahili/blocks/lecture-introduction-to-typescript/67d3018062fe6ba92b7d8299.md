# --description--

U ovoj lekciji, naučićete kako da radite sa osnovnim primitivnim tipovima i kako funkcionišu anotacije tipova za varijable.

Za primitive tipove podataka ``string``, ``null``, ``undefined``, ``number``, ``boolean``, i ``bigint``, TypeScript nudi odgovarajuće ključne reči tipova.

Za ovu lekciju, fokusiraćemo se na uobičajeno korišćene primitive i neće ući u previše detalja za `bigint`. Tip podataka `bigint` je numerički tip podataka u JavaScriptu za predstavljanje brojeva većih od standardnog celobrojnog tipa.

Pogledajmo svaki od ovih primitivnih tipova, počevši sa `string`. Evo kako možete definisati niz znakova u TypeScriptu:

```ts
let firstName: string = "Angie";
```

Za razliku od čistog JavaScripta, varijabla `firstName` očekuje tip `string`. Dakle, ako pokušate da je ponovo dodelite vrednošću broja, dobićete grešku kao što je ovo:

```ts
let firstName: string = "Angie";
firstName = 9 // Type 'number' is not assignable to type 'string'.
```

Sledeći tip na koji ćemo pogledati je tip `number`. Evo primera:

```ts
let age: number = 16;
```

Možete dodeliti cele brojeve i floatove tipu `number`. Ali ako pokušate da ponovo dodelite vrednost koja je tipa `bigint`, dobićete grešku kao što je ova:

```ts
let age: number = 16;

// Type 'bigint' is not assignable to type 'number'.
age = 123n;
```

Sledeći tip koji ćemo pogledati je `boolean` tip. Evo nekih primera:

```ts
let isLoggedIn: boolean = true;
let isAdmin: boolean = false;
```

Poslednje dve vrste su tipovi ``null`` i ``undefined``.

Kao što ste se setili iz ranijih lekcija o JavaScriptu, `null` predstavlja namerno odsustvo vrednosti. `undefined` znači da je varijabla deklarisana, ali joj nije dodeljena vrednost.

Videćete primere kako da radite sa ovim tipovima kasnije kada naučite o funkcionim tipovima i tipovima unije.

Sada kada ste videli kako tipne anotacije rade sa varijablama, postavlja se pitanje da li je potrebno uvek eksplicitno definisati vaše varijable.

Ako imate jednostavnu tipsku anotaciju kao ovo, onda možda nećete morati da koristite eksplicitni tip.

```ts
let firstName: string = "Angie";
```

Razlog je taj što će TypeScript pokušati automatski da izvede tipove na osnovu njegove vrednosti. Dakle, ako ovo napišete, TypeScript će razumeti da je to string.

```js
let firstName = "Angie";
```

Trebalo bi da eksplicitno tipizirate svoje varijable kada TypeScript ne može jasno izvesti tip ili kada želite da nametnete određeni tip za situacije kao što su parametri funkcija ili složeni objekti.
