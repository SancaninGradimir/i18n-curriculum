# --description--

U ovoj lekciji, naučićete kako da dodate tipove za objekte i nizove.

Prvo ćemo pogledati nizove.

Evo primera niza programskih jezika:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

Ako želite eksplicitno definisati taj niz kao niz stringova, onda ga možete definisati kao `string[]`:

```ts
const programmingLanguages:string[] = ["Java", "C++", "Python"];
```

Kao što ste ranije naučili, TypeScript će biti dovoljno pametan da izvede tip kao niz stringova ako samo napišete ovo:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

Ali eksplicitno njegovo kucanje će pomoći u situacijama ako pokušate dodati drugi tip u niz na ovaj način:

```ts
// Type 'number' is not assignable to type 'string'.(2322)
const programmingLanguages:string[] = ["Java", 9, "Python"];
```

Drugi način za tipiziranje nizova je korišćenje ove sintakse:

```ts
const programmingLanguages:Array<string> = ["Java", "C++", "Python"];
```

U suštini, ovi dva sintaksa su isti, a izbor između njih je generalno pitanje preferencije.

Sada ćemo pogledati tipizovanje objekata.

Ovo je primer objektnog literala:

```js
const developer = {
  firstName: "Jessica",
  isEmployed: true
}
```

Da eksplicitno tipizirate svojstva objekta, možete koristiti ovu sintaksu:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

Ako pokušate da dodate ili uklonite svojstva iz objekta, dobićete grešku kao što je ova:

```ts
// Property 'isEmployed' is missing in type '{ firstName: string; }' but required in type '{ firstName: string; isEmployed: boolean; }'
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
}
```

Možete učiniti svojstva objekta opcionalnim korišćenjem `?` nakon imena svojstva, kao što je ovo:

```ts
const developer: { firstName: string, isEmployed?: boolean } = {
  firstName: "Jessica",
}
```

TypeScript više neće prikazivati grešku jer je svojstvo `isEmployed` označeno kao opcionalno.

Do sada, svi primeri objekata imali su restriktivna svojstva. To znači da smo eksplicitno naveli svako svojstvo koje je objekt dozvoljeno da ima:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

Ali šta ako ne znate sva imena svojstava unapred? Šta ako želite objekat koji može imati bilo koji broj svojstava, ali moraju svi podaci da prate određeni tip?

Pa, postoji dva načina za postizanje ovoga. Prvi način je korišćenje `Record`, koji je ugrađeni generički utilitarni tip u TypeScriptu.

Evo osnovne sintakse:

```ts
Record<Keys, ValueType>
```

Prvi tip predstavlja tip ključeva, dok drugi tip predstavlja tip vrednosti.

Ovo je primer korišćenja `Record` za objekat `userRoles`:

```ts
const userRoles: Record<string, string> = {
  admin: "full-access",
  editor: "limited-access",
  viewer: "read-only"
};
```

Ovaj `userRoles` objekat može imati ključeve bilo kog tipa niza (string), a sve vrednosti moraju biti nizovi (string). Ako pokušate da dodelite vrednost koja nije niz (string), dobićete grešku:

```ts
// Type 'number' is not assignable to type 'string'
const userRoles: Record<string, string> = {
  admin: 1
};
```

Drugi način za pisanje iste ideje je sa indeksnom potpisom:

```ts
const obj: { [key: string]: string } = {};
```

Sintaksa iznad navodi da svako svojstvo sa string ključem mora imati string vrednost. Evo primera za objekat `settings`:

```ts
const settings: { [key: string]: string } = {
  theme: "dark",
  language: "en",
  layout: "grid"
};
```

Kao i ranije, ako imate pogrešne tipove vrednosti, TypeScript će prikazati poruku o grešci:

```ts
// Type 'boolean' is not assignable to type 'string'
const settings: { [key: string]: string } = {
  darkMode: true
};
```

I `Record` i indeks potpisi su fundamentalno isti. Zato je izbor između njih uglavnom pitanje preferencije.
