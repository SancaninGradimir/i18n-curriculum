## Veb servisi i API-ji

- **API (Application Programming Interface)**: Opšti način na koji softver komunicira sa drugim softverom.
- **Web service**: Specifičan tip API-ja koji koristi veb tehnologije poput HTTP za razmenu podataka preko mreže. Svaki web servis je API, ali nije svaki API web servis.
- **Key characteristics of a web service**: radi na serveru, omogućava komunikaciju mašina sa mašinama (machine-to-machine), je nezavisan od jezika (language-agnostic), nezavisan od platforme (platform-agnostic) i koristi standardne veb protokole (HTTP/HTTPS, FTP, SMTP).
- **REST (Representational State Transfer)**: Najčešće korišćeni tip web servisa danas. Izlaže resurse identifikovane putem URL putanja kao što su `/users` ili `/users/20`. Odgovori su obično u JSON formatu, ali mogu biti i XML ili običan tekst (plain text).
- **SOAP (Simple Object Access Protocol)**: Koristi samo XML. Čest je u korporativnim sistemima koji zahtevaju strože standarde i bezbednost.
- **XML-RPC**: Koristi XML za enkardiranje (encoding) zahteva i odgovora. Lagan je, ali manje fleksibilan od REST-a.
- **`Accept` header**: Koristi ga klijent da kaže serveru koji željeni format odgovora, na primer `Accept: application/json`.

## REST Arhitektura

- **Bezstanost (Statelessness)**: Svaki zahtev mora da sadrži sve informacije koje je server potreban za njegovu obradu. Server ne pamti prethodne zahteve.
- **Resurs**: Komadi podataka koji se mogu pristupiti ili modifikovati, identifikovani putem URL-a (na primer, `/products/1`).
- **HTTP metode u REST-u**:
  - `GET` — dohvaćanje resursa
  - `POST` — kreiranje novog resursa
  - `PUT` / `PATCH` — ažuriranje postojećeg resursa
  - `DELETE` — brisanje resursa

## HTTP Status Kodovi

Nakon obrade zahteva, server odgovara sa **status kodom** koji ukazuje rezultat:

- `200` **(OK)**: Zahtev je uspešan
- `201` **(Created)**: Krešiv resurs je kreiran
- `400` **(Bad Request)**: Klijent je poslao nevažeće podatke
- `404` **(Not Found)**: Zahtevani resurs ne postoji
- `500` **(Internal Server Error)**: Došlo je do greške na serveru

## Monoliti naspram mikroservisa

- **Monolit**: Jedna, velika baza koda gde UI (interfejs korisnika), baza podataka i API funkcionišu zajedno. Dobro je za male aplikacije, ali postaje sve teže za održavanje i skaliranje kako sistem raste.
- **Mikroservisi**: Arhitektura u kojoj je velika aplikacija razbijena na manje, nezavisne servise. Svaki servis obrađuje jednu poslovnu funkcionalnost (na primer, proizvodi, porudžbine, plaćanja) i može imati svoju bazu koda, bazu podataka i tim.
- **Kako mikroservisi komuniciraju**: Kroz API-je ili sisteme zasnovane na porukama.
- **Prednosti mikroservisa**: Brži razvoj, bolja izolacija grešaka i sloboda korišćenja različitih jezika ili framework-ova po servisu.
- **Izazovi mikroservisa**: Kompleksnije debugovanje, teže upravljanje infrastrukturom i veći mrežni nadopci između servisa.

## Šta je Express.js?

- **Express**: Minimalan i fleksibilan web framework baziran na Node.js koji olakšava izgradnju web servera i API-ja.
- **Instaliranje Express-a**: `npm install express`
- **Šta možete izgraditi sa Expressom**: veb aplikacije, RESTful API-ji, SPA backend-ovi, servisi sa mnogo middleware-a i sve što je na strani servera sa HTTP protokolom.
- **Middleware**: Kod koji se izvršava između prijema zahteva i slanja odgovora. Koristi se za rukovanje greškama, validaciju ulaznih podataka, modifikovanje podataka zahteva ili odgovora i još mnogo toga. Više middleware funkcija može biti povezane (chainovane) jedna sa drugom.
- **Plugins**: Eksterni npm moduli koji proširuju Express, obično implementirani kao middleware.

## Kreiranje osnovne Express aplikacije

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

- `require("express")` — učitava biblioteku Express.
- `express()` — fabrička funkcija koja kreira i vraća novu instancu aplikacije Express.
- `app` — glavni objekat servera sa pristupom metodama kao što su `.get()`, `.post()` i `.listen()`.
- `port` — mrežna tačka (endpoint) na kojoj aplikacija sluša. Port `3000` je uobičajen za lokalni razvoj.
- `app.listen(port, callback)` — pokreće server i počinje da sluša dolazne HTTP zahteve.

## Rutiranje u Express-u

- **Ruta**: Pravilo koje određuje kako će aplikacija odgovoriti kada klijent traži određeni URL koristeći specifičnu HTTP metodu.
- **Metode rute**: `app.get()`, `app.post()`, `app.put()`, `app.delete()` — svaka obrađuje zahteve sa odgovarajućom HTTP metodom.
- **Putanja statične rute**: Fiksni URL poput `/home` ili `/menu/drinks`.
- **Putanja dinamičke rute (parametar rute)**: Placeholder u URL-u koji hvata vrednost, prefiksiran sa `:`, na primer `/post/:postId`.
- **`req.params`**: Objekt koji sadrži dinamičke vrednosti uhvaćene iz URL-a. Na primer, zahtev ka `/post/42` vraća `req.params.postId === "42"`.

### Ručnici za rute

- **Rukovatelji rute** (route handler) je funkcija povratnog poziva ``(req, res) => { ... }`` koja se izvršava kada zahtev odgovara ruti.
- ``req`` — sadrži informacije o dolaznom zahtevu (parametri, query stringovi, zaglavlja, telo).
- ``res`` — koristi se za slanje odgovora nazad klijentu.

## Metode odgovora

| Metoda | Šta radi |
| --- | --- |
| `res.send()` | Šalje odgovor — običan tekst, HTML, JSON ili binarni podaci. |
| `res.json()` | Šalje JSON odgovor; automatski konvertuje JavaScript objekat. |
| `res.status(code)` | Postavlja HTTP status kod; obično se koristi uz `res.send()` ili `res.json()`. |
| `res.redirect(url)` | Preusmerava klijenta na drugačiji URL. |
| `res.render(view, data)` | Renderuje prikaz (view) pomoću mašine za šablone i šalje rezultujući HTML. |

## Lančovani handleri za rute sa `app.route()`

``app.route(path)` grupiše više rukovaoca HTTP metoda za istu putanju kako bi se izbeglo ponavljanje.`

```javascript
app
  .route("/user")
  .get((req, res) => { res.send("Fetching user data"); })
  .post((req, res) => { res.send("Creating a user"); })
  .put((req, res) => { res.send("Updating the user"); })
  .delete((req, res) => { res.send("Deleting the user"); });
```

## Modularno rutiranje sa `express.Router()`

- ``express.Router()`` kreira router objekat — slično mini Express aplikaciji — na kojem možete definisati rute, a zatim ga montirati u glavnu aplikaciju.
- Montirajte router pomoću ``app.use(path, router)``. Rute unutar rutera su relative u odnosu na putanju montaže (mount path).
- **Prednosti**: razdvaja odgovornosti (separates concerns), održava glavni fajl čistim i čini rute ponovo upotrebljivima i lakšim za skaliranje.

```javascript
// userRoutes.js
const express = require("express");
const router = express.Router();
router.get("/", (req, res) => { res.send("List of users"); });
router.get("/:id", (req, res) => { res.send(`User ${req.params.id}`); });
module.exports = router;

// app.js
app.use("/users", userRoutes); // mounts the router at /users
```

## Služenje statičkih fajlova

- **Statički fajlovi**: Resursi kao što su slike, CSS i JavaScript na strani klijenta koji se serviraju direktno bez obrade na strani servera.
- `express.static(directory)` — ugrađeni middleware koji služi fajlove iz definisane foldere.
- Ako traženi fajl ne postoji, Express automatski odgovara sa 404.
- Za pojedinačne statičke fajlove nisu potrebne definicije ruti (route definitions).

```javascript
app.use(express.static(path.join(__dirname, "public")));
```

- Možete servirati iz više direktorija pozivanjem ``app.use()`` više puta, opcionalno sa putanjom za montažu:

```javascript
app.use(express.static(path.join(__dirname, "public")));
app.use("/images", express.static(path.join(__dirname, "images")));
```
