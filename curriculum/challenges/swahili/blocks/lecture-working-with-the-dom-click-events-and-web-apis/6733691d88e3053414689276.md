# --interactive--

U prethodnim lekcijama, naučili ste kako da radite sa događajima koristeći metod ``addEventListener()``. Ali postoji drugi, ne preporučeni način za rad sa događajima u JavaScript-u.

Inline rukovači događajem su specijalni atributi na HTML elementu koji se koriste za izvršavanje JavaScript koda kada dođe do događaja.

Evo primera `button` elementa sa ugrađenim rukovaocem za klik događaj:

:::interactive_editor

```html
<button onclick="alert('Hello World!')">Show alert</button>
```

:::

Kada korisnik klikne na dugme, funkcija `alert` se poziva i prikazan je dijalog upozorenja sa porukom `Hello World!`.

Drug način za korišćenje ugrađenih hendlera događaja je pozivanje funkcije koja je definisana u tagu `script` u HTML dokumentu.

Evo primera definisanja funkcije nazvane `changeBgColor` i pozivanja je iz ugrađenog hendlera događaja za klik:

:::interactive_editor

```html
<script>
  function changeBgColor() {
    document.body.style.backgroundColor = "lightblue";
  }
</script>

<button onclick="changeBgColor()">Change background color</button>
```

:::

Kada korisnik klikne na dugme, funkcija `changeBgColor` se poziva i promena se pozadinska boja stranice u svetlo plavu. Iako je moguće koristiti inline rukovače događaja na ovaj način, ne smatra se najboljom praksom.

Po jednoj razlozi, ugrađeni rukovaoci događajima mogu se koristiti samo za dodavanje jednog slušaoca događaja elementu. Ako želite da dodate više slušaoca događaja istom elementu, moraćete da koristite `addEventListener()`. Drugi razlog je što ugrađeni rukovaoci događajima mešaju HTML i JavaScript kod zajedno, što može učiniti vaš kod teškim za čitanje i održavanje. Bolje je da odvojite svoj HTML kod i JavaScript kod korišćenjem `addEventListener()` za dodavanje slušaoca događaja elementima.

Inline rukovaoci događajima nisu preporučljivi za upotrebu u modernom JavaScriptu. Stoga je najbolja praksa se pridržavati metode `addEventListener()` kada radite sa događajima u JavaScriptu.
