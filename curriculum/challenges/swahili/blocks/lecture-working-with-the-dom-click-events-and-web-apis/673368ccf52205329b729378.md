# --interactive--

Naučićemo o `innerText`, `textContent`, i `innerHTML`.

Ovo su svojstva koja možete pristupiti u JavaScript-u da biste dobili ili promenili sadržaj HTML elementa. Čak i ako na prvi pogled izgledaju veoma slično, imaju ključne razlike. Izbor pravog zavisi od vašeg specifičnog slučaja upotrebe, pa hajde da zaronimo.

Počinjemo sa `innerText`.

`innerText` predstavlja vidljivo tekstualno sadržajno delo HTML elementa i njegovih potomaka. Ova svojstva ne uključuje skriveni tekst niti HTML oznake, već samo prikazani tekst.

Na primer, ovde možete videti element `div` koji sadrži dva pasusa:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

Ako dobijemo referencu na ovaj HTML element u našem JavaScript kodu koristeći `getElementById()`, možemo pristupiti svojstvu `innerText` ovog elementa:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

Ovo je unutrašnji tekst ovog elementa:

```md
Hello, World!
I'm learning JavaScript
```

Svojstvo vraća niz sa tekstom sadržanim unutar elementa, uključujući i tekst iz njegovih potomaka.

Trebalo bi da znate da `innerText` vraća samo tekst koji je vidljiv u određenom trenutku kada se niz traži. Ako je dete element skriveno, njegov tekst neće biti vidljiv.

Ovo je primer gde je drugi pasus sakriven:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
```

:::

Ako pokušamo da logujemo ``innerText`` ponovo, sada izlaz neće imati tekst drugog pasusa:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

Možete postaviti ``innerText`` za HTML element na ovaj način, ali ovo će zameniti postojeći tekst i dodati element za prekid linije (``br``) element za svaki prekid linije:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.innerText = "JavaScript is awesome!";
```

:::

Pošto ``innerText`` uzima u obzir vidljivost, dobijanje njegove vrednosti pokreće proces nazvan "reflow", koji ponovo računa poziciju određenih elemenata na veb stranici. Ovaj proces može biti računski zahtevan, pa biste trebali da izbegnete njegovo pokretanje ako je to moguće.

Odlično. Sada hajde da pričamo o `textContent`.

`textContent` vraća sadržaj čistog teksta elementa, uključujući sav tekst unutar njegovih potomaka.

Najvažnija razlika između `innerText` i `textContent` jeste da `textContent` uvek vraća pun sadržaj teksta HTML elementa i njegovih potomaka, bez obzira na to da li je vidljiv ili skriven.

TEKST:

Ovde imamo isti primer u HTML-u:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

Ako pokušate da pristupite ovoj svojstvu, videćete tekst elementa i njegovih potomaka kao izlaz, zadržavajući uvlačenje i razmak:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::

Ako HTML element nije vidljiv, kao što možete videti ovde, gde smo sakrili drugi pasus, njegov tekst će i dalje biti uključen u ovo svojstvo:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::
  
`textContent` takođe će uključivati sadržaj elemenata kao što su `script` i `style`.

Ako pokušate da zamenite vrednost `textContent` na čvoru, on će ukloniti sve njegove potomke čvore i zameniti ih jednim tekstualnim čvorom koji sadrži novi niz znakova:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.textContent = "New content";
```

:::

I na kraju, hajde da pričamo o tome kako `textContent` i `innerText` se razlikuju od `innerHTML`.

Zapamtite da sa ``innerHTML`` možete postaviti unutrašnji HTML sadržaj elementa. Ovo je korisno za dinamično ubacivanje novog HTML-a u DOM.

Međutim, zapamtite da ovo predstavlja sigurnosni rizik ako nemate kontrolu nad nizom (stringom), kao što su nizovi koji sadrže podatke koje je unet korisnik. Ako su ti podaci zlonamerni, mogu dovesti do ozbiljnih sigurnosnih problema.

Da bi se ovo izbeglo, preporučuje se korišćenje svojstva `textContent` za umetanje običnog teksta umesto toga.

Svojstva `innerText`, `textContent` i `innerHTML` u JavaScript-u pružaju različite načine za pristup i manipulaciju sadržajem HTML elemenata. Trebalo bi da razumete razlike između ovih svojstava ako je vaš cilj efikasno raditi sa HTML sadržajem u JavaScript-u.
