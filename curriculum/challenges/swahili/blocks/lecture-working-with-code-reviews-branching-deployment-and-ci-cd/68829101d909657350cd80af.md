# --description--

GitHub Pages je veoma pouzdan izbor za postavljanje statičkih sajtova ili aplikacija koje ne zahtevaju *back-end* server za obradu logike. Drugim rečima, aplikacije koje rade potpuno na strani klijenta (*client-side*), ili u pregledaču korisnika, mogu se u potpunosti postaviti na ovoj platformi.

Pored toga, GitHub Pages su potpuno besplatni za projekte otvorenog koda. Postoji mnogo dobrih razloga da učinite svoj projekat otvorenim kodom, a Pages dodaje još jedan.

Ali kako zapravo implementirati na GitHub Pages? Pa, to zavisi od toga kako vaša aplikacija radi.

Počnimo sa osnovnom HTML stranicom. Evo jedne koju možete dodati u svoj postojeći repozitorijum ako želite:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>Welcome to freeCodeCamp's Lessons!</h1>
    <p>Let's have some fun learning about Git and GitHub!</p>
  </body>
</html>
```

Sada moramo da posetimo naš repozitorijum i kliknemo na karticu "Podešavanja" da bismo otišli na podešavanja repozitorijuma. Zatim moramo da izaberemo "Stranice" sa bočne trake.

In početku ima samo nekoliko opcija, ali će se to promeniti kako počnemo da konfigurišemo podešavanja. Prvo, moramo reći GitHubu od čega da deploje.

Želećemo da ostavimo ovo kao „Deploy from a branch“, jer ne koristimo okvir i uopšte nismo konfigurisali nikakve GitHub akcije. Naučićete više o tome šta su to u sledećoj lekciji.

Zatim, podesite granu sa koje vršimo deployment na `main` da deployujete ono što je na njoj.

Zatim ćete imati izbor da objavite iz koriscenog foldera vašeg projekta ili iz foldera ``docs``. Ova funkcija je korišćena od strane projekata kao što je freeCodeCamp - ranije smo spakovali dokumentaciju sa našom aplikacijom, tako da bismo mogli objaviti samo dokumentaciju na GitHub stranicama putem foldera ``docs``. Trenutno, ostavite ga kao ``/`` (korisnički) folder i pritisnite dugme "Sačuvaj" pored ovog padajućeg menija da sačuvate svoje promene.

Nakon što sačuvate, videćete novo polje za konfigurisanje prilagoovanog domena. Ako već imate jedan i želite da ga koristite, možete pratiti link ka dokumentaciji kako biste to uradili. Međutim, za ovu lekciju ćemo ovo ostaviti prazno.

Takođe možete videti domen na koji je vaš novi sajt na GitHub Pages-u implementiran. Bez prilagođenog domena, ovo se podrazumeva na `<username>.github.io`, gde je `<username>` ime vašeg naloga (ili nalog organizacije ako deployujete sa repozitorijuma koji pripada organizaciji). A repozitorijum će biti serviran na putanji koja odgovara imenu repozitorijuma, npr. `<username>.github.io/<repo-name>`. Ako posetite URL, videćete svoj sajt!

Sada, kada potaknete nove promene u vašu `main` granu, GitHub Pages će automatski ažurirati naš razgrađeni sajt najnovijim promenama. Ovaj proces može potrajati nekoliko minuta, u zavisnosti od opterećenja servera – i možda ćete morati da uradite hard refresh kako biste obrisali keš.

I time, uspešno smo objavili osnovnu veb aplikaciju na GitHub Pages. Takođe možete uključiti CSS i JavaScript fajlove u vaše Page objavljivanje, tako da možete napraviti lepe i interaktivne sajtove. I kada naučite okvir poput Reacta ili Angulara, čak možete to objaviti i na GitHub Pages!
