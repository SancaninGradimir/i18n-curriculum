## Uvod u Kontrolu Verzija

- **Definicija**: Sistem za kontrolu verzija omogućava vam da pratite i upravljate promenama u vašem projektu. Primeri sistema za kontrolu verzija koji se koriste u softveru su Git, SVN ili Mercurial.

## Pružatelji sistema kontrole verzija u oblaku

- **Lista pružalaca kontrole verzija baziranih na oblaku**: GitHub i GitLab su popularni primeri pružaoca kontrole verzija baziranih na oblaku koji omogućavaju softverskim timovima da sarađuju i upravljaju repozitorijumima.

## Instalacija i konfigurisanje Gita

- **Instalacija Gita**: Da biste proverili da li je Git već instaliran na vašoj mašini, možete pokrenuti sledeću komandu u terminalu:

```sh
git --version
```

Ako vidite broj verzije, to znači da je Git instaliran. U suprotnom, moraće vam se instalirati.

Za Linux sisteme, Git je često predinstaliran sa većinom distribucija. Ako nemate Git predinstaliran, trebalo bi da ga instalirate pomoću komandi vašeg menadžera paketa, kao što su `sudo apt-get install git` ili `sudo pacman -S git`.

Za korisnike Mac-a, možete instalirati Git preko Homebrewa sa `brew install git`, ili možete preuzeti izvršni instalater sa Git-ovog sajta.

Za Windows, možete preuzeti izvršni instalater sa Git-ovog sajta. Ili, ako ste postavili Chocolatey, možete pokrenuti `choco install git.install` u PowerShellu. Napomena da na Windows-u možda želite da preuzmete i Git Bash kako biste imali dostupan Unix-sličan shell okruženje.

Kako biste bili sigurni da je instalacija uspela, ponovo pokrenite komandu ``git --version`` u terminalu.

- **Konfiguracije Git-a**: ``git config`` se koristi za postavljanje varijabli konfiguracije koje određuju kako Git radi na vašoj mašini. Da biste videli trenutne varijable podešavanja i gde su sačuvane u vašem sistemu, možete pokrenuti sledeću komandu:

```sh
git config --list --show-origin
```

Trenutno bi trebalo da vidite samo sistemska podešavanja konfiguracije ako ste tek instalirali Git prvi put.

Da biste postavili korisničko ime, možete pokrenuti sledeću komandu:

```sh
git config --global user.name "Jane Doe"
```

Argument `--global` se ovde koristi za postavljanje korisničkog imena za sve projekte na vašem sistemu koji koriste Git. Ako trebate nadjačati korisničko ime za određeni projekat, možete pokrenuti komandu u tom konkretnom direktorijumu projekta bez argumenta `--global`.

Za postavljanje e-mail adrese korisnika, možete pokrenuti sledeću komandu:

```sh
git config --global user.email janedoe@example.com
```

Još jedna konfiguracija koju možete podesiti je preferirani editor koji želite da Git koristi. Evo primera kako da podesite vaš preferirani editor na Emacs:

```sh
git config --global core.editor emacs
```

Ako ne postavite preferirani editor, Git će koristiti podrazumevani editor vašeg sistema.

## Softver otvorenog naspram zatvorenog izvora

- **Definicija**: "Open-source" znači da ljudi mogu da vide kod koji objavite, predlažu izmene, prijavljuju probleme i čak pokrenu modifikovanu verziju. "Closed-source" znači da su jedine osobe koje mogu da vide i interaguju sa projektom one koje vi eksplicitno ovolaštavate.

## GitHub

- **Definicija**: GitHub je rešenje zasnovano na oblaku koje nudi skladištenje projekata pod kontrolom verzija u nešto što se zove "repositories", i omogućava funkcije za saradnju za korišćenje sa tim projektima.
- **GitHub CLI**: Ovaj alat se koristi za obavljanje zadataka specifičnih za GitHub bez napuštanja komandne linije (command line). Ako ga nemate instaliranog, možete dobiti uputstva kako da to uradite iz dokumentacije GitHub-a – ali trebalo bi da ga imate dostupan u menadžeru paketa vašeg sistema.
- **GitHub Pages**: GitHub Pages je opcija za *deployovanje* statičkih sajtova ili aplikacija koje ne zahtevaju *back-end* server za obradu logike. To znači da se aplikacije koje rade isključivo na klijentskoj strani (*client-side*), ili u pregledaču korisnika, mogu potpuno *deployovati* na ovoj platformi.
- **GitHub Actions**: GitHub Actions je funkcija koja vam omogućava da automatizujete radne tokove (*workflows*) direktno u vašem GitHub repozitorijumu (repository), uključujući izgradnju (*building*), testiranje i *deployovanje* vašeg koda.

## Uobičajene Git Komande

- **`git init`**: Ovo će inicijalizovati prazan Git repozitorijum kako bi Git mogao da počne praćenje promena za ovaj projekat. Kada inicijalizujete prazan Git repozitorijum za projekat, dodiraće se novi skriven direktorijum `.git`. Ovaj `.git` direktorijum sadrži važne informacije koje su potrebne Git-u da upravlja vašim projektom.
- **`git status`**: Ova komanda se koristi za prikaz trenutnog stanja vašeg radnog direktorijuma – ovu komandu ćete mnogo koristiti u svom radnom procesu.
- **`git add`**: Ova komanda se koristi za pripremu (staging) promena. Sve što je u stazingu će biti dodato za sledeći commit. Ako želite da pripremite sve nepripremljene promene, možete koristiti `git add .`. Period (`.`) je alias za trenutni direktorijum u kojem se nalazite.
- **`git commit`**: Ova komanda se koristi za commit vaših promena. Commit je snimak stanja vašeg projekta u dato vreme. Ako pokrenete `git commit`, otvoriće vam omiljeni editor koji ste podesili u Git konfiguraciji. Kada je editor otvoren, možete priložiti detaljnu poruku o svojim promenama. Takođe možete odlučiti da pružite kraću poruku koristeći komandu `git commit -m` na ovaj način:

```sh
git commit -m "short message goes here"
```

- **`git log`**: Ovo će listati sve prethodne commit-ove sa korisnim informacijama kao što su autor, datum commita, poruka commita i hash commita. Hash commita je dugačak niz koji služi kao jedinstveni identifikator za commit.
- **`git remote add`**: Ova komanda se koristi za podešavanje udaljene veze (remote connection) na vaš repozitorijum.
- **`git push`**: Ova komanda se koristi za slanje (push) promena na udaljeni repozitorijum.
- **`git pull`**: Ova komanda se koristi za preuzimanje (pull) najnovijih promena sa vašeg udaljenog repozitorijuma u lokalni repozitorijum.
- **`git clone`**: Ova komanda će klonirati (clone) repozitorijum. To znači da ćete dobiti kopiju repozitorijuma. Ova kopija uključuje istoriju repozitorijuma, sve fajlove/foldere i commit-ove na vašem lokalnom uređaju.
- **`git remote -v`**: Ova komanda će prikazati listu udaljenih repozitorijuma povezanih sa vašim lokalnim Git repozitorijumom.
- **`git branch`**: Ova komanda će listati sve vaše lokalne grane (branches).
- **`git fetch upstream`**: Ova komanda govori Git-u da preuzme najnovije promene sa vašeg upstream udaljenog repozitorijuma (koji je originalni repo).
- **`git merge upstream/main`**: Ova komanda govori Git-u da spoji (merge) najnovije promene iz grane `main` u upstream udaljenom repozitorijumu u vašu trenutnu granu.
- **`git reset`**: Ova komanda vam omogućava da resetujete trenutno stanje grane. Prosleđivanje flag-a `--hard` govori Git-u da prisili lokalne fajlove da odgovaraju stanju grane. Ovo osigurava da imate čist početak sa kojeg možete raditi.
- **`git rebase`**: Rebase u Git-u je način za pomeranje ili kombinovanje sekvence commit-ova sa jedne grane na drugu.

## Rad sa granama

- **Definicija**: Grana u Git-u je zasebno radno okruženje gde možete napraviti promene. Grana `main` često predstavlja glavnu ili produkcionu granu u stvarnoj aplikaciji. Timovi programera će kreirati više grana za nove funkcije i popravke grešaka, a zatim te promene spojiti nazad u granu `main`.
- **Kreiranje nove grane**: Da biste kreirali novu granu, možete pokrenuti sledeću komandu:

```sh
git branch feature
```

Da biste prešli na tu granu, možete pokrenuti sledeću komandu:

```sh
git checkout feature
```

Većina razvojnika će koristiti skraćenu komandu za kreiranje i prebacivanje na granu, a koja je sledeća:

```sh
git checkout -b new-branch-name
```

Noviji i alternativni komanda bi bio ``git switch`` komanda. Evo primera za kreiranje i prebacivanje na novu granu:

```sh
git switch -c new-branch-name
```

- **Strategije grana**: Vaša `main` grana je vaša podrazumevana grana i obično je prilično stabilna. Zato je najbolje da se od nje odvojite kako biste kreirali nove grane za stavke kao što su popravke grešaka (bug fixes), nove funkcije (new features) ili drugi razni rad.
- **Konflikti spajanja**: To se dešava kada Git pokušava automatski da spoji promene sa različitih grana, ali ne može da odluči koje promene zadržati. Ovo obično nastaje kada postoje sukobne promene za isti deo fajla.

## Pet stanja za fajl koji je praćen od strane Git-a

- **"Untracked"**: To znači da je fajl nov u repozitorijumu, i da ga Git ranije nije "video".
- **"Modified"**: Ovaj fajl je postojao u prethodnom komitu i ima promene koje nisu komitovane.
- **"Ignored"**: Verovatno nećete videti ignorisane fajlove u Git-u, ali vaš IDE može imati indikator za njih. Ignorisani fajlovi su isključeni iz Git operacija, obično zato što su uključeni u fajl `.gitignore`.
- **"Deleted"**: Izbrisan fajl je suprotan nepraćenom fajlu – to je fajl koji je ranije postojao i uklonjen.
- **"Renamed"**: Preimenovani fajl je fajl čiji sadržaj nije promenjen, ali mu je ime ili lokacija modifikovana. U nekim slučajevima, fajl se može smatrati preimenovanim čak i ako ima malu količinu promena.

## `.gitignore` Datoteke

- **Definicija**: Fajl ``.gitignore`` je poseban tip fajla povezan sa operacijama u Git-u. Ime sugeriše da se ovaj fajl koristi za reći Git-u da ignoriše stvari, i to je uobičajena upotreba. Međutim, ono što zapravo radi jeste da kaže Git-u da prestane praćenje fajla.

## Rad sa repozitorijumima

- **Definicija**: Repozitorijum je kao kontejner za projekat – ako radiš na aplikaciji, sve fajlove za tu aplikaciju bi držao zajedno u repozitorijumu. Repozitorijumi mogu biti lokalni na tvom računaru ili udaljeni na servisu poput GitHub-a.
- **Javni i privatni repozitorijumi**: Javni repozitorijum može videti i preuzeti bilo ko. Privatni repozitorijum mogu pristupiti samo ti, i svako kome eksplicitno dodeliš pristup.
- **Kreiranje repozitorijuma na GitHubu**: Da bi kreirao novi repozitorijum na GitHubu, možeš kliknuti na dugme `"New Repository"` i proći kroz korisnički interfejs (UI) GitHub-a za podešavanje novog repozitorijuma.
- **Pushovanje lokalnih repozitorijuma na GitHub**: Ako imaš lokalni projekat na računaru, možeš ga "pushovati" (uploadovati) na GitHub. Evo pregleda procesa korak po korak:

1. Inicijalizujte prazan Git repozitorijum u direktorijumu projekta (`git init`).
2. Napravite promene na vašem projektu.
3. Pokrenite komandu `git status` da vidite sve napravljene promene koje prati git.
4. Pripremite (stage) vaše promene (`git add`).
5. Izvršite commit promenama (`git commit`).
6. Podesite udaljenu konekciju (`git remote add`).
7. Pushujte svoje promene na GitHub (`git push`).

## Zahtevi za povlačenje

- **Pull Requests**: Pull request je zahtev da se promene povuku sa vaše grane u ciljnu granu. Pull requestovi su tok koji koristite kada želite da doprinesete promenama koda nekom projektu. Ovaj pristup omogućava održavateljima projekta da preglede vaše izmene. Oni mogu ostaviti komentare, postaviti pitanja i predložiti prilagodbe. Kada je proces pregleda završen, on može biti odobren i spojen u glavnu granu.

## Doprinoseći drugim repozitorijumima

- **Proces**: Postoji hiljada projekata kojima možete doprineti. Evo osnovnog procesa kako da doprinesete drugom repozitorijumu:

1. Pročitajte dokumentaciju za doprinose
2. Pronađite dostupan *issue* na kojem možete raditi
3. Napravite *fork* repozitorijuma
4. Klonirajte svoju napravljenu kopiju (*fork*) repozitorijuma
5. Kreirajte novu granu (*branch*)
6. Izvršite promene u skladu sa *issue*-om
7. Napravite PR (Pull Request)
8. Sačekajte pregled (*review*) za taj PR

## Rad sa SSH i GPG ključevima

- **GPG Ključevi**: GPG, ili Gnu Privacy Guard, ključevi se obično koriste za potpisivanje fajlova ili commit-ova. Neko zatim može da koristi vaš javni GPG ključ kako bi proverio da li je potpisanje fajla izvađeno iz vašeg ključa i da li sadržaj fajla nije modifikovan ili narušen.

Za generisanje GPG ključa, morate pokrenuti:

```sh
gpg --full-generate-key
```

- **SSH Ključevi**: SSH ključevi, ili ključevi za Secure Shell, obično se koriste za autentifikaciju udaljenog povezivanja na server – putem utilitete ``ssh``. Takođe možete koristiti SSH ključ za potpisivanje commit-ova.

Za SSH ključ, pokovaćete:

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
```

``ed25519` je savremen/moderni algoritam potpisa javnog ključa.`

- **Potpisivanje commita pomoću GPG ključa**: Da biste potpisali svoje commit-ove sa vašim GPG ključem, morate da učitate svoj javni ključ, a ne privatni ključ, na svoj GitHub nalog. Da biste naveli svoje javne ključeve, moraćete da pokrenete sledeće:

```sh
gpg --list-secret-keys --keyid-format=long
```

Zatim, da biste dobili javni ključ, koristite:

Zatim, uzmite kratki ID koji ste dobili prilikom spravljanja ključeva i pokrenite ovu komandu da ga postavite kao vaš git signing key:

```sh
git config --global user.signingkey <your_gpg_key_id>
```

Zatim, možete proslediti zastavicu `-S` vašoj komandi `git commit` da potpišete određeni commit – moraćete da unesete svoju lozinku za frazu. Alternativno, ako želite automatski potpisati svaki commit, možete podesiti konfiguraciju autosign na `true`:

```sh
git config --global commit.gpgsign true
```

- **Potpisivanje Commit-ova SSH Ključevima**: Da potpišete pomoću SSH ključa, što je relativno nova funkcija na GitHubu, prvo morate učitati taj ključ na svoj GitHub nalog. Zatim ćete morati postaviti režim potpisivanja za git da koristi SSH:

```sh
git config --global gpg.format ssh
```

Zatim, da postavite ključ za potpisivanje, prosleđujete putanju fajla umesto ID-a:

```sh
git config --global user.signingkey <path_to_your_ssh_keys>
```
