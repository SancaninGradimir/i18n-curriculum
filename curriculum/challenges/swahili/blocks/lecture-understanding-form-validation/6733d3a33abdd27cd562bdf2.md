# --interactive--

Naučićemo o nameni metode `preventDefault()` na događajima.

Svaki događaj koji se aktivira u DOM-u ima neku vrstu podrazumevanog ponašanja. Događaj ``click`` na čekboksu prebacuje stanje tog čekboksa, po defaultu. Pritiskanje razmaknice na fokusiranom dugmetu aktivira dugme. Metoda ``preventDefault()`` na ovim objektima događaja zaustavlja da se to ponašanje dogodi.

Pogledajmo primer. Definisati ćemo element `input` za korisnika da unese:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<label>Enter some characters:
  <input type="text">
</label>
```

```css
label {
  display: flex;
  flex-direction: column;
  font-family: Arial, sans-serif;
  font-size: 14px;
  color: #333;
  margin-bottom: 12px;
}

input[type="text"] {
  padding: 8px 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 14px;
  margin-top: 4px;
  width: 100%;
  box-sizing: border-box;
  transition: border-color 0.2s, box-shadow 0.2s;
}

input[type="text"]:focus {
  border-color: #0078d4;
  box-shadow: 0 0 3px rgba(0, 120, 212, 0.5);
  outline: none;
}

```

:::

Iako pogledamo rezultat, možemo uneti u polje za unos kako je očekivano. Ali možda to ne želimo. Možda, umesto toga, želeli bismo da prikažemo karakter koji korisnik unosi u zasebnom elementu. Prvo, definišemo naš element za to:

```html

  <input type="text">

```

```css
  margin: 0;

  font-size: 1.2rem;
  color: #333;
  margin-bottom: 1rem;
  display: flex;

  padding: 0.5rem;
  font-size: 1rem;
  border: 2px solid #ccc;
  border-radius: 5px;
  margin-top: 0.5rem;
  outline: none;

  border-color: #007BFF;
}

  margin-top: 1rem;
  font-size: 1.1rem;
  color: #555;

```

Zatim, moramo se priključiti na `keydown` događaj da slušamo kucanje karaktera na tastaturi. Napomena da ne želimo `change` ili `input` događaje ovde, jer nam trebaju informacije o tastaturi.

```js
const input = document.querySelector("input");

input.addEventListener("keydown", (e) => {
    
})
```


:::

`e.key` daje vam vrednost pritisnute tipke, kao što je `a` za tipku `a` ili `Enter` za tipku `Enter`.

Sa gornjim kodom, kada ukucate u `input`, znak koji ukucate biće prikazan u elementu `p`.

Ovo je odlično, ali ne želimo da prikažemo i karaktere u `input`. Ovde nam dolazi na pomoč metoda našeg `preventDefault()`. Podrazumevano ponašanje `keydown` je da prikaže karakter iz unosa. Izbegaćemo to pozivanjem `e.preventDefault()`:

:::interactive_editor

```html

  <input type="text">
</label>
<p id="output"></p>
<script src="index.js"></script>
```

```css

  font-size: 1.2rem;
  color: #333;
  margin-bottom: 1rem;
  display: flex;
  flex-direction: column;
}

input[type="text"] {
  padding: 0.5rem;
  font-size: 1rem;
  border: 2px solid #ccc;
  border-radius: 5px;
  margin-top: 0.5rem;
  outline: none;
}

  border-color: #007BFF;
```

```js
const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  e.preventDefault();
  output.innerText = `You pressed the ${e.key} key`;
});
```

:::

I tako, sprečili ste podrazumevano ponašanje kako biste sebi omogućili implementaciju sopstvene prilagođene obrade događaja.

Još jedan uobičajen primer kada koristiti metod ``e.preventDefault`` je onaj koji mora da radi sa podmetajima formulara. Podrazumevano, slanje formulara šalje podatke na server i ponovo učitava stranicu. Korišćenjem ``e.preventDefault()`` sprečava se da se to dogodi.

```js
const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  // rest of code goes here
});
```

Sprečavanje podrazumevanog ponašanja je odlično kada vam treba više kontrole nad načinom na koji korisnik interaguje sa stranicom, ali je važno imati u vidu stvari kao što su pristupačnost – vaše prilagođeno ponašanje bi trebalo da pruži iste funkcije kao i podrazumevano.
