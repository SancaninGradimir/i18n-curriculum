# --interactive--

U prethodnoj lekciji ste naučili kako da koristite HTML za ograničavanje vrednosti koje vaši korisnici mogu poslati u vaš formular. Ali ponekad to nije dovoljno. Ako želite da postignete nešto kompleksnije, kao što je prikazivanje sopstvenih poruka o greškama korisniku, moraćete da koristite JavaScript.

Određeni HTML elementi, kao što su elementi `textarea` i `input`, izlažu API za validaciju ograničenja (Constraint Validation API). Ovaj API vam omogućava da potvrdite da vrednost koju je korisnik unio za taj element prolazi svaku validaciju na nivou HTML-a koju ste napisali, kao što su minimalna dužina ili podudarnost uzorka.

Ali kako ga zaista možete koristiti? Recimo da želite da zaposleni u kompaniji pošalju povratne poruke putem obrasca kao ovoga:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>Enter your email: </label>
  <input required type="email" />

  <label>Enter your feedback: </label>
  <textarea required placeholder="Your feedback here..."></textarea>

  <button type="submit">Submit Feedback</button>
</form>
```

Koristimo e-mail `input` koji dolazi sa ugrađenom validacijom za proveru osnovne validacije, kao što je da li unos sadrži znak @ (`@`).

Ali šta ako korisnik unese e-mail adresu poput `example@email.com`? Ovo bi prošlo osnovnu validaciju, ali želimo da budemo specifičniji u vezi sa prihvatanjem e-mailova od onih koji imaju poslovnu e-mail adresu.

U gornjem primeru, upitali smo naš unos iz DOM-a i dodali slušalicu događaja unosa.

Znamo da se ``e.target`` odnosi na element koji je pokrenuo događaj. U ovom slučaju, naš ``input``. Ali šta je metoda ``checkValidity()``?

Ovo je deo API-ja za validaciju ograničenja. Metoda ``checkValidity()`` vraća ``true`` ako element odgovara svim HTML validacijama (na osnovu njegovih atributa), i ``false`` ako ne uspe.

Kada pokušamo sa nevažećim unosom, vidimo da se ``false`` loguje u konzolu. Sada kada znamo da je unos nevažeći, hajde da izveštamo o nevažećoj vrednosti:

```js
const input = document.querySelector("input");

input.addEventListener("input", (e) => {
  if (!e.target.checkValidity()) {
    e.target.setCustomValidity(
      "You must use a company email address that ends in @sampleCompany.com"
    );
  }
});
```

Svojstvo ``validity`` je instanca objekta ``ValidityState``. Evo primera kako bi objekat mogao izgledati u pretraživaču:

```js
ValidityState {
  badInput: false,
  customError: false,
  patternMismatch: true,
  rangeOverflow: false,
  rangeUnderflow: false,
  stepMismatch: false,
  tooLong: false,
  tooShort: false,
  typeMismatch: true,
  valueMissing: false,
  valid: true
}
```

Postoje nekoliko korisnih svojstava koja sve nose vrednost booleana od `true` ili `false`.

Neke od ovih korisnih svojstava koja možete dalje istražiti sami bi bile svojstvo ``valueMissing`` koje je ``true`` kada je obavezno polje za unos ostavljeno prazno. Ili ``patternMismatch`` koje je ``true`` ako vrednost ne odgovara navršenom regex obrascu.
