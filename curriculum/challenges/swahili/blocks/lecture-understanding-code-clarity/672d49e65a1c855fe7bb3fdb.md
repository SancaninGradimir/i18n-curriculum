# --description--

Komentari u programiranju se koriste za pružanje dodatnog konteksta za kod ili ostavljanje napomena za sebe i druge.

Komentari su linije ili blokovi teksta koje JavaScript interpretator ignoriše kada se vaš kod izvršava. Oni postoje samo radi koristi ljudi koji čitaju kod, bilo da ste to vi ili neko drugi.

JavaScript nudi dva načina za dodavanje komentara u vaš kod: jednoredne komentare i višelinijske komentare.

Komentari u jednoj liniji se kreiraju korišćenjem dve kose crte (`//`). Evo primera:

```js
// I am a single line comment in JavaScript
```

Ovaj tip komentara je odličan za kratke objašnjenja ili pojašnjenja.

TEKST:

Evo primjera iz stvarnog svijeta iz fajlova projekata kurikuluma freeCodeCamp-a:

```js
// This is to allow English to build without having to download the i18n files.
// It fails when trying to resolve the i18n-curriculum path if they don't exist.
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

Ne brinite zbog pokušaja razumevanja šta kod zapravo radi jer je napredniji od onoga što ste do sada naučili. Umesto toga, fokusirajte se na komentar koji je ostavio programer. Ovaj komentar pruža važan kontekst zašto ovaj kod postoji.

Komentari poput ovih su važni za one koji rade u timovima iz dva razloga:

1. Drugi programeri koji rade na projektu razumeće svrhu ovog koda.

1. Pomaže u sprečavanju nepotrebnih promena ili brisanja bez konsultacija sa timom, što bi moglo dovesti do grešaka ili problema.

Drugi tip komentara su višelinijski komentari. Evo osnovne sintakse:

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

Višelinijski komentari su korisni kada trebate napisati duže opise, objašnjenja ili napomene u svom kodu.

Pogledajmo ponovo na projekat fajlove kurikuluma freeCodeCamp-a da bismo videli kako se višelinijski komentari mogu koristiti u stvarnom svetu.

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

Kao i ranije, ignoriši sav JavaScript kod jer koristi koncepte koji još nisu naučeni. Umesto toga, fokusiraj se na komentar koji je ostavio programer.

Razvojnik u timu, ili čak novi doprinosilac koji radi na projektu, može razumeti zašto je ovaj deo koda ovde i imati pun kontekst pre rada na ovom delu projekta.

Iako su komentari korisni u programiranju, važno je izbegavati prekomerno komentarisanje. Ne morate da komentarišete svaku pojedinačnu liniju koda, posebno ako je kod jednostavan i sam po sebi objašnjavajući.

Evo primera korišćenja komentara za objašnjavanje očiglednog:

```js
// This code uses the const keyword to create a new variable called price.
// We are assigning the number 10 to the price variable.
const price = 10;
```

U ovoj situaciji, nema potrebe da se ovde dodaju komentari jer je kod sam po sebi objašnjen. Cilj je poboljšati čitljivost, a ne da se kod prepuni nepotrebnim objašnjenjima.

Ako želiš da dodaš komentare svojim ličnim projektima dok učiš kodiranje, to je u redu. Ali kada počneš da radiš na projektima iz stvarnog sveta sa drugim programerima, važno je da ne koristiš komentare za kod koji je sam po sebi objašnjavajući.

Takođe je važno ne koristiti komentare kako biste pomogli da objasnite zbunjujući, prekomerno komplikovan ili loše napisan kod. U takvim situacijama, najbolje je refaktorišati, ili promeniti, svoj kod tako da drugi programeri bolje razumeju šta se dešava.

Komentari su moćni alati za dokumentovanje vašeg koda i olakšavanje razumevanja istog. Trebalo bi da koristite komentare kako biste pružili kontekst ili ostavili napomene za sebe i druge.
