## Uobičajene greške u Pythonu

- **SyntaxError**: Greška koju Python izbacuje kada vaš kod ne prati njegove sintaksičke pravila. Na primer, kod `print("Hello there"` će dovesti do sintaksne greške sa porukom `SyntaxError: '(' was never closed`, jer nedostaje zatvorena zagrada.
- **NameError**: Python izvlači `NameError` kada pokušate da pristupite promenljivoj ili funkciji koju niste definisali. Na primer, ako imate liniju `print(username)` u svom kodu bez prethodno definisanja promenljive `username`, dobićete grešku imena sa porukom `NameError: name 'username' is not defined`.
- **TypeError**: Ovo je greška koju Python baca kada izvršavate operaciju na dve ili više nekompatibilne tipove podataka. Na primer, ako pokušate da dodate string i broj, dobićete grešku `TypeError: can only concatenate str (not "int") to str`.
- **IndexError**: Dobići ćete `IndexError` ako pristupite indeksu koji ne postoji u listi ili drugim sekvencama kao što su tuple i string. Na primer, u stringu `Hello world`, indeks poslednjeg karaktera je `11`. Ako nastavite i pristupite karakteru na ovaj način, `greet = "hello world"; print(greet[12])`, dobićete grešku sa porukom `IndexError: string index out of range`.
- **AttributeError**: Python izvlači ovu grešku kada pokušate da koristite metodu ili svojstvo koje ne postoji u objektu tog tipa. Na primer, pozivanje `.append()` na string kao što je `"hello".append("!")` dovesti će do greške sa porukom `AttributeError: 'str' object has no attribute 'append'`.

## Dobre tehnike debagovanja u Pythonu

- **Korišćenje funkcije `print`**: Ubacivanje izjava `print()` oko različitih tačaka u vašem kodu tokom debagovanja pomaže vam da vidite vrednosti promenljivih i kako teče izvršavanje vašeg koda.
- **Korišćenje ugrađenog debagera Pythona (`pdb`)**: Python pruža modul `pdb` za debagovanje. To je deo standardne biblioteke Pythona, pa je uvek dostupan za korišćenje. Sa `pdb`, možete postaviti trag pomoću funkcije `set_trace()` kako biste mogli da počnete sa korakovanjem kroz kod i interaktivno inspekcionisati promenljive.
- **Iskorišćavanje alata za debagovanje u IDE-ovima**: Mnogi integrisani razvojni okruženja (IDE) i uređivači koda, kao što su Pycharm i VS Code, nude alate za debagovanje sa tačkama prekida (breakpoints), korakovanjem izvršavanja, inspekcijom promenljivih i drugim funkcijama za debagovanje.

## Upravljanje izuzecima

- **`try...except`**: Ovo se koristi za izvršavanje bloka koda koji može da izazove izuzetak. Blok `try` je mesto gde predviđate da bi mogla doći greška, dok blok `except` prima specificovan izuzetak i izvršava se ako je ta specifikovana greška izazvana. Evo primera:

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

Takođe možete povezati više `except` blokova kako biste mogli da rukujete sa većim brojem tipova izuzetaka:

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you enter a string  
  ```

- **`else` i `finally`**: Ovi blokovi proširuju (extend) `try...except`. Ako ne dođe do izuzetka, blok `else` se izvršava. Blok `finally` uvek radi bez obzira na greške.

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **Objekat Izuzetka**: Omogućava vam pristup samom izuzetku radi bolje dijagnostike (debugging) i štampanja direktne poruke o grešci. Da biste pristupili objektu izuzetka, morate koristiti ključnu reč ``as``. Evo primera:

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **Izjava `raise`**: Ovo vam omogućava da ručno izazovete izuzetak. Možete ga koristiti za bacanje izuzetka kada se ispuni određeni uslov. Evo primera:

```markdown
  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```
  
## Signalizacija Izuzetaka
  
  Naredba ``raise`` je takođe korisna kada kreirate sopstvene, prilagođene izuzetke, jer je možete koristiti za bacanje izuzetka sa prilagođenom porukom. Evo primera toga:
```

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

Evo kako možete koristiti funkciju ``login`` sa izuzetkom ``InvalidCredentialsError``:

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

Izjava ``raise`` se takođe može koristiti sa ključnom rečju ``from`` za lančano povezivanje izuzetaka, pokazujući odnos između različitih grešaka:

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
  ```
