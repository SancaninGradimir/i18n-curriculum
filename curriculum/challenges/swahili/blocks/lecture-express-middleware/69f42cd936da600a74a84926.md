# --description--

Middleware je esencijalni koncept u Express-u i omogućava nam da obrađujemo zahteve i odgovore između klijenta i servera. Middleware funkcije imaju pristup objektu zahteva, objektu odgovora i sledećoj funkciji u ciklusu zahtev-odgovor aplikacije.

Razmotrimo to:

## Šta je Middleware?

Middleware je suštinski funkcija koja se izvršava tokom životnog ciclo zahteva ka serveru. Obavlja različite zadatke, kao što su logovanje zahteva, rukovanje greškama, parsiranje dolaznih zahteva ili serviranje statičkih fajlova.

Express pruža praktičan način za definisanje middleware-a koji obrađuje dolazne zahteve pre nego što stignu do rutinskih hendlera (route handlers).

Kako Middleware radi:

Kada se pošalje zahtev na Express aplikaciju, middleware se izvršava u redosledu kojim je dodan. Tok izgleda ovako:

*   Zahtev je poslat serveru.
*   Middleware funkcije obrađuju zahtev.
*   Ako middleware pozove `next()`, prenosi kontrolu na sledeći middleware ili rutinski hendler.
*   Ako middleware ne pozove `next()`, ciklus zahtev-odgovor se zaustavlja.

```js
const express = require('express')
const app = express()

// Simple middleware that logs request info
app.use((req, res, next) => {
  console.log('Request URL:', req.url)
  console.log('Request Method:', req.method)
  next() // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

`app.use()`: Dodaje middleware globalno vašoj aplikaciji.

U ovom primeru, middleware loguje detalje zahteva i poziva `next()` da bi nastavio ka sledećem middleware-u ili rutinskom hendleru.

Middleware je moćna funkcija u Express-u koja vam omogućava da presecate i modifikujete zahteve na različitim tačkama. Možete koristiti middleware za rukovanje stvarima kao što su logovanje, autentikacija, pa čak i modifikovati objekte zahteva i odgovora pre nego što stignu do vaših rutinskih hendlera.
