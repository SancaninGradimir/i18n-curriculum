# --interactive--

Flotovi u CSS-u su tehnika prvobitno dizajnirana da omogući tekstu da se savija oko elementa, kao što je slika. Međutim, sa vremenom, programeri su pronašli nove načine korišćenja flotova, primenjujući ih na dizajn rasporeda na kreativne načine. Iako se moderne metode raspoređivanja kao što su Flexbox i Grid sada češće koriste, razumevanje flotova je i dalje važno, posebno kada radite sa starijim kodom ili kada vam treba postići specifične efekte rasporeda.

Kada je element "floatovan", izvlači se iz normalnog protoka dokumenta i gura se levo ili desno od svog kontejnera. Sadržaj koji sledi će se obavijati oko floatovanog elementa, popunjavajući preostali prostor. Jedna klasična upotreba je obvijanje teksta oko slika, gde se slika floatuje na jednu stranu, a tekst se obavija oko nje. Ova tehnika se i dalje široko koristi, posebno u člancima i blogovima gde je potrebno postaviti slike pored teksta. Evo primjera koda:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
</div>
```

```css
.container {
  border: 1px solid black;
  padding: 10px;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

U gornjem primeru, kontejner se ne obavija oko plutajuće slike. Slika je van normalnog dokumentovog protoka, a kontejner se srušio na visinu od nula jer ne "vidi" plutajuće elemente dece.

Flotovi su bili takođe popularni za kreiranje višestubičnih rasporeda pre nego što su Flexbox i Grid postali široko rasprostranjeni. Flotiranjem elemenata jedan pored drugog, programeri su mogli kreirati kolone koje su se horizontalno poravnale. Međutim, prilikom korišćenja flotova, važno je rešiti problem kolapsiranja roditeljskih elemenata kada su njihovi detiljni elementi flotirani. Rešenje tehnike 'clearfix' primenjuje se na element sa klasom `container` kako bi se rešio ovaj problem.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
  <p> This is an example of text flowing around a floated image.</p>
</div>
```

```css
.container {
  border: 1px solid black;
}

/* Clearfix CSS */
.container::after {
  content: "";  
  display: block;
  clear: both;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

- `::after` je pseuoelement koji dodaje nevidljivi blok nakon sadržaja kontejnera.
- `content: ""` osigurava da je pseuoelement prisutan, ali ne prikazuje nikakav sadržaj.
- `display: block` čini pseuoelement blok-nivaj elementom.
- `clear: both` osigurava da pseuoelement čisti obe strane bilo kojih plutajućih elemenata iznad njega.

U gornjem primeru, dodali smo novi paragraf element kako bismo učinili kolaps vidljivijim. Pošto paragraf ostaje u normalnom dokumentovnom protoku, kontejner se proširi dovoljno da ga obuhvati. Zatim primenjujemo tehniku `clearfix` kako bismo popravili kolaps i učinili da se granica kontejnera ispravno prikaže. Možete pokušati da uklonite pravilo `clearfix` da biste videli kako se kontejner ponovo kolapsira.

Tehnika clearfix osigurava da roditeljski element pravilno obuhvati svoje plovajuće potomke. Clearfix prisiljava roditeljski kontejner da "vide" plovajuće elemente deteta dodavanjem svojstva `clear` nakon plovajućeg sadržaja.

Iako floti više nisu primarni metod za složene rasporede zbog savremenijih tehnika kao što su Flexbox i Grid, oni i dalje igraju esencijalnu ulogu u određenim scenarijima. Bilo da tekst obavijate oko slika ili radite na starijim projektima, razumevanje kako floti rade i kako ih pravilno očistiti ključno je za održavanje dobro strukturiranih i responsivnih rasporeda.
