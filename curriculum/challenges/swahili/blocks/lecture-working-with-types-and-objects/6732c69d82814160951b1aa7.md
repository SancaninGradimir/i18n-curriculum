# --interactive--

Konstruktor `Number()` se koristi za kreiranje objekta broja. Objekat broja sadrži nekoliko korisnih svojstava i metoda kao što su `isNaN` i metoda `toFixed`. Evo primera korišćenjem konstruktora `Number()` sa ključnom reči `new`:

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object" 
```

:::

U ovom primeru prosleđujemo string literalu konstruktoru `Number()` i tip povratne vrednosti je tip `object` umesto `string`.

Kada se konstruktor `Number()` pozove kao funkcija bez ključne reči `new`, povratna vrednost će biti primitivni tip broja. Većinom ćete koristiti konstruktor `Number()` za konverziju drugih tipova podataka u tip podataka broja. Evo primera konverzije niza (stringa) u broj:

:::interactive_editor

```js
const myNum = Number("100");
console.log(myNum); // 100

console.log(typeof myNum); // number
```

:::

Ovo je korisno kada dobijate unos od korisnika i morate da konvertujete taj tekstualni unos u broj kako biste mogli da izvršite matematičke proračune.

Ako pokušate da pozovete konstruktor `Number()` preko praznog stringa, onda će rezultat biti broj `0`:

:::interactive_editor

```js
const num = Number("");
console.log(num); // 0
```

:::

Ovo je zato što će JavaScript pokušati da parsira niz i pošto ne sadrži nikakve cifre, rezultat će biti nula.

Ako pokušate da prosledite niz sa nasumičnim karakterima, onda rezultat će biti `NaN` ili "Nije broj".

:::interactive_editor

```js
const num = Number("random");
console.log(num); // NaN
```

:::

Kada radite sa booleanima, `true` vraća `1` jer `true` se konvertuje u jedan i `false` vraća `0` jer `false` se konvertuje u nulu.

:::interactive_editor

```js
const boolTrue = Number(true);
const boolFalse = Number(false);

console.log(boolTrue); // 1
console.log(boolFalse); // 0
```

:::

Ako unesete `null`, rezultat će biti `0` i ako unesete `undefined`, rezultat će biti `NaN`.

:::interactive_editor

```js
const undefinedNum = Number(undefined);
const nullNum = Number(null);

console.log(undefinedNum); // NaN
console.log(nullNum); // 0
```

:::

Kada radite sa nizovima, postoji nekoliko stvari koje je potrebno uzeti u obzir.

Prazan niz će vratiti `0`. Niz sa jednom cifrom vraća taj broj. Niz sa više brojeva vraća `NaN`. I niz sa stringovima takođe će vratiti `NaN`.

:::interactive_editor

```js
const emptyArr = Number([]);
const arrOneNum = Number([7]);
const arrMultiNum = Number([7, 36, 12]);
const arrStr = Number(["str1"]);
const arrMultiStr = Number(["str1", "str2"]);

console.log(emptyArr); // 0
console.log(arrOneNum); // 7
console.log(arrMultiNum); // NaN
console.log(arrStr); // NaN
console.log(arrMultiStr); // NaN
```

:::

Kada radite sa objektima, rezultat je uvek `NaN`.

:::interactive_editor

```js
const obj1 = Number({});
const obj2 = Number({2: 2});
const obj3 = Number({key: "val"});
const obj4 = Number({key: true});

console.log(obj1); // NaN
console.log(obj2); // NaN
console.log(obj3); // NaN
console.log(obj4); // NaN
```

:::

Zaključno, najviše ćete koristiti konstruktor ``Number()`` za konverziju tipa nego kreiranje broja ili objekta broja.
