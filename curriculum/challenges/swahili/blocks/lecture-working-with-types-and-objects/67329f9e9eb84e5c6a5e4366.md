# --interactive--

U prethodnim modulima ste se navikili na rad sa string literalima kao što je ovo:

```js
const greeting = "Hello, World!";
```

Ali JavaScript takođe ima string objekte. I string objekti i string primitivi se koriste za rukovanje tekstom, ali funkcionišu drugačije u svojoj strukturi. String objekat se kreira korišćenjem funkcije konstruktora za stringove, koja umotava primitivnu vrednost u objekat. Evo kako biste kreirali string objekat:

:::interactive_editor

```js
const greetingObject = new String("Hello, World!");

console.log(typeof greetingObject); // "object"
```

:::

Kada koristimo operater `typeof`, možemo videti da je rezultat tipa objekat umesto tipa string. Jedna od stvari o kojima ste možda razmišljali je kako možete koristiti svojstva poput svojstva `.length` na string primitivima.

Kada koristite svojstvo `length` na primitivu niza karaktera, JavaScript privremeno umotava primitivni niz karaktera u objekat za niz karaktera kako bi izvršio operaciju. Zato možete koristiti svojstvo `length` i različite metode kao što su `repeat()`, `concat()` i `slice()`. Ove vrste metoda i svojstava nazivaju se instancijske metode, instancijska svojstva i statičke metode. Naučićete detalje o tome kako to funkcioniše u budućim modulima.

Jedna ključna razlika između objekta za niz znakova i primitivnog niza znakova je kako se odnosi na memoriju i performanse. Primitivni nizovi znakova su obično efikasniji po memoriji i brži u poređenju sa objektima za niz znakova. Iako ćete primarno raditi sa primitivnim nizovima znakova u svom kodu, i dalje je korisno razumeti kako objekti za niz znakova funkcionišu.
