# --description--

U Reactu, inline stilovi se koriste za primenu CSS stilova direktno na React elemente unutar vašeg JSX koda, umesto da ih definišete u zasebnim CSS fajlovima.

Pristup React-a za ugrađene stilove podrazumeva korišćenje JavaScript objekata za definisanje stilova, umesto tradicionalnih CSS stringova. To znači da, umesto pisanja stilova kao što biste to uradili u CSS fajlu, kreirate JavaScript objekat gde su ključevi *camel case* verzije imena CSS svojstava, a vrednosti su stringovi CSS vrednosti.

Ovo je primer kako možete koristiti ugrađene stilove za komponentu `Button`:

```jsx
function Button({ buttonText }) {
  const defaultStyles = {
    backgroundColor: "#007BFF",
    color: "white",
    border: "none",
    borderRadius: "4px",
    padding: "10px 20px",
    fontSize: "16px",
    fontWeight: "bold",
    cursor: "pointer",
    transition: "background-color 0.3s ease",
  };

  return <button style={defaultStyles}>{buttonText}</button>;
}
```

U ovom primeru, definišemo objekat stila nazvan `defaultStyles`. Zatim primenjujemo ove stilove na element dugmeta koristeći atribut `style`. React se brine o primeni ovih stilova na element kada ga renderuje.

Takođe možete izabrati da prosledite objekat direktno atributu ``style``. Evo kako bi izgledao revidirani primer:

```jsx
function Button({ buttonText }) {
  return (
    <button
      style={{
        backgroundColor: "#007BFF",
        color: "white",
      }}
    >
      {buttonText}
    </button>
  );
}
```

Primećujte dvostruke vitičaste zagrade `{{}}` u atributu `style`. Spoljašnje zagrade ukazuju na JavaScript izraz u JSX-u, dok unutrašnje zagrade definišu JavaScript objektnu literalu. Ova sintaksa vam omogućava da direktno umetnete JavaScript objekte u JSX atribute.

Ponekad možda želite da direktno prosledite objekat ako ima samo nekoliko svojstava, kao što je prikazano ovde. Inače, bolje je proslediti ime objektu kao u prvom primeru.

Važno je napomenuti da, iako su imena svojstava CSS-a obično napisana u kebab slučaju, kao što je `font-size`, u React-ovim inline stilovima koristimo camel slučaj, kao što je `fontSize`. To je zato što je objekat stila JavaScript objekat, i imena u kebab slučaju nisu važeća kao ključevi objekta u JavaScriptu bez korišćenja navodnika.

Velika prednost ugrađenih stilova u Reactu je što podržavaju dinamičko stilizovanje zasnovano na stanju komponente ili propovima. Na primer:

```jsx
function DynamicButton({ isActive }) {
  const buttonStyles = {
    backgroundColor: isActive ? "green" : "red",
    color: "white",
    padding: "10px 15px",
    border: "none",
    cursor: "pointer",
  };

  return <button style={buttonStyles}>Login</button>;
}
```

U ovom primeru, pozadinska boja dugmeta se menja na osnovu prop-a `isActive`. Ovaj tip dinamičkog stilizovanja može biti moćan za kreiranje interaktivnih i responsivnih korisničkih interfejsa.

Ukratko, ugrađeni stilovi u Reactu pružaju moćan način za primenu i manipulaciju stilovima direktno unutar vaših komponenti. Oni koriste JavaScript objekte umesto CSS stringova, zahtevaju imena svojstava u formatu 'camel case', i mogu lako da integrišu dinamičke vrednosti. Oni su neophodan alat u repertoaru React developera, posebno za kreiranje visoko prilagođenih i interaktivnih korisničkih interfejsa.
