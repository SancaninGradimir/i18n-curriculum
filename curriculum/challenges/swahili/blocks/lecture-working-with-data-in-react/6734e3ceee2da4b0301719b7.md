# --description--

U prethodnim lekcijama, naučili smo kako da izgradimo male komponente u React-u ovako:

```jsx
function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

Možemo izabrati da ugnježdilimo ovu komponentu unutar druge roditeljske komponente ili korene komponente na ovaj način:

```jsx
function App() {
  return <Greeting />;
}

function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

Iako će ovaj kod normalno raditi i prikazati rezultat od `Hi Jessica!` na ekranu, nije to previše fleksibilan komponent.

Šta ako želimo da prikažemo druga ime kao što su Naomi, Tom ili Oliver? Ovde dolaze React props. Props, što je skraćenica za 'properties' (svojstva), je način na koji roditeljske komponente prosleđuju podatke dete komponenti. Props mogu biti bilo kog tipa: nizovi znakova (strings), brojevi (numbers), logički vrednosti (booleans), objekti ili nizovi (arrays).

Ažurirajmo naš primer iz ranije da sada prihvata i `name` prop:

```jsx
function App() {
  return <Greeting name="Jessica" />;
}

export default App;

function Greeting(props) {
  console.log(props);
  return <h1>Hi {props.name}!</h1>;
}
```

Za podkomponentu nazvanu `Greeting` sada koristimo `props.name` umesto da namenski kodiramo ime `"Jessica"`. Takođe logujemo `props` u konzolu, što se prikazuje kao objekat.

Zatim, unutar roditeljskog `App` komponente, prosleđujemo vrednost na prop `name` tako da se može proslediti detetu. Rezultat će biti isti na ekranu kao i ranije, ali smo sada kreirali fleksibilniji komponent.

Sada imamo mogućnost ponovnog korišćenja detiljne komponente više puta i prosleđivanja različitih imena svaki put:

```jsx
function App() {
  return (
    <>
      <Greeting name="Naomi" />
      <Greeting name="Tom" />
      <Greeting name="Jessica" />
      <Greeting name="Oliver" />
    </>
  );
}
```

Takođe možete izabrati da koristite dekonstrukciju objekata u propovima kako biste ga učinili čitljivijim. Evo kako biste mogli ponovo napisati komponentu `Greeting`:

```jsx
function Greeting({ name }) {
  return <h1>Hi {name}!</h1>;
}
```

Ovaj kod postiže isti rezultat, ali čini jasnijim koje props komponenta očekuje da primi.

React takođe pruža poseban prop nazvan `children`. Bilo koji JSX koji postavite između otvarajućih i zatvarajućih tagova komponente prosleđuje se komponenti kao `children`.

Na primer:

```jsx
function Card({ children }) {
  return <div className="card">{children}</div>;
}

function App() {
  return (
    <Card>
      <h2>Hello</h2>
    </Card>
  );
}
```

U ovom slučaju, JSX unutar `<Card>...</Card>` postaje `children` svojstvo (prop). Izlaz koji se renderuje u DOM-u biće:

```html
<div class="card">
  <h2>Hello</h2>
</div>
```

Ovaj obrazac se često koristi za kompoziciju komponenti, gde komponenta obuhvata druge UI elemente.

Ponekad možete imati mnogo svojstava koja morate proslediti kao propse. Umesto da ih prosleđujete jedno po jedno, možete koristiti operater širenja (`...`), nakon što ih konvertujete u objekat.

Here is an example of a new child component called `DeveloperCard`:

```jsx
function DeveloperCard({ name, age, country }) {
  return (
    <div className="developer-card">
      <h1>Developer: {name}</h1>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}
```

Ova `DeveloperCard` komponenta prihvata tri prop-a: `name`, `age`, i `country`.

U roditeljskoj komponenti `App` , možemo koristiti spread sintaksu za prosleđivanje svih svojstava iz objekta kao pojedinačnih propova u komponentu deteta:

```jsx
function App() {
  const developerObj = {
    name: "Alice",
    age: 30,
    country: "USA",
  };

  return (
    <div className="App">
      <DeveloperCard {...developerObj} />
    </div>
  );
}
```

Ovo je posebno korisno kada radite sa nizovima objekata i prosleđujete više skupova svojstava podkomponente. Na primer, možete imati listu programera gde svaki objekat u nizu ima istu strukturu, ali predstavlja različ osobu.

Naučićete više o tome kako da renderujete liste u nizovima u budućim lekcijama.

Korišćenje propova u Reactu čini vaše komponente fleksibilnijim i ponovo upotrebljivijim, omogućavajući vam da izgradite složenije korisničke interfejse (UI). Međutim, važno je napomenuti da su propovi imutabilni, što znači da se ne mogu promeniti nakon što se proslede komponenti. Ako morate da rukujete unosom korisnika i modifikujete podatke, trebalo bi da koristite stanje (state) umesto toga. Vi ćete saznati više o upravljanju stanjem u budućim lekcijama.
