# --interactive--

Lančano pozivanje metoda je tehnika u kojoj pozivate nekoliko metoda jedna za drugom. Možete koristiti lančano pozivanje metoda na mnogo tipova vrednosti u JavaScriptu, uključujući nizove znakova, nizove i objekte. Iako su nizovi znakova primitivne vrednosti, JavaScript ih privremeno umotava u objekat `String` kada koristite metodu za nizove znakova.

Pogledajmo primer korišćenjem različitih metoda za nizove (stringova):

:::interactive_editor

```js
const result = "  Hello, World!  "
  .trim()
  .toLowerCase()
  .replace("world", "JavaScript");

console.log(result); // "hello, JavaScript!"
```

:::

U ovom primeru, počinjemo sa nizom i izvršavamo tri operacije po redu: uklanjanje razmaka, konverzija u mala slova i zamena niza `world` sa `JavaScript`. Svaka metoda vraća novi niz, koji postaje cilj sledećeg poziva metode.

Metodsko lančanje može značajno poboljšati čitljivost koda prilikom rada sa kompleksnim operacijama.

Na primer, razmotrite ovaj primer korišćenjem metoda `filter`, `map` i `reduce`:

:::interactive_editor

```js
const transactions = [
  { amount: 100, type: "credit" },
  { amount: 20, type: "cash" },
  { amount: 150, type: "credit" },
  { amount: 50, type: "cash" },
  { amount: 75, type: "credit" }
];

const totalCreditWithBonus = transactions
  .filter((transaction) => transaction.type === "credit")
  .map((transaction) => transaction.amount * 1.1)
  .reduce((sum, amount) => sum + amount, 0);

console.log(totalCreditWithBonus); // 357.5
```

:::

U ovom primeru, imamo niz transakcija gde svaki objekat ima iznos i tip kreditne kartice ili gotovine.

Prvo filtriramo transakcije i kreiramo novi niz samo kartičnih transakcija. Zatim, lančujemo metod `map` na filtrirani rezultat i za svaki iznos transakcije, množimo ga sa `1.1`, koji predstavlja bonus `10%`.

Zatim, uzimamo taj rezultat i lančujemo metodu ``reduce`` da bismo sabrali svaku od iznosa što rezultira u ``357.5``.

Iako lančanje metoda može učiniti kod kraćim i čitljivijim, važno je da se koristi promišljeno.

Vrlo duge lance mogu postati teški za debagovanje, jer nije odmah jasno koji korak u lancu može uzrokovati problem. Često je dobra praksa razbiti vrlo duge lance na više koraka radi bolje jasnoće i lakšeg debagovanja.

Takođe možete lančovati metode na objektu. U ovom slučaju, svaka metoda vraća ``this``, što se odnosi na trenutni objekat, omogućavajući sledećoj metodi u lancu da radi na njemu.

:::interactive_editor

```js
const calculator = {
  total: 0,
  add(n) {
    this.total += n;
    return this;
  },
  multiply(n) {
    this.total *= n;
    return this;
  },
  subtract(n) {
    this.total -= n;
    return this;
  },
  getResult() {
    return this.total;
  }
};

const result = calculator.add(5).multiply(2).subtract(3).getResult();
console.log(result); // 7
```

:::

Na ovaj način, svaka metoda vraća isti objekat, tako da možete nastaviti lančanjem poziva jedan za drugim.
