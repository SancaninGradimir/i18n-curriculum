# --interactive--

Petlja ``for...in`` je najbolje korišćena kada vam treba da iterirate preko svojstava objekta. Ova petlja će iterirati kroz sva enumarabilna svojstva objekta, uključujući nasleđene svojstva i nenumerička svojstva.

Nasleđena svojstava je svojstvo koje se nasleđuje iz prototipne lance objekta. Neprečono numeričko svojstvo je svojstvo koje nije broj ili niz koji se može pretvoriti u broj.

Ovo je osnovna sintaksa petlje `for...in`:

```js
for (variable in object) {
  // code block to be executed
}
```

Varijabla u primeru predstavlja trenutno svojstvo objekta preko kojeg se prolazi petljom (ili: preko kojeg se iterira).

Pogledajmo nekoliko primera kako biste bolje razumeli kako radi petlja `for...in`.

U ovom prvom primeru imamo `fruit` objekat i želimo da iteriramo kroz svako svojstvo i logujemo vrednost u konzolu.

:::interactive_editor

```js
const fruit = {
  name: 'apple',
  color: 'red',
  price: 0.99
};

for (const prop in fruit) {
  console.log(fruit[prop]);
}
```

:::

Varijabla `prop` predstavlja trenutno svojstvo objekta. `fruit[prop]` se koristi za pristup vrednosti svakog svojstva.

Za prvu iteraciju, `prop` će biti `name`. Za drugu iteraciju, `prop` će biti `color`, i tako dalje.

Rezultati zapisani u konzolu će biti `apple`, `red` i `0.99`.

U ovom drugom primeru, imamo ugnježeni objekat i želimo da iteriramo kroz svako svojstvo i logujemo vrednost na konzolu.

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

for (const prop in person) {
  console.log(person[prop]);
}
```

:::

Svojstvo `address` je sam po sebi objekat. Petlja `for...in` će takođe iterirati kroz svojstva objekta `person` i logovati ceo objekat `address` u konzolu.

Ovo je kako će rezultat izgledati u konzoli:

```md
John
30
{ street: '123 Main St', city: 'Anytown', state: 'CA' }
```

Ako želite da iterirate po svojstvima objekta `address`, možete ugnježiti petlju `for...in` unutar prve.

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

function isObject(obj) {
  return typeof obj === 'object' && !Array.isArray(obj) && obj !== null;
}

for (const prop in person) {
  if (isObject(person[prop])) {
    for (const nestedProp in person[prop]) {
      console.log(person[prop][nestedProp]);
    }
  } else {
    console.log(person[prop]);
  }
}
```

:::

U ovom primeru, imamo prilagođenu funkciju `isObject` koja proverava da li je vrednost objekat.

Metoda ``Array.isArray`` koristi se za proveru da li je vrednost niz. Postavljanjem logičkog NOT operatora (``!``) ispred metode, proveravamo da li vrednost nije niz.

Razlog zašto ne možemo samo da koristimo ``typeof`` jednako ``'object'`` je zato što su nizovi takođe smatrani objektima u JavaScriptu. Želimo da isključimo nizove iz provere.

Takođe, zbog istorijskog buga u JavaScriptu, `typeof null` vraća `'object'`. Zato želimo da takođe isključimo vrednosti `null` iz provere.

Ako je uslov tačan, gnježimo još jednu petlju `for...in` koja će prolaziti kroz svojstva ugnježdjenog objekta i logovati vrednost na konzolu.

Varijabla `nestedProp` predstavlja trenutno svojstvo ugnježdene objekte.

Evo kako će modifikovani rezultat izgledati u konzoli:

```md
"John"
30
"123 Main St"
"Anytown"
"CA"
```

Petlja ``for...in`` je korisna kada vam treba da iterirate kroz svojstva objekta.

Preporučuje se da se ne koristi `for...in` petlja za iteriranje kroz elemente niza. Umesto toga, koristite `for...of` petlju ili druge metode nizova kao što su `forEach`, `map`, `filter`, i `reduce`, o kojima ćete naučiti u budućim lekcijama.

## --text--

Zašto se generalno ne preporučuje korišćenje petlje ``for...in`` za iteraciju kroz elemente niza?
## --answers--

`for...in` iterira samo nad metodama nizova, a ne nad elementima.
### --feedback--

`for...in` iterates over all enumerable properties, including inherited properties and non-numeric properties, which may lead to unexpected results.

---

`for...in` loops are slower compared to other array iteration methods.
