# --description--

U poslednjoj lekciji, naučili ste o memoizaciji i kako radi kuka ``useMemo``.

U ovoj lekciji, naučićete kako funkcionišu `useCallback` kuka i `React.memo`.

U prošloj lekciji smo takođe pomenuli da `useCallback` je za memorizovanje referenci funkcija.

Za `React.memo`, omogućava vam da memoizujete komponentu kako bi sprečio nepotrebne ponovne renderovanja kada se njegova svojstva (prop) nisu promenila.

Evo osnovne sintakse za `useCallback` hook:

```js
const handleClick = useCallback(() => {
  // code goes here
}, [dependency]);
```

Evo osnovne sintakse za `React.memo`:

```jsx
const MemoizedComponent = React.memo(({ prop }) => {
  return (
    <>
      {/* Presentation */}
    </>
  )
});
```

Pogledajmo primer kuka ``useCallback``:

```jsx
import { useState, useEffect } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    setCount((prevCount) => prevCount + 1);
  };

  useEffect(() => {
    console.log("useEffect runs");
  }, [handleClick]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </div>
  );
}

export default Counter;
```

U komponenti, efekat se pokreće svaki put kada `handleClick` promeni jer funkcija `handleClick` se ponovo kreira pri svakom renderovanju.

Da biste ovo popravili, morate reći React-u da tretira funkciju `handleClick` kao istu tokom renderovanja memorizovanjem je pomoću `useCallback` hooka, tako da se ne kreira ponovo:

```jsx
import { useState, useEffect, useCallback } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  // Memoize the handleClick function with useCallback
  const handleClick = useCallback(() => {
    setCount((prevCount) => prevCount + 1);
  }, []);

  useEffect(() => {
    console.log("useEffect runs");
  }, [handleClick]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </div>
  );
}

export default Counter;
```

Sada funkcija ``handleClick`` se više ne kreira pri svakom renderovanju.

Da vam pokažem kako funkcija višeg reda `React.memo` (ili `memo`) radi i kako se kuka `useCallback` rade u tandemu, evo komponente `Counter` sa funkcijom `handleClick` koja treba `useCallback` ali trenutno to ne koristi:

```jsx
import { useState, useEffect, useCallback } from "react";
import CounterChild from "./CounterChild";

function Counter() {
  const [count, setCount] = useState(0);
  const [timer, setTimer] = useState(new Date().toLocaleTimeString());

  const handleClick = () => {
    setCount(count + 1);
  };

  useEffect(() => {
    const interval = setInterval(() => {
      setTimer(new Date().toLocaleTimeString());
    }, 1000);

    return () => clearInterval(interval);
  }, []);

  return (
    <div>
      <h1>Time: {timer}</h1>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
      <CounterChild onClick={handleClick} />
    </div>
  );
}

export default Counter;
```

Ova funkcija takođe ima tajmer u stanju koji se ažurira svake sekunde. To uzrokuje da komponenta ponovo renderuje svaki put kada se `timer` promeni, što rezultira kreiranjem funkcije `handleClick` pri svakom renderovanju.

Zato što `handleClick` mora da bude memoizovan sa `useCallback`.

Evo `CounterChild` komponente:

```jsx
const CounterChild = ({ onClick }) => {
  console.log("CounterChild component rendered");
  return <button onClick={onClick}>Increment from Child</button>;
};

export default CounterChild;
```

Ovaj `CounterChild` komponent uzima `onClick` prop, dajući vam mogućnost da takođe inkrementirate brojač od njega.

Pošto je komponenta `CounterChild` dete komponente `Counter`, takođe će se renderovati svaki put kada `Counter` ponovo renderuje zbog menjajućeg tajmera. Dakle, i `CounterChild` takođe treba da bude memorizovan.

Bez memorizacije, jer se komponenta ponovo renderuje zbog tajmera koji se ažurira svake sekunde, komponenta `CounterChild` takođe se ponovo renderuje.

Da biste ovo sprečili, morate da memoizujete komponentu `CounterChild` sa `React.memo`:

```jsx
import React from "react";

const CounterChild = React.memo(({ onClick }) => {
  console.log("CounterChild component rendered");
  return <button onClick={onClick}>Increment from Child</button>;
});

export default CounterChild;
```

Stvari još uvek ne rade optimalno čak ni nakon memoizovanja `CounterChild` sa `React.memo`.

Ovo se dešava zato što funkcija ``handleClick`` se ponovo kreira pri svakom renderovanju, pa je takođe potrebno memorizovati pomoću ``useCallback``, radi reaktovanja da mu kažete da vam je potrebna funkcija koja ostaje ista tokom renderovanja:

```js
const handleClick = useCallback(() => {
  setCount((prevCount) => prevCount + 1);
}, [count]);
```

Sada, komponenta se ponovno renderuje samo kada stanje `count` promeni.
