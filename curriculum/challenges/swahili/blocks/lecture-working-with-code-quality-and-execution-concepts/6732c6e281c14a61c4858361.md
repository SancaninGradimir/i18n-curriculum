# --interactive--

Klosure su jedna od najmoćnijih i često nesporazumljivih karakteristika u JavaScriptu. U svojoj srži, klosura je funkcija koja ima pristup varijablama u svom spoljašnjem obuhvatnom leksičkom opsegu, čak i nakon što je spoljna funkcija vratila kontrolu. Ovo možda zvuči komplikovano, ali je to fundamentalni koncept koji omogućava mnoge napredne programerske obrasce u JavaScriptu.

Da bismo razumeli zatvaranje (closures), počnimo sa primerom:

:::interactive_editor

```js
function outerFunction(x) {
    let y = 10;
    function innerFunction(){
        console.log(x + y);
    }
    return innerFunction;
}

let closure = outerFunction(5);
console.log(closure()); // 15
```

:::

U ovom primeru, `outerFunction` uzima parametar `x` i definiše lokalnu varijablu `y`. Zatim definiše `innerFunction` koji koristi i `x` i `y`. Konačno vraća `innerFunction`. Kada pozovemo `outerFunction(5)` on vraća `innerFunction`, koje dodeljujemo varijabli `closure`. Kada kasnije pozovemo `closure()`, i dalje ima pristup `x` i `y` iz `outerFunction` iako `outerFunction` je već završio izvršavanje. Ovo je suština zatvaranja (closure).

Unutrašnja funkcija održava referencu na svoje spoljašnje leksičko okruženje, čuvajući pristup varijablama u tom okruženju čak i nakon što je spoljna funkcija završena.

Zaključanja su posebno korisna za kreiranje privatnih varijabli i funkcija. Razmotrite ovaj primer:

:::interactive_editor

```js
function createCounter() {
    let count = 0;
    return function () {
        count++;
        return count;
    };
}

let counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

:::

U ovom slučaju, `createCounter` vraća funkciju koja inkrementira i vraća varijablu `count`. Varijabla `count` nije direktno dostupna izvan `createCounter`, ali vraćena funkcija (naš zatvorenik/closure) ima pristup njoj. Svaki put kada pozovemo `counter()`, on inkrementira i vraća `count`.

Zaključanja takođe mogu da uhvate više promenljivih iz svog spoljnog opsega. Na primer:

:::interactive_editor

```js
function multiply(x) {
    return function (y) {
        return x * y;
    };
}

let double = multiply(2);
console.log(double(5)); // 10
```

:::

Ovde unutrašnja funkcija hvata parametar `x` iz `multiply`. Kada kreiramo `double` pozivanjem `multiply(2)`, vraća funkciju koja uvek množi svoj argument sa `2`.

Jedna važna stvar koju treba napomenuti o zatvaranjima je da one hvataju varijable, po referenci, a ne po vrednosti. To znači da ako se vrednost uhvaćene varijable promeni, zatvaranje će videti novu vrednost. Na primer:

:::interactive_editor

```js
function createIncrementer() {
    let count = 0;
    return function () {
        count++;
        console.log(count);
    };
}

let increment = createIncrementer();
increment(); // 1
increment(); // 2
```

:::

Svaki put kada pozovemo ``increment``, radi sa istom varijablom ``count``, a ne kopijom njenog početne vrednosti. Zatvorenosti su moćan alat u JavaScriptu. Kako nastavite da radite sa JavaScriptom, shvatićete da razumevanje i efikasno korišćenje zatvorenosti može značajno poboljšati vašu sposobnost pisanja čistog, efikasnog i moćnog koda.
