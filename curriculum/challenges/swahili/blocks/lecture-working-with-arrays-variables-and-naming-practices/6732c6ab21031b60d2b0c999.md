# --description--

Imenovanje promenljivih i funkcija je važan aspekt pisanja čistog, čitljivog i održivog koda. Dobre prakse imenovanja čine vaš kod samodokumentovanim, smanjujući potrebu za opštnim komentarima i olakšavajući drugim programerima, uključujući vas u budućnosti, razumevanje vašeg koda.

Hajde da počnemo sa opštim konvencijama imenovanja u JavaScript-u. U prethodnim lekcijama naučili ste o korišćenju *camel case*-a za imena promenljivih. Za booleanske promenljive, uobičajna praksa je korišćenje prefiksa kao što su `is`, `has` ili `can`. Ovo čitaocu odmah govori da li je promenljiva boolean:

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

Za funkcije, ime bi trebalo jasno da ukazuje šta funkcija radi. Često je korisno početi glagolom:

```js
function getUserData(){
  /* ... */
}

function calculateTotal(){
  /* ... */
}

function validateInput(){
  /* ... */
}
```

Za funkcije koje vraćaju boolean, često nazvane predikati, možete koristiti iste prefikse `is`, `has` ili `can`:

```js
function isValidEmail(email) {
  /* ... */
}

function hasRequiredFields(form) {
  /* ... */
}
```

Kada imate funkcije koje dohvaćaju podatke, uobičajeno je početi rečju `get`:

```js
function getProductDetails(productId) {
  /* ... */
}

function getUserProfile(userId) {
  /* ... */
}
```

Kada imate funkcije koje postavljaju podatke, uobičajeno je početi rečju `set`:

```js
function setUserPreferences(preferences) {
  /* ... */
}

function setPageTitle(title) {
  /* ... */
}
```

Za funkcije za rukovanje događajima (*event handler*), možete dodati prefiks `handle` ili sufiks `handler`:

```js
function handleClick(){
  /* ... */
}

function onSubmit(){
  /* ... */
}

function keyPressHandler(){
  /* ... */
}
```

Rukovaoc događaja (*event handler*) je akcija koja se dešava nakon što je neki događaj nastao, kao što je klik na dugme. Naučićete o tome u budućim lekcijama.

Kada imenujete iteratore promenljive i petlje, uobičajeno je koristiti pojedinačna slova kao što su `i`, `j` ili `k`, ali za ugnježđene petlje ili kompleksnije iteracije mogu biti korisna opisnija imena:

```js
for (let i = 0; i < array.length; i++) {
  /* ... */
}

for (let studentIndex = 0; studentIndex < students.length; studentIndex++) {
  /* ... */
}
```

Za nazive nizova, razmislite o korišćenju množine imenica kako biste ukazali da promenljiva sadrži više stavki:

```js
const colors = ['red', 'green', 'blue'];
const userNames = ['Alice', 'Bob', 'Charlie'];
```

Zapamtite da je cilj učiniti vaš kod što samoujašnjenijim. Dobro pravilo je da ako morate dodati komentar kako biste objasnili šta promenljiva ili funkcija radi, možda želite razmotriti njeno preimenovanje na nešto opisnije.

Napokon, budite dosledni u svojoj bazi koda ili timu. Ako vaš tim ima uspostavljene konvencije imenovanja, pridržavajte se njih. Doslednost čini kod čitljivijim i održivijim za sve uključene.
