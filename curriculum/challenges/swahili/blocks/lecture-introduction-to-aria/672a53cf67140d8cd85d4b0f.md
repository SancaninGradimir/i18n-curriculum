# --interactive--

Činjenje statičkog sadržaja dostupnim može biti relativno jednostavno, ali dinamički sadržaj može biti izazovniji. Ovde ulazi u igru WAI-ARIA.

Pogledaćemo šta je WAI-ARIA, njegova svrha, kako funkcioniše i neke primere.

WAI-ARIA znači Web Accessibility Initiative - Accessible Rich Internet Applications. To je specifikacija koja poboljšava pristupačnost dinamičnog sadržaja i komponenti korisničkog interfejsa (UI).

Napomena da WCAG i WAI-ARIA nisu isto. WCAG pruža opšte smernice za pristupačnost veb stranica, dok WAI-ARIA nudi specifična pravila za omogućavanje pristupačnosti dinamičnog i interaktivnog sadržaja korisnicima asistivnih tehnologija.

Dakle, primarna svrha WAI-ARIA je poboljšanje pristupačnosti za dinamički sadržaj i komponente korisničkog interfejsa koje nemaju ugrađene HTML ekvivalente.

WAI-ARIA funkcioniše uvođenjem skupa atributa koje možete dodati na HTML elemente kako biste pružili dodatne semantičke informacije. Ovi atributi su kategorizovani u uloge, stanja i svojstva.

Uloga ARIA definiše svrhu elementa unutar veb sajta ili web aplikacije. Evo primera postavljanja uloge na `button` za element `div`.

```html
<div role="button">Click Me</div>
```

Time ovim ukazujete asistivnoj tehnologiji da je element dugme. Međutim, uloge ne pružaju nikakvu funkcionalnost. Jednostavno davanje ovog `div` atributa `role` od `button` neće ga učiniti da se ponaša kao dugme. Da biste ga učinili tako da izgleda i ponaša se kao dugme, morali biste koristiti CSS i JavaScript kako biste postigli željeni rezultat.

Ovo je primer korišćenja HTML-a, CSS-a i JavaScript-a za kreiranje prilagođenog `button` elementa.

**NAPOMENA**: Ne brinite zbog pokušaja razumevanja CSS i JavaScript koda. Naučićete te jezike u budućim modulima.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">
<div id="custom-btn" role="button">Click Me</div>
<script src="index.js"></script>
```

```css
#custom-btn {
  display: inline-block;
  padding: 0.4em 1em;
  font-size: 1rem;
  font-family: sans-serif;
  color: buttontext;
  background-color: buttonface;
  border: 1px solid #ccc;
  border-radius: 4px;
  cursor: pointer;
  user-select: none;
  text-align: center;
}

#custom-btn:focus {
  outline: 2px solid Highlight;
  outline-offset: 2px;
}

#custom-btn:active {
  background-color: #ddd;
}
```

```js
const button = document.getElementById("custom-btn");

button.addEventListener("click", () => {
  alert("Button clicked!");
});

button.addEventListener("keydown", (e) => {
  if (e.key === "Enter" || e.key === " ") {
    e.preventDefault(); 
    button.click();
  }
});
```

:::

Uvek je bolje koristiti nativni `button` ili `input` element sa `type="button"` umesto toga.

ARIA svojstva pružaju dodatne detalje o elementima. Na primer, svojstvo `aria-labelledby` vam omogućava da povežete element sa specifičnom oznakom:

```html
<h2 id="header-id">About freeCodeCamp</h2>
<button id="button-id" aria-labelledby="header-id button-id">Learn More</button>
```

Ovo će učiniti elemente razumljivim i navigabilnim za korisnike asistivnih tehnologija.

Da biste najbolje iskoristili WAI-ARIA, pokušajte da se držite nativnog HTML-a kad god je moguće, jer generalno pruža veću pristupačnost "iz kutije".

Koristite WAI-ARIA samo kada je HTML nedovoljan, i ne zaboravite da testirate sa asistivnim tehnologijama kao što su čitači ekrana, ili neka osobe sa invaliditetom testiraju vaš rad. Takođe, uverite se da se stanja i svojstva vašeg WAI-ARIA ažuriraju sa sadržajem u realnom vremenu. Izbegavajte prekomerno korišćenje ARIA, jer to često može biti zbunjujuće.
