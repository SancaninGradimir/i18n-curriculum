# --interactive--

**NAPOMENA**: Neki interaktivni primeri mogu koristiti CSS i JavaScript koji još niste naučili. Ne brinite zbog pokušaja razumevanja celog koda. Cilj primera je da vam pokažu predgleđaje ovih dizajnerskih koncepata kako biste bolje razumeli kako stvari funkcionišu. Da biste videli predgleđaje, moraćete da omogućite interaktivni uresnik.

Beskonačno skrolovanje je dizajnerski obrazac koji učitava više sadržaja kako korisnik skroluje dole po stranici. Često se ovo koristi na sajtovima za društvene mreže kao što su Twitter. Na primer, ako ste prijavljeni i želite da vidite više tvitova, možete skrolovati dole i učitaće se više tvitova. Ovo je primer beskonačnog skrolovanja.

// Simulate loading more content as the user scrolls
window.addEventListener('scroll', () => {
  if (window.innerHeight + window.scrollY >= document.body.offsetHeight) {
    loadMorePosts();
  }

function loadMorePosts() {
  const container = document.querySelector('.infinite-scroll');
  for (let i = 0; i < 3; i++) {
    const post = document.createElement('div');
    post.className = 'post';
    post.textContent = `Post ${container.children.length + 1}`;
    container.appendChild(post);
  }
}

Beskonačno skrolovanje se takođe koristi kao zamena za paginaciju. Paginacija je dizajnerski obrazac koji deli sadržaj na stranice. Ovo se često koristi kada postoji mnogo sadržaja za prikazivanje. Primer paginacije je kada pretražujete nešto na Google-u i vidite rezultate pretrage na više stranica. Sa paginacijom, morate kliknuti na dugme da biste otišli na sledeću stranicu. Sa beskonačnim skrolovanjem, samo nastavite sa skrolovanjem nadole i učitaće se više sadržaja.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="pagination">
  <div class="post">Post 1</div>
  <div class="post">Post 2</div>
  <div class="post">Post 3</div>
  <!-- More posts will be on the next pages -->
</div>
<div class="pagination-controls">
  <button class="prev" disabled>Previous</button>
  <button class="next">Next</button>
</div>
<script src="index.js"></script>
```

```css
.pagination {
  display: flex;
  flex-direction: column;
  gap: 12px;
}

.post {
  padding: 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
}

.pagination-controls {
  margin-top: 12px;
}

.pagination-controls button {
  padding: 8px 16px;
  margin-right: 8px;
}
```

```js
let currentPage = 1;
const postsPerPage = 3;
const totalPosts = 9; 
const totalPages = Math.ceil(totalPosts / postsPerPage);
const container = document.querySelector('.pagination');
const prevButton = document.querySelector('.prev');
const nextButton = document.querySelector('.next'); 

function renderPosts() {
  container.innerHTML = '';
  const start = (currentPage - 1) * postsPerPage;
  const end = start + postsPerPage;
  for (let i = start; i < end && i < totalPosts; i++) {
    const post = document.createElement('div');
    post.className = 'post';
    post.textContent = `Post ${i + 1}`;
    container.appendChild(post);
  }
  prevButton.disabled = currentPage === 1;
  nextButton.disabled = currentPage === totalPages;
}

prevButton.addEventListener('click', () => {
  if (currentPage > 1) {
    currentPage--;
    renderPosts();
  }
});

nextButton.addEventListener('click', () => {
  if (currentPage < totalPages) {
    currentPage++;
    renderPosts();
  }
});

renderPosts();
```

Kada integrišete beskonačno skrolovanje u svoj dizajn, postoji nekoliko najboljih praksi koje treba imati na umu. Prvo razmatranje je pružanje dugmeta „Učitaj više“ koje učitava sledeći set rezultata kada korisnik klikne na njega. Ovo je dobar način da se korisniku pruži kontrola nad onim kada želi da vidi više sadržaja.

Razmislio bi se o dodavanju dugmeta "Nazad". To korisnicima omogućava povratak na prethodnu stranicu bez potrebe da skroluju sve do samog vrha. Ovo stvara bolje korisničko iskustvo i daje im više kontrole nad njihovim preglednim iskustvom.

Ponekad ćete videti dizajne sa dugmetom "Nazad na vrh" koje korisnike vraća na vrh stranice rezultata. Druga razmatranost je pružanje indikatora učitavanja. Korisnici bi trebalo da imaju jasan pokazatelj da se učitava više sadržaja; inače, mogli bi smatrati da je stranica pokvarena.

```js
const container = document.querySelector('.infinite-scroll');
const loadingIndicator = document.querySelector('.loading-indicator');
const backToTopBtn = document.getElementById('back-to-top');

window.addEventListener('scroll', () => {
  if (window.scrollY > 400) {
    backToTopBtn.style.display = 'block';
  } else {
    backToTopBtn.style.display = 'none';
  }

  if (window.innerHeight + window.scrollY >= document.body.offsetHeight) {
    loadMorePosts();
  }
});

backToTopBtn.addEventListener('click', () => {
  window.scrollTo({ top: 0, behavior: 'smooth' });
});

function loadMorePosts() {
  loadingIndicator.style.display = 'block';
  setTimeout(() => {
    for (let i = 0; i < 3; i++) {
      const post = document.createElement('div');
      post.className = 'post';
      post.textContent = `Post ${container.children.length + 1}`;
      container.appendChild(post);
    }
    loadingIndicator.style.display = 'none';
  }, 1000);
}
```
