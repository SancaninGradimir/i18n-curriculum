# --description--

React 19 je došao sa dve značajne nove karakteristike nazvane server komponente i server akcije.

Od te verzije pa nadalje, server komponente su postale podrazumevane u okviromima kao što je Next.js koji ih lako podržavaju.

Server akcije, s druge strane, su funkcije koje se izvršavaju na serveru da bi omogućile obradu formulara direktno na serveru bez potrebe za API endpointima.

Server akcija izgleda ovako:

```js
"use server";

async function submitForm(formData) {
  const name = formData.get("name");
  return { message: `Hello, ${name}!` };
}
```

Ova serverska akcija izvlači polje `name` iz obrasca i vraća niz pozdrava tog imena.

Da bi se pojednostavio upravljanje stanjem za server akcije i uklonio potreba za klijentskim JavaScriptom za jednostavne forme, tim Reacta uveo je kuku `useActionState` u verziji 19.

Pogledajmo bliže ovu kukicu i vidimo kako radi.

Dokumentacija za React opisuje kuku ``useActionState`` kao kuku koja "omogućava ažuriranje stanja na osnovu rezultata akcije forme."

Ali to ne znači da možete koristiti samo ``useActionState`` hak sa formularima. Takođe ga možete koristiti za upravljanje klikovima na dugmad i drugim događajima, sve dok imate akciju postavljenu.

I imajte na umu da, pošto je `useActionState` kuka, ne možete ga koristiti unutar server komponente.

Evo osnovne sintakse za `useActionState` hook:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` je trenutno stanje koje akcija vraća.

- `action` je funkcija koja pokreće server akciju.

- `isPending` je boolean koji ukazuje da li je akcija trenutno u toku ili ne.

- Parametar `actionFunction` jeste sama serverska akcija.

- `initialState` je parametar koji predstavlja početnu tačku stanja pre nego što akcija prođe.

- `permalink` je opcionalni niz znakova koji sadrži jedinstveni URL stranice koji formular modifikuje.

Da biste koristili ``useActionState`` kuka (hook), prvo se uverite da imate akciju. Koristićemo akciju iz prethodnog primera za ovo, sa malim preokretom:

```js
"use server";

export async function submitForm(_, formData) {
  const name = formData.get("name");

  const hour = new Date().getHours();
  let greeting;

  if (hour < 12) {
    greeting = "Good morning";
  } else if (hour < 18) {
    greeting = "Good afternoon";
  } else {
    greeting = "Good evening";
  }

  return { message: `${greeting}, ${name}` };
}
```

U vašoj komponenti, zatim morate uvesti kuka ``useActionState`` i pozvati ga na najvišem nivou tela komponente (pre izjava za povratak) baš kao i druge kuke. Takođe biste trebali uvesti akciju:

```jsx
"use client";

// Import the useActionState hook
import { useActionState } from "react";

// Import the submitForm action
import { submitForm } from "./actions/submitForm";

const Greeter = () => {

 // Initialize the hook
 const [state, submit, isPending] = useActionState(submitForm, {
   message: "",
 });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      {/* Rest of component */}
    </div>
  );
};

export default Greeter;
```

Evo kako izgleda ceo kod sa malo stilizovanja:

```jsx
"use client";

import { useActionState } from "react";
import { submitForm } from "./actions/submitForm";

const Greeter = () => {
  const [state, submit, isPending] = useActionState(submitForm, {
    message: "",
  });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      <form
        action={submit}
        className="bg-white p-6 rounded-2xl shadow-md w-full max-w-md"
      >
        <h2 className="text-2xl text-center font-semibold text-gray-700 mb-4">
          Greet Someone
        </h2>

        <input
          type="text"
          name="name"
          placeholder="Enter your name"
          required
          className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-green-400"
        />

        <button
          type="submit"
          disabled={isPending}
          className="w-full mt-4 p-3 bg-green-500 text-white font-semibold rounded-lg hover:bg-green-600 disabled:bg-gray-400 transition-all"
        >
          {isPending ? "Greeting..." : "Greet"}
        </button>

        {state.message && (
          <p className="mt-4 text-green-600 text-center font-medium">
            {state.message}
          </p>
        )}
      </form>
    </div>
  );
};

export default Greeter;
```

U pretraživaču, videli biste da se vaš dugme za formular menja sa `Greet` na `Greeting...` dok akcija `isPending` – a pozdrav bi prikazao `Good morning, {name}`, `Good afternoon, {name}`, ili `Good evening, {name}`, u zavisnosti od toga u kom vremenskom periodu je formular poslat.

Sećate li se kako smo pomenuli da možete koristiti i ``useActionState`` hak van forme?

U ovom primeru, dohvatićemo pet korisnika sa JSONPlaceholder-a klikom na dugme:

```js
"use server";

export async function getUsers() {
  const res = await fetch(
    "https://jsonplaceholder.typicode.com/users?_start=0&_limit=5/"
  );
  return await res.json();
}
```

Evo stilizovanog korisničkog interfejsa:

```jsx
"use client";

import { useActionState } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        onClick={fetchAction}
        disabled={isPending}
        className="px-4 py-2 cursor-pointer bg-green-500 text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400 font-bold"
      >
        {isPending ? "Fetching Users..." : "Fetch Users"}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

U pretraživaču, videćete da tekst dugmeta nikada se ne ažurira na `Fetching Users...` nakon klika.

Ovo se dešava zato što React tretira dohvaćanje podataka i renderovanje kao viši prioritet od stanja `isPending`, koje blokira `isPending` u procesu i baca grešku.

Da biste rešili ovaj problem, morate umotati akciju u `startTransition`:

```jsx
"use client";

// import startTransition from React
import { useActionState, startTransition } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        {/* wrap fetchAction in startTransition */}
        onClick={() => startTransition(() => fetchAction())}
        disabled={isPending}
        className="px-4 py-2 bg-green-500 font-bold cursor-pointer text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400"
      >
        {isPending ? 'Fetching Users...' : 'Fetch Users'}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

Ako se pitate šta je `startTransition`, to je funkcija koja govori Reactu da je ažuriranje stanja niskoprioritno i da ga je moguće prekinuti. Ovo održava korisnički interfejs (UI) reaktivnim dok obrađuje asinhrono ažuriranja poput akcija na serveru.

Tako se koristi kuka `useActionState` unutar i van obrasca.
