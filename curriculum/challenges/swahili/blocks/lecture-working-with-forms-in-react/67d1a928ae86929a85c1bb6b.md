# --description--

Formulari su osnova svake veb aplikacije jer vam omogućavaju da obrađujete unos korisnika, prikupljate podatke i pokrećete akcije.

U Reactu, forme se upravljaju korišćenjem stanja ili referenci, što vam daje potpunu kontrolu nad njihovim ponašanjem i validacijom. Ova dva načina upravljanja formama nazivaju se "controlled" i "uncontrolled" unos.

Pogledajmo šta su kontrolisani i nekontrolisani ulazi.

Kontrolisani unos je najviše "React-sličan" način za rukovanje unosima forme. Sa kontrolisanim unosima, skladištite vrednost polja za unos u stanje i ažurirate je kroz događaje `onChange`. Ovo vam daje potpunu kontrolu nad podacima forme i omogućava trenutnu validaciju i uslovno renderovanje.

Proces funkcioniše ovako: React održava stanje forme sa `useState` hook-om, a vi ga ažurirate pri svakoj promeni. Kada korisnik ukuca u polje za unos, događaj `onChange` se aktivira, ažurira stanje, a React ponovo renderuje komponentu sa novom vrednošću.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form> 
    </>
  );
}

export default App;
```

Prednosti kontrolisanih ulaza uključuju sledeće:

- Trenutni pristup podacima forme.

- Možete implementirati trenutnu validaciju.

- Možete uslovno onemogućiti dugme za slanje.

- Možete programski kontrolisati vrednost unosa.

Nekontrolisani unosi, s druge strane, su češće viđeni u tradicionalnim HTML formularima. Dakle, umesto da se unosi obrađuju putem `useState` hakova (hook-a), nekontrolisani unosi u HTML održavaju svoj sopstveni interni status uz pomoć DOM-a.

Pošto DOM kontroliše vrednosti unosa, ono što vam je potrebno jeste da dohvatite vrednosti polja za unos sa `ref`. Ovaj pristup zahteva manje koda i bolje radi jer `refs` ne uzrokuju ponovno renderovanje Reacta.

Evo primera nekontrolisanih ulaza:

```jsx
import { useRef } from "react";

function App() {
  const nameRef = useRef();

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(nameRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label htmlFor="name">Your</label>{" "}
      <input type="text" ref={nameRef} id="name" />
      <button type="submit">Submit</button>
    </form>
  );
}

export default App;
```

Jedna vrlo primetna prednost nekontrolisanih ulaza je što zahtevaju manje koda. Takođe bolje funkcionišu i osećaju se prirodnije za početnike u React-u koji su upoznati sa HTML-om.

Pa, koji bi trebalo da koristite između kontrolisanih i nekontrolisanih ulaza?

Koristite kontrolisane ulaze kada vam trebaju dinamičke ažuracije forme, validacija u realnom vremenu, ili kada želite da uskladite vrednosti unosa sa stanjem. Oni pružaju bolju kontrolu, ali zahtevaju više ponovnih renderovanja.

Koristite nekontrolisane inpute kada vam trebaju jednostavniji obrasci, želite da pristupite vrednostima samo prilikom podnošenja forme, ili kada radite sa kodom koji nije React.

Bez obzira na to da li koristite kontrolisane ili nekontrolisane ulaze, evo nekih najboljih praksi kojima bi trebalo da se pridržavate prilikom kreiranja formulara u React-u:

- Uvek sprečite podrazumevano slanje forme.

- Uverite se da validirate unose pre slanja.

- Uvek pružajte jasan povratni odgovor korisnicima sa greškama učitavanja, validacije ili drugim srodnim stanjima.
