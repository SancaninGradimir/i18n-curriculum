# --interactive--

Operator `typeof` u JavaScript-u je jednostavan, ali moćan alat koji vam omogućava da vidite tip podataka varijable ili vrednosti. Uvek vraća niz koji ukazuje na tip.

Pogledajmo nekoliko primera:

:::interactive_editor

```js
let num = 42;
console.log(typeof num); // "number"
```

:::

U ovom prvom primeru, kreirali smo varijablu nazvanu `num` i dodelili joj broj `42`. Kada koristite operator `typeof` na varijabli nazvanoj `num`, vraćaće niz karaktera `number`.

Evo još jednog primera korišćenja operatora `typeof` na varijabli nazvanoj `isUserLoggedIn`:

:::interactive_editor

```js
let isUserLoggedIn = true;
console.log(typeof isUserLoggedIn); // "boolean"
```

:::

Kada se koristi operator `typeof` na varijabli `isUserLoggedIn`, vraćaće string `boolean` jer je booleanski `true` dodijeno varijabli.

Korišćenje operatera ``typeof`` može biti posebno korisno kada debagujete ili pokušavate da razumete kakve vrste podataka radite u svom kodu.

Međutim, postoji poznata neobičnost u JavaScriptu kada je reč o `null`.

Pogledajmo primer:

:::interactive_editor

```js
let exampleVariable = null;
console.log(typeof exampleVariable); // "object"
```

:::

U ovom primeru, imamo varijablu nazvanu `exampleVariable` i dodelili smo joj vrednost `null`. Ali kada koristimo operater `typeof`, vraća niz znakova `object`.

Ovo se široko smatra greškom u JavaScriptu, koja potiče iz njegovih ranih dana. Razlog za ovo ponašanje ukorenjen je u načinu na koji je JavaScript prvobitno dizajniran.

Kada je jezik prvi put implementiran, vrednosti poput `null` su bile predstavljene kao specijalni tip objekta, što je dovelo do ovog neočekivanog rezultata.

Nažalost, ovo je postalo deo jezika, i iako je zbunjujuće, to je nešto o čemu morate biti svesni.
