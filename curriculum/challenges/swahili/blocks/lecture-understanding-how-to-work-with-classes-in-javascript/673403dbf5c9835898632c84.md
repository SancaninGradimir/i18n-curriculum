# --interactive--

Statička svojstva i metode pripadaju samoj klasi, a ne pojedinačnim instancama te klase. Možete im pristupiti direktno preko imena klase bez kreiranja instance te klase. Definisani su unutar klasa kako bi enkapsulirali srodnu funkcionalnost.

Možete definisati statičku metodu upisivanjem ključne reči ``static`` pre imena metode.

```js
class MyClass {
  static staticMethod() { ... }
}
```

Zatim, možete pozvati statičku metodu na klasi direktno, koristeći notaciju tačke i prosleđujući sve potrebne argumente:

```js
MyClass.staticMethod();
```

Primećujete da možete pozvati metodu bez kreiranja instanci klase. To je jedna od ključnih karakteristika statičkih metoda.

Evo primera. Recimo da kreiramo aplikaciju za filmove i želimo da budemo u mogućnosti da poredimo filmove na osnovu njihovog rejtinga/ocene.

Mogli bismo smatriti ovu metodu poređenja kao metodom višeg nivoa koja nije specifična za bilo koji film:

```js
if (movieA.rating < movieB.rating) {
  console.log(`${movieB.title} has a higher rating.`);
} 
```

To je kao opštiji metod vezan za `Movie` klasu.

Radi čitljivosti i održivosti, bilo bi korisno ga definisati unutar klase `Movie` da se svi povezani metodi drže relativno blizu jedni drugima.

Stoga, ovo je savršen kandidat za statičku metodu. Možete ga videti ovde, odmah ispod konstruktora:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
console.log(movieA);
```

:::

Statička metoda je definisana sa ključnom reči `static` i zove se `compareMovies`. Ima dva parametra: `movieA` i `movieB`. Ovo će biti instancama klase `Movie`.

Uporediće ih na osnovu njihove ocene, u opsegu od `0` do `100`. Ova logika je implementirana sa uslovom i štampaće odgovaraju poruku na osnovu toga koji film ima višu ocenu.

Nakon što je metoda definisana, možete je pozvati na klasu. Ali prvo, morate imati argumente spremne i definisane u vašem programu.

U ovom slučaju, metoda prima dve instance filmova kao argumente, pa definišemo ove instance ovde:

```js
let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
```

Možete videti da `movieA` ima viši rejting od `movieB`. Pogledajmo izlaz ove metode.

Da biste pozvali metodu, morate samo da koristite notaciju tačke na samoj klasi.
Napišete ime klase (`Movie`), praćeno tačkom, i zatim ime statičke metode (`compareMovies`).

Zatim, prosleđujete argumente unutar zagrada. U ovom slučaju, to su dve instance filmova koje metoda zahteva.

```js
Movie.compareMovies(movieA, movieB);
```

Evo ažuriranog primera:

:::interactive_editor

  }

    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);

Movie.compareMovies(movieA, movieB);

Statičke metode su takođe korisne za implementaciju "fabričkih" metoda. Fabrička metoda je metoda koju definišete pored konstruktora kako biste kreirali objekte na osnovu specifičnih kriterijuma.

Evo primera sa klasom `Pizza`. Statička metoda `createMargherita` je fabrička metoda koju možete pozvati da biste kreirali instancu pizze Margherita sa već definisanim tipom i cenom.

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}
```

Ovo takođe podiže nešto veoma važno u vezi sa statičkim metodama.

Vrednost ključne reči ``this`` u statičkim metodama je sama klasa, jer statička metoda pripada klasi.

Zato možemo koristiti `this` da kreiramo novu instancu klase `Pizza`.

Ako pozovete ovu metodu na samoj klasi `Pizza` i dodelite vraćenu instancu promenljivoj, kao u ovom primeru:

```js
let myPizza = Pizza.createMargherita();
```

Možete ga koristiti gde god vam zatreba u vašem kodu. Na primer, možete ga ispisati na konzolu:

```js
console.log(myPizza);
```

TEKST:

Ovo je izlaz:

```js
Pizza { type: 'Margherita', price: 6.99 }
```

Takođe možete koristiti notaciju tačke za pozivanje njegovih metoda i pristup njegovim svojstvima, kao što je ovo:

```js
console.log(myPizza.type);
```

Evo celog primera:

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);
console.log(myPizza.type);
```

Pored metoda, možete definisati statičke svojstva sa ključnom reči `static`.

U ovom primeru, imamo statičko svojstvo `numberOfPizzasSold`.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}
```

Ono je statično jer ne pripada nijednom konkretnom primeru pizze, pripada klasi samoj.

Ima početnu vrednost od `0` i ažurira se svaki put kada se kreira nova instanca.

Ako kreirate dve instance pizze, vrednost će biti ažurirana dva puta:

```js
let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");
```

Da biste pristupili vrednosti statičke svojstva, samo je potrebno da koristite notaciju tačke na samoj klasi, jer svojstvo pripada klasi.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}

let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");

console.log(Pizza.numberOfPizzasSold);
```

U ovom slučaju, izlaz je `2` jer su prodate dve pice.

Ovo su osnove statičkih svojstava i metoda u JavaScriptu. Razumevanje statičkih članova je esencijalno za kreiranje ponovo upotrebljivih i efikasnih klasa.

## --text--

Koja je glavna razlika između statičke metode i instance metode u JavaScriptu?
## --answers--

Statičke metode se mogu koristiti samo unutar klase, dok se instancijske metode mogu koristiti izvan klase.
### --feedback--

Razmislite o opsegu i dostupnosti statičkih i instancijalnih metoda.
---

Static methods can access instance properties, while instance methods cannot access instance properties.

### --feedback--

Razmislite o opsegu i dostupnosti statičkih i instancijalnih metoda.
---

Static methods are associated with the class itself, while instance methods are associated with specific instances of the class.

---

Static methods are always public, while instance methods can be public or private.
