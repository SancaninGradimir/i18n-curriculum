# --interactive--

U programiranju, nasleđivanje vam omogućava da definišete klase koje nasleđuju svojstva i metode iz drugih klasa.

Na primer, `Car` je tip od `Vehicle`, pa biste mogli definisati klasu `Vehicle` sa najopštijim svojstvima i metodama koje su zajedničke svim vrstama vozila i zatim definisati klasu `Car` koja nasleđuje svojstva i metode klase `Vehicle`.

Ovaj odnos "je deo" je karakterističan za nasleđivanje. U ovom primeru, `Car` predstavlja specijalizovaniju formu od `Vehicle`.

Postoje dva glavna koncepta o kojima biste trebali znati kako biste počeli da radite sa nasleđem: roditeljska klasa i dete klasa.

Roditeljska klasa je klasa koja deluje kao nacrt za druge klase. Ona definiše svojstva i metode koje nasleđuju druge klase.

Detinijska klasa je klasa koja nasleđuje svojstva i metode druge klase. Detinijske klase takođe mogu proširiti funkcionalnost svojih roditeljskih klasa dodavanjem novih svojstava i metoda.

U našem prethodnom primeru, `Vehicle` bi bio roditeljska klasa za klasu `Car`, dok bi klasa `Car` bila dete klase `Vehicle`.

Roditeljska klasa može imati više podklasa.

U JavaScriptu, koristimo ključnu reč ``extends`` za implementaciju nasleđivanja. Ova ključna reč ukazuje da je klasa podklasa druge klase.

U ovom primeru, u kojem imamo samo prve linije definicija klasa, možete videti da `Car` klasa nasleđuje `Vehicle` klasu, pa `Car` nasleđuje od `Vehicle`.

```js
class Vehicle {
  // Implementation of Vehicle...
}

class Car extends Vehicle {
  // Implementation of Car...
}
```

Recimo da klasa `Vehicle` ima dve svojstva: `brand` i `year`. Ovo možete videti u ažuriranom kodu.

`Car` ima metod nazvan `honk`. Međutim, primetite da klasa `Car` nema svoj konstruktor. Pogledaćemo da li nasleđuje svojstva.

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}
```

Da biste definisali instancu od `Car`, moraćete da prosledite dva argumenta: `brand` i `year`. U ovom slučaju, marka je `freeCodeCamp Motors` a godina je `2019`.

```js
let myCar = new Car("freeCodeCamp Motors", 2019);
```

Ako pokušate da pristupite ovim svojstvima i pozovete metodu koristeći notaciju tačke:

```js
console.log(myCar.brand);
console.log(myCar.year);
myCar.honk();
```

Zaista, izlaz je tačan, pa su ova svojstva i metoda definisana u instanci `Car` i upravo ste sebi uštedeli mnogo ponavljanja koda nasleđujući ova svojstva iz klase `Vehicle`.

U ovom primeru, podklasa nije imala nikakve dodatne svojstva. Zbog toga klasa nije imala konstruktor, već samo metodu.

Ako zaista treba da dodate dodatna svojstva, moraćete da definišete konstruktor.

Pogledajmo primer.


```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}
```

Ovog puta, dodaćemo svojstvo `numDoors` klasičnoj podklasi `Car`.

Možete videti da ova klasa sada ima konstruktor u ažuriranom kodu. Unutar konstruktora, postoji poziv na `super()`, koji prosleđuje argumente `brand` i `year`.

`super()` poziva konstruktor superklase, tako da korišćenjem `super(brand, year)` zapravo definišete svojstva superklase u podklasi. Ovo je još jedna terminologija koju možete koristiti- superklasa je roditeljska klasa, a podklasa je dete klasa.

Zatim, `this.numDoors = numDoors` linija definiše novu svojstvo koje pripada isključivo klasi `Car`, broj vrata (`numDoors`).

Ovo je način za proširenje klase `Vehicle`, dodajući svojstvo koje je ekskluzivno za instance `Car`.

To je primer ekstensibilnosti, jedna od fundamentalnih prednosti nasleđivanja.

Sada, ako kreirate instancu klase `Car` , videćete da ima tri svojstva: `brand` , `year` , i `numDoors`. Takođe ćete morati da prosledite neophodne argumente kada kreirate instancu.

```js
let myCar = new Car("freeCodeCamp Motors", 2019, 4);
```

Ako ispišete vrednosti ovih svojstava sa `console.log()`:

```js
console.log(myCar.brand);
console.log(myCar.year);
console.log(myCar.numDoors);
```

Glavne prednosti nasleđivanja su ponovna upotreba koda, modularnost, ekspanzivnost i poboljšana struktura koda.

Implementiranjem hijerarhije, možete ponovo koristiti kod koji ste već napisali za roditeljsku klasu u ćerkolskoj klasi, izbegavajući ponavljanje.

Nasleđivanje takođe podstiče modularnost razbijanjem složenih sistema na jednostavnije komponente u hijerarhiji.

Takođe, mogućnost proširivanja funkcionalnosti roditeljske klase olakšava prilagođavanje promenljivim zahtevima i dodavanje novih funkcija kasnije tokom procesa razvoja.

Konačno, hijerarhijska struktura može učiniti vaš kod lakšim za razumevanje i održavanje.
