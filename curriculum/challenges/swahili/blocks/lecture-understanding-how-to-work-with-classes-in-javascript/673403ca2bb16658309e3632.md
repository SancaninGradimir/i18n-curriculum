# --interactive--

Naučili ste da su klase kao nacrti koje možete koristiti za kreiranje objekata.

Ovi nacrti zahtevaju opšti način za referisanje na specifični objekat koji se kreira, pristupa ili modifikuje unutar koda. To je upravo za šta se koristi ključna reč `this`. Odnosi se na kontekst u kojem bi kod trebalo da radi. Ako ga koristite unutar metode, vrednost `this` će biti referenca na objekat povezan sa tom metodom. Na ovaj način, možete pristupiti svojstvima i metodama objekta unutar klase i ponovo koristiti metodu na različitim objektima.

Na primer, ovde imamo `Dessert` klasu. Ovim putem, definišemo `hasPeanuts` svojstvo.

```js
class Dessert {
  constructor(hasPeanuts) {
    this.hasPeanuts = hasPeanuts;
  }
}
```

Primećujte da koristimo ključnu reč ``this`` unutar konstruktora. Zapamtite da se konstruktor automatski poziva kada se kreira novi objekat klase.

Ključna reč ``this`` govori programu da ``hasPeanuts`` treba da bude svojstvo objekta koji se trenutno kreira.

Desna strana izjave za dodelu vrednost parametra `hasPeanuts` dodeljuje svojstvu `hasPeanuts` novog objekta.

```js
this.hasPeanuts = hasPeanuts;
```

To je u kontekstu konstruktora, ali možete koristiti i ključnu reč ``this`` unutar drugih metoda.

Ovo je naša `Dessert` klasa sa nekoliko promena. Sada imamo dve osobine: `name` i `price`.

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is ${this.price}.`);
  }
}
```

Takođe imamo novu metodu `showPrice`. U ovoj metodi, ispišemo deskriptivnu poruku na konzolu sa imenom i cenom objekta deserta.

Primećujete kako koristimo ključnu reč `this` za pristup svojstvima objekta unutar metode `showPrice`.

Koja vrednost će ključna reč `this` imati u kontekstu ove metode? Odnosiće se na objekat na kojem se poziva metoda. U ovom slučaju, kreiramo novu instancu klasi Dessert sa imenom `"Brownie"` i cenom `$5.99`:

```js
const brownie = new Dessert("Brownie", 5.99);
```

Ako pozovemo metodu `.showPrice()` na ovom objektu:

```js
brownie.showPrice();
```

Evo celog primera:

:::interactive_editor

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is ${this.price}.`);
  }
}

const brownie = new Dessert("Brownie", 5.99);
brownie.showPrice();
console.log(brownie);
```

:::

Ime i cena trenutnog objekta (`brownie`) se zamenjuju u nizu karaktera.

Ovo je sve zahvaljujući ključnoj riječi `this`, koja vam daje opšti način da se osvrnete na trenutni objekat unutar klase. Ovo vam omogućava ponovno korišćenje metode na različitim objektima.

Ovo je u kontekstu metoda objekata, ali ključna reč ``this`` će takođe imati vrednost u samostalnim funkcijama i streličitim funkcijama.

Njegova vrednost unutar samostalne funkcije obično se odnosi na global objekat (u ne-strogom režimu) ili `undefined` (u strogom režimu).

Vrednost ključne reči ``this`` je malo drugačija u streličitim funkcijama. Streličite funkcije nasleđuju vrednost ``this`` iz okružnog opsega gde su definisane. One ne kreiraju svoje sopstveno vezivanje (binding) ``this``. To znači da vrednost ``this`` zavisi od konteksta gde je streličita funkcija definisana, a ne od mesta ili načina na koji se streličita funkcija poziva. To čini streličite funkcije korisnim za povratne pozive (callbacks) ili za očuvanje konteksta.

Međutim, zbog toga, vrednost `this` neće ukazivati na trenutni objekat u objekt metodama, pa je ovo nešto što biste trebali imati na umu kada radite sa streličitim funkcijama.

Isto važi i za strelične funkcije definisane unutar drugih funkcija. Nasleđiće vrednost ``this`` iz svog roditeljskog opsega.

Ključna reč `this` je moćan alat za pristup i manipulisanje objektima. Razumevanje kako funkcioniše je esencijalno za savladavanje objekto-usmerenog programiranja u JavaScriptu.
