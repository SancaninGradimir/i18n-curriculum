# --description--

U Express-u, `express.Router()` je moćna funkcija koja vam pomaže da organizujete svoje rute u manje, modularne delove. Ovo je posebno korisno kako vaša aplikacija raste.

Umesto definisanja svih vaših ruta u jednom velikom fajlu, možete ih razdvojiti u zasebne fajlove koristeći rutere (routers). Ovo čini vaš kod lakšim za čitanje, održavanje i skaliranje.

Metoda `express.Router()` kreira novi router objekat.

Router radi kao mini Express aplikacija. Može da obrađuje sopstvene rute i middleware-e, ali nije punopravna Express aplikacija. Kada definišete rute na routeru, možete ga montirati u vašoj glavnoj aplikaciji koristeći `app.use()`.

Drugim rečima, router vam omogućava da grupišete povezane rute zajedno, a kasnije ih integrišete u svoju glavnu aplikaciju.

Korišćenje rutera ima nekoliko prednosti:

**Razdvajanje briga (Separation of concerns)**

Možete definisati rute u zasebnim fajlovima, što poboljšava organizaciju i čitljivost vaše aplikacije.

**Ponovna upotreba koda (Code reusability)**
Ruteri se mogu ponovo koristiti u različitim delovima aplikacije ili čak preko projekata.

**Čistiji glavni fajl (Cleaner main file)**
Umesto da popunjavate svoj glavni `app.js` ili `server.js` fajl sa mnogo definicija ruta, možete ih prebaciti u rutere i zadržati vaš glavni fajl fokusiranim na konfiguraciju i podešavanje.

Pogledajmo jednostavan primer gde koristimo `express.Router()` za definisanje ruta za putanju `/users`.

Prvo, evo fajla `app.js`:

```javascript
const express = require("express");
const app = express();
const port = 3000;

// Importing the user routes
const userRoutes = require("./userRoutes");

// Using the userRoutes router for /users path
app.use("/users", userRoutes);

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

A ovo je `userRoutes.js`:

```javascript
const express = require("express");
const router = express.Router();

// GET request for /users
router.get("/", (req, res) => {
  res.send("List of users");
});

// POST request for /users
router.post("/", (req, res) => {
  res.send("Creating a new user");
});

// GET request for /users/:id
router.get("/:id", (req, res) => {
  const userId = req.params.id;
  res.send(`User details for user with id: ${userId}`);
});

module.exports = router;
```

Sada hajde da pogledamo bliže kako sve funkcioniše.

U `app.js`, uvozimo `express` i kreiramo instancu Express aplikacije, dodeljujući je promenljivoj pod nazivom `app`.

Zatim, uvozimo router `userRoutes` iz `userRoutes.js` i koristimo `app.use("/users", userRoutes)` da montiramo (mount) router. Ovo govori Express-u da, ako zahtev počinje sa `/users`, prosledi ga `userRoutes` routeru.

Zatim u `userRoutes.js`, kreiramo novi router koristeći `express.Router()` i definišemo rute korišćenjem `router.get()`, `router.post()` i `router.get("/:id")`.

Napomena da su definisane rute ovde relative (relativne) u odnosu na `/users` jer se router tamo montira. Dakle:

*   `router.get("/")` → `GET /users`: Odgovara listom korisnika
    
*   `router.post("/")` → `POST /users`: Odgovara porukom o kreiranju novog korisnika
    
*   `router.get("/:id")` → `GET /users/:id`: Odgovara detaljima korisnika za dati ID korisnika
    

Korišćenje `express.Router()` održava vaš glavni fajl čistim i fokusiranim na konfiguraciju. Sve rute vezane za korisnike grupisane su u `userRoutes.js`.

Kako vaša aplikacija raste, možete kreirati dodatne rutere za druge sekcije, kao što su `/products`, `/orders` ili `/auth`.

Svaka sekcija ostaje organizovana u svom fajlu, što čini aplikaciju lakšom za održavanje i skaliranje.
