# --description--

Sada kada ste izgradili osnovnu Express aplikaciju, vreme je da detaljnije pogledamo rutiranje i kako funkcioniše.

U Express-u, rutiranje je način na koji definišete krajnje tačke (endpoints) web aplikacije. Možete razmišljati o rutama kao o pravilima koja aplikaciji govore kako da odgovori kada korisnik poseti određenu URL.

Pogledajmo ponovo našu jednostavnu Express aplikaciju:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

U ovom primeru, aplikacija definiše rutu za korenske URL (`/`). Kada klijent pošalje GET zahtev na ovu putanju, server odgovara tekstom `Hello World!`. Aplikacija zatim počinje da sluša dolazne zahteve na portu `3000`.

Ali postoje i druge metode rute.

Express vam omogućava da rukujete različitim tipovima HTTP zahteva korišćenjem različitih metoda rutiranja. Najčešće su:

* `app.get()` – dohvaća podatke
    
* `app.post()` – kreira nove podatke
    
* `app.put()` – ažurira postojeće podatke
    
* `app.delete()` – briše podatke
    

Svaka metoda definiše kako vaša aplikacija treba da odgovori na određeni HTTP zahtev na određenoj ruti.

Evo nekoliko jednostavnih primera:

```javascript
app.post("/submit", (req, res) => {
  res.send("Form submitted!");
});

app.put("/update", (req, res) => {
  res.send("Item updated!");
});

app.delete("/delete", (req, res) => {
  res.send("Item deleted!");
});
```

U svakom slučaju, Express sluša zahtev sa određenom HTTP metodom i URL putanjom, a zatim pokreće odgovarajuću funkciju rukovatelja (handler function) kako bi poslao odgovor nazad klijentu.

Možete testirati ove rute koristeći alate kao što su Postman ili `curl`, koji vam omogućavaju da pošaljete različite tipove HTTP zahteva na vaš server.

I to je rutiranje u Express-u – jednostavno je definisanje načina na koji vaša aplikacija odgovara na različite zahteve.
