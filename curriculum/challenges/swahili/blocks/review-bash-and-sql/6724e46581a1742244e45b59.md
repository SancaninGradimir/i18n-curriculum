## Normalizacija baze podataka

Radi se o procesu organizovanja relacijske baze podataka radi smanjenja redundancije podataka i poboljšanja integriteta.

Njegove prednosti uključuju:

- Minimalizovanje dupliranih podataka, što štedi skladište i smanjuje nekonzistentnosti.
- Održavanje integralnosti podataka korišćenjem primarnih i stranih ključeva.
- Činjenje baza podataka lakšim za održavanje i razumevanje.

### Normalne forme

- **1NF (Prva normalna forma)**
  - Svaka ćelija sadrži jednu (atomsku) vrednost.
  - Svaki zapis je jedinstven (što je nameće primarni ključ).
  - Redosled redova/kolona nije relevantan.
  - Primer: Premestite više telefonskih brojeva iz tabele `students` u zasebnu tabelu `student_phones`.

- **2NF (Druga Normalna Forma)**
  - Ispunjava zahteve 1NF.
  - Nema **delimičnih zavisnosti**: svaki atribut koji nije ključ mora zavisiti od celog kompozitnog primarnog ključa.
  - Primer: Podelite tabelu `orders` na `order_header` i `order_items` kako biste izbegli da atributi zavise samo od dela ključa.

- **3NF (Third Normal Form)**
  - Ispunjava zahteve 2NF.
  - Nema **tranzitivnih zavisnosti**: atributi koji nisu ključni ne smeju zavisiti od drugih atributa koji nisu ključni.
  - Primer: Premestite `city_postal_code` u tabelu `cities` umesto da ga čuvate sa svakom porudžbinom.

- **BCNF (Norma Boyce-Codda)**
  - Ispunjava zahteve 3NF.
  - Svaki determinator (leva strana funkcionalne zavisnosti) mora biti superključ.

**Savet**: Ciljajte na 3NF u većini dizajna radi dobrog balansa između integriteta i performansi.

## Ključni SQL Koncepti

- SQL je skraćenica za Structured Query Language, jezik za komunikaciju sa relacionalnim bazama podataka.
- **Osnovne komande** → `SELECT`, `INSERT`, `UPDATE`, `DELETE`, `CREATE TABLE`, `ALTER TABLE`, itd.
- **Spajanja (Joins)** → Kombinuje podatke iz više tabela (`INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `FULL JOIN`).

## Pokretanje SQL Komandi u Bash-u

Možete izvršiti SQL komande direktno sa komandne linije koristeći klijenta za komandnu liniju `psql` za PostgreSQL ili slične alate za druge baze podataka.

Na primer, da biste izvršili SQL fajl u PostgreSQL-u:

```bash
psql -U username -d database_name -c "SELECT * FROM students;"
```

Takođe možete izvršiti MySQL komande direktno:

```bash
mysql -u username -p database_name -e "SELECT * FROM students;"
```

### Izvršavanje SQL upita iz fajla

```bash
# PostgreSQL
psql -U username -d database_name -f script.sql

# MySQL
mysql -u username -p database_name < script.sql
```

### Ugrađivanje SQL upita u Bash skriptu

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Insert student data
psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age, major) VALUES ('Alice', 20, 'CS');"
```

## Dohvatanje i korišćenje rezultata SQL upita u Bash-u

Kada pokrenete SQL upite preko ``psql``, možete prikupiti i obraditi vraćene vrednosti u vašim Bash skriptama.

### Dobijanje jedne vrednosti

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get total student count
STUDENT_COUNT=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -c \
"SELECT COUNT(*) FROM students;")

echo "Total students: $STUDENT_COUNT"
```

Izlaz → 42

### Dohvatanje više kolona

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get top 3 students' names and ages
RESULTS=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, age FROM students LIMIT 3;")

echo "Top 3 students:"
echo "$RESULTS"
```

Izlazak

```bash
Alice,20
Bob,21
Charlie,22
```

### Iteriranje kroz rezultate upita

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get student names and majors
psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, major FROM students;" | while IFS="," read -r name major
do
    echo "Student: $name | Major: $major"
done
```

Oblik izlaza

```bash
Student: Alice | Major: CS
Student: Bob   | Major: Math
Student: Carol | Major: Physics
```

## SQL Injekcija

To je web sigurnosna ranjivost u kojoj napadači ubacuju zlonamerni SQL kod u polja za unos kako bi manipulisali bazom podataka.

Ovo može dovesti do rizičnih akcija, kao što su:

- Zaobilazak autentikacije.
- Krađa osetljivih podataka.
- Izmenjivanje ili brisanje zapisa.

Primer napada tipa SQL injekcije:

```sql
SELECT * FROM users WHERE username = ' " " OR "1"="1" -- ' AND password = 'anything';
```

Ovaj upit bi vratio sve korisnike jer je uslov ``OR "1"="1"`` uvek tačan, što omogućava napadačima zaobilaženje provera prijavljivanja.

### Sprečavanje SQL injekcije

1. **Koristite pripremljene izjave**: One razdvajaju SQL kod od podataka, sprečavajući injekciju. Evo primera (Node.js sa pg):

    ```js
    client.query('SELECT * FROM users WHERE username = $1 AND password = $2', [username, password]);
    ```

2. **Validacija unosa**: Sanitizujte i validirajte sve korisničke unose kako biste osigurali da odgovaraju očekivanim formatima.

3. **Princip najmanjih privilegija**: Koristite račune baze podataka sa minimalnim dozvolama koje su neophodne za aplikaciju.

**Napomena**: Nikada ne dodeljujte administratorska prava aplikacionim nalogima.

## Problem N+1

Problem N+1 nastaje kada aplikacija izvrši jedan upit radi dohvaćanja liste stavki (N), a zatim za svaku stavku izvrši dodatni upit radi dohvaćanja povezanih podataka, što rezultira sa ukupno N+1 upita.

**Zašto je loše**

- Svaki upit dodaje mrežni i obradni nadopadak.
- Više malih upita je sporije nego jedan optimizovani upit.

### Primer N+1 obrasca

```sql
-- 1: Get list of orders
SELECT * FROM orders LIMIT 50;

-- N: For each order, get customer
SELECT * FROM customers WHERE customer_id = ...;
```

**Rešenje**: Koristite `JOINs` ili druge operacije zasnovane na skupovima.

```sql
SELECT 
  orders.order_id,
  orders.product,
  orders.quantity,
  customers.customer_id,
  customers.name,
  customers.email,
  customers.address
FROM orders
JOIN customers 
  ON orders.customer_id = customers.customer_id
WHERE orders.order_id IN (SELECT order_id FROM orders LIMIT 50);
```

Uvek potražite mogućnosti da kombinujete povezane podatke u jedan upit.
