# --interactive--

JavaScript je jezik u kojem stvari ponekad rade na iznenađujući, pa čak i čudan način. Takva iznenađenja se dešavaju kada pomešate brojeve i nizove karaktera u proračunima. Prva stvar koju ćete verovatno pokušati je dodavanje broja i niza karaktera. U JavaScriptu, operater ``+`` obavlja dvostruku ulogu. On obrađuje i sabiranje i konkatenaciju nizova karaktera, što je način spajanja dva niza karaktera zajedno.

Kada koristite `+` sa brojem i nizom karaktera (string-om), JavaScript odlučuje da ih oba tretira kao nizove karaktera i spaja ih. Ako proverite tip rezultata sa operaterom `typeof`, videćete da je zaista niz karaktera:

:::interactive_editor

```js
const result = 5 + '10';

console.log(result); // 510
console.log(typeof result); // string
```

:::

Šta misliš da će se desiti ako promeniš redosled između ``5`` i ``'10'``?

:::interactive_editor

```js
const result = '10' + 5;

console.log(result); // 105
console.log(typeof result); // string
```

:::

Možete videti da se desilo isto. JavaScript vidi niz u `'10'` i broj u `5`, pa konvertuje broj u niz i spaja ih. Ovo je poznato kao tipsko prisiljavanje. Tipsko prisiljavanje je kada se vrednost iz jednog tipa podataka konvertuje u drugi.

Stvari postaju zanimljivije kada pokušate da izvršite druge aritmetičke operacije kao što su oduzimanje, množenje ili deljenje sa nizom i brojem. U ovim slučajevima, JavaScript pokušava da konvertuje niz u broj pre nego što obavi matematiku – još jedna vrsta koercije tipa! Evo šta se dešava:

:::interactive_editor

```js
const subtractionResult = '10' - 5;
console.log(subtractionResult); // 5
console.log(typeof subtractionResult); // number

const multiplicationResult = '10' * 2;
console.log(multiplicationResult); // 20
console.log(typeof multiplicationResult); // number

const divisionResult = '20' / 2;
console.log(divisionResult); // 10
console.log(typeof divisionResult); // number
```

:::

U gornjim primerima, JavaScript uspešno konvertuje niz (string) `'10'` ili `'20'` u broj, a zatim izvrši izračunavanje. Kao rezultat, `'10' - 5` rezultira sa `5`, `'10' * 2` daje `20`, i `'20' / 2` rezultira sa `10`.

Ali šta ako string ne izgleda kao broj? Pogledajmo šta se dešava u tom slučaju:

:::interactive_editor

```js
const subtractionResult = 'abc' - 5;
console.log(subtractionResult); // NaN
console.log(typeof subtractionResult); // number

const multiplicationResult = 'abc' * 2;
console.log(multiplicationResult); // NaN
console.log(typeof multiplicationResult); // number

const divisionResult = 'abc' / 2;
console.log(divisionResult); // NaN
console.log(typeof divisionResult); // number
```

:::

U gornjim primerima, string `'abc'` ne predstavlja validnu numeričku vrednost, pa JavaScript ne može da ga konvertuje u smislen broj. Kada ta konverzija ne uspe, JavaScript vraća `NaN`, što označava "Not a Number" (Nije broj). `NaN` je posebna vrednost tipa `Number` koja predstavlja nevažeći ili nereprezentabilan broj.

Šta ako izvršite aritmetičke operacije sa booleanom (`true` ili `false`)? Pogledajmo šta se dešava. JavaScript tretira booleane kao brojeve u matematičkim operacijama: `true` postaje `1`, a `false` postaje `0`.

:::interactive_editor

```js
const result1 = true + 1;
console.log(result1); // 2
console.log(typeof result1); // number

const result2 = false + 1;
console.log(result2); // 1
console.log(typeof result2); // number

const result3 = 'Hello' + true;
console.log(result3); // "Hellotrue"
console.log(typeof result3); // string
```

:::

U prvih dva primera, `true + 1` rezultovalo je u `2`, i `false + 1` rezultovalo je u `1`. U trećem primeru, `'Hello' + true`, JavaScript konvertovao `true` u niz i spaja ga sa `'Hello'`, što rezultira `'Hellotrue'`, koji je niz.

Za `null` i `undefined`, JavaScript tretira `null` kao `0` i `undefined` kao `NaN` u matematičkim operacijama:

:::interactive_editor

```js
const result1 = null + 5;
console.log(result1); // 5
console.log(typeof result1); // number

const result2 = undefined + 5;
console.log(result2); // NaN
console.log(typeof result2); // number
```

:::

JavaScript često vrši koerciju tipova, automatski konvertujući tipove podataka kao što su brojevi, nizovi i logičke vrednosti na ponekad neočekivane načine. Razumevanje ovih konverzija je ključno za izbegavanje grešaka i pisanje robusnog koda u vašim projektima.
