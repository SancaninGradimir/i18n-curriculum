# --interactive--

Skrivanje sadržaja na veb stranici je uobičajena praksa u web razvoju, ali je ključno to uraditi na način koji ne kompromituje pristupačnost. Različite tehnike skrivanja mogu imati različite uticaje na način na koji asistivne tehnologije tumače i predstavljaju sadržaj korisnicima. Istražićemo neke najbolje prakse za skrivanje sadržaja uz održavanje pristupačnosti. Jedan uobičajen metod za skrivanje sadržaja je korišćenje ``display: none``. Evo primera:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  display: none;
}
```

:::

Iako ovo vizuelno skriva sadržaj, uklanja ga i iz stabla pristupačnosti. Stablo pristupačnosti je struktura koju koriste asistivne tehnologije, kao što su čitači ekrana, za tumačenje i interakciju sa sadržajem na veb stranici. Ono predstavlja sadržaj i njegovo semantičko značenje na način koji asistivna tehnologija može razumeti i predstaviti korisniku.

Korišćenje `display: none` znači da čitači ekrana i druge asistivne tehnologije neće moći pristupiti ovom sadržaju, jer nije uključen u stablo pristupačnosti. Stoga je važno koristiti ovu metodu samo kada želite da potpuno uklonite sadržaj iz vizuelne prezentacije i pristupačnosti.

Drugi pristup za skrivanje sadržaja je korišćenjem `visibility: hidden`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  visibility: hidden;
}
```

:::

`visibility: hidden` skriva sadržaj vizuelno, ali ga zadržava u protoku dokumenta, što znači da i dalje zauzima prostor na stranici. Kao `display: none`, `visibility: hidden` takođe uklanja sadržaj iz stabla pristupačnosti. To znači da asistivne tehnologije, kao što su čitači ekrana, neće moći pristupiti skrivenom sadržaju. Koristite samo `visibility: hidden` kada želite da sakrijete sadržaj od svih, uključujući ljude koji koriste asistivne tehnologije.

Za sadržaj koji treba da bude vizuelno sakriven, ali ostane dostupan čitačima ekrana, možete koristiti tehniku koja se često naziva „vizuelno sakriveno“ ili „samo za čitače ekrana“. Evo primera korišćenjem CSS klase ``.sr-only``, što je uobičajena tehnika koja se koristi za vizuelno skrivanje sadržaja, zadržavajući ga dostupan čitačima ekrana:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="sr-only">Hidden text</p>
<p>Visible text</p>
```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

:::

U ovom primeru, koristimo svojstva kao što su `position`, `clip` i `white-space`, o kojima ćete naučiti u budućim lekcijama. Za sada, samo znajte da ovo CSS pravilo efektivno skriva sadržaj vizuelno, a istovremeno ga održava dostupnim za čitače ekrana (screen reader-e). Korisno je za pružanje dodatnog konteksta korisnicima čitača ekrana bez uticaja na vizuelni raspored.

Za uključivanje/isključivanje vidljivosti sadržaja, razmislite o korišćenju atributa `hidden`:

:::interactive_editor

```html
<p hidden>This content is hidden</p>
<p>This content is visible</p>
```

:::

Atribut ``hidden`` je podržan od strane većine modernih pregledača i skriva sadržaj kako vizuelno, tako i iz stabla pristupačnosti. Može se lako kontrolisati pomoću JavaScript-a. Napokon, budite oprezni sa skrivanjem važnog sadržaja. Ako su informacije ključne za razumevanje ili korišćenje veb-sajta, one bi trebalo da budu vidljive i dostupne svim korisnicima. Skrivajte sadržaj samo kada to zaista poboljšava korisničko iskustvo. Prateći ove najbolje prakse, možete osigurati da vaš sadržaj ostane dostupan svim korisnicima, bez obzira na način na koji sajt koriste.
