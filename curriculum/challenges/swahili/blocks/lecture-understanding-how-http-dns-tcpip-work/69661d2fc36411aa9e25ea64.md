# --description--

Naučićemo se o uobičajenim HTTP kodovima odgovora.

HTTP kodovi odgovora su trocifreni numerički kodovi koje web serveri koriste za komunikaciju sa klijentima kako bi ih obavestili šta se dogodilo sa njihovim zahtevima.

Oni pružaju informacije o tome da li je zahtev uspešno obrađen ili da je došlo do greške tokom obrade zahteva.

Pokrićemo neke od najčešćih u ovoj lekciji.

## Kodovi uspeha (Success Codes)

Počnimo sa **kodovima uspeha**. Ovi kodovi počinju brojem 2.

Kod **200 OK** je najčešći HTTP kod odgovora. To znači da je zahtev uspešan i da je server vratio informacije koje je tražio klijent.

Kod **201 Created** vraća ga server kada klijent traži kreiranje resursa, a proces se uspešno završi. Na primer, ako se uspešno kreira novi korisnički nalog, server vraća ovaj kod odgovora klijentu.

Kod **204 No Content** vraća se kada je zahtev uspešno obrađen, ali server nema sadržaj za vraćanje. Ovo se može koristiti za procese koji brišu ili ažuriraju resurs u bazi podataka, ali nemaju podatke za vraćanje.

## Kodovi preusmeravanja (Redirection Codes)

Sada ćemo pričati o **kodovima preusmeravanja**.

Ovi kodovi ih server vraća kada je resurs koji traži klijent trajno premešten na druga mesto ili kada klijent mora da preduzme dodatne korake kako bi zavr시오o zahtev.

Ovi kodovi počinju brojem 3.

Kod **301 Moved Permanently** vraća se kada je resurs koji je tražio klijent trajno premešten na drugo mesto. U ovom slučaju, server bi trebalo da uključi URL do novog mesta u odgovor i nakon što klijent primi odgovor, korisnik bi trebalo da bude preusmeren na taj URL.

Kod **302 Found** ukazuje da je resurs koji je tražio klijent privremeno premešten na drugo mesto. Server bi trebalo da pruži novi URL u odgovoru i klijent bi trebalo da preusmeri korisnika na taj URL. Međutim, originalni URL možda će biti dostupan u budućnosti jer je promena privremena. To je glavna razlika između kodova odgovora 301 i 302.

Kod **304 Not Modified** vraća ga server kako bi rekao klijentu da resurs koji je tražen nije modifikovan od poslednjeg puta kada je zahtevan. Server šalje ovaj kod kako bi izbegao slanje podataka koji su već poslati ako se sadržaj nije promenio. To je tehnika optimizacije performansi.

## Kodovi greške klijenta (Client Error Codes)

Sada ćemo početi da istražujemo **kodove greške**.

Greške mogu biti uzrokovane od strane klijenta i od strane servera. Prvo ćemo pričati o kodovima greške klijenta.

Kodovi greške klijenta vraćaju se kada zahtev nije mogao biti završen zbog problema generisanog od strane klijenta.

Ovi kodovi greške počinju brojem 4.

Kod **400 Bad Request** vraća se kada server nije mogao da razume zahtev poslat od strane klijenta. Ovo se može dogoditi kada klijent pošalje zahtev sa neispravnom sintaksom, sa nevažećim parametrima ili nevažećim podacima.

Kod **401 Unauthorized** je vezan za autentifikaciju. Ukazuje na to da klijent mora da pruži određene kredencijale za autentikaciju kako bi pristupio traženom resursu. Ovi kredencijali mogu uključivati korisničko ime i lozinku.

Kod **403 Forbidden** vraća se kada klijent nije ovlašćen za pristup traženom resursu. Razlika između 401 i 403 je u tome što kod 403 znači da klijent nema neophodna prava za određeni resurs koji zahteva, čak i ako ima važeće kredencijale za autentikaciju.

Kod **404 Not Found** vraća se kada nije pronađen resurs na serveru. Možda je obrisan ili postoji tipfeler u URL-u koji sprečava server od pronalaženja resursa. Server takođe može doživeti privremeni problem koji rezultira ovim kodom greške.

## Kodovi greške servera (Server Error Codes)

Zatim, imamo **kodove greške servera**. Ovi kodovi ih server vraća kada zahtev nije mogao biti obrađen zbog problema na strani servera.

Kod greške **500 Internal Server Error** ukazuje da je došlo do neočekivane greške na serveru. Ova greška može biti uzrokovana širokim spektrom problema, od problema sa konfiguracijom do privremenih problema.

Kod greške **502 Bad Gateway** vraća se kada je server pokušavao da komunicira sa drugim serverom, ali je primio nevažeći odgovor od tog servera. Postoji širok spektar uzroka za ovaj tip greške.

Kod greške **503 Service Unavailable** ukazuje na to da server neće moći privremeno da obradi zahtev, pa bi klijent trebalo da pokuša ponovo kasnije. Uobičajeni uzroci za ovu grešku uključuju zadatke održavanja i visok volumen zahteva koje server trenutno ne može da obradi.

HTTP kodovi odgovora su esencijalni za razumevanje komunikacije između klijenata i web servera. Razumevanjem ovih kodova, možete rešavati probleme (troubleshoot) i otklanjati greške (debug) u svojim web aplikacijama.
