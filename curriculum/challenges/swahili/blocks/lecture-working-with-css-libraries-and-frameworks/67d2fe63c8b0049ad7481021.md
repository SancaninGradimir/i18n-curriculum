# --description--

CSS preprocesor je alat koji proširuje standardni CSS moćnim funkcijama kao što su varijable, miksini (mixins), ugnježđivanje (nesting) i nasleđivanje selektora.

Neke od ovih funkcija, poput varijabli i ugnježđivanja, sada su podržane ili dobijaju sve veću podršku u nativnom CSS-u, a ovaj trend je vrlo verovatan da će se nastaviti. Međutim, ranije verzije nativnog CSS-a nisu podržavale ove funkcije.

Zato su CSS preprocesori postali toliko široko korišćeni. Uzimaju CSS pravila i stilove koje pišete, zajedno sa njihovom proširenom sintaksom, i kompajluju ih u nativnu CSS datoteku koju pregledači mogu razumeti.

Korišćenjem CSS preprocesora, možete strukturirati svoj CSS kod na način koji je reusabilniji i logičniji. Takođe ćete imati pristup moćnim funkcijama, kao što su miksini, koje CSS ne podržava direktno.

Neki od najpopularnijih CSS preprocesora su Sass, Less i Stylus.

Pričajmo o Sass-u.

Sass je skraćenica za "Syntactically Awesome Style Sheets" (Sintaktički neverovatni stilovi listova). Kompatibilan je sa svim verzijama CSS-a i održava ga velika zajednica programera.

Sass podržava funkcije kao što su:

- Varijable, tako da možete skladištiti i ponovo koristiti vrednosti širom tabele (spreadsheet).

- Ugnježđeni CSS pravila, tako da možete kreirati vizuelnu hijerarhiju u vašoj datoteci.

- Moduli, tako da možete razdvojiti svoje stilove na više listova stila.

- Miksini, tako da možete ponovo koristiti deklaracije CSS širom vaše stranice.

- Nasleđivanje (Inheritance), tako da više CSS selektora može deliti svojstva.

- I operateri, za osnovne matematičke operacije.

Sass takođe podržava dve sintakse.

SCSS sintaksa je skraćenica za Sassy CSS, što je superset CSS-a. To znači da SCSS proširuje osnovnu sintaksu CSS-a.

SCSS je najčešća sintaksa koju ćete koristiti i pronaći kada radite sa Sass-om. Zahteva korišćenje vitičastih zagrada (`{` i `}`) oko CSS svojstava i tačka-zarez (`;`) na kraju CSS deklaracija, baš kao nativni CSS.

Ove datoteke imaju ekstenziju `.scss`. Evo primera:

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

Što vidite ovde, na vrhu, je varijabla definisana u SCSS-u. Ova varijabla se koristi u CSS pravilu ispod.

Uporedimo SCSS sa manje korišćenom sintaksom – ugnježdenom sintaksom (indented syntax). To je takođe poznato kao "Sass sintaksa" jer je to bila originalna Sass sintaksa.

Ova sintaksa se oslanja na uvlačenje (indentation) za definisanje pravila. Evo primera:

```scss
$primary-color: #3498eb

header
  background-color: $primary-color
```

Primećujte da nema vitičastih zagrada oko CSS pravila, niti tačka-zarez na kraju CSS deklaracija. Na vrhu takođe možete videti varijablu. Primećujte da njeno ime počinje sa dolar znakom (`$`) u ovoj sintaksi, što je drugačije od nativnih CSS varijabli, ali radi vrlo slično.

Sass takođe podržava moćnu funkciju nazvanu miksini (mixins).

Miksini vam omogućavaju da grupišete više CSS svojstava i njihove vrednosti pod imenom i ponovo koristite taj blok CSS koda širom vašeg lista stila.

Ovo čini vaš CSS kod manje repetitivnim i lakšim za održavanje, jer ako promenite nešto u tom bloku, promena se primenjuje svuda gde koristite miksin.

Pošto miksini mogu imati prilagođena i opisna imena, mogu biti korisni za razumevanje šta svaki blok CSS koda radi, što može učiniti vaše listove stila lakšim za razumevanje.

I miksini takođe promovišu doslednost u listovima stila primenjujući uniformne stilove.

Ovo je primer miksina u SCSS sintaksi:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

Da biste definisali miksin, počinjete pisanjem `@mixin` at-pravila, praćeno imenom miksina. U ovom slučaju, miksin se zove `center-flex`. Ima tri CSS svojstva za centriranje elemenata korišćenjem Flexbox-a.

Zatim, kada definišete svoj miksin, koristili biste `@include` at-pravilo da uključite ta svojstva u CSS pravilo. Samo treba da napišete `@include`, praćeno imenom miksina. U ovom slučaju, ime je `center-flex`:

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

Ovo je kompletan kod sa miksinom i CSS pravilom:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

Možete uključiti miksin u koliko god je potrebno CSS pravila. Ako morate napraviti bilo kakve promene, morali biste samo da promenite sam miksin, a promene će biti automatski primenjene svuda. Ovaj primer je u SCSS sintaksi. Primećujte da ima vitičaste zagrade i tačka-zarez.

Evo ekvivalenta u ugnježdenoj sintaksi (indented syntax), takođe poznatoj kao Sass sintaksa, bez vitičastih zagrada ili tačka-zarez:

```scss
@mixin center-flex
  display: flex
  justify-content: center
  align-items: center

section
  @include center-flex
  height: 500px
  background-color: #3289a8
```

A ovo je kompajlovani CSS kod, ono što će pregledač zapravo interpretirati nakon što se fajl kompajluje u nativni CSS:

```css
section {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 500px;
  background-color: #3289a8;
}
```

Primećujte da uključuje tri miksin svojstva na vrhu: `display`, `justify-content` i `align-items`.

Ovo je samo jednostavan primer onoga čemu su sposobni CSS preprocesori. Imaju mnogo moćnih funkcija koje vredi naučiti i primeniti u vašem svakodnevnom radnom toku.

Međutim, imaju potencijalne nedostatke o kojima biste trebali da znate.

Prvo, CSS preprocesori moraju kompajlovati CSS pravila u standardni CSS. Iako je potencijalni režijski uticaj ovog procesa kompilacije često minimalan u smislu vremena i resursa, nešto je što bi trebalo da razmotrite za projekte iz stvarnog sveta.

Zatim, mogu stvoriti izazove debagovanja (debugging challenges), jer pregledači koriste kompajlovani CSS direktno. Utvrđivanje šta generiše problematičan stil iz proširene sintakse može zahtevati nekoliko dodatnih koraka u poređenju sa standardnim CSS-om.

Međutim, prednosti CSS preprocesora obično nadmašuju njihove nedostatke, posebno za složene projekte. Mogu biti veoma korisni za pisanje čistijeg, reusabilnijeg, manje repetitivnog i skalabilnog CSS-a.
