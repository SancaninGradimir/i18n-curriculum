# --description--

U prethodnoj lekciji naučili ste o atributima kao varijablama koje pripadaju objektu. To znači da čuvaju podatke koji opisuju stanje ili ponašanje objekta.

Na primer, automobil bi normalno trebalo da ima marku i model. Marka i model bi mogli da kreiraju atribute za klasu `Car`:

```python
class Car: 
    def __init__(self, brand, model): 
        self.brand = brand 
        self.model = model 

my_car = Car('Lamborghini', 'Gallardo') 
print(my_car.brand) # Lamborghini 
print(my_car.model) # Gallardo 
```

Ali ponekad možda ne znate koji su vam atributi potrebni dok vaš program radi. Zamislite da pišete skriptu koja prima imena atributa od korisnika ili iz konfiguracionog fajla. To nisu atributi koje možete hardkodovati unapred.

Tu je gde ulazi u igru dinamičko rukovanje atributima. Na ovaj način, možete pristupiti, modifikovati, proveriti ili čak obrisati atribute koristeći njihova imena kao varijable, a ne kao fiksna imena u vašem kodu. Ovo daje vašem programu fleksibilnost da reaguje na različite podatke ili unos korisnika "u letu" (on the fly).

Python vam nudi četiri praktične ugrađene funkcije za dinamičko rad sa atributima objekta. To su `getattr()`, `setattr()`, `hasattr()` i `delattr()`.

One vam omogućavaju da pristupate, kreirate, proveravate i uklanjate atribute koristeći varijabilna imena. Pogledajmo svaku od njih u akciji.

`getattr()` čini mogućim čitanje atributa sa objekta kada ne znate njegovo ime do vremena izvršavanja (runtime). Ako atribut ne postoji, on generiše `AttributeError`, osim ako ne pružite podrazumevanu vrednost.

Da biste ga koristili, prosleđujete objekat, ime atributa i opcionalnu podrazumevanu vrednost:

```python
getattr(object, attribute_name, default_value) 
```

Evo primera:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30) 
 
print(getattr(person, 'name')) # John Doe 
print(getattr(person, 'age')) # 30 
print(getattr(person, 'city', 'Milano')) # Milano
```

U gornjem primeru, `Milano` je podrazumevana vrednost jer ne postoji u klasi `Person`.

Kao što smo ranije rekli, prava snaga `getattr()` vidljiva je kada dolazi do imena atributa iz varijable, kao što je iz unosa korisnika ili nekog fajla.

U tom slučaju, ne možete koristiti regularnu sintaksu `object.ime_atributa` jer ime atributa nije fiksno.

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

attr_name = input('Enter the attribute you want to see: ')
print(getattr(person, attr_name, 'Attribute not found'))
```

U ovom slučaju, ako korisnik ukuca `name`, vidi `John Doe`, a ako ukuca `age`, vidi `30`. A ako ukuca nešto što ne postoji u klasi kao što je `email`, vidi poruku `Attribute not found`.

Ovo je upravo gde dinamičko rukovanje atributima blista. Omogućava vašem kodu da reaguje na unos i podatke koje ranije nije video.

Pored toga, možda želite da pregledate sve atribute koje objekat ima, a ne samo one koje već znate. Ugrađena funkcija `dir()` to vam omogućava. Ona vraća listu svih imena atributa na objektu. Evo kako je koristiti:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

# Loop through all attributes of the person object with dir() function
for attr in dir(person):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(person, attr)): 
        value = getattr(person, attr)
        print(f'{attr}: {value}')

# Output
# age: 30
# name: John Doe
```

U petlji iznad, `callable()` je ugrađena funkcija koja vraća `True` ako objekat prosleđen joj može biti pozvan kao funkcija ili metoda, i `False` inače. Proveravanjem `not callable(getattr(person, attr))`, petlja preskače metode i ispisuje samo podatkowe atribute poput `name` i `age`.

Funkcija `setattr()` omogućava kreiranje novog atributa ili ažuriranje postojećeg dinamički. Sintaksa izgleda ovako:

```python
setattr(object, attribute_name, value) 
```

Evo primera koji postavlja atribute konfiguracije na osnovu podataka iz nekog fajla sa konfiguracijom ili varijable okruženja:

```python
class Configuration:
    pass

# Data loaded at runtime (like from a config or env file)
settings_data = {
    'server_url': 'https://api.example.com',
    'timeout_sec': 30,
    'max_retries': 5
}

config_obj = Configuration()

# Dynamically set attributes using dictionary keys and values
for attr_name, attr_value in settings_data.items():
    setattr(config_obj, attr_name, attr_value)

print(config_obj.server_url) # https://api.example.com
print(config_obj.timeout_sec) # 30
```

Postoji i `hasattr()`. Pre nego što nešto uradite sa atributom ili ga obrišete, dobra praksa je proveriti da li on postoji. To je ono što vam omogućava `hasattr()`. Proverava da li atribut postoji i vraća `True` ili `False` na osnovu rezultata.

Evo osnovne sintakse:

```python
hasattr(object, attribute_name)  
```

I evo primera koji dinamički proverava postojanje atributa u instanci klase `Product`:

```python
class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

product_a = Product('T-Shirt', 25)

required_attributes = ['name', 'price', 'inventory_id']

for attr in required_attributes:
    if not hasattr(product_a, attr):
        print(f"ERROR: Product is missing the required attribute: '{attr}'")
    else:
        # Access the attributes dynamically once their existence is confirmed
        print(f'{attr}: {getattr(product_a, attr)}')

# Output:
# name: T-Shirt
# price: 25
# ERROR: Product is missing the required attribute: 'inventory_id'
```

Izlaz ERROR je nastao jer `inventory_id` nedostaje iz klase i instance `Product`.

Konačno, `delattr()` omogućava dinamičko uklanjanje atributa:

```python
delattr(object, attribute_name) 
```

Na primer, zamislite da je objekat potpuno obrađen, pa odlučite da očistite bilo koje osetljive ili privremene atribute koji možda postoje pre snimanja finalne verzije. Nakon toga, možete koristiti `dir()` za petliranje kroz preostale atribute:

```python
class UserSession:
    def __init__(self, user_id, token):
        self.user_id = user_id
        self.auth_token = token # sensitive
        self.temp_counter = 0 # temporary

session = UserSession(101, 'a1b2c3d4e5')

# List of attributes to remove dynamically before "saving" the session
attributes_to_clean = ['auth_token', 'temp_counter']

# Dynamically remove specified attributes
for attr in attributes_to_clean:
    if hasattr(session, attr):
        delattr(session, attr)
        print(f'Removed attribute: {attr}')

print('\nFinal attributes remaining:')

# Loop through the remaining attributes with dir()
for attr in dir(session):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(session, attr)):
        print(f' - {attr}: {getattr(session, attr)}')

# Output:
# Removed attribute: auth_token
# Removed attribute: temp_counter

# Final attributes remaining:
#  - user_id: 101
```
