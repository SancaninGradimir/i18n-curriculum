# --description--

Specijalne metode u Pythonu, poznate i kao „magične metode“ ili „dunder metode“, su specijalne funkcije Pythona koje počinju i završavaju se dvostrukim donjom linijom (`__`). Sama reč „dunder“ potiče od dvostrukih donjih linija (**d** za double, **under** za donje linije).

Verovatno ste već koristili specijalne metode bez da znate. Svaki put kada napišete nešto poput `3 + 4`, Python tiho izvršava `3.__add__(4)` u pozadini. To je akcija specijalne metode. Dakle, iako *možete* direktno pozvati specijalne metode, retko to radite. Nešto kao što je `3 + 4` je mnogo jasnije i lakše za čitanje od samog pozivanja `3.__add__(4)`.

Pored `__add__`, `__init__()` je još jedna specijalna metoda koju ćete najviše videti i koristiti, jer je inicijalizator klase. Postoje i druge, kao što su `__len__()` i `__str__()`.

Razmislite o specijalnim metodama kao o direktorima aktivnosti između programera koji piše kod i samog interpretatora jezika Python.

Zapamtite, ne morate pozivati specijalne metode direktno. Umesto toga, Python ih automatski poziva kada se dogode određene radnje. Ove operacije uključuju:

- **Aritmetičke operacije kao što su sabiranje, oduzimanje, množenje, deljenje i druge**. Za sabiranje, poziva se `__add__()`, za oduzimanje `__sub__()`, za množenje `__mul__()` i za deljenje `__truediv__()`.
    
- **Operacije sa nizovima znakova (string) kao što su konkatenacija, ponavljanje, formatiranje i konverzija u tekst**. Za konkatenaciju se poziva `__add__()`, za ponavljanje `__mul__()`, za formatiranje `__format__()`, a za konverziju teksta se koriste `__str__()` i `__repr__()`, itd.
    
- **Operacije poređenja kao što su jednakost, manje od, veće od i druge**. Za provere jednakosti poziva se `__eq__()`, za manje od `__lt__()`, za veće od `__gt__()`, itd.
    
- **Operacije iteracije kao što je činjenje objekta iterabilnim i napredovanje kroz stavke**. Poziva se `__iter__()` da bi se vratio iterator, a `__next__()` da bi se dohvatila sledeća stavka.
    

Normalno, tipovi podataka Pythona poput nizova znakova i brojeva već znaju kako da sabiraju stvari, vrše konkatenaciju, porede na jednakost, koriste se u petljama i druge operacije.

Ali kada kreirate svoju klasu, Python neće automatski znati kako da to rukuje.

Ovde ulaze specijalne metode — one vam omogućavaju da prilagodite ugrađeno ponašanje Pythona.

Recimo da želite da dobijete broj stranica u objektima knjiga kreiranim sa sledećom klasom, ili da ih uporedite i dobijete čitljiv string objekata. Evo šta se dešava bez specijalnih metoda:

```python
class Book:
   def __init__(self, title, pages):
       self.title = title
       self.pages = pages

book1 = Book("Built Wealth Like a Boss", 420)
book2 = Book("Be Your Own Start", 420)

print(len(book1)) # TypeError: object of type 'Book' has no len()
print(str(book1)) # <__main__.Book object at 0x102ed2900>
print(book1 == book2) # False even though they have the same number of pages
```

U primeru:

- `len(book1)` je propao jer Python ne zna kako da dobije dužinu vašeg objekta knjige bez `__len__()`
    
- `str(book1)` je ispisao nešto poput `<__main__.Book object at 0x102ed2900>` jer je to podrazumevana reprezentacija kada ne koristite `__str__()`
    
- `book1 == book2` rezultirao je `False` jer Python samo proverava da li su oba objekta isti u memoriji, a ne po sadržaju.
    

Evo kako možete definisati sopstvene specijalne metode `__len__()`, `__str__()` i `__eq__()` kako biste olakšali rad sa objektima kreiranim iz klase `Book`:

```python
class Book:
   def __init__(self, title, pages):
       self.title = title
       self.pages = pages

   def __len__(self):
       return self.pages

   def __str__(self):
       return f"'{self.title}' has {self.pages} pages"

   def __eq__(self, other):
       return self.pages == other.pages
  
book1 = Book("Built Wealth Like a Boss", 420)
book2 = Book("Be Your Own Start", 420)

print(len(book1)) # 420
print(len(book2)) # 420
print(str(book1)) # 'Built Wealth Like a Boss' has 420 pages
print(str(book2)) # 'Be Your Own Start' has 420 pages
print(book1 == book2) # True
```

Drugi primer je korpa za kupovinu gde radite sledeće:

- Dodavanje stavki u korpu
    
- Uklanjanje stavki iz korpe
    
- Dobijanje broja stavki u korpi
    
- Provera kojih stavki se nalaze u korpi
    
- Provera da li je određena stavka u korpi
    
- Vraćanje ili prikaz stavke na određenom indeksu u korpi
    

Iako možda imate metodu koja dodaje stavke u korpu i uklanja određene stavke iz nje, možete kreirati specijalne metode za sve ostale funkcionalnosti:

- `__len__()` za dobijanje dužine stavki u korpi
    
- `__iter__()` za iteriranje kroz stavke u korpi kako biste ih videli
    
- `__contains__()` za proveru da li je određena stavka u korpi
    
- `__getitem__()` za vraćanje ili prikaz stavke na određenom indeksu u korpi
    

Evo primera klase `Cart` sa ovim definisanim metodama i specijalnim metodama:

```python
class Cart:
   def __init__(self):
       self.items = []

   def add(self, item):
       self.items.append(item)

   def remove(self, item):
       if item in self.items:
           self.items.remove(item)
       else:
           print(f'{item} is not in cart')

   def list_items(self):
       return self.items

   def __len__(self):
       return len(self.items)

   def __getitem__(self, index):
       return self.items[index]

   def __contains__(self, item):
       return item in self.items

   def __iter__(self):
       return iter(self.items)
```

I evo kako ih možete koristiti:

```python
cart = Cart()
cart.add('Laptop')
cart.add('Wireless mouse')
cart.add('Ergo keyboard')
cart.add('Monitor')

for item in cart:
   print(item, end=' ') # Laptop Wireless mouse Ergo keyboard Monitor

print(len(cart)) # 4
print(cart[3]) # Monitor

print('Monitor' in cart) # True
print('banana' in cart) # False

cart.remove('Ergo keyboard')

print(cart.list_items()) # ['Laptop', 'Wireless mouse', 'Monitor']

cart.remove('banana') # banana is not in cart
```
