# --interactive--

Binarni logički operateri vam pomažu da evaluirate dva izraza i vrate rezultat na osnovu njihove istinitosti. Pogledajmo tri najčešća binarna logička operatera: logički AND, logički OR i operater za spajanje nulnih vrednosti.

Logički AND operater je predstavljen dvostrukim ampersandom (`&&`). On proverava da li oba operanda su tačna i vraća rezultat. Ako oba operanda jesu istični, on vraća drugu vrednost, tj., ona sa desne strane:

:::interactive_editor

```js
const result = true && 'hello';

console.log(result); // hello
```

:::

U gornjem primeru, tekst `hello` je logovan u konzolu jer su oba operanda `true`. Ako je bilo koji operand lažan (falsy), vraća lažno (falsy) vrednost:

:::interactive_editor

```js
const result = 0 && 3;

console.log(result); // 0
```

:::

Pošto je ``0`` lažno vrednost, broj ``0`` se loguje u konzolu. I ako su oba operanda lažna, vraća prvu lažnu vrednost:

:::interactive_editor

```js
const result = false && 0;

console.log(result); // false
```

:::

Pošto je `false` falsna vrednost, onda se `false` loguje u konzolu. Logički AND operator je koristan kada želite proveriti više uslova i osigurati da su svi istiniti pre nastavka. Evo primera:

:::interactive_editor

```js
if (2 < 3 && 3 < 4) {
 console.log('The if block runs'); 
} else {
 console.log('The else block runs');
} 
```

:::

U uslovu, pošto je `2` manje od `3` I `3` je manje od `4`, onda će rečenica `The if block runs` biti logovana u konzolu.

Logički OR operator proverava da li je barem jedan od operanata istinit. Ako prvi operand nije lažan (truthy), vraća tu vrednost:

:::interactive_editor

```js
const result = 'This is truthy' || false;

console.log(result); // This is truthy
```

:::

Ako je prvi operand lažan, ali drugi istinit, druga vrednost će biti logovana u konzolu:

:::interactive_editor

```js
const result = 0 || 'This is truthy';

console.log(result); // This is truthy
```

:::

Uobičajeno je koristiti logički OR operater u izjavama `if/else` na ovaj način:

:::interactive_editor

```js
let userInput;

if (userInput || 'Guest') {
 console.log('A user is present');
} else {
 console.log('No user detected');
}
```

:::

Pošto nismo dodelili vrednost promenljivoj `userInput`, trenutno je `undefined`. Uslov u izjavi `if` proverava da li su ili promenljiva `userInput` ili niz `Guest` istiniti (truthy). Pošto je niz `Guest` tačan u boološkom kontekstu kao što je ovo, niz `A user is present` će biti logovan na konzolu.

Operator za spajanje nulnih vrednosti je sofisticiraniji od logičkog ILI i logičkog I. Predstavljen dvostrukim znakom za upitanje (`??`), pomaže u scenarijima gde želite da vratite vrednost samo ako je prva `null` ili `undefined`. Evo primera rada sa operatorom za spajanje nulnih vrednosti:

:::interactive_editor

```ts
const result = null ?? 'default';

console.log(result); // default
```

:::

Pošto je ``null`` nulizativna vrednost, string ``default`` biće logovan u konzolu. Operator za spajanje nulizativnih vrednosti je neverovatno koristan u situacijama gde su ``null`` ili ``undefined`` jedine vrednosti koje bi trebalo da pokrenu rezervnu ili podrazumevanu vrednost. Evo primera rukovanja podešavanjima preferencija korisnika:


:::interactive_editor

```ts
const userSettings = {
 theme: null,
 volume: 0,
 notifications: false,
};

let theme = userSettings.theme ?? 'light';
console.log(theme); // light
```

:::

U gornjem primeru, imamo objekat nazvan `userSettings` koji sadrži svojstva `theme`, `volume` i `notifications`. Pristupamo `theme` koristeći notaciju tačke kao `userSettings.theme`. Naučićete više o tome kako raditi sa objektima u budućoj lekciji. Pošto je korisnikov `theme` trenutno podešen na `null`, onda će se niz `light` logovati u konzolu.
