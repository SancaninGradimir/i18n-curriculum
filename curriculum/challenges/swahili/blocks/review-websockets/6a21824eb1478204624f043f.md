```markdown
## WebSockets naspram HTTP
```

- **HTTP** prati model zahtev-odgovor: klijent šalje zahtev, server šalje odgovor, nakon čega se konekcija zatvara.
- **WebSockets** kreiraju trajnu, dvosmernu konekciju između klijenta i servera koja ostaje otvorena dok obe strane ne pošalju signal zatvaranja.
- HTTP je odlično pogodan za učitavanje stranica ili dohvaćanje podataka, ali postaje spor i neefikasan za aplikacije u realnom vremenu.
- Svaka WebSocket konekcija počinje sa **HTTP rukovanjem (handshake)**: klijent zahteva nadogradnju, a server odgovara sa `101 Switching Protocols`.

### Tok povezivanja WebSocket-om

1. Klijent se povezuje na server putem HTTP-a i zahteva nadogradnju.
2. Server prihvata i odgovara sa `101 Switching Protocols`.
3. Poveznica ostaje otvorena.
4. Obje strane razmenjuju podatke u bilo kom trenutku dok se konekcija ne zatvori.

### HTTP naspram WebSocket komunikacije

- **HTTP**: `Client → Request → Server → Response` (jedan ciklus, nakon čega se zatvara)
- **WebSocket**: `Client ↔ Server` (trajan, dvosmeran)

### Koristna analogija

- **HTTP** je kao razmena pisama: ti napišeš jedno, čekaš odgovor, i konverzacija se završava.
- **WebSockets** su kao telefonski poziv: obe strane razgovaraju u realnom vremenu koliko je potrebno.

### Praktične primene WebSocketsa

- Aplikacije za razgovor
- Online višeigračke igre
- Aktualizacija rezultata sportskih događaja uživo
- Kontrolne table (dashboardi) akcionog tržišta
- Alati za saradnju (npr. Google Docs)

## Arhitektura Pub/Sub (Publisher/Subscriber)

- **Pub/Sub** (Objavljivanje/Pretplata) je arhitektura poruka koja omogućava delovima sistema da komuniciraju bez direktnog poznavanja jednih o drugima.
- Postoje tri glavne dela: **objavljivači** (publishers), **teme** (topics) i **pretplatnici** (subscribers).
  - **Objavljivač** šalje poruke na temu.
  - **Tema** (takođe nazvana kanal ili grupa) deluje kao posrednik.
  - **Pretplatnik** prima poruke sa tema na koje je pretplaćen.

```markdown
### Tok poruka u modelu Objava/Pretplata
```

1. Pretplatnik se pretplaćuje na temu.
2. Izдаваč šalje poruku na tu temu.
3. Sistem za poruke isporučuje poruku svakom pretplatniku te teme.

### Direktna komunikacija naspram Pub/Sub

- **Bez Pub/Sub**: `Application A → Application B` — čvrsto vezano (tightly coupled); promena u B može pokvariti A.
- **Sa Pub/Sub**: `Application A → Topic → Subscribers` — Samo A objavljuje; ne mora da zna ko prima.

### Stvarni primeri upotrebe Pub/Sub

- Aplikacije za chat sa više soba
- Online multiplayer igre
- Sistemi obaveštenja
- Kontrolne table uživo (Live dashboards)
- Aplikacije vođene događajima

### WebSockets naspram Pub/Sub

- **WebSockets** pružaju komunikacioni kanal (put).
- **Pub/Sub** pruža obrazac poruka koji određuje ko prima specifične poruke (saobraćajni sistem).
- Mnoge aplikacije u realnom vremenu koriste oba zajedno: WebSockets održavaju konekciju otvorenom, dok Pub/Sub usmerava poruke ka odgovarajućim pretplatnicima.
