## Algoritmi i Notacija Big O

- **Algoritmi**: Skup neuključljivih uputstava za rešavanje problema ili izvršavanje zadatka. Algoritmi moraju da se završe u finitom broju koraka, a svaki korak mora biti precizan i neuključljiv.

- **Big O Notation**: Opisuje performanse u najgorem slučaju, ili stopu rasta, algoritma kako veličina ulaza raste. Fokusira se na to kako rast potrošnje resursa sa veličinom ulaza, ignorišući konstante faktore i članove nižeg reda.

### Uobičajene kompleksnosti vremena

- **O(1) - Konstantno vreme**: Algoritam zahteva isto vreme bez obzira na veličinu ulaza.

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) - Logaritamsko vreme**: Vreme raste sporo kako ulaz raste. Često je u algoritmima koji ponavljajuće smanjuju veličinu problema frakcionim stepenom (kao što je Binarna pretraga).

- **O(n) - Linearno vreme**: Vreme izvršavanja raste proporcionalno veličini ulaza.

```python
for grade in grades:
    print(grade)
```

- **O(n log n) - Log-Linear Time**: Uobičajena kompleksnost vremena efikasnih algoritama sortiranja, kao što su Merge Sort i Quick Sort.

- **O(n²) - Kvadratna složenost**: Vreme izvršavanja raste kvadratno. Često se susreće u ugnježdene petlje.

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### Prostorna kompleksnost

- **O(1) - Konstantni prostor**: Algoritam koristi istu količinu memorije bez obzira na veličinu ulaznih podataka.
- **O(n) - Linearni prostor**: Korišćenje memorije raste proporcionalno sa veličinom ulaza.
- **O(n²) - Kvadratični prostor**: Korišćenje memorije raste kvadratično u odnosu na veličinu ulaza.

## Tehnike rešavanja problema

- **Razumevanje problema**: Pročitajte problemsko opisanje više puta. Identifikujte ulaz, očekivani izlaz i način transformacije ulaza u izlazu.

- **Pseudokod**: Opis na visokom nivou logike algoritma koji je nezavisan od jezika. Koristi uobičajen pisani jezik mešano sa programskim konstruktima kao što su `IF`, `ELSE`, `FOR`, `WHILE`.

```md

  ADD character TO THE BEGINNING OF reversed_string

```

- **Granični slučajevi**: Specifični, važeći unosi koji se javljaju na granicama onoga što algoritam treba da obradi. Uvek razmotrite i testirajte granične slučajeve.

## Nizovi

- **Statički nizovi**: Imaju fiksnu veličinu određenu prilikom inicijalizacije. Elementi su pohranjeni na susednim memorijskim lokacijama. Veličina se ne može promeniti tokom izvršavanja programa.

- **Dinamički nizovi**: Mogu automatski rasti ili smanjiti tokom izvršavanja programa. Upravljaju promenljivom veličinom kroz automatsko kopiranje u veće nizove kada je to potrebno.

### Python Liste (Dinamički Nizovi)

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### Kompleksnosti vremena za dinamičke nizove

- **Pristup**: O(1)
- **Dodavanje na kraj**: O(1) prosečno, O(n) kada je potrebno skaliranje (resizing needed)
- **Dodavanje u sredinu**: O(n)
- **Brisanje**: O(n) za sredinu, O(1) za kraj

## Staplišta

- **Stapanji (Stacks)**: Struktura podataka tipa „Poslednji uđi, prvi izađe“ (LIFO). Elementi se dodaju i uklanjaju samo sa samog vrha.

- **Operacija Push**: Dodavanje elementa na vrh steka. Vremenska složenost: O(1).

- **Operacija Pop**: Uklanja element sa vrha steka. Vremenska složenost: O(1).

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## Kore

- **Redovi**: Struktura podataka "Prvi uđi, prvi izađe" (FIFO). Elementi se dodaju na zadnji kraj i uklanjaju sa prednjeg kraja.

- **Operacija Enqueue**: Dodaje element na zadnji kraj reda. Vremenska složenost: O(1).

- **Operacija Dequeue**: Uklanja element sa prednje strane reda. Vremenska složenost: O(1).

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## Povezane liste

- **Spojene liste**: Linearna struktura podataka gde svaki čvor sadrži podatke i referencu na sledeći čvor. Čvori su povezani kao lanac.

### Jednostručno povezane liste

- **Struktura**: Svaki čvor ima podatke i jednu referencu na sledeći čvor.
- **Traversiranje**: Može se kretati samo napred, od glave do repa.
- **Čvor Glava**: Prvi čvor u listi, obično jedini direktno dostupan čvor.
- **Čvor Rep**: Poslednji čvor u listi, ukazuje na `None`.

### Operacije i vremenske složenosti

- **Umetanje na početak**: O(1)
- **Umetanje na kraj**: O(n) - mora proći do kraja
- **Umetanje u sredinu**: O(n) - mora proći do poziciju
- **Brisanje sa početka**: O(1)
- **Brisanje sa kraja**: O(n) - mora proći da pronađe prethodni čvor
- **Brisanje iz sredine**: O(n) - mora proći da pronađe čvor

### Dvosmerne spojene liste

- **Struktura**: Svaki čvor ima podatke i dve reference: na sledeći čvor i prethodni čvor.
- **Traversiranje**: Može se kretati u oba smera.
- **Memorija**: Zahteva više memorije od jednosmernih lista povezanih čvorova zbog dodatne reference.

## Hash mape i skupovi

### Karte i Hash mape

- **Mapa (Apstraktni tip podataka)**: Upravlja kolekcijama parova ključ-vrednost. Svaki ključ mora biti jedinstven, ali vrednosti mogu biti ponovljene.

- **Hash Map**: Konkretna implementacija apstraktnog tipa podataka mape (map ADT) koja koristi tehniku hashovanja. Koristi funkciju hashovanja za generisanje vrednosti hashovanja za ključeve, koje određuju lokaciju skladištenja u osnovnom nizu.

### Python Rečnici (Hash Maps)

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### Skupovi

- **Skupovi**: Nerangordirane kolekcije jedinstvenih elemenata. Nisu dozvoljeni duplikati, niti se održava određeni redosled.

- **Samo nepromenljivi elementi**: Skupovi mogu sadržavati samo nepromenljive tipove podataka (brojevi, stringovi, tuple) jer hash vrednosti moraju ostati konstantne.

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
set_a.issuperset(set_b)
set_a.isdisjoint(set_b)

# Membership testing
5 in numbers
```

## Kolizije hasha

- **Kolizija hasha**: Nastaje kada dva različita ključa proizvedu istu vrednost hasha.

- **Strategije rešavanja kolizija**:
  - **Chaining**: Svaki indeks niza ukazuje na spajanu listu koja skladišti sve elemente sa istom vrednošću haša
  - **Open Addressing**: Pretraživanje sledećeg dostupnog indeksa korišćenjem unapred definisane sekvence

## Kada koristiti svaku strukturu podataka

- **Liste**: Kada vam je potreban direktan, indeksiran pristup i ne znate veličinu unapred
- **Stapovi (Stacks)**: Za LIFO operacije (funkcionalnost poništavanja, evaluacija izraza, praćenje povratka)
- **Redovi (Queues)**: Za FIFO operacije (planiranje zadataka, pretraga po širini - breadth-first search)
- **Povezane liste (Linked Lists)**: Kada je često dodavanje/brisanje na početku, nepoznata veličina i nije potreban nasumičan pristup
- **Hash mape (Hash Maps)**: Za brze pretrage po ključu-vrednostima, brojanje pojavljivanja, keširanje
- **Setovi (Sets)**: Za proveru jedinstvenosti, matematičke operacije sa skupovima, uklanjanje duplikata
