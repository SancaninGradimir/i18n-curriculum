# --description--

Svaki računarski program koji radi na vašem uređaju ima specifičan set instrukcija, koje se izvršavaju u određenom redosledu kako bi se zadatak završio.

Zadatak može biti sortiranje skupa brojeva, modifikovanje slike, praćenje zaliha ili čak pokretanje vaše omiljene video igre.

Ovde ulaze u igru algoritmi. **Algoritam** je set nedvosmislenih instrukcija za rešavanje problema ili obavljanje zadatka.

Možete razmišljati o algoritmima kao o „receptima“. Kada kuvarstvo, recepti navode sve sastojke koji će vam trebati i pružaju uputstva korak po korak kako pripremiti jelo.

Ekvivalentno, možete razmišljati o algoritmima kao o „receptima“ koji računarima tačno govore šta treba da se uradi i kako to učiniti.

Algoritmi imaju dve ključne karakteristike:

* Ne mogu trajati beskonačno. Moraju da završe u konačnom broju koraka.
    
* Svaki korak mora biti precizan i nedvosmislen.
    

Mogu imati nula, jedan ili više ulaza (input) i generisati jedan ili više izlaza (output).

Koraci algoritma su nezavisni od bilo kog programskog jezika.

Ali da bi zaista radili na računaru, morate ih implementirati u programskom jeziku, kao što su Python ili JavaScript.

Ako je algoritam ispravan, izlaz za svaki validan ulaz treba da odgovara očekivanom izlazu.

Pored toga što mora biti ispravan, algoritmi takođe moraju biti efikasni.

Efikasnost algoritma se može meriti po tome koliko vremena im je potrebno za izvršavanje i koliko prostora zahtevaju u memoriji kako bi završili zadatak.

Poznavanje efikasnosti algoritma veoma je važno jer vam daje ideju o tome kako će funkcionisati kako raste veličina ulaza.

Na primer, sortiranje 15 celih brojeva nije isto što i sortiranje 1 milion celih brojeva.

Kako proces raste u veličini i složenosti, ako algoritam nije dovoljno efikasan da ga obradi, možete završiti sa veoma sporim programom koji čak može dovesti do pada celog sistema.

Zato je veoma važno razviti i izabrati najefikasnije moguće algoritme.

Ovde postaje veoma važna Big O notacija.

Big O notacija opisuje performanse u najgorem slučaju, ili stopu rasta, algoritma kako veličina ulaza raste.

Stopa rasta algoritma odnosi se na to kako rastu resurse koje zahteva kako veličina ulaza raste.

Big O notacija fokusira se na performanse u najgorem slučaju jer je ovaj slučaj veoma važan za razumevanje koliko algoritam može biti efikasan, čak i u scenariju najgoreg slučaja, bez obzira na ulaz.

Vraćajući se na naš primer sortiranja, sortiranje 1 miliona celih brojeva bi intuitivno trebalo da zahteva više vremena i resursa nego sortiranje 15 celih brojeva.

Ali koliko više?

To zaista zavisi od algoritma koji izaberete za njihovo sortiranje.

Big O notacija neće dati tačan broj za opis efikasnosti algoritma, ali će vam dati ideju o tome kako skalira kako raste veličina ulaza, na osnovu broja operacija koje algoritam obavlja.

U Big O notaciji, obično označavamo veličinu ulaza slovom `n`. Na primer, ako je ulaz lista, `n` bi označavao broj elemenata u toj listi.

Konstantni faktori i članovi nižeg reda ne uzimaju u obzir kada se određuje vremenska složenost algoritma na osnovu broja operacija. To je zato što kako raste veličina `n`, uticaj ovih manjih članova na ukupan broj izvršenih operacija postaje sve manji i manji.

Član koji će dominirati ukupnim ponašanjem algoritma biće najviši redni član sa `n`, veličinom ulaza.

Na primer, ako algoritam obavlja `7n + 20` operacija da bi se završio, uticaj konstante `20` na konačni rezultat biće sve manji i manji kako raste `n`. Član `7n` će tendirati ka dominaciji i to će definisati ukupno ponašanje i efikasnost algoritma.

Još jedan primer bi bio algoritam koji zahteva `20n² + 15n + 7` operacija da bi se završio. Član `20n²` će tendirati ka dominaciji kako raste `n`, pa bi ovaj algoritam imao kvadratnu vremensku složenost jer dominantni član sadrži `n²`.

Kvadratna vremenska složenost je jedan od mnogih različitih tipova vremenske složenosti koje možete pronaći u svetu algoritama.

Učimo o nekim od najčešćih.

**`O(1)`** je poznat kao „Konstantna vremenska složenost“. Kada algoritam ima konstantnu vremensku složenost, zahteva isto vreme za izvršavanje, bez obzira na veličinu ulaza.

Na primer, proveravanje da li je broj paran ili neparan uvek će trajati isto vreme, bez obzira na sam broj.

```js
function checkEvenOrOdd(number) {
  if (number % 2 === 0) {
    return "Even";
  } else {
    return "Odd";
  }
}
```

**`O(log n)`** je poznat kao „Logaritamska vremenska složenost“. To znači da vreme koje algoritam zahteva polako raste kako veličina ulaza raste. Ovo je uobičajeno kod problema u kojima se veličina problema ponavljano smanjuje konstantnim frakcionim brojem.

Na primer, popularni algoritam pretrage nazvan Binarna pretraga ima Big O vremensku složenost `O(log n)`. To je zato što eliminiše polovinu preostalih elemenata u svakom poređenju, što ga čini efikasnijim ukupno.

**`O(n)`** je poznat kao „Linearna vremenska složenost“. Vremenski rad algoritama sa ovom vremenskom složenošću raste proporcionalno veličini ulaza.

Na primer, petlja `for` koja iterira kroz sve elemente liste obavljaće više iteracija kako broj elemenata u listi raste. Ako se lista udvostručuje po veličini, broj operacija će takođe približno udvostručiti.

```js
for (const grade of grades) { // grades is an array
  console.log(grade);
}
```

**`O(n log n)`** je poznat kao „Log-linearna vremenska složenost“. Ovo je uobičajena vremenska složenost efikasnih algoritama za sortiranje, kao što su Merge Sort i Quick Sort.

**`O(n²)`** je poznat kao „Kvadratna vremenska složenost“. Vremenski rad ovih algoritama raste kvadratno u odnosu na veličinu ulaza, što generalno nije efikasno za probleme iz stvarnog sveta.

Ugnježdeni petlje su uobiştak primer kvadratne vremenske složenosti. Unutrašnja petlja će obaviti `n` iteracija za svaku od `n` iteracija spoljašnje petlje, što rezultira sa `n` na kvadrat iteracija.

```js
for (let i = 0; i < n; i++) {
  for (let j = 0; j < n; j++) {
    console.log("Hello, World!");
  }
}
```

Ostale vremenske složenosti uključuju „Eksponencijalnu vremensku složenost“, označenu kao `O(2^n)`, i „Faktorialnu vremensku složenost“, označenu kao `O(n!)`. Oba su neefikasna za scenarije iz stvarnog sveta.

Na ovoj grafici, možete uporediti rast matematičkih funkcija koje predstavljaju najčešće vremenske složenosti. Razmišljajte o x-osi (horizontalno) kao o veličini ulaza i y-osi (vertikalno) kao o vremenskom radu algoritma.

Možete videti da Kvadratna vremenska složenost (`O(n²)`) (žuta) raste mnogo brže od ostalih, dok Konstantna vremenska složenost (`O(1)`) (crvena) ostaje konstantna, čak i ako ulaz postaje veći.
