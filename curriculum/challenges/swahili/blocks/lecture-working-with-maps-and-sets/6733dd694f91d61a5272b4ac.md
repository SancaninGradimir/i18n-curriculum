# --description--

U JavaScript-u, `Map` je ugrađeni objekat koji skladišti parove ključ-vrednost, slično objektu. Međutim, razlikuje se od standardnih JavaScript objekata po tome što dozvoljava ključeve bilo kog tipa, uključujući objekte i funkcije.

`WeakMap` je kolekcija parova ključ-vrednost, slična `Map`-u, ali koristi slabe reference za svoje ključeve. Ključevi moraju biti objekti, dok vrednosti mogu biti bilo kog tipa.

Razmislite o odnosu između `WeakMap`-a i `Map`-a kao da je sličan odnosu između `WeakSet`-a i `Set`-a, kako ste naučili u prethodnoj lekciji.

Da biste kreirali `Map`, koristite konstruktor `Map()` predznakom `new`:

```js
const myFirstMap = new Map();
```

Možete inicijalizovati Map vrednostima:

```js
const myTreesMap = new Map([
 [{ type: 'deciduous' }, 'Maple tree'],
 [['forest', 'grove'], 'Pine tree'],
 [42, 'Oak tree'],
 [true, 'Birch tree'],
 [function() { return 'I am a function key'; }, 'Willow tree'],
]);
```

Ako niste inicijalizovali `Map` vrednostima, možete koristiti metodu `set()` za dodavanje:

```js
const myTreesMap = new Map();
myTreesMap.set({ type: 'deciduous' }, 'Maple tree');
myTreesMap.set([1, 2], 'Pine tree');
myTreesMap.set(42, 'Oak tree');
myTreesMap.set(true, 'Birch tree');
myTreesMap.set(function() { return "I'm a function key"; }, 'Willow tree');

console.log(myTreesMap);
```

Drugim metodama koje možete koristiti za rad sa `Map`-om su:

- `get(key)` za dohvaćanje vrednosti povezane sa navedenim `key`.
- `has(key)` za proveru da li postoji `key` u `Map`-u.
- `delete(key)` za uklanjanje para ključ-vrednost iz `Map`-a.
- `clear()` za uklanjanje svih parova ključ-vrednost.
- `entries()` za pregled stavki (vraća stavke u `MapIterator`).
- `forEach()` za iteriranje kroz stavke `Map`-a.
- `size` za indikaciju broja parova ključ-vrednost u `Map`-u.
    

Postoji konstruktor `WeakMap()` koji možete koristiti za kreiranje `WeakMap`-a:

```js
const myFirstWeakMap = new WeakMap();
```

Metode `set()`, `get()`, `has()` i `delete()` takođe su dostupne za korišćenje sa `WeakMap`-om. Na primer, evo kako možete dodeliti stavke u `Map` pomoću metode `set()`:

```js
const myTreeWeakMap = new WeakMap();

myTreeWeakMap.set({ id: 1 }, 'Maple tree');
myTreeWeakMap.set({ id: 2 }, 'Pine tree');
myTreeWeakMap.set({ id: 3 }, 'Oak tree');
myTreeWeakMap.set({ id: 4 }, 'Birch tree');
myTreeWeakMap.set({ id: 5 }, 'Willow tree');

console.log(myTreeWeakMap);
```

Evo kako izgleda `WeakMap` u konzoli:

```js
/*
WeakMap {{…} => 'Willow tree', {…} => 'Maple tree', {…} => 'Pine tree', {…} => 'Oak tree'}
  [[Entries]]
    No properties
  [[Prototype]]: WeakMap
*/
```

Evo razlika između `Map`-a i `WeakMap`-a:

| **Funkcija** | **Map** | **WeakMap** |
| --- | --- | --- |
| **Tip ključa** | Ključevi mogu biti bilo kog tipa podataka, uključujući nizove, brojeve, objekte ili čak funkcije. | Ključevi moraju biti objekti. |
| **Slučaj upotrebe** | Koristite `Map` kada vam je potrebno povezati podatke sa bilo kojim tipom ključa. | Koristite `WeakMap` kada vam je potrebno povezati podatke samo sa objektima. |
| **Iteracija** | Možete iterirati kroz `Map` koristeći `forEach()`, `keys()`, `values()` ili `entries()`. | `WeakMap` nije iterabilan. |
| **Svojstvo veličine** | `Map` ima svojstvo `size` za dobijanje broja parova ključ-vrednost. | `WeakMap` nema svojstvo `size`. |

U tabeli iznad, možete videti sažet razlike uključujući tip ključa, slučaj upotrebe, iteraciju i svojstvo veličine. Molimo vas da odvojite trenutak za čitanje sadržaja ove tabele kako biste saznali više o njihovim razlikama.

## --text--

Koji od ovih razlikuje `Map` od `WeakMap` u JavaScriptu?
## --answers--

``Map` dozvoljava samo objekte kao ključeve, dok `WeakMap` dozvoljava ključeve bilo kog tipa.`
### --feedback--

### --feedback--
