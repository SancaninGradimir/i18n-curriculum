- **Synchronous JavaScript** se izvršava sekvencijalno i čeka da prethodna operacija završi pre nego što pređe na sledeću.
- **Asynchronous JavaScript** omogućava izvršavanje više operacija u pozadini bez blokiranja glavnog *thread*-a (niti).
- **Thread** je sekvenca instrukcija koja se može izvršiti nezavisno od glavnog toka programa.
- **Callback functions** su funkcije koje se prosleđuju kao argumenti drugim funkcijama i izvršavaju se nakon završetka operacije ili kao rezultat događaja.

## JavaScript engine i JavaScript runtime

- **JavaScript engine** je program koji izvršava JavaScript kod u web pretraživaču. Radi kao konverter koji uzima vaš kod i pretvara ga u instrukcije koje računar može razumeti i po kojima se ponašati.
- V8 je primer **JavaScript engine**-a razvijenog od strane Google-a.
- **JavaScript runtime** je okruženje u kojem se izvršava JavaScript kod. Uključuje **JavaScript engine** koji obrađuje i izvršava kod, kao i dodatne funkcije poput web pretraživača ili Node.js-a.

## Fetch API

- Fetch API omogućava web aplikacijama da izvrše mrežne zahteve, obično radi preuzimanja ili slanja podataka na server. Pruža metod `fetch()` koji možete koristiti za izvođenje ovih zahteva.
- Koristeći Fetch API, možete preuzeti tekst, slike, audio, JSON i druge tipove podataka.

## HTTP metode za Fetch API

Fetch API podržava različite HTTP metode za interakciju sa serverom. Najčešće metode su:

- **GET**: Koristi se za dohvaćanje podataka sa servera. Podrazumevano, Fetch API koristi metodu `GET` za dohvaćanje podataka.

```js
fetch('https://api.example.com/data')
```

Da bi se koristili preuzeti podaci, mora biti konvertovan u JSON format koristeći metod `.json()`:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

U ovom kodu, `fetch()` vraća Promise, a `.then()` handler konvertuje rešeni odgovor u JSON format.

- **POST**: Koristi se za slanje podataka na server. Metoda `POST` se koristi za kreiranje novih resursa na serveru.

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

U ovom primeru, šaljemo zahtev `POST` radi kreiranja novog korisnika. Specifikovali smo metodu kao `POST`, postavili odgovarajuće zaglavlja (headers) i uključili telo (body) sa podacima koje želimo da pošaljemo. Telo mora biti string, pa koristimo `JSON.stringify()` da konvertujemo naš objekat u JSON string.

- **PUT**: Koristi se za ažuriranje podataka na serveru. Metoda `PUT` se koristi za ažuriranje postojećih resursa na serveru.

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

U ovom primeru, ažuriramo ID `45` koji je naveden na kraju URL-a. Upotrebili smo metod `PUT` na kodu i takođe naveli podatke kao telo (body), koji će se koristiti za ažuriranje identifikovanih podataka.

- **DELETE**: Koristi se za brisanje podataka na serveru. Metoda `DELETE` se koristi za brisanje resursa na serveru.

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE'
})
```

U ovom primeru, šaljemo `DELETE` zahtev radi uklanjanja korisnika sa ID-om `45`.

## Promise i lančevanje Promise-a

Objekat Promise je temeljni koncept u modernom JavaScript-u koji se koristi za rukovanje asinhronim operacijama. Omogućava vam pisanje čistijeg, čitljivijeg koda upravljanjem eventualnim vrednostima promise-a.

Zapamtite da su Promise-i dizajnirani da obrađuju buduće vrednosti. Oni predstavljaju vrednost koja će biti dostupna u nekom trenutku u budućnosti.

Napišite funkciju koja prima niz URL-ova i vraća Promise koji se rešava kada su svi zahtevi uspešno završeni. Koristite `Promise.all()`

- **Promises** su objekti koji predstavljaju konačan završetak ili neuspeh asinhrone operacije i njenu rezultativnu vrednost. Vrednost promise-a je poznata tek kada se asinhrona operacija završi.
- Evo primera kako kreirati jednostavan promise:

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Data received successfully');
  }, 2000);
});
```

- Metoda `.then()` se koristi u Promise-u da definiše šta bi trebalo da se desi kada je Promise uspešno izvršen, dok se `.catch()` koristi za rukovanje svim greškama koje nastanu.
- Evo primera korišćenja `.then()` i `.catch()` sa Promise-om:

```js
promise
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });
```

U gornjem primeru, metoda `.then()` koristi se za logovanje podataka primljenih od Promise-a, dok metoda `.catch()` koristi se za logovanje bilo kakvih grešaka koje nastanu.

- **Lančanoznačnost sa Promiseima**: Jedna od moćnih karakteristika Promise-a je što možemo povezati više asinhronih operacija zajedno. Svaki ``.then()`` može vratiti novi Promise, omogućavajući vam da izvršite sekvencu asinhronih operacija jedna za drugom.
- Evo primera lančanoznačnosti sa Promiseima:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/other-data');
  })
  .then(response => response.json())
  .then(otherData => {
    console.log(otherData);
  })
  .catch(error => {
    console.error(error);
  });
```

U navedenom primeru, prvo dohvatamo podatke sa jednog URL-a, zatim dohvatamo podatke sa drugog URL-a na osnovu prvog odgovora, i konačno logujemo drugi primljeni podatak.

Metoda ``catch`` bi obrađivala sve greške koje nastanu tokom procesa. To znači da ne morate dodavati obradu grešaka na svaki korak, što može značajno pojednostaviti vaš kod.

## Korišćenje `async/await` za rukovanje promise-ima

``async`/`await`` olakšava pisanje i čitanje asinhronog koda, koji je baziran na Promises-ima.

- **async**: Ključna reč ``async`` se koristi za definisanje asinhronne funkcije. Funkcija ``async`` vraća Promise, koji se rezolviranjem (resolves) vraća vrednost koju je vratila funkcija ``async``.
- **await**: Ključna reč ``await`` se koristi unutar funkcije ``async`` da bi privremeno zaustavila izvršavanje funkcije dok Promise nije rezolvovan. Može se koristiti samo unutar funkcije ``async``.
- Evo primera korišćenja ``async/await``:

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

U gornjem primeru, funkcija `delayedGreeting` je `async` funkcija koja pauzira na 2 sekunde pre štampanja poruke pozdrava. Ključna reč `await` se koristi za pauziranje izvršavanja funkcije dok `Promise` nije rezolvisan.

Jedna od najvećih prednosti `async/await` je rukovanje greškama putem blokova `try/catch`. Evo primera:

```js
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

fetchData();
```

U gornjem primeru, blok `try` sadrži kod koji može da izazove grešku, a blok `catch` obrađuje tu grešku ako se ona dogodi. To čini obradu grešaka jednostavnijom i čitljivijom.

## Atribut `async`

Atribut ``async`` nalaže pretraživaču da asinhrono preuzme skript fajl dok nastavlja sa parsiranjem HTML dokumenta.
Nakon što se skript preuzme, parsiranje HTML-a je pauzirano, skript se izvršava, a zatim parsiranje HTML-a ponovo počinje.
Trebalo bi da koristite ``async`` za nezavisne skripte gde redosled izvršavanja nije bitan.

## Atribut `defer`

- Atribut ``defer`` takođe preuzima skriptu asinhrono, ali odlaže izvršavanje skripte do nakon što je HTML dokument potpuno parsiran.
- Skripte ``defer`` održavaju redosled izvršenja kako se pojavljuju u HTML dokumentu.

Važno je napomenuti da oba atributa ``async`` i ``defer`` se ignorišu za ulinijske skripte (inline scripts) i rade samo za eksterne skriptne fajlove (external script files).

- Kada su prisutni i atribut `async` i atribut `defer`, atribut `async` ima prednost.

## API za geolokaciju

- Geolocation API omogućava web sajtovima da zahtevaju lokaciju korisnika.

- Primer ispod demonstrira metodu `getCurrentPosition()` API-ja, koja se koristi za dobijanje trenutne lokacije korisnika.

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

U ovom kodu, pozivamo ``getCurrentPosition`` i prosleđujemo mu funkciju koja će biti pozvana kada se pozicija uspešno dobije.

Objekat ``position`` sadrži raznovrsne informacije, ali smo ovde izabrali samo ``latitude`` i ``longitude``.

Ako postoji problem sa dobijanjem ``position``, greška će biti logovana u konzolu.

- Važno je poštovati privatnost korisnika i tražiti njihovu lokaciju samo kada je to neophodno.
