## Šta je Middleware?

- **Middleware** je funkcija koja se izvršava tokom životnog ciklusa zahteva ka serveru.
- Middleware ima pristup **objektu za zahtev** (``req``), **objektu za odgovor** (``res``) i funkciji ``next``.
- Uobičajene upotrebe: logovanje zahteva, rukovanje greškama, parsiranje dolaznih podataka i serviranje statičkih fajlova.
- Middleware se izvršava u redosledu kojim je dodan aplikaciji.

### Kako Middleware funkcioniše

1. Za server se šalje zahtev.
2. Middleware funkcije obrađuju zahtev.
3. Ako middleware pozove ``next()``, kontrola se prenosi na sledeći middleware ili rukovaoca rute (route handler).
4. Ako middleware ne pozove ``next()``, ciklus zahtev-odgovor (request-response cycle) se zaustavlja.

```javascript
app.use((req, res, next) => {
  console.log('Request URL:', req.url);
  console.log('Request Method:', req.method);
  next();
});
```

- `app.use()` dodaje *middleware* globalno na aplikaciju.

## Middleware na nivou aplikacije

- **Middleware na nivou aplikacije** je vezan za instancu Express aplikacije.
- Registruje se korišćenjem `app.use()` ili metoda specifičnih za rute kao što su `app.get()`, `app.post()`, itd.
- Izvršava se za svaki dolazni zahtev, osim ako nije ograničen na određenu rutu.
- Primene: logovanje (logging), autentifikacija i rukovanje greškama (error handling) preko više ruta.

```javascript
app.use((req, res, next) => {
  console.log('Request received at:', new Date());
  next();
});
```

## Middleware na nivou rutera

- **Middleware na nivou rutera** je vezan za instancu ``express.Router()`` umesto glavne aplikacije.
- Primena se pomoću ``router.use()`` i utiče samo na rute koje obrađuje taj ruter.
- Pomaže u modularizaciji aplikacije u izolovane module ruta.
- Montirajte ruter sa ``app.use('/path', router)`` kako biste ograničili (scope) middleware na određene rute.

```javascript
const router = express.Router();

router.use((req, res, next) => {
  console.log('Request made to /menu route');
  next();
});

router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!');
});

app.use('/menu', router);
```

## Sredstvo za rukovanje greškama

- **Middleware za obradu grešaka** je poseban middleware koji se koristi za hvatanje grešaka tokom ciklusa zahtev-odgovor.
- Ima **četiri parametra**: `err`, `req`, `res`, i `next`. Četvorostruk potpis ukazuje Expressu da je ovo rukovaoc grešaka (error handler).
- Poziva se kada mu se prosledi greška na `next(err)`.
- Treba ga postaviti **na kraju** steka middleware-a, nakon svih ostalih middleware-a i ruta.

```javascript
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something went wrong!');
});
```

## Ugrađeni i Middleware od Trećih Strana

### Ugrađeni Middleware

- **`express.json()`**: parsira dolazne zahteve sa JSON opterećenjima.

```javascript
app.use(express.json());
```

- **`express.static()`**: pruža statičke fajlove kao što su slike, CSS i JavaScript.

```javascript
app.use(express.static('public'));
```

- **`express.urlencoded()`**: Parsira dolazne zahteve sa URL-enkodiranim opterećenjima (format koji se šalje putem HTML formulara); čini parsirane podatke dostupnim na `req.body`.

```javascript
app.use(express.urlencoded({ extended: true }));
```

### Middleware Trećih Strana

- **`cors`**: omogućava Deljenje resursa sa različitih izvora (Cross-Origin Resource Sharing).

```javascript
app.use(cors());
```

- **`morgan`**: beleži HTTP zahteve.

```javascript
app.use(morgan('tiny'));
```

I ugrađene i one od trećih strana middleware funkcije dodaju se po redosledu kojim su pozvane sa `app.use()`.
