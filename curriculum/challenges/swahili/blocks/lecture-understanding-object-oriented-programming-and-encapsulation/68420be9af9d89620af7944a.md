# --description--

Objekto-orijentisano programiranje, poznato i kao OOP, je stil programiranja u kojem razvijači tretiraju sve u svom kodu kao objekat iz stvarnog sveta.

Klasa je kao nacrt za kreiranje objekata. Svaki pojedinačni objekat kreiran iz klase ima atribute koji definišu podatke i metode koje definišu ponašanja objekata.

U prethodnoj lekciji ste naučili kako da kreirate klase. Evo podsetnika za sintaksu:

```py
class ClassName:
   def __init__(self, parameters):
       attribute = value

   def method_name(self):
       # method logic
```

Evo primera klase koja koristi specijalnu metodu `__init__` za inicijalizaciju atributa brenda i boje svaki put kada se objekat kreira korišćenjem klase:

```py
class Car:
   def __init__(self, brand, color):
       self.brand = brand
       self.color = color

# create two objects from the Car class
car1 = Car('Toyota', 'red')
car2 = Car('Lambo', 'green')

print('Car 1 Brand:', car1.brand) # Car 1 Brand: Toyota
print('Car 1 Color:', car1.color) # Car 1 Color: red

print('Car 2 Brand:', car2.brand) # Car 2 Brand: Lambo
print('Car 2 Color:', car2.color) # Car 2 Color: green
```

Objekto-usmereno programiranje ima četiri ključna principa koji vam pomažu da efikasno organizujete i upravljate kodom. To su enkapsulacija, nasleđivanje, polimorfizam i apstrakcija.

Preostali deo ove lekcije će se fokusirati na to kako enkapsulacija funkcioniše.

Enkapsulacija je grupisanje atributa i metoda objekta u jednu jedinicu, klasu.

Uz enkapsulaciju, možete sakriti interni stanje objekta iza jednostavnog skupa javnih metoda i atributa koji deluju kao vrata. Iza tih vrata nalaze se privatni atributi i metode koje kontrolišu kako se podaci menjaju i ko ih može videti.

Recimo da želite pratiti stanje novčanika. Želite omogućiti ljudima da depozitiraju ili povuku novac sa novčanika, ali niko ne bi trebalo da može direktno manipulisati stanjem.

U tom slučaju, možete napraviti ``deposit()`` i ``withdraw()`` javne metode, i sakrivate stanje pod atributom ``_balance``:

```py
class Wallet:
   def __init__(self, balance):
       self._balance = balance # For internal use by convention

   def deposit(self, amount):
       if amount > 0:
           self._balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self._balance:
           self._balance -= amount # Remove from the balance safely
```

Po konvenciji, prefiksiranje atributa i metoda sa jednom donjom crtom znači da su namenjeni za interno korišćenje. Nikome ne bi trebalo direktno pristupiti izvan klase jer to krši principe enkapsulacije, što može dovesti do grešaka (bugova).

Iako je prefiks sa jednom donjom crtom samo konvencija, dodeljivanje atributima i metodama dvostrukom donjom crtom efektivno sprečava da se pristupe izvan njihove klase, čime te atribute i metode čini privatnim.

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

Da biste dobili trenutnu vrednost `__balance`, možete definisati metod `get_balance`. Na primer:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount
  
   def get_balance(self):
       return self.__balance


acct_one = Wallet(100)
acct_one.deposit(50)
print(acct_one.get_balance()) # 150

acct_two = Wallet(450)
acct_two.withdraw(28)
print(acct_two.get_balance()) # 422

acct_two.deposit(150)
print(acct_two.get_balance()) # 572
```

Takođe možete definisati privatnu metod ``__validate`` da proverite da li je svaki iznos depozita ili povlačenja pozitivan broj:

```py
class Wallet:
   def __init__(self):
       self.__balance = 0

   def __validate(self, amount):
       if amount < 0:
           raise ValueError('Amount must be positive')

   def deposit(self, amount):
       self.__validate(amount)
       self.__balance += amount

   def withdraw(self, amount):
       self.__validate(amount)
       if amount > self.__balance:
           raise ValueError('Insufficient funds')
       self.__balance -= amount

   def get_balance(self):
       return self.__balance

acct_one = Wallet()
acct_one.deposit(3)
print(acct_one.get_balance()) # 3

acct_one.deposit(50)
print(acct_one.get_balance()) # 53

acct_one.deposit(-4)  # ValueError: Amount must be positive
acct_one.withdraw(-8) # ValueError: Amount must be positive
acct_one.withdraw(58) # ValueError: Insufficient funds
```

Kao što možete da vidite, metoda `__validate` je privatna i radi u pozadini u javnim metodama `deposit()` i `withdraw()` kako bi se osiguralo da je iznos uvek validan.

U sledećoj lekciji, naučićete više o tome kako funkcionišu atributi koji su prefiksirani dvostrukim donjom crtom.

Zaključno, enkapsulacija zaključava interne podatke iza jasnih javnih metoda. Na taj način čuvate svoje klase od manipulacije i centralizujete validaciju na jednom mestu. Možete slobodno ažurirati ili proširiti svoj kod, znajući da spoljni kod dodiruje samo interfejse koje izlažete.
