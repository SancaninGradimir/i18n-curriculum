# --description--

Getteri i seteri su metode koje vam omogućavaju da kontrolišete kako se pristupače i modifikuju atributi klase. Getterima povlačite vrednost, a setterima postavljate vrednost.

Ove akcije se obavljaju putem onoga što je poznato kao svojstva (properties). Oni povezuju getter i setere i omogućavaju pristup podacima.

Svojstva deluju kao atributi, ali se ponašaju kao metode "ispod haube". Zamislite ih kao podatke koje definišete kao metode, ali koji rade kao atributi. To znači da možete pristupati svojstvima notacijom tačke umesto zagrada ili okruglih zagrada.

Glavna stvar koju svojstva rade je to što izvršavaju dodatnu logiku u pozadini kada dohvatate, postavljate ili brišete vrednosti sa njima. Zbog toga su savršeni izbor kada želite da pristupite ili manipulišete podacima unutar objekata.

Pa zašto koristiti svojstva umesto metoda? Uglavnom je reč o čitljivosti i konvenciji. Čine vaš kod čistijim i lakšim za čitanje.

Kada koristite metodu, uvek morate da je pozovete sa zagradama. Ali sa svojstvom, možete pristupiti njemu kao normalnom atributu koristeći notaciju tačke. To čini vaš kod jednostavnim čak i kada radi dodatni posao u pozadini.

Na primer, možda želite da izračunate vrednost ili proverite da li je nova vrednost validna pre nego što je sačuvate. Umesto pozivanja metode za to, možete koristiti način sličan atributu za to.

U Pythonu, dekorator (decorator) je funkcija koja modifikuje funkcionalnosti drugih funkcija ili klasa, bez menjanja njihovog originalnog koda.

Iako je moguće kreirati prilagođene dekoratore, to je izvan opsega ove lekcije. Samo znajte da biste kreirali svojstvo, definišete metodu i postavite dekorator `@property` iznad nje. Ovo pretvara metodu u svojstvo, tako da se može pristupiti kao atribut, a istovremeno interni poziva dekorisanu metodu.

To nas dovodi do getterija. Evo kako kreirati jedan sa dekoratorom `@property`:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

Ovaj primer dohvaća radijus i površinu kruga.

Primećujte kako smo koristili `_radius` umesto `radius` unutar klase. Podvučena linija je uobičajena Python konvencija za pokazivanje da je atribut namenjen da bude privatan. Drugim rečima, to ukazuje da je za internu upotrebu i ne bi trebalo direktno pristupiti njime izvan klase.

Da biste napravili setter za kreiranje radijusa, na primer, morate definisati drugu metodu istog imena i koristiti `@<property_name>.setter` iznad nje:

Korišćenje `self.radius` unutar `__init__` osigurava da se setter pokrene tokom kreiranja objekta, tako da su nevažeći radijus vrednosti odmah uhvaćeni.

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

U ovom primeru, setter za radijus ne samo da postavlja radijus za krug, već izvršava i validaciju koja osigurava da radijus nije negativan broj.

Nakon što definišete getter i setere, Python ih automatski poziva u pozadini svaki put kada koristite normalnu sintaksu atributa:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

Napomena je da unutar seta ne možete koristiti isto ime svojstva prilikom dodeljivanja nove vrednosti. To je zato što će `self.radius = value` pozvati setter unutar samog setter metoda, što dovodi do beskonačne rekurzije i `RecursionError`. Stoga uvek morate koristiti formu sa prefiksom podvučene linije: `self._radius = value`.

Upravo kao što možete kontrolisati kako se atribut pristupa preko getter-a i kako se modifikuje preko setter-a, tako možete kontrolisati i kako se briše koristeći deleter.

Deleter izvršava prilagođenu logiku kada koristite naredbu `del` na svojstvu. Da biste ga kreirali, koristite dekorator `@<property_name>.deleter`:

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius

    # Getter
    @property
    def radius(self):
        return self._radius

    # Setter
    @radius.setter
    def radius(self, value):
        if value <= 0:
            raise ValueError("Radius must be positive")
        self._radius = value

    # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

Evo kako se deleter može primeniti:

```py
# Create circle object with a radius
my_circle = Circle(33)
print("Initial radius:", my_circle.radius)  # 33

# Delete the radius
# This calls the deleter
del my_circle.radius # Deleting radius...
print("Radius deleted!") # Radius deleted!

# Try to access radius after deletion
try:
    print(my_circle.radius)
except AttributeError as e:
    print("Error:", e) # Error: 'Circle' object has no attribute '_radius'
```

Zaključak iz ovoga je da:

- Getteri vam omogućavaju da dohvatite vrednost ili čak izračunate vrednost na letu.
- Seteri vam omogućavaju sigurno modifikovanje vrednosti pokretanjem provera pre dodeljivanja.
- Svojstva su ono što spaja ove getter i setere zajedno, tako da možete pisati logiku a istovremeno koristiti notaciju tačke.
- Deleteri vam omogućavaju da definišete šta se dešava kada je atribut obrisan.
