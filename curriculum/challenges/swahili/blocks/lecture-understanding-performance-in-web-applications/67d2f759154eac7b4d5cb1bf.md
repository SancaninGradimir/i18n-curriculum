# --description--

Kao što znamo, percepcija performansi nije samo o tome koliko je sajt zapravo brz — već o tome kako korisnicima deluje da je. Dakle, na primer, da li ste ikada koristili veb-sajt koji se osećao sporo, iako to nije bilo tako? To je percepcija performansi — kako korisniku deluje brzina sajta.

Male manevri mogu napraviti ogromnu razliku u korisničkom iskustvu. Istražimo neke praktične načine za poboljšanje percepcije performansi!

Spinneri mogu učiniti da korisnici osećaju da čekaju duže nego što zapravo jesu. Umesto toga, pokušajte sa skeletnim ekranima — sivi naslovi koji imitiraju pravi sadržaj i postepeno se učitavaju:

```html
<div class="skeleton"></div>
<script>
  setTimeout(() => {
    document.querySelector(".skeleton").innerHTML = "✅ Content Loaded!";
  }, 2000);
</script>
```

Korisnici osećaju da se stranica učitava progresivno, umesto čekanja u mraku.

Pored ovoga, korisnici ne bi trebalo da čekaju da se sve učita pre nego što mogu početi da interaguju. Prvo učitajte važan sadržaj, a zatim ostatak u pozadini. Ovo smo već razmatrali sa opuštenim učitavanjem (lazy loading):

```html
<img src="placeholder.jpg" data-src="real-image.jpg" class="lazy">
<script>
  document.addEventListener("DOMContentLoaded", function () {
    const lazyImages = document.querySelectorAll("img.lazy");
    lazyImages.forEach(img => {
      img.src = img.dataset.src;
    });
  });
</script>
```

Ovo čini stranicu da izgleda responzivno, čak i ako još uvek nije učitano sve.

Sada - šta ako bi vaš sajt mogao da predvidi na šta će korisnik sledeći kliknuti? Korišćenjem pre-opterećenja (preloading), možete dohvatiti sledeću stranicu pre nego što je traže:

```html
<link rel="preload" href="next-page.html" as="document">
```

Ovo smanjuje percipirano vreme učitavanja na skoro nulu kada korisnici navigiraju.

A konačno, korisnici ne vole neizvesnost. Kada kliknu na dugme, trenutno potvrdite njihovu akciju:

```html
<button onclick="this.innerText='⏳ Processing...'; setTimeout(() => this.innerText='✅ Done!', 2000);">
  Click Me
</button>
```

Ovo znači da, čak i ako dođe kašnjenje, korisnik oseća da se nešto dešava.

Dakle, u zaključku, poboljšanje percepcije performansi nije samo o sirovoj brzini, već je o tome kako korisnik doživljava brzinu. Koristite skeletne ekrane, prioritetizujte važan sadržaj, pametno predmemorizujte i dajte trenutnu povratnu informaciju kako bi vaš sajt izgledao brže.
