# --description--

Iako su alati poput Lighthousa i Chrome DevTools-a odlični, ponekad vam trebaju prilagođeni uvidi za vaš sajt.

Tamo dolaze Performance Web API-ji!

U ovoj lekciji, razbrajaćemo kako da koristite tri ključna Web API-a za merenje i analizu brzine vaše veb stranice.

Ali prvo, šta su Performance Web API-ji?

Performansni Web API-ji omogućavaju programerima da prate koliko efikasno se web stranica učitava i reaguje, direktno iz koda. Ovi API-ji vam omogućavaju da merite vreme učitavanja stranice, pratite kašnjenja renderovanja i interakcije, te analizirate vreme izvršavanja JavaScripta.

Sa ovim API-jima, možete napraviti sopstvene alate za praćenje performansi bez oslanjanja na softver trećih strana!

Pogledajmo tri moćna Web API-ja koje danas možete koristiti.

Prvi na redu je, `performance.now()`_.

Ovaj API vam pruža vremenske oznake visoke preciznosti (u milisekundama) kako biste izmerili koliko dugo različiti delovi vašeg sajta traju da se učitaju.

Recimo da želite da izmerite koliko brzo radi funkcija:

```js
const start = performance.now();  
// Run some code here  
const end = performance.now();  

console.log(`Execution time: ${end - start}ms`);
```

Ovo je preciznije od korišćenja `Date.now()` jer meri vreme u milisekundama sa mikrosekondnom preciznošću, izbegavajući probleme sa pomeranjem sata (clock drift). Možete ga koristiti za praćenje vremena izvršavanja skripti, kašnjenja odgovora na događaje i performansi animacije.

Sledeće je Performance Timing API.

Ovaj API vam daje detaljan pregled svakog pojedinačnog koraka učitavanja stranice, od DNS pretrage do `DOMContentLoaded`.

Želite li da izmerite koliko vremena vam treba da se vaša stranica potpuno učita?

```js
let [navigationTiming] = performance.getEntriesByType("navigation");

if (navigationTiming instanceof PerformanceNavigationTiming) {
  // Calculate time from navigation start to DOM content loaded
  const pageLoadTime =
    navigationTiming.domContentLoadedEventEnd - navigationTiming.startTime;

  console.log("DOM Content Loaded Time:", pageLoadTime, "ms");
}
```

Ključne metrike koje možete pratiti sa ovim API-jem su vreme DNS pretrage - ili rečeno, brzina konekcije, Vreme do prvog bajta (TTFB) - ili brzina odgovora servera, i `DOMContentLoaded` - ili rečeno, kada je stranica spremna za interakciju.

Ako su vam vremena učitavanja stranice spora, ovaj API tačno određuje gde nastaje kašnjenje!

I konačno, hajde da pričamo o `PerformanceObserver`.

Ovaj API prati događaje performansi kao što su pomaci rasporeda, duge zadatke i interakcije korisnika.

Želite li da nadgledate dugotrajne JavaScript zadatke?

```js
const observer = new PerformanceObserver((list) => {  
  list.getEntries().forEach((entry) => {  
    console.log(`Long task detected: ${entry.duration}ms`);  
  });  
});  

observer.observe({ type: "longtask", buffered: true });
```

A šta ovaj API može da prati? Pa, on može da prati duge zadatke – ili rečima, JavaScript koji blokira renderovanje, promene rasporeda za detekciju 'UI janka', i Prilika kašnjenja unosa (FID) – ili koliko brzo stranica reaguje na unos korisnika.

Ako želite praćenje performansi u realnom vremenu, ovaj API menja igru.

Dakle, koji API bi trebalo da koristite? Evo kratkog poređenja:

| API za performanse | Najbolje za |
| :--- | :--- |
| `performance.now()` | Precizno određivanje vremena funkcija i skripti |
| API za vremensko merenje performansi | Merenje performansi učitavanja cele stranice |
| Observer performansi | Praćenje interakcija i renderovanja u realnom vremenu |

Kombinacijom ovih API-ja, možete izgraditi sopstvene alate za merenje performansi i pratiti šta je najvažnije za vaš sajt.
