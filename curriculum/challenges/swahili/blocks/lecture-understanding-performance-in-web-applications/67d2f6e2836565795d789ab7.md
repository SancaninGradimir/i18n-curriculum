# --description--

Jeste li se ikada pitali šta se dešava iza kulisa kada unesete URL i pritisnete Enter? Kako pretraživač pretvara kod u interaktivne stranice koje vidimo?

Razumevanje kako pregledač renderuje veb stranicu je esencijalno za programere koji nastoje da optimizuju performanse i poboljšaju korisničko iskustvo.

U ovoj lekciji ćemo objasniti proces renderovanja, razbijanjem ga na jasne i razumljive korake.

Prvo pregledač parsira HTML i gradi DOM.

Proces počinje što pretraživač dohvati HTML sadržaj stranice. On parsira ovaj HTML kako bi konstruisao Document Object Model, ili DOM — strukturu sličnu drvetu koja predstavlja sadržaj stranice.

Razmotrite ovaj jednostavan HTML isječak:

```html
<html>
  <body>
    <h1>Hello, World!</h1>
    <p>Welcome to our website.</p>
  </body>
</html>
```

Pretraživač ovo parsira kako bi kreirao DOM stabljiku sa `html` kao korenom, koja sadrži `body`, koji zauzvrat sadrži elemente `h1` i `p`.

Zatim, pregledač obrađuje CSS, konstruišući CSS Object Model, ili CSSOM. Ovo je još jedna struktura drveta koja određuje kako bi elementi trebalo da budu stilizovani.

S obziма на ovaj CSS:

```css
h1 {
  color: blue;
}
p {
  font-size: 16px;
}
```

CSS Object Model određuje da `h1` elementi su plavi i `p` elementi imaju veličinu fonta od 16 piksela.

Pregledač zatim kombinuje DOM i CSS Object Model kako bi stvorio render drvo. Ovo drvo uključuje samo čvorove potrebne za prikaz stranice, svaki uparen sa svojim odgovarajućim stilovima.

Za naš primer, render stab se sastoji od `h1` i `p` elemenata, odgovarajuće stilizovanih.

Sa render stabljem u ruci, pretraživač izračunava tačnu poziciju i veličinu svakog elementa u procesu koji se naziva raspored (layout) ili ponovno razređivanje (reflow). Ovo osigurava da elementi izgledaju na ispravnoj lokaciji i dimenzijama na ekranu.

Konačno, pregledač crta piksele na ekran, renderujući svaki element na osnovu izračunatih stilova i raspoređivanja. U kompleksnim stranicama, ovo može uključivati više slojeva koji se kompozituju zajedno kako bi se stvorio konačni vizuelni izlaz.

Zaključno, od parsiranja HTML-a i CSS-a do slikanja pikselima na ekranu, proces renderovanja pregledača je sofisticirana sekvenca koraka.
