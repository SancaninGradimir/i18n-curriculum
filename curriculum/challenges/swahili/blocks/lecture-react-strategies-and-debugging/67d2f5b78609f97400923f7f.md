# --description--

Kako vaša aplikacija raste, upravljanje protokom podataka između komponenti može postati složeno.

Na početku, React-ov `useState` hook možda je dovoljan, ali kako dodajete funkcije, možete naići na probleme sa:

- Prosleđivanje svojstava kroz komponente koje ih ne trebaju, poznato i kao "prop drilling"
- Održavanje sinhronizacije podataka u različitim delovima vaše aplikacije
- Upravljanje kompleksnim ažuriranjima koja utiču na više komponenti istovremeno

Ove i druge izazove mogu nastati, što može dovesti do baze koda koja je teža za održavanje, debagovanje i testiranje. Tu dolaze biblioteke za upravljanje stanjem – one pružaju centralizovano mesto gde komponente mogu dobiti ili ažurirati podatke koji im trebaju.

Pogledajmo nekoliko različitih opcija za upravljanje stanjem koje imate i kada ih koristiti.

Context API je menadžer stanja ugrađen u React koji vam omogućava deljenje stanja između komponenti bez korišćenja biblioteke treće strane. To je dobro uspostavljen nadogradnja u odnosu na `useState` hook, pa je savršen za slučajeve kao što su prebacivanje tema ili status autentifikacije korisnika.

Međutim, Context API nije dobar u rukovanju učestalim ažuriranjima, i može izazvati nepotrebno ponovno renderovanje, zbog čega je manje pogodan za složene potrebe stanja u aplikacijama kao što su e-trgovina i platforme društvenih medija.

Evo komponenta brojača koja demonstrira osnovnu upotrebu Context API-ja:

```jsx
import { useState, createContext } from 'react';

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

Ovaj kod kreira kontekst i provajdera za deljenje stanja `count` širom aplikacije.

`CounterProvider` koristi `useState` kuku da inicijalizuje i upravlja stanjem `count` i njegovim seterom. Oboje se zatim prosleđuju u komponente dece putem `Provider`.

Dakle, kada obuhvatite celu aplikaciju sa `CounterProvider`, stanje `count` je dostupno svuda u vašoj aplikaciji.

Evo kako možete obuhvatiti `CounterProvider` oko vaše aplikacije:

```jsx
import { CounterProvider } from './context/CounterContext';

function App() {
  return (
    <CounterProvider>
        {/* App components */}
    </CounterProvider>
  );
}

export default App;
```

Ite kako možete koristiti stanje `count`:

```jsx
import React, { useContext } from 'react';
import { CounterContext } from '../context/CounterContext';

const Counter = () => {
  const { count, setCount } = useContext(CounterContext);

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Context API Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => setCount(count - 1)}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => setCount(count + 1)}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

Kao što možete da vidite, `count` i njegova funkcija za postavljanje, `setCount`, se inicijalizuju putem funkcije `useContext`.

Trenutni status `count` se zatim prikazuje, a `setCount` se koristi za povećanje i smanjenje stanja `count` kada korisnik respektivno klikne na dugmad za smanjenje i povećanje.

Još jedna popularna biblioteka za upravljanje stanjem je Redux, koja je jedna od najpopularnijih biblioteka za upravljanje stanjem za korišćenje sa React-om. Postoji već dugo i idealna je za veće aplikacije kao što su e-trgovina (eCommerce) i platforme društvenih medija, forumi, itd.

Redux upravlja upravljanjem stanjem pružanjem centralnog skladišta i stroge kontrole nad ažuriranjima stanja. Koristi predvidljiv obrazac sa akcijama, redukcerima i middlewerom.

Akcije su opterećenja informacija koja šalju podatke iz vaše aplikacije u Redux skladište, često pokrenute interakcijama korisnika.

Redukceri su funkcije koje određuju kako bi se stanje trebalo promeniti u odgovor na te akcije, osiguravajući da je stanje ažurirano imutabilnim načinom.

Prosljeđivač, s druge strane, deluje kao most između dispečiranja akcija i reducer, omogućavajući vam da proširite funkcionalnost Redux-a (na primer, logovanje, rukovanje asinhkronnim operacijama) bez modifikovanja osnovnog toka.

Najčešća žalba za Redux je sve onim boilerplate kodom koji vam je potreban da započnete. Kao odgovor na to, tim Reduxa uveo je "Redux Toolkit" i "RTK Query", koji značajno pojednostavljuju proces podešavanja.

Obično definišete i akcije i reducere u jedinom fajlu koristeći funkciju `createSlice()`. Uobičajeno je nazvati fajl tako da se završava rečju `Slice`, na primer, `productSlice`, `userSlice`, `counterSlice`, i tako dalje.

Evo `counterSlice` fajla da vam pokaže osnove:

```jsx
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',

  initialState: { count: 0 },

  reducers: {
    increment: (state) => {
      state.count += 1;
    },
    decrement: (state) => {
      state.count -= 1;
    },
  },
});

export const { increment, decrement } = counterSlice.actions;

export default counterSlice.reducer;
```

Odavde, zatim morate obaviti omotavanje cele aplikacije sa `Provider`, izaberite deo stanja iz slice-a pomoću `useSelector()`, a zatim koristite `useDispatch()` kako biste učinili stanje aktivnim.

Još jedna opcija za razmatranje je Zustand.

Zustand je biblioteka za upravljanje stanjem koja je lagana i ima jednostavan API. Zasnovan je na hookovima, pa ima manje dosadnog koda (boilerplate) u poređenju sa Redux-om, što ga čini lakšim i bržim za podešavanje.

Zustand je idealan za aplikacije malog do srednjeg obima. Radi se korišćenjem ``useStore`` hook-a za direktan pristup stanju u komponentama i stranicama. Ovo vam omogućava da modifikujete i pristupite podacima bez potrebe za akcijama, reducerima ili provajderom.

Evo jednog `useCounterStore` koji implementira drugu funkcionalnost brojača:

```jsx
import { create } from 'zustand';

const useCounterStore = create((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
  decrement: () => set((state) => ({ count: state.count - 1 })),
}));

export default useCounterStore;
```

I ovde je kako da inicijalizujete i koristite stanja u vašoj aplikaciji:

```jsx
// Import the useCounterStore (it's just a hook)
import useCounterStore from '../useCounterStore';

const Counter = () => {
  // Initialize the states with the useCounterStore hook
  const { count, increment, decrement } = useCounterStore();

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Zustand Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => decrement()}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => increment()}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```
