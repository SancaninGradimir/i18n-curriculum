# --description--

React Server Components (RSCs) je novi trend koji je promenio način na koji se React programeri pristupi stvarima. Sa RSC-ovima, više posla se prebacuje na server, što ima mnogo prednosti.

Pogledajmo šta su server komponente, kako funkcionišu i šta je dovelo do uvođenja server komponenti.

React Server Components su React komponente koje se renderuju isključivo na serveru, koji šalje samo konačni HTML klijentu. To znači da te komponente mogu direktno pristupiti resursima sa strane servera i dramatično smanjiti količinu JavaScript-a poslanog u pretraživač.

React aplikacije su tradicionalno koristile sistem „klijentske komponente“ koji obrađuje sve u tipičnoj React aplikaciji, kao što su renderovanje, interaktivnost i nuspojave. Termin „klijentska komponenta“ retko je korišćen do nedavnog uvođenja React server komponenti.

Ali sistem klijentskih komponenti dolazi sa nekim manama, kao što su veliki JavaScript paketi i sporije početno učitavanje.

React okviri poput Next.js i Gatsby pronašli su rešenja za prebacivanje nekih procesa na server kako bi popravili te probleme, ali nijedan od njih nije bio standardizovan. Ako ste koristili bilo koji od ovih okvira, verovatno ste čuli za `getServerSideProps` i `getServerData`.

Zatim su došli React Server Components, koji vam omogućavaju da pokrenete neke komponente u potpunosti na serveru kako biste mogli da radite stvari kao što su dohvaćanje podataka i izračunavanje pre nego što bilo koji kod radi u pregledaču korisnika.

Server komponente su prvi popularizovane i lako dostupne u Next.js-u. Drugi okviri poput Remiksa i Gatsby prate korak, a postoji eksperimentalni dodatak za Vite nazvan `vite-plugin-react-server` koji vam omogućava da gradite server komponente.

Pa kako server komponente funkcionišu?

Jedan od najboljih načina za demonstraciju React Server Components je dohvaćanje podataka.

U tradicionalnim React klijentskim komponentama, dozvoljavate pretraživaču da obrađuje zahtevove API-ja. Pošto je dohvaćanje podataka nuspojava (side effect), taj API poziv vršite u `useEffect` hooku.

Takođe je dobra praksa postaviti varijable stanja kao što su `loading`, `data` i `error` kako biste mogli da naznačite da se podaci učitavaju, prikažete podatke kada su spremni ili prikažete grešku u vašoj aplikaciji.

Sa React Server Components, možete prebaciti celu komponentu na server i dohvaćati podatke tamo bez potrebe za korišćenjem `useState` ili `useEffect`:

```jsx
const Users = async () => {
  const res = await fetch("https://jsonplaceholder.typicode.com/users");
  const users = await res.json();

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users;
```

Pošto se React Server Components pokreću samo na serveru, možete jednostavno dohvatati podatke sa API-ja i renderovati ih samo jednom. Takođe, pošto se dohvaćanje podataka dešava na serveru, bliže izvoru, vaša aplikacija može bolje da radi, posebno za ljude sa sporim mrežnim konekcijama.
