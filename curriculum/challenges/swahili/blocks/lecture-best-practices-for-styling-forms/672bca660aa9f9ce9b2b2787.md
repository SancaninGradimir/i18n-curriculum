# --interactive--

Naučićemo se o nekim uobičajenim problemima kada pokušavamo da stilizujemo posebne elemente za unos kao što su unosi `datetime-local` i `color`.

Ovi specijalni tipovi ulaznih polja se oslanjaju na složene pseudo-elemente za kreiranje stvari kao što su birači datuma i boja. Ovo predstavlja značajan izazov za stilizovanje ovih ulaznih polja. Jedan od izazova je što, pošto zavisnost podrazumevanog stilizovanja zavisi u potpunosti od pregledača, CSS koji čini da birač izgleda ispravno u jednom pregledaču može proizvesti veoma različit rezultat u drugom.

Evo primera unosa boje:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="favorite-color">Pick your favorite color:</label>
  <input type="color" id="favorite-color" name="favorite-color">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="color"] {
  width: 60px;
  height: 40px;
  padding: 0;
  border: 2px solid #555;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

Drugim može biti složenost pseudo-elementa. Razmislite o selektoru datuma; ovde ima mnogo pokretnih delova, a složena struktura pseudo-elementa može predstavljati značajan izazov u primeni stilizovanja na prave oblasti.

Here is an example of a date input:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="birthdate">Select your birthdate:</label>
  <input type="date" id="birthdate" name="birthdate">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="date"] {
  padding: 6px 10px;
  border: 2px solid #555;
  border-radius: 4px;
  font-size: 14px;
  cursor: pointer;
}

input[type="date"]::-webkit-calendar-picker-indicator {
  background-color: #4CAF50;
  color: white;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

Naravno, sa ovim kompleksnim elementima, takođe rizičite od slučajnog gubitka važnih funkcionalnosti kada ih ručno stilizujete. Ne samo da možete izgubiti važne indikatore kao što su fokus stanje ili izabrani stav, već potencijalno možete potpuno pokvariti selektor.

Iz ovih razloga, mnogi programeri se oslanjaju na JavaScript biblioteke ili prilagođene komponente umesto korišćenja ugrađenih komponenti pregledača.
