# --description--

JavaScript je prvobitno kreiran da radi u web pregledačima, gde je njegova glavna uloga bila učiniti veb stranice interaktivnim. Zbog toga se rani JavaScript fokusirao prvenstveno na rukovanje tekstom u formularima i manipulisanje Document Object Model-om (DOM).

Međutim, nisu podaci sa celog veba samo tekst. Fajlovi, slike i video zapisi su binarni podaci, koji zahtevaju različite mehanizme za obradu. U pregledačima, ovakvi tipovi podataka se obično obrađuju od strane specijalizovanih komponenti samog pregledača, a ne direktno od strane JavaScript-a.

Moderni pregledači koriste renderovanje motore i JavaScript motore za upravljanje ovim zadacima — na primer, Blink (sa V8) u Chrome-u, WebKit (sa JavaScriptCore) u Safariju i Gecko (sa SpiderMonkey) u Firefox-u.

Node.js ne radi u pregledaču, pa mu je bio potreban sopstven način za rukovanje binarnim podacima, posebno prilikom obrade ulaza i izlaza fajlova (I/O) i TCP tokova, gde dolaze podaci u paketima (chunks). Tu dolazi modul `Buffer`.

Node.js `Buffer` modul vam omogućava da radite sa binarnim podacima kao što su fajlovi, slike ili mrežni tokovi direktno. Sa njime možete skladištiti i manipulisati binarnim podacima direktno u memoriji.

Kao i `fs` modul, `Buffer` je jedan od osnovnih Node.js modula, tako da ga ne morate instalirati zasebno pre korišćenja.

Da biste ga koristili, prvo ga uvezite destrukturiranjem:

```js
const { Buffer } = require("buffer");
```

Zatim pozovite `Buffer` sa metodama koje pruža. Na primer, `Buffer.from()` omogućava vam da kreirate buffer iz stringa, niza ili drugih sirovih podataka.

Evo kako kreirati `Buffer` iz stringa i niza:

```js
// Create a buffer from a string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// Create a buffer from an array of numbers
const myNumBuffer = Buffer.from([
  70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80,
]);

console.log(myNumBuffer); // <Buffer 46 52 45 45 43 4f 44 45 43 41 4d 50>
```

Iako je moguće koristiti neke metode iz `Buffer` modula bez prethodnog uvoza, druge metode nisu dostupne osim ako eksplicitno ne uvezete `Buffer`. Stoga se preporučuje da uvek uvezite Buffer kada ga koristite u svojim projektima.

Možete pristupiti pojedinačnim elementima `buffer`-a baš kao nizu:

```js
console.log(myNumBuffer[0]); // 70
console.log(myStrBuffer[0]); // 102
```

Takođe možete koristiti metodu `toString()` na bufferima kako biste videli kako zaista izgledaju:

```js
console.log(myStrBuffer.toString()); // freeCodeCamp
console.log(myNumBuffer.toString()); // FREECODECAMP
```

`Buffer.alloc()` omogućava vam da kreirate novi buffer određenog veličine (broj bajtova). Svaki bajt unutar njega automatski je popunjen sa `0`:

```js
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

Možete videti da je buffer inicijalizovan nulama, na osnovu veličine prosleđene u metodu `alloc()`.

Možete nastaviti i koristiti metodu `Buffer.write()` za pisanje u ovaj buffer:

```js
someBuffer.write("Hello fCC");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 43 43 00>
console.log(someBuffer.toString()); // Hello fCC
```

Ako napišete više podataka nego što buffer može da primi, on će biti skraćeno (truncated):

```js
someBuffer.write("Hello freeCodeCamp");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 72 65 65>
console.log(someBuffer.toString()); // Hello free
```

Na kraju, možete koristiti `Buffer.byteLength()` da pokažete broj bajtova potrebnih za skladištenje stringa u određenom kodiranju (encoding):

```js
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

Ostale `Buffer` metode uključuju:

- `Buffer.isBuffer()`: proverava da li je dati objekat buffer
- `Buffer.compare()`: poredi dva buffera i vraća njihov redosled sortiranja
- `Buffer.concat()`: spaja više buffer-a zajedno u jedan
