# --description--

Kripto je još jedan osnovni modul ugrađen u Node.js. Uključuje alate za stvari kao što su хеšovanje, šifrovanje, dešifrovanje i kreiranje digitalnih potpisa, a sve se koristi za zaštitu osetljivih informacija i održavanje sigurnosti vaše aplikacije.

Rekli smo da `crypto` pruža niskonivove građevne blokove, a ne "plug-and-play" bezbednost. Kreiranje sopstvenog koda za šifrovanje ili autentifikaciju može biti nesigurno ako niste pažljivi. U većini slučajeva, najbolje je koristiti dobro testirane biblioteke kao što su `bcrypt` za хеšovanje lozinki ili `jsonwebtoken` (JWT) za upravljanje prijavama i tokenima.

Ipak, korisno je razumeti kako rade neki od metoda u kripto modulu.

Da biste koristili ove metode, morate uvesti modul `crypto`:

```
```js
const crypto = require("crypto");
```
```

Neke metode se koriste radi transformacije podataka, kao što su one za хеšovanje i šifrovanje podataka, dok druge služe za upravljanje ključevima i bezbednošću, kao što su one za generisanje nasumičnih vrednosti i kreiranje tajni.

Pogledajmo prvo metode za хеšovanje i šifrovanje podataka.

Metoda `createHash()` kreira objekat za хеš tako što prima algoritme poput `sha256`, `sha512` ili `md5`. To je jednosmerna operacija, pa ne možete da ga obrnete.

`createHash()` je koristan za хеšovanje lozinki i otiskivanje fajlova. Da biste ga koristili, prosleđujete svoj algoritam, koristite metodu `update()` da unesete podatke, a na kraju koristite `digest()` sa kodiranjem kako biste dobili vrednost хеša:

```
```js
const crypto = require("crypto");

const hashedPassword = crypto
  .createHash("sha256")
  .update("myStrongPassword")
  .digest("hex");

console.log("createHash result:", hashedPassword);
// createHash result: f92c9cfa0ead1bcec05ca75888a4074ba994ad237e5e2a8c7cc6a620378c061d
```
```

`createHmac()` radi skoro isto kao i `createHash()`, ali ga vodi na viši nivo jer prihvata tajni ključ, tako da samo neko ko poseduje taj ključ može da verifikuje хеš. Idealno je za autentikaciju i proveru integriteta podataka:

```
```js
const crypto = require("crypto");

const hashedMessage = crypto
  .createHmac("sha256", "secretkey")
  .update("important-secret-message")
  .digest("hex");

console.log("createHmac result:", hashedMessage);
// createHmac result: da48d6f026b6036286b1fb872c63264130d5cc4271f3a213bb6ddca5a023e77e
```
```

Metode `createCipheriv()` i `createDecipheriv()` šifruju i dešifruju podatke. Obje prihvataju algoritam, ključ i `iv`, koji je blok nasumičnih ili jedinstvenih podataka korišćen na početku procesa šifrovanja:

```
```js
createCipheriv(algorithm, key, iv);
createDecipheriv(algorithm, key, iv);
```
```

Da biste dešifrovali podatke, ključ mora biti isti, inače će dešifrovanje propasti i dobićete grešku ili nečitljiv žargon umesto originalne poruke.

Evo ova dva u akciji:

```
```js
const crypto = require("crypto");

// A key must match the algorithm length. Here AES-256 is 32 bytes
const key = Buffer.from("12345678901234567890123456789012");

// A fixed IV, 16 bytes for AES
const iv = Buffer.from("1234567890123456");

const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);

let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

console.log("Encrypted data:", encrypted);
// Encrypted data: 4ee93aa398ab44e3540e4a67ca96bc8c

// Decrypt the "Hello campers!" message
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");

console.log("Decrypted data:", decrypted);
// Decrypted data: Hello campers!
```
```

Druga kripto metoda za transformaciju podataka je `sign()` i `verify()`. 

`sign()` kreira digitalni potpis iz nekih podataka koristeći privatni ključ. Ovaj potpis dokazuje da su podaci došli od vlasnika privatnog ključa i da nisu bili manipulisani.

Zatim, `verify()` proverava taj potpis, i propada ako se podaci ili potpis ne poklapaju.

Sada pogledajmo kripto metode za generisanje nasumičnih vrednosti i kreiranje tajni.

`randomBytes()` prima `size` i generiše kriptografski sigurne tokene. To ga čini dobrim za generisanje UUID-ova (universalno jedinstveni ID-ovi). Pored toga, dobra zamena je za `Math.random()`, koji nije siguran za tokene i ključeve.

```
```js
console.log("Random Bytes:", crypto.randomBytes(16));
// Random Bytes: <Buffer 01 88 aa 1e 2c 38 48 39 26 e1 6b a9 d8 c5 ed 49>
```
```

Izlaz je podrazumevano buffer. Kao što ste naučili u lekciji o Buffer modulu, možete taj Buffer konvertovati u string metodom `toString()`:

```
```js
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Random Bytes: a6154ef5a296fa176ad0f332bd94d712
```
```

Argument `'hex'` u `toString('hex')` ovde govori Node-u da kodira binarne podatke iz Buffera kao heksadecimalni string.

Metoda `randomInt()` prima vrednosti `min` i `max` i generiše siguran nasumičan integer između njih. Koristan je za OTP (One-Time Passwords) i nasumično odabiranje.

```
```js
console.log("Random Int:", crypto.randomInt(0, 100)); // 89
```
```

Opet, nadogradnja u odnosu na `Math.random()` i `Math.floor()` jeste što metoda koristi kriptografski siguran nasumičnost ispod haube, tako da napadači ne mogu predvideti rezultujući nasumični broj.

Druga metoda je `createSecretKey()`. Prima buffer i generiše sirovi bajt umotan u `KeyObject`:

```
```js
const crypto = require("crypto");

const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret); // SecretKeyObject [KeyObject] { [Symbol(kKeyType)]: 'secret' }
```
```

Zatim možete koristiti metodu `export` da pošaljete taj `KeyObject`:

```
```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export());
// <Buffer 53 06 a1 c7 75 69 8b 38 8b a4 b2 f7 1b bc b8
// ae e2 d1 bf 67 af 1a 6a 0a 6e a0 29 62 bb 52 52 32>
```
```

I na kraju, koristite `toString()` na bufferu da vidite njegov string predstavnik:

```
```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
// 32dfe5917668580160986f1623bf8152913329c71163be9c3404a110cd78efd6
```
```

Pored ovoga postoje:

- `createPublicKey()` i `createPrivateKey()` koji vam omogućavaju rad sa ključevima generisanim negde drugde
- `createDiffieHellman()` za dve strane da generišu zajednički tajni bez direktnog slanja tog tajnog koda
- `Certificate()` za rad sa onim korišćenim u HTTPS, tako da možete parsirati, eksportovati i proveriti sadržaj sertifikata

Koja od ovih tvrdnji o `sign()` i `verify()` je tačna?
## --answers--

`sign()` koristi javni ključ za kreiranje potpisa, i `verify()` ga proverava privatnim ključem.

Both `sign()` and `verify()` require only a private key.
