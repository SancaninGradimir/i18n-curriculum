# --description--

Mnogi jezici i okviri nude načine za rad sa fajlovima i direktorijumima. Node.js vam omogućava da to uradite sa `fs` ili modulom sistema fajlova.

Ova lekcija će se fokusirati na modul Node `fs`, kako radi, i tri različita načina na koja možete koristiti metode koje pruža.

Modul Node `fs` vam pruža metode za rad sa fajlovima i direktorijumima, uključujući operacije otvaranja i zatvaranja, čitanja i pisanja, te brisanje.

Modul ``fs`` je standardni Node.js modul, pa je dostupan za korišćenje sve dok imate instaliran Node.js u okruženju. Da biste koristili modul, uvezete ga na ovaj način:

```js
const fs = require("fs");
```

Fleksibilnost koju modul `fs` nudi je u tome što vam omogućava da koristite njegove metode kako sinhrono, tako i asinhrono. Metode modula `fs` su asinhrone po defaultu, ali za svaku metodu postoji sinhrona forma:

```js
fs.writeFile() // Asynchronous file writing
fs.writeFileSync() // Synchronous file writing

fs.readFile() // Asynchronous file reading
fs.readFileSync() // Synchronous file reading

fs.open() // Opens a file
fs.openAsBlob() // Opens as blob
fs.openSync() // Synchronous open
fs.opendir() // Opens directory
fs.opendirSync() // Synchronous directory open
```

To nije kraj. U zavisnosti od vaših potreba, možete koristiti metode na tri načina:

- sa povratnim pozivima, na primer `fs.writeFile()` (asinhrono)
- sa obećanjima ako više volite sintaksu `async/await`, na primer `fs.promises.writeFile()`
- sinhrono, na primer, `fs.writeFileSync()`

Evo osnovne sintakse za asinhrono korišćenje metoda:

```js
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) {
    throw err;
  }
  console.log("File written to!");
});
```

Za verziju obećanja, možete ga uvesti iz `fs/promises` ili povezati `promises` sa vašim `fs` uvozom. Evo sintakse:

```js
async function promisesExample() {
  try {
    await fs.promises.writeFile("filePath", "content", "utf8");
    console.log("File written to!");
  } catch (err) {
    console.error("Error:", err);
  }
}

promisesExample();
```

Evo sinhronizovane verzije:

```js
try {
  fs.writeFileSync("filePath", "content", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error("Error:", err);
}
```

Napomena da korišćenje bilo koje od metoda sinhrono predstavlja blokiranje. Drugim rečima, vaš program će prestati sa radom i čekati dok operacija ne bude završena pre nego što pređe na sledeći red koda.

Za brze skripte, male projekte ili jednokratne zadatke, sinhroni metodi su verovatno u redu. Ali u aplikacijama iz stvarnog sveta, blokirajući I/O (ulaz/izlaz) postaje problem jer može zamrznuti druge delove vaše aplikacije dok Node čeka da sistem za fajlove završi svoje operacije. To usporava stvari i šteti performansi i korisničkom iskustvu.

Zato postoje asinhroni metodi. Oni omogućavaju ostatku vašeg programa da nastavi sa radom dok Node obavlja operacije sa fajlovima u pozadini. Tradicionalno se ovo koristilo sa povratnim pozivima (callbacks), ali to može brzo postati neuredno kako vaša aplikacija raste.

Obećanja i `async/await` rešavaju taj problem. I dalje nisu blokirajući, ali kod izgleda kao normalni sinhroni kod i mnogo je lakše za održavanje.

Iz tog razloga, pristup zasnovan na obećanjima je danas generalno preferiran, i to ćemo koristiti u ostatku ove lekcije.

Metoda `writeFile()` vam omogućava da pišete u postojeći fajl. Ako fajl ne postoji, on ga kreira u trenutnom direktorijumu, a zatim mu piše navedeni sadržaj.

Evo kako da se koristi metoda `writeFile()`:

```js
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile(
      "article.md",
      "## Node `fs` Module: The Complete Guide",
      "utf8",
    );
    console.log("File written to!");
  } catch (err) {
    console.error("Error writing to file:", err);
  }
}

writeToFile(); // File written to!
```

Evo sadržaja fajla article.md nakon toga:

```md
## Node `fs` Module: The Complete Guide
```

Metoda `appendFile()` vam omogućava da dodate sadržaj postojećem fajlu. Evo kako ga koristiti:

```js
const fs = require("fs/promises");

async function appendToFile() {
  try {
    await fs.appendFile(
      "article.md",
      "\nIn this article, you will learn all there is to know about the Node fs module...",
      "utf8",
    );
    console.log("File appended to!");
  } catch (err) {
    console.log("Error appending to file:", err);
  }
}

appendToFile(); // File appended to!
```

Metoda `readFile()` vam omogućava da vidite kakav je sadržaj fajla:

```js
const fs = require("fs/promises");

async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md", "utf8");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: ## Node `fs` Module: The Complete Guide

In this article, you will learn all there is to know about the Node fs module...
*/
```

Ako ne navedete kodiranje znakova `utf8`, dobićete sadržaj fajla kao keš:

```js
async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*

 54 68 65 20 43 6f 6d 70 6c 65 74 65 20 47 75 69 64 65 0a 0a 49 6e 20 74 68 69 73 20 ... 
72 more bytes>
*/
```

U sledećoj lekciji, naučićete kako da rukujete baferom kao ovakvim sa Node.js Buffer modulom.

Konačno, metoda `unlink()` vam omogućava brisanje fajla:

```js
const fs = require("fs/promises");

async function deleteFile() {
  try {
    await fs.unlink("article.ts");
    console.log("File deleted successfully!");
  } catch (err) {
    console.error("Error deleting file:", err);
  }
}

deleteFile();
```

Ako fajl ne postoji, dobijate poruku o grešci sličnu ovoj:

```bash
Error deleting file: Error: ENOENT: no such file or directory, unlink 'article.ts'
```

Ako postoji, datoteka se briše i dobijate poruku o uspehu koju ste postavili.
