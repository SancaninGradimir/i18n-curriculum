# --description--

Node.js nema ugrađeni WebSocket server. Programeri obično koriste eksterne pakete kao što je ``ws`` za implementaciju WebSocket protokola i rukovanje komunikacijom u stvarnom vremenu između klijenata i servera.

Biblioteka ``ws`` je lagana, brza i laka za korišćenje. Implementira WebSocket protokol tako da možete kreirati i upravljati WebSocket konekcijama.

Možete ga instalirati sa npm-a:

```bash
npm install ws
```

Nakon instalacije, kreirajte WebSocket server uvozi root `ws` biblioteku, pozivanjem njene metode `Server`, i prosleđivanjem porta:

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

Ovo pokreće WebSocket server koji sluša konekcije na portu `3000`.

Kada se klijent poveže, server emituje `connection` događaj.

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

Smatrajte ovaj događaj kao serverov način da kaže: "pridružio se novi klijent".

Parametar ``socket`` predstavlja konekciju između servera i tog klijenta. Koristite ga za slanje i primanje poruka.

Kada povezani klijent pošalje podatke, server emituje događaj `message`.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

U ovom primeru, server sluša dolazne poruke i loguje ih u konzolu.

Možete poslati podatke nazad klijentu korišćenjem metode `send()`:

```js
socket.send("Hello from the server");
```

Ovo šalje poruku putem otvorene WebSocket veze.

Dok konekcija ostane otvorena, i klijent i server mogu nastaviti razmenu poruka u realnom vremenu.

Osim od ``connection`` i ``message``, WebSocket konekcije takođe emituju događaje ``close`` i ``error``.

`close` se pokreće kada se konekcija zatvori:

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error` se aktivira kada nešto pođe po zlu:

```js
socket.on("error", (error) => {
  console.error(error);
});
```

Zajedno, ovi događaji čine temelj većine WebSocket aplikacija.

Čest zahtev u aplikacijama za ćaskanje je broadcastovanje. Broadcastovanje znači slanje poruke više povezanih klijenata umesto samo jednog.

Zamislite čat sobu sa tri korisnika po imenu **John Doe**, **Jane Doe**, i **Wes Smith**.

Ako John Doe pošalje poruku, server možda mora da prosledi tu poruku Jane Doe i Wesu Smith-u.

Da bi to uradio, server prati povezani klijente i prolazi kroz njih kada stigne poruka.

Pre slanja poruke, server proverava da li je konekcija klijenta još uvek otvorena:

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

Svaka WebSocket konekcija ima svojstvo `readyState` koje opisuje njen trenutni status.

``WebSocket.OPEN` znači da je konekcija aktivna i spremna za slanje i prijem podataka.`

Proveravanjem toga, server izbegava slanje poruka klijentima koji su se već odjavili ili čije veze još uvek uspostavljaju:

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

U ovom primeru, svaki povezani klijent prima dolaznu poruku. Ovo se zove broadcastovanje.

Mnoge aplikacije u stvarnom vremenu koriste broadcastovanje za dostavljanje podataka kao što su poruke iz ćaskanja, živa obaveštenja, sportska ažuriranja i događaji višeg igrača.

Sa `ws` bibliotekom, nije komplikovano izgraditi WebSocket servere u Node.js-u.

Sve što treba da uradite je kreiranje servera, slušanje događaja, razmena poruka i emitovanje ažuriranja povezanih klijentima.

Ovi građevinski blokovi čine temelj mnogih aplikacija u stvarnom vremenu.
