# --description--

WebSoketi omogućavaju klijentu i serveru da komuniciraju u realnom vremenu.

Obično, aplikacije na strani klijenta, kao što su veb i mobilne aplikacije, koriste HTTP. Sa HTTP-om, klijent šalje zahtev, a server vraća odgovor. Ovo se naziva ciklus zahteva i odgovora, kao što ste naučili.

Nakon što se uspostavi konekcija i podaci se razmene, konekcija se zatvara. Ovo dobro funkcioniše za većinu stvari, kao što je učitavanje stranice, preuzimanje ili postavljanje podataka. Ali postaje sporo i nepouzdano za aplikacije u realnom vremenu.

Na primer, razmislite o aplikaciji za ćaskanje. Ako odlučite da koristite HTTP za nju, klijent mora stalno tražiti podatke od servera s vremena na vreme, sa otvaranjem veze, dolaskom podataka i zatvaranjem veze nakon toga za svaku poruku.

Ovo je problem koji rešavaju WebSockets. Kreiraju trajno povezivanje između klijenta i servera. Kada se konekcija otvori, obe strane mogu razmenjivati podatke u bilo kom trenutku putem kanala u realnom vremenu sve dok jedna od strana ne pošalje signal zatvaranja drugoj.

Iako su WebSoketi različiti od HTTP-a, svaka WebSocket konekcija počinje sa HTTP-om, u procesu koji se naziva "handshake" (ruko popunjavanje).

Klijent zatim zahteva da se konekcija nadogradi na WebSocket. Kada server pristane, šalje odgovor ``101 Switching Protocols`` koji nadograđuje konekciju na WebSocket. Nakon toga, konekcija ostaje otvorena i počinje komunikacija u realnom vremenu.

Tok izgleda ovako:

1. Klijent se povezuje na server putem HTTP-a
2. Server prihvata i nadograđuje konekciju na WebSocket
3. Konekcija ostaje otvorena
4. Obje strane razmenjuju podatke u bilo kojem trenutku dok se konekcija ne zatvori

Ovaj tok je dvosmeran, a nije poput modela zahtev-odgovor koji je karakterističan za HTTP.

```md
Client <---> Server
```

Sa HTTP-om, tok obično prati model zahtev-odgovora:

```md
Client ---> Request
Server ---> Response
```

O tome kako se WebSockets razlikuju od HTTP-a, HTTP je kao razmena pisama, a WebSocket je kao telefonski poziv. Napišete pismo, čekate odgovor, i razgovor se završava. Ali telefonskim pozivom, vi i primaoc razgovarate u realnom vremenu.

WebSoketi pokreću aplikacije u realnom vremenu, kao što su chat aplikacije, online multiplayer igre, uživo sportski ažuramenti, kontrolne table berze trağı i kolaborativni alati poput Google Dokumenta.
