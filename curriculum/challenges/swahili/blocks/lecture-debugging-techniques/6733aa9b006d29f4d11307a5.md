# --description--

Kako ste programirali u JavaScript-u, neizbežno ste naišli na poruke o greškama. Razumevanje uobičajenih poruka o greškom će vam pomoći da bolje rešavate probleme (debugujete) i razvijete se u jačeg programera. Četiri uobičajena tipa poruka o greškom su `SyntaxError`, `ReferenceError`, `TypeError` i `RangeError`.

`SyntaxError` nastaje kada nešto pogrešno napišete u svom kodu, kao što je nedostajuća zagrada ili uglasti zagradnik. Razmislite o tome kao o gramatičkoj grešci u rečenici. Evo uobičajene greške koju programeri prave prilikom kreiranja nizova (arrays):

```js
const arr = ["Beau", "Quincy" "Tom"]
```

Svaki element niza mora biti razdvojen zarezom, inače će rezultirati porukom o greškom.

Drugi uobičajeni JavaScript greška je `ReferenceError`. Postoji nekoliko tipova `ReferenceError`-a, koji se aktiviraju na različite načine. Prvi tip `ReferenceError`-a bi bile neizdefinovane varijable.

```js
console.log(price);
```

U ovom primeru pokušavamo da logujemo (ispisemo) varijablu `price` u konzolu, ali ona nije definisana. Ovo će rezultirati `ReferenceError`.

Još jedan primer `ReferenceError`-a je pokušaj pristupa varijabli deklarisanoj sa `let` ili `const`, pre nego što je definisana:

```js
console.log(b);
const b = 50;
```

Gornji primer će rezultovati greškom `Cannot access 'b' before initialization`.

Treći uobičajeni greška bi bio `TypeError`. Ove greške nastaju kada pokušate da izvršite operaciju na pogrešnom tipu podataka. Evo primera pokušaja korišćenja metode `map` na objektu:

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

Ovaj primer će rezultovati greškom `developerObj.map is not a function` jer se metoda `map` koristi za nizove (arrays), a ne za objekte.

Poslednja uobičajena greška koju ćemo pogledati je `RangeError`.

`RangeError` nastaje kada vaš kod pokušava da koristi vrednost koja je izvan opsega onoga što JavaScript može da obradi. Evo primera dodeljivanja nevažeće vrednosti dužini niza:

```js
const arr = [];
arr.length = -1; 
```

Pošto mora biti celobrojni broj koji nije negativan, postavljanje na `-1` pokreće `RangeError`.

Kako nastavite sa programiranjem u JavaScript-u, samo budite svesni ovih različitih tipova grešaka koje ćete verovatno naići i zašto se dešavaju.
