# --description--

U prethodnoj lekciji videli smo šta je Fetch API i kako ga koristiti. U ovoj lekciji ćemo razgovarati o HTTP metodama `GET`, `POST`, `PUT` i `DELETE` iz Fetch API-ja.

Početi ćemo sa najčešćom HTTP metodom, a to je metoda `GET`. Ona se koristi za dohvaćanje podataka sa servera. Kada koristite `fetch()` bez specifikovanja metode, podrazumevano je na `GET`.

```js
fetch('https://api.example.com/data')
```

U ovom kodu pravimo `GET` zahtev ka `https://api.example.com/data`. Sada, molim vas, imajte na umu da ne možete koristiti ove podatke direktno; morate konvertovati odgovor u JSON format. Tek tada ga možete koristiti bilo gde u vašem projektu. Evo primera kako to učiniti:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

U ovom kodu, odgovor koji dolazi iz Fetch API-ja je obećanje (promise), i koristimo `.then` rukovatelja za konverziju odgovora u JSON format. U prethodnoj lekciji ste naučili da je `Promise` objekat koji predstavlja eventualno završavanje (ili neuspeh) asinhronog procesa i njegovu vrednost.

Vrednost obećanja nije poznata kada se promise kreira. Poznata je tek kada se asinhroni proces završi. Kada lančamo dva `.then` rukovatelja na `fetch` poziv, to je nešto što se zove lančano povezivanje obećanja (promise chaining), što ćemo naučiti u sledećoj lekciji.

Do sada smo dohvatali resurse sa servera. Ali, znate li da takođe možemo poslati podatke serveru? `POST` metoda se koristi za slanje podataka na server radi kreiranja resursa. Evo primera `POST` metode koja se koristi za kreiranje resursa na serveru:

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

U ovom primeru, šaljemo `POST` zahtev radi kreiranja novog korisnika. Specifikujemo metodu kao `POST`, postavljamo odgovarajuće zaglavlja i uključujemo telo (body) sa podacima koje želimo poslati. Telo mora biti string, pa koristimo `JSON.stringify()` da konvertujemo naš objekat u JSON string.

`PUT` metoda se koristi za ažuriranje postojećih resursa servera. Evo primera:

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

U ovom primeru, obratite pažnju na URL; vidite `45` na kraju. Ovo se obično koristi kao jedinstveni ID za identifikaciju podataka koje pokušavamo da ažuriramo. Koristili smo `PUT` metodu u kodu i takođe specifikovali podatke kao `body`, koji će se koristiti za ažuriranje identifikovanih podataka.

`DELETE` metoda se koristi za brisanje resursa sa servera. Evo primera:

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE',
})
```

U ovom kodu, uključujemo `DELETE` metodu i ID na kraju URL-a kako bismo identifikovali podatke koje treba obrisati.
