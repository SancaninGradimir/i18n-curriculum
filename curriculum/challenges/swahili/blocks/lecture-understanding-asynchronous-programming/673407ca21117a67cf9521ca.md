# --description--

`Promise` je objek koji predstavlja eventualno završavanje ili neuspeh asinhronih operacija. Početno je u stanju čekanja (`pending`). Zatim može da pređe u stanje ispunjenosti (`fulfilled`) kada je operacija uspešna, ili u stanje odbijanja (`rejected`) kada operacija ne uspe. Evo primera kreiranja `Promise`-a:

```js
const aPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("Operation successful!");
  }, 1000);
});
```

U ovom primeru, kreiramo obećanje (promise) koje simulira asinhronu operaciju korišćenjem `setTimeout`. Nakon jedne sekunde, obećanje se rešava porukom „Operation successful!“ (Operacija uspešna!).

Drug način za rad sa obećanjima je korišćenje metoda `.then` i `.catch`.

Metoda `.then()` koristi se u obećanju da bi se specificiralo šta treba da se desi kada je obećanje ispunjeno, dok se `.catch()` koristi za rukovanje svim greškama koje nastanu. Pogledajmo kako možemo koristiti `.then` i `.catch` sa našim obećanjem:

```js
aPromise.then((result) => {
  console.log(result);  // Outputs: "Operation successful!"
}).catch((error) => {
  console.error(error);
});
```

U ovom kodu, funkcija prosleđena `.then()` biće pozvana sa rešćenom vrednošću obećanja. Ako dođe do greške, umesto toga će biti pozvana funkcija prosleđena `.catch()`.

Sada, hajde da pričamo o lančovanju obećanja (promise chaining). Jedna od moćnih karakteristika obećanja je što možemo povezati više asinhronih operacija zajedno. Svaki `.then()` može vratiti novo obećanje, omogućavajući vam da izvršite sekvencu asinhronih operacija jedna za drugom. Evo primera:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/data2');
  })
  .then(response => response.json())
  .then(data2 => console.log(data2))
  .catch(error => console.error('Error:', error));
```

U ovom primeru, vršimo dva API poziva u sekvenci. Prvi `.then()` parsira odgovor kao JSON. Drugi `.then()` loguje podatke i pravi još jedan API poziv. Treći `.then()` parsira odgovor drugog API poziva, a četvrti `.then()` loguje te podatke. Ako dođe do greške bilo gde u ovoj lancu, ona će biti uhvaćena od strane `.catch()` na kraju.

Važno je napomenuti da će `.catch()` uhvatiti greške iz bilo kog prethodnog koraka u lancu. To znači da ne morate dodavati rukovanje greškama svakom pojedinačnom koraku, što može značajno pojednostaviti vaš kod.
