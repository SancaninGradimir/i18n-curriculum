# --interactive--

U prethodnim lekcijama, naučili ste o asinhronom programiranju koje omogućava drugom kodu da radi dok čekamo da se završe neki vremenski zahtevni zadaci, kao što je dohvaćanje podataka sa servera, čitanje podataka iz fajla i tako dalje.

`async`/`await`, izgrađen na osnovu obećanja (promises), olakšava pisanje i čitanje asinhronog koda. Kada stavite ključnu reč `async` pre funkcije, to znači da će ta funkcija uvek vratiti `Promise`. Ključna reč `await`, koja vam omogućava da sačekate da se `Promise` reši pre nego što nastavite sa sledećim redom koda, može se koristiti samo unutar asinhronih funkcija ili u glavnim telima modula. Evo primera kako radi `async`/`await`:

:::interactive_editor

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

:::

U ovom kodu, definišemo funkciju nazvanu `async` koja se zove `delayedGreeting`. Unutar ove funkcije, koristimo `await` da pauziramo izvršenje na 2 sekunde. Nakon kašnjenja, ispisuje pozdrav.

Kada pozovemo ovu funkciju, videćete da se `First Printed Message!` pojavi pre pozdrava. To je zato što je funkcija asinhrona – ne blokira izvršavanje ostatka koda.

Jedna od najvećih prednosti `async`/`await` je rukovanje greškama. Sa obećanjima (promises), često smo morali da koristimo `.catch()` metod za rukovanje greškama. Sa `async`/`await`, možemo koristiti `try`/`catch` blokove. Evo primera:

```js
async function fetchUserData() {
  try {
    let response = await fetch(`https://api.example.com/users`);
    let userData = await response.json();
    console.log(userData);
  } catch (error) {
    console.log("Error fetching user data:", error);
  }
}

fetchUserData();
```

U ovom primeru, koristimo `async`/`await` za dohvaćanje podataka korisnika sa API-ja. Ključna reč `await` je korišćena dvaput: jednom da sačekaju završetak operacije dohvaćanja, i ponovo da sačekaju završetak parsiranja JSON-a. Ako dođe do bilo kakve greške tokom ovog procesa, ona će biti uhvaćena u bloku `catch`.
