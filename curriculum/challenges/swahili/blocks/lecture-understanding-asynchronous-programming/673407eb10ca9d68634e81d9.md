# --description--

Geolokacioni API pruža način da sajtovi traže lokaciju korisnika. Važno je napomenuti da iz razloga privatnosti, korisnik mora dati dozvolu pre nego što se njegova lokacija može pristupiti putem sajta.

Glavna metoda na koju ćemo danas da se fokusiramo je `getCurrentPosition`. Ova metoda se koristi za prikupljanje geografske lokacije uređaja. Evo primera kako biste mogli da koristite `getCurrentPosition`:

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

U ovom kodu, pozivamo `getCurrentPosition` i prosleđujemo mu funkciju koja će biti pozvana kada se pozicija uspešno dobije. Ovaj objekat pozicije sadrži razne komadićke informacije, ali fokusiramo se samo na `latitude` i `longitude`.

Ako postoji problem sa dobijanjem pozicije, onda će greška biti logovana u konzolu.

Metoda `getCurrentPosition` koristi geolokaciju putem GPS-a, Wi-Fi mreža ili IP adrese, u zavisnosti od uređaja i njegovih podešavanja. Kada se lokacija pronađe, poziva se funkcija povratne pozive za uspeh sa objektom pozicije.

Objekat pozicije sadrži različita svojstva, gde su najčešće korišćena `latitude` i `longitude`, ali može da uključuje `altitude`, `accuracy`, `speed`, `heading`, i tako dalje.

Jedan važan aspekt kada se koristi geolokacija jeste privatnost korisnika. Obavezno objasnite svojim korisnicima zašto vam je potrebni podaci o njihovoj lokaciji i kako ćete ih koristiti.
