# --description--

Model zahtev-odgovor HTTP je fundamentalni komunikacioni obrazac koji pokreće World Wide Web (WWW). Ima mnogo komponenti koje ga čine efikasnim, tako da svi možemo pristupiti materijalima sa veba.

Pogledajmo šta je model zahtev-odgovor HTTP, koje komponente ga čine i kako te komponente zajedno rade da unesu efikasnost u ono što pokreće veb.

Model zahtev-odgovor HTTP je komunikacioni obrazac u interakcijama sa vebom u kojem klijent, obično pregledač veba, šalje **HTTP zahtev** na server, a server obrađuje zahtev i vraća **HTTP odgovor**.

Evo komponenti koje čine model zahtev-odgovor HTTP:

* **Klijent**: Pregledač, mobilna aplikacija ili bilo koji drugi softver koji šalje HTTP zahteve na server i tumači HTTP odgovore koje prima nazad.
    
* **Zahtev (Request)**: Poruka koju klijent šalje serveru. Uključuje metodu, URL, zaglavlja i opcionalno telo (body).
    
* **HTTP metoda**: Akcija koja se izvodi — `GET` za preuzimanje podataka, `POST` za slanje podataka, `PATCH` i `PUT` za modifikovanje podataka, i `DELETE` za uklanjanje podataka.
    
* **URL (Uniform Resource Locator)**: Resurs koji se traži sa servera.
    
* **Zaglavlja (Headers)**: Metadata poslata uz zahtev ili odgovor. Na primer, `Content-Type` i `Authorization`.
    
* **Telo (Body)**: Podaci poslati sa zahtevima `POST`, `PUT`, `PATCH`, `DELETE`, često u JSON ili formatu podataka obrasca (form data).
    
* **Server**: Back-end sistem koji obrađuje zahtev i generiše odgovor. Može biti napisan na bilo kom programskom jeziku koji podržava server-side programiranje.
    
* **Odgovor (Response)**: Poruka koju server vraća. Sadrži kod stanja, zaglavlja i opcionalno telo.
    
* **Kod stanja ili Kod odgovora**: Numerički kod koji ukazuje na uspeh, greške ili preusmeravanja.
    
* **Telo odgovora (Response Body)**: Stvarni podaci poslati nazad, obično u HTML, JSON ili XML formatu.

Pa kako sve ove komponente rade zajedno u ciklusu zahtev-odgovor HTTP?

1. **Klijent inicira zahtev**
    
    * Pregledač, mobilna aplikacija ili API klijent šalje HTTP zahtev namenjen serveru.
        
    * Zahtev uključuje metodu kao što su `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, URL, zaglavlja i ponekad telo (body).
        
2. **Zahtev se šalje na server**
    
    * Zahtev putuje preko interneta do servera, back-end sistema napisanog programskim jezikom.
        
    * Server prima zahtev i izvlači relevante informacije iz njega.
        
3. **Server obrađuje zahtev**
    
    * Server proverava metodu zahteva i URL.
        
    * Možda je potrebno validirati autentifikaciju (preko zaglavlja kao što je `authorization`).
        
    * Ako je potrebno, on komunicira sa bazom podataka kako bi preuzeo ili modifikovao podatke.
        
4. **Server priprema odgovor**
