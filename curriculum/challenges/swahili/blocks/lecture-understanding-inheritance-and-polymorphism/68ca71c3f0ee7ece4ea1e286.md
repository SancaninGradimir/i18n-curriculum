# --description--

U prethodnoj lekciji ste naučili o prefiksiranju atributa sa jednim i sa dvostrukim donjom crtom.

Da biste se podsetili na razliku između njih, jedna donja crta je konvencija koja znači da je atribut namenjen za interno korišćenje u klasi i ne bi trebalo direktno pristupiti izvan klase. Dvostruka donja crta, s druge strane, sprečava da se tom atributu pristupi direktno izvan klase.

Evo primera koji demonstrira kako oba funkcionišu:

```py
class Example:
    def __init__(self):
        self._internal = 'I can be accessed from outside the class, but should not'
        self.__private = 'You cannot access me directly from outside the class'

obj = Example()

print(obj._internal) # I can be accessed from outside the class, but should not
print(obj.__private)  # AttributeError: 'Example' object has no attribute '__private'
```

Prefiksiranje atributa dvostrukom donjom crtom pokreće proces "name mangling"-a u Pythonu, tokom kojeg Python interno preimenovava atribut dodavanjem donje crte i imena klase kao prefiksa, pretvarajući `__atribut` u `_ImeKlase__atribut`.

Da biste ovo videli u akciji, kreirate instancu klase i koristite specijalni atribut `__dict__` te instance, koji je rečnik koji sadrži atribute objekta:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)

print(example1.__dict__)
```

Rezultat bi bio:

```py
{
  '_internal': 'I can be accessed from outside the class, but should not',
  '_Example__private': 'I cannot be accessed directly from outside the class'
}
```

Kao što vidite, atribut `__private` je sačuvan kao `_Example__private`. To znači da i dalje možete pristupiti tom atributu izvan klase na ovaj način:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)
example2 = Example(
    'I should not be accessed from outside the class',
    'But I can be accessed from outside the class with name mangling'
)

print(example1._Example__private) # I cannot be accessed directly from outside the class
print(example2._Example__private) # But I can be accessed from outside the class with name mangling
```

Pa, zašto Python radi "name mangling"?

Glavna svrha "name mangling"-a je sprečavanje slučajnog prepisivanja atributa i metoda kada koristite nasleđivanje. Evo primera koji to čini jasnim:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

Možete videti da i klasa `Parent` i `Child` koja nasleđuje od nje imaju svoje zasebne atribute `_class__data`. Ovo je moguće zahvaljujući "name mangling"-u. Inače, `Child` bi slučajno prepisao podatke Parent-a.

Evo šta bi se dogodilo bez dozvoljavanja Pythonu da uradi "name mangling", tj. ako ne prefiksirate atribute u obe klase dvostrukom donjom crtom:

```py
class Parent:
   def __init__(self):
       self.data = 'Parent data'

class Child(Parent):
   def __init__(self):
       super().__init__()
       self.data = 'Child data'

c = Child()
print(c.__dict__)  # {'data': 'Child data'}
```

Dakle, koji biste trebali koristiti za prefiksiranje atributa između jedne donje crte (`_`) i dvostruke donje crte (`__`)? Zavisi. Ako je atribut namenjen samo za interno korišćenje unutar klase, držite se jedne donje crte.

Ali ako radite sa klasom koja će biti nasleđena, trebalo bi da koristite dvostruku donju crtu kako se atribut iz roditeljske klase ne bi prepisao.
