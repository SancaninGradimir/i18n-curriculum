# --description--

Polimorfizam je sledeći ključni koncept objektno orijentisanog programiranja (OOP) o kojem ćemo govoriti.

Polimorfizmom, imate pristup interfejsu gde možete interagovati sa mnogim objektima iste vrste.

Pogledajmo bliže na polimorfizam i kako vam omogućava ponovno korišćenje koda.

Polimorfizam omogućava metodama u različitim klasama da dele isto ime, ali obavljaju različite zadatke. Pozivate isti naziv metode na različitim objektima, a svaki reaguje na svoj način.

Evo osnovnog primera polimorfizma:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

Evo primera korišćenjem različitih životinjskih zvukova za prikaz polimorfizma:

```py
class Cat:
   def speak(self):
       return "A cat meow"

class Bird:
   def speak(self):
       return "A bird tweet"
  
class Monkey:
   def speak(self):
       return "A monkey ooh ooh aah aah ooh ooh aah aah"

def animal_sound(animal):
   print(animal.speak())

animal_sound(Cat())
animal_sound(Bird())
animal_sound(Monkey())
```

U ovom primeru, `animal_sound()` je funkcija koja prima bilo koji objekat sa metodom `speak()`.

Kada prosledite `Cat`, `Bird` ili `Monkey`, on poziva metodu `speak()` objekta i ispisuje rezultat. Pošto svaka klasa definiše `speak()` drugačije, dobijate različite izlaze iz iste funkcije. To je polimorfizam u delovanju.

Evo još jednog primera, ovog puta sa instancama i atributom:

```py
class Twitter:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"🐦 Tweet: '{self.content}' (280 chars max)"

class Instagram:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"📸 Instagram Post: '{self.content}' + ✨ filters"

class LinkedIn:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"💼 LinkedIn Article: '{self.content}' (Professional Mode)"

def start(social_media):
   print(social_media.post())  # Calls .post() on any object

# Instances
tweet = Twitter('Just learned Python polymorphism!')
photo = Instagram('Sunset vibes 🌅')
article = LinkedIn('Why OOP matters in 2024')

# The polymorphic calls - same function, different outputs
start(tweet) # 🐦 Tweet: 'Just learned Python polymorphism!' (280 chars max)
start(photo) # 📸 Instagram Post: 'Sunset vibes 🌅' + ✨ filters
start(article) # 💼 LinkedIn Article: 'Why OOP matters in 2024' (Professional Mode)
```

Postoji takođe vrsta polimorfizma nazvana **polimorfizam zasnovan na nasleđivanju**.

U polimorfizmu zasnovanom na nasleđivanju, roditeljska klasa definiše metodu, a više potomaka klasa prepisuje tu metodu na svoj način. Zatim možete pozvati istu metodu na bilo kom objektu potomka, i ona se ponaša drugačije zavisno od toga koji je to klasa potomaka.

Evo primera:

```py
class Animal:
   def speak(self):
       return 'Some generic sound'

class Cat(Animal):
   def speak(self):
       return 'A cat meow'

class Dog(Animal):
   def speak(self):
       return 'A dog barks woof woof'

class Monkey(Animal):
   def speak(self):
       return 'A monkey ooh ooh aah aah ooh ooh aah aah'
  
print(Cat().speak()) # A cat meow
print(Dog().speak()) # A dog barks woof woof
print(Monkey().speak()) # A monkey ooh ooh aah aah ooh ooh aah aah
print(Animal().speak()) # Some generic sound
```

Možete videti da svaka podklasa roditeljske ``Animal`` klase nadjačava metodu ``speak()`` kako bi pružila svoju implementaciju. Dakle, kada pozovete metodu ``speak()`` na instanci svake podklase, ona vraća specifičan zvuk povezan sa tim životinjom.

Takođe možete i ići dalje i izvršiti pozivanje u listi, a zatim proći kroz listu kako biste prikazali šta metoda ``speak()`` vraća za svaki:

```py
animals = [Cat(), Dog(), Monkey()]

for animal in animals:
   print(animal.speak())

# Output:
# A cat meow
# A dog barks woof woof
# A monkey ooh ooh aah aah ooh ooh aah aah
```
