# --description--

U ranijim lekcijama, naučili ste da je React aplikacija jedne stranice. Aplikacije jedne stranice su aplikacije koje sadrže jedan HTML fajl i koriste JavaScript za dinamičko ažuriranje bilo kog sadržaja na stranici.

Šta se dešava kada morate dodati više "stranica" u svoju React aplikaciju? Kako biste pristupili navigaciji ka tim različitim prikazima?

Pa, tu je mesto gde ulazi React Router.

React Router je biblioteka treće strane koja vam omogućava dodavanje rutiranja vašim React aplikacijama. Da biste počeli, moraćete da instalirate React Router u postojeći React projekat na ovaj način:

```sh
npm i react-router
```

Ako proverite fajl `package.json`, videćete da je `react-router` dodat u listu zavisnosti:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1",
  "react-router": "^7.2.0"
}
```

Zatim unutar vašeg `main.jsx` ili `index.jsx` fajla, moraćete da uvezete `BrowserRouter` i prikazati `BrowserRouter` oko vaše `App` komponente:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>
);
```

Da biste omogućili rute u vašoj aplikaciji, moraćete da ažurirate svoj izraz `import` kako biste uključili komponente `Routes` i `Route` na ovaj način:

```js
import { BrowserRouter, Routes, Route } from "react-router";
```

Zatim, unutar `BrowserRouter`, dodajte komponente `Routes` i `Route`:

```jsx
createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

<code>`path`</code> i <code>`element`</code> se koriste za povezivanje URL i UI komponenti zajedno. U ovom slučaju, postavljamo rutu za početnu stranicu koja ukazuje na <code>`App`</code> komponentu.

Uobičajeno je u većim aplikacijama da se ima više prikaza i ruta postavljeno ovako:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

Prop `index` u ovim primerima namenjen je da predstavlja podrazumevan put za dati segment puta. Dakle, komponenta `Home` će biti prikazana na korenom putu (`/`) dok će komponenta `ProductsHome` biti prikazana na putanji `products`.

Možda ste primetili i da gnježimo nekoliko ruta unutar druge rute na ovaj način:

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

Ovo znači da će putanja det route-a biti spajana sa putanjom roditeljskog route-a. Dakle, u ovom primeru, ``path`` za trendi proizvode biće ``products/trending``.

Ako `path` počne sa dvotačkom (`:`) onda to predstavlja dinamički segment u ruti:

```jsx
<Route path=":category" element={<Category />} />
```

U ovom primeru imamo dinamički segment nazvan `category`. Kada korisnik navigira ka URL-u poput `products/brass-instruments`, onda će prikaz promeniti na komponentu `Category` i možete dinamički dohvatiti odgovarajuće podatke na osnovu segmenta.

Možete pristupiti vrednosti dinamičnog segmenta korišćenjem kuka ``useParams`` unutar komponente za dete, na ovaj način:

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
   {/* Accessing the category param: params.category */}
   {/* rest of code goes here */}
}
```

Dinamičke rute su korisne jer vam omogućavaju da kreirate fleksibilne i ponovno upotrebljive komponente koje mogu prikazati različit sadržaj na osnovu parametara u URL-u. Umesto definisanja fiksne liste putanja za svaku moguću rutu, možete koristiti dinamičke segmente za prikaz raznolikog sadržaja na osnovu onoga što je korisnik zatražio.
