# --description--

npm treba način za skladištenje važnih detalja o vašem projektu – stvari kao što su paketi koje koristi, verzija tih paketa i komande koje može pokrenuti. To radi sa fajlom `package.json`.

Razmišljajte o fajlu ``package.json`` kao o npm konfiguracionom fajlu za projekat. To je jednostavan JSON fajl koji opisuje vaš projekat koristeći parove ključ-vrednost kako bi pratio stvari kao što su:

* ime i verzija projekta
    
* autor
    
* glavni fajl koji treba pokrenuti prvi
    
* licenca
    
* repozitorijum koda, ključne reči i opisi
    
* bilo koje zavisnosti kojima je projekat potreban

Evo kako izgleda datoteka `package.json`:

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "description": "A simple NPM project",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "test": "echo \"Running tests...\""
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/username/project.git"
  },
  "keywords": ["fCC", "js"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "react": "^19.2.1"
  },
  "devDependencies": {
    "cypress": "^15.7.1"
  }
}
```

Polje `main` se često postavlja na `index.js`, i govori Node.js-u koji fajl da pokrene prvi.

Polje `license` se koristi za određivanje pravnih uslova na kojima drugi ljudi mogu da koriste, modifikuju i distribuiraju kod u vašem paketu. Postoji mnogo različitih licenci kao što su `MIT`, `ISC` ili `GPL`, koje sve imaju različite uslove i dozvole. Podrazumevano je `ISC`, Licenca Internetskog sistema konsortijuma, koja je veoma permisivna open source licenca.

Kako vaš projekat raste i instalirate pakete, npm ih dodaje pod jednu od dve sekcije:

* `dependencies`, koji su paketi kojima je vaša aplikacija potreban za rad u produkciji
    
* `devDependencies`, koji su paketi koji vam trebaju samo tokom razvoja

Na primer, ako vaš projekat koristi React za svoj UI, to ide u `dependencies`, jer je potrebno da vaš projekat radi nakon implementacije. Alat poput Cypressa, koji se koristi samo za end-to-end testiranje pre implementacije u produkciju, pripada `devDependencies`.

Postoje nekoliko jednostavnih načina za kreiranje datoteke `package.json`. Najčešći način je pokretanje sledeće komande:

```bash
npm init
```

Trebalo bi da ovo pokrenete u korenom folderu vašeg projekta. Ova komanda započinje interaktivni proces podešavanja gde vas npm pita niz pitanja. Vaši odgovori se koriste za popunjavanje polja kao što su naziv projekta, verzija, opis, skripte, ključne reči, autor i licenca.

Ako želite da preskočite pitanja i kreirate fajl ``package.json`` sa podrazumevanim vrednostima, možete pokrenuti:

```bash
npm init -y
```

ili

```bash
npm init --yes
```

Ovo trenutno kreira fajl `package.json` za vas sa podrazumevanim vrednostima u mnogim poljima.

Obično je najbolje koristiti `npm init` bez zastavice `-y` ili `--yes` kako biste prošli kroz svako polje i prilagodili informacije vašeg projekta. Ako ne odgovorite na pitanje, npm jednostavno koristi podrazumevanu vrednost. Na primer, ako ostavite pitanje o licenci prazno, npm će ga po podrazumevanom postaviti na `ISC`.

Možda primetite da `dependencies` i `devDependencies` još nisu navedeni. To je zato što se ovi odeljci dodaju tek nakon instalacije vašeg prvog paketa.

Takođe ćete primetiti da je `repository` polje nedostajuće. Iako je npm pitao o tome tokom `npm init`, ne popunjava ga automatski kasnije. Na primer, kada pokrenete komande kao što su `git remote add origin` ili `git push`, npm neće ažurirati polje repozitorijuma za vas — morate ga ručno dodati ako želite da bude uključen.
