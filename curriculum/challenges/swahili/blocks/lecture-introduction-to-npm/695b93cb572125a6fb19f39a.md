# --description--

U ranijoj lekciji ste naučili šta su zavisnosti (dependencies). Ovde ćete saznati više o njima i o stvarima koje treba uzeti u obzir prilikom izbora eksternog paketa za vaš projekat.

Zavisnost je još jedan deo koda koji vaš projekat treba da bi radio. To su obično eksterni paketi koje su drugi programeri kreirali i podelili u npm registru.

Korišćenje zavisnosti štedi vam mnogo vremena. Umesto da sve sami gradite, možete instalirati paket koji već rešava zajednički problem – stvari poput kreiranja veb servera, rukovanja rutiranjem (routing), preuzimanja podataka, upravljanja autentifikacijom i još mnogo toga.

U vašoj datoteci `package.json`, svi paketi na koje se vaš projekat oslanja su navedeni pod ključem `dependencies`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  }
}
```

U gornjoj primeru datoteci `package.json`, `chalk` i `lodash` su navedeni kao zavisnosti projekta. To znači da se projekat oslanja na ove pakete da bi radio. Ako nisu instalirani – i navedeni u datoteci `package.json` – sve funkcije koje ih koriste će prestati sa radom (će puknuti).

Postoji i druga kategorija zavisnosti nazvana `devDependencies`. To su paketi koji vam trebaju samo dok gradite ili testirate svoj projekat, a ne kada radi u produkciji. Uobičajeni primeri uključuju:

* Nodemon za automatsko ponovno pokretanje projekta prilikom ažuriranja
    
* Jest ili Mocha za testiranje
    
* ESLint za lintovanje koda
    

Kao što očekujete, `devDependencies` se pojavljuju pod sekcijom `devDependencies` u datoteci `package.json`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^30.2.0",
    "nodemon": "^3.1.10"
  }
}
```

Kada birate koje pakete koristiti u vašem projektu, postoji nekoliko važnih stvari o kojima treba pamtiti.

Bezbednost bi trebalo uvek da je na prvom mestu. Čak i jedan nebezbeban paket može stvoriti probleme za ceo vaš projekat. Takođe biste trebali uzeti u obzir performanse, jer veliki ili loše optimizovan paket može usporiti vaš projekat.

Takođe pomaže birati pakete koji su popularni i dobro podržani. Popularni paketi obično imaju aktivne zajednice, dobru dokumentaciju i česte ažuriranja. Jednostavan način da proverite popularnost je da pogledate broj nedeljnih preuzimanja na npm-u. Na primer, paket `chalk` prima preko 300.000 preuzimanja svake nedelje:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-package-dependencies-and-what-are-some-considerations-for-choosing-an-external-package-1.webp" alt="screenshot from npm of chalk package statistics">

Održavanje je takođe važno. Dobro održavani paketi se redovno ažuriraju i bolje rade sa novim verzijama drugog softvera.

Konačno, ne zaboravite na dokumentaciju. Jasna, sveobuhvatna dokumentacija čini mnogo lakšim razumevanje kako koristiti paket i štedi vam mnogo vremena prilikom integrisanja u vaš projekat.
