# --interactive--

Rekurzija je složena karakteristika koja vam omogućava da pozovete funkciju ponovljeno sve dok se ne stigne do baznog slučaja. Za razliku od tradicionalne petlje, rekurzija vam omogućava da rukujete nečim sa nepoznatom dubinom, kao što su duboko ugnježdeni objekti/nizovi ili stablo fajlova. Ali možete je koristiti i za jednostavnije zadatke, kao što je brojanje nizanjem od određenog broja.

Kreiraćemo funkciju koja će tačno to uraditi. Nazvaćemo našu funkciju `recursiveCountdown`, i ona mora da primi broj. Nateraćemo je da odštampa ovaj broj na konzolu:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(number);
};

recursiveCountdown(5);
```

:::

Sada ako ovo pozovemo i prosledimo broj 5, videćemo da se broj ispiše na našem terminalu. Ali ništa drugo ne dešava – a broj 5 definitivno nije odbrojavanje.

Pre nego što počnemo da gradimo rekuzivni deo naše funkcije, moramo prvo uspostaviti bazni slučaj. Ako nemate uspostavljen bazni slučaj, vaš kod će se izvršiti sve dok ne prekorači dodelu memorije i sruši.

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
};

recursiveCountdown(5);
```

:::

Za naš bazni slučaj, želimo da prebrojavanje prestane ako je broj manji od 1. Kada dođemo do tog baznog slučaja, možemo vratiti (return) da bismo izašli iz izvršavanja funkcije.

Sada što smo sigurno pripremili osnovni slučaj, možemo postaviti rekurziju. Ključna tačka koja čini funkciju rekurzivnom jeste to što se ona poziva sama u svom izvršavanju. U ovom slučaju, želimo da pozovemo funkciju nakon što ispišemo broj. Ali kako bismo brojali nadole, naš novi broj mora biti jedan manji:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
  recursiveCountdown(number - 1);
};

recursiveCountdown(5); 
```

:::

Ovo bi logovalo brojeve 5, 4, 3, 2 i 1 na konzolu.

Dobijamo naše pet brojeva! Ali šta ako bismo hteli da brojimo rastući umesto toga? Umesto pisanja potpuno nove funkcije, možemo da zamenimo redosled našeg log-a i našeg rekurzivnog poziva:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
    if (number < 1) {
        return;
    }
    recursiveCountdown(number - 1);
    console.log(number);
  };

recursiveCountdown(5);
```

:::

Ovo bi ispisalo brojeve 1, 2, 3, 4 i 5 na konzolu.

Ali zašto to radi? Pa, da biste razumeli ovo morate da razumete pilu poziva. Pila poziva je način na koji JavaScript prati i rešava pozive funkcija. Stak funkcioniše kao red nekoga рода tipa "poslednji u, prvi iz". Da biste ovo bolje razumeli, dodajmo neku logovanje našoj funkciji:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(`Function execution started for number: ${number}`);
  if (number < 1) {
    console.log(`Base case reached, begin resolving stack`);
    return;
  }
  console.log(`Calling recursiveCountdown with number: ${number - 1}`);
  recursiveCountdown(number - 1);
  console.log(`Function execution completed for number: ${number}`);
};

recursiveCountdown(5);
```

:::

Dodali smo četiri ključne izjave ovde. Prvi log se izvršava kada počinje izvršavanje poziva funkcije. Treći log se izvršava neposredno pre nego što se pozove rekurzivna funkcija. A četvrti log radi kada je izvršavanje funkcije završeno. Rezultat je:

```md
Function execution started for number: 5
Calling recursiveCountdown with number: 4
Function execution started for number: 4
Calling recursiveCountdown with number: 3
Function execution started for number: 3
Calling recursiveCountdown with number: 2
Function execution started for number: 2
Calling recursiveCountdown with number: 1
Function execution started for number: 1
Calling recursiveCountdown with number: 0
Function execution started for number: 0
Base case reached, begin resolving stack
Function execution completed for number: 1
Function execution completed for number: 2
Function execution completed for number: 3
Function execution completed for number: 4
Function execution completed for number: 5
```

Ali kako se ovo dešava? Pa, ovde ulazi u igru stek poziva. Kada pozovemo ``recursiveCountdown(5)``, taj poziv funkcije dodaje se u stek poziva.

Kada taj poziv funkcije stigne do tačke gde mora da pozove `recursiveCountdown(4)`, on mora da se zaustavi i sačeka taj rezultat. U međuvremenu, naš `recursiveCountdown(4)` se dodaje na stogu poziva (call stack), preko `recursiveCountdown(5)`.

Kada taj poziv funkcije stigne do tačke gde mora da pozove `recursiveCountdown(3)`, on mora da se zaustavi i sačeka taj rezultat. U međuvremenu, naš `recursiveCountdown(3)` se dodaje na stogu poziva (call stack), preko `recursiveCountdown(4)`.

Kada taj poziv funkcije stigne do tačke gde mora da pozove `recursiveCountdown(2)`, on mora da se zaustavi i sačeka taj rezultat. U međuvremenu, naš `recursiveCountdown(2)` se dodaje na stogu poziva (call stack), preko `recursiveCountdown(3)`.

Kada taj poziv funkcije stigne do tačke gde mora da pozove `recursiveCountdown(1)`, on mora da se zaustavi i sačeka taj rezultat. U međuvremenu, naš `recursiveCountdown(1)` se dodaje na stogu poziva (call stack), preko `recursiveCountdown(2)`.

I na kraju, kada taj poziv funkcije stigne do tačke gde mora da pozove `recursiveCountdown(0)`, mora da se zaustavi i sačeka taj rezultat. U međuvremenu, naš `recursiveCountdown(0)` dodaje se na stogu poziva (call stack), preko `recursiveCountdown(1)`.

Ali ``recursiveCountdown(0)`` ne poziva drugu funkciju – on stiže do našeg baznog slučaja, gde se rano vraća. Pošto je izvršavanje te funkcije završeno, poziv funkcije može se smatrati „rešenim”. Kada je poziv rešen, uklanja se sa steka.

Sada naš `recursiveCountdown(1)` više ne čeka taj poziv – nalazi se na vrhu steka i može nastaviti izvršavanje.

`recursiveCountdown(1)` rešava, uklanja se sa steka, i omogućava da `recursiveCountdown(2)` nastavi izvršavanje.

`recursiveCountdown(2)` rešava, uklanja se sa steka, i omogućava da `recursiveCountdown(3)` nastavi izvršavanje.

`recursiveCountdown(3)` rešava, uklanja se sa steka, i omogućava da `recursiveCountdown(4)` nastavi izvršavanje.

`recursiveCountdown(4)` rešava, uklanja se sa steka, i omogućava da `recursiveCountdown(5)` nastavi izvršavanje.

I `recursiveCountdown(5)` rešava se i uklanja sa steka. Naš stek poziva je sada prazan, pa je rekurzija završena!

Ovo je osnovni pregled kako rekurzija radi u JavaScriptu. To je komplikovan koncept, i trebalo bi da eksperimentiš sa nekim kodom i `log` izjavama dok ne budeš udoban/udobna sa ponašanjem steka poziva (call stack).

Za mali zanimljivost, razgovarali smo o tome kako nedostatak osnovnog slučaja može uzrokovati da vaš kod padne kada mu zmanji memorije. To je zato što rekurzija stalno dodaje sve više i više poziva funkcija u stak složak dok ne dođe do preljeva steka (stack overflow). Upravo kao naziv popularne programerske zajednice.
