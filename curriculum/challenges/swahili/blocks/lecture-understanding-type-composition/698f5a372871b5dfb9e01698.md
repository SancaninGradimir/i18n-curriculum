# --description--

U prethodnim lekcijama, naučili ste kako da radite sa primitivnim tipovima kao što su `string`, i `boolean`. Ali TypeScript vam omogućava da kreirate nove tipove iz postojećih. Jedan primer ovoga je tip unije (union type).

Tip unije je tip koji kombinuje dva ili više drugih tipova.

Evo osnovne sintakse:

```ts
type1 | type2 | type3
```

Pogledajmo neke primere kako bismo bolje razumeli koncept/pojam.

Evo varijable nazvane `id` sa unijet tipom za `string` ili `number`:

```ts
let id: string | number = "user_783";
```

Vrednost se može ažurirati na broj koji je i dalje validan za tip unije:

```ts
let id: string | number = 1024;
```

Ako pokušate da dodelite drugu vrednost koja nije `string` ili `number`, onda će TypeScript prikazati poruku o grešci:

```ts
// Type 'string[]' is not assignable to type 'string | number'.
let id: string | number = ["Java", "Python"];
```

Kada radite sa tipovima unije, važno je koristiti metode koje rade za sve tipove, inače će TypeScript baciti grešku kao ovo:

```ts
function getId(id: string | number) {
  // Property 'toUpperCase' does not exist on type 'string | number'.
  return id.toUpperCase();
}
```

Postoji rešenje za ovo koje uključuje sužavanje unije. Ali naučićete više o tome, kao i kako tipovi unije rade sa aliasima tipova u budućim lekcijama.

Tipovi uniona su takođe korisni kada radite sa vrednostima `null` i `undefined`.

Evo primera gde promenljiva može da sadrži vrednost `string`, ili može biti `null` ako još uvek nije postavljena nikakva vrednost:

```ts
let username: string | null;

// Type 'number' is not assignable to type 'string | null'.
username = 42;
```

U ovom slučaju, `username` može biti ili `string` ili `null`, ali ništa drugo. Možete koristiti `null` kada želite namerno predstaviti odsustvo vrednosti.

Evo još jednog primera korišćenjem `undefined`:

```ts
let score: number | undefined;

score = undefined; 

// Type 'string' is not assignable to type 'number | undefined'.
score = "high";
```

U ovom primeru, `score` može biti ili `number` ili `undefined`. Možete koristiti `undefined` kada vrednost još nije dodeljena.

Korišćenje tipova unije sa `null` i `undefined` čini jasno kada vrednost može biti nedostupna, a istovremeno omogućava TypeScript-u da uhvati nevažeće asignacije.
