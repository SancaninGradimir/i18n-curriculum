# --description--

U prethodnim lekcijama, naučili ste kako da radite sa primitivnim tipovima, kao i sa unijun tipovima i interfejsima.

Ali TypeScript ima neke posebne tipove o kojima biste trebali znati. Počnimo sa pregledom tipa `any`.

Tip `any` se koristi za predstavljanje bilo kog tipa vrednosti.

```ts
let randomValue: any;

randomValue = 42;           

randomValue = "Hello";     
 
randomValue = true;        

randomValue = { name: "Alice" }; 
```

Sa ovim tipom, morate biti oprezni jer ga je lako previše koristiti i pogrešno koristiti. Ne želite da sve tipizujete sa tipom `any` samo da ućutite poruke o greškama TypeScript-a. To poništava svrhu korišćenja TypeScript-a i dodavanja sigurnosti tipova uopšte.

Bezbedniji ekvivalent tipu `any` bio bi tip `unknown`. `unknown` je sličan `any`, ali sa tipom `unknown`, morate izvršiti proveru tipa za varijablu pre nego što je koristite.

Evo primera:

```ts
function doubleValue(value: unknown) {
  if (typeof value === "number") {
    console.log(value * 2);
  } else if (typeof value === "string") {
    console.log(value + value);
  }
}

doubleValue(10);    
doubleValue("Hi "); 
doubleValue(true);   
```

U ovom primeru, provere tipa osiguravaju da se operacije izvode samo na vrednostima ispravnog tipa. Vrednosti koje ne odgovaraju očekivanim tipovima ignorišu se.

Sledeći tip na koji ćemo pogledati je tip `never`. Ovaj tip predstavlja nešto što nikada neće uskočiti.

U većini situacija, nećete pisati tipne anotacije koristeći `never`. Umesto toga, najverovatnije ćete videti tip `never` u porukama o greškama kao što je ovo:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}
```

Unutar `else` klauzule gornjeg primera, promenljiva `value` ima tip `never`. To je zato što grana `else` sada postaje nemoguća za dostići i nema preostalog tipa. Ako pokušate da pozovete funkciju sa vrednošću koja nije `string` ili `number`, prikazaćće se greška kao ova:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}

// Argument of type 'boolean' is not assignable to parameter of type 'string | number'.
processValue(true); 
```

Poslednji tip na koji ćemo pogledati je tip `void`. Kada imate funkciju koja ne vraća vrednost, to bi bio tip `void`.

Evo primera:

```ts
type Status = "loading" | "success" | "error";

type Handler = {
  status: Status;
  onChange: (newStatus: Status) => void; 
};
```

U ovom primeru, svojstvo `onChange` za tip `Handler` ništa ne vraća, zbog čega se koristi `void`.
