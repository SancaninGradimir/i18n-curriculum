# --description--

U prethodnoj lekciji, naučili ste kako da radite sa tipovima aliasima i videli ste primere kako da imenujete tipove objekata.

Kao što se sećate, jedan način za imenovanje tipa objekta bio bi korišćenje ključne reči `type` na ovaj način:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};
```

Drugi način za imenovanje tipa objekta bilo bi korišćenje deklaracije `interface` na ovaj način:

```ts
type ID = number | string;

interface User {
  id: ID;
  name: string;
}
```

`type` i `interface` su slični, ali ključna razlika je u tome što interfejsi koriste ključnu reč `extends` da bi se oslanjali na druge interfejse ili tipove, dok aliasi `type` koriste intersektivne tipove (`&`) za kombinovanje tipova.

Pogledajmo neke primere:

```ts
interface User {
  id: number | string;
  name: string;
}

interface Admin extends User {
  role: string;
}

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

U ovom primeru, `User` počinje sa `id` i `name`. Interfejs `Admin` proširuje `User`, što znači da uključuje `id` i `name` i dodaje novo svojstvo nazvano `role`.

Ako želite da proširite ``type``, koristili biste tip preseka (``&``) kao što je ovo:

```ts
type User = {
  id: number | string;
  name: string;
};

type Admin = User & {
  role: string;
};

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

`Admin` kombinuje tip `User` sa novim tipom objekta koji uključuje `role`. Rezultat je isti oblik kao interfejs primer, što je objekat sa `id`, `name` i `role`.

Oba `type` i `interface` mogu opisati oblik objekta. Ako niste sigurni koji koristiti, zapamtite da `interface` je specifično dizajniran za objekte oblika, dok je `type` fleksibilniji i može predstavljati tipove unija, primitive i mnogo kompleksnije kombinacije tipova.
