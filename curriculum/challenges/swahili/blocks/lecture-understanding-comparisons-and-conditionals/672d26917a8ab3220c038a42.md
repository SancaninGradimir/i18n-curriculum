# --interactive--

U JavaScriptu, `null` i `undefined` su dva različita tipa podataka koji predstavljaju odsustvo vrednosti, ali se drugačije ponašaju u poređenjima. Razumevanje kako ovi tipovi interaguju u različitim scenarijima poređenja ključno je za pisanje robusnog i bez grešaka koda.

Počinjemo sa `undefined` tipom. Varijabla je `undefined` kada je deklarisana, ali nije dobila vrednost. To je podrazumevana vrednost nedefinisovanih varijabli i parametara funkcija kojima nije dato argument.

Tip ``null``, s druge strane, je vrednost dodeljivanja koja predstavlja namernu ne-vrednost. Često se koristi za ukazivanje na to da varijabla namerno nema vrednost.

Prilikom poređenja `null` i `undefined` koristeći operator jednakosti (`==`), JavaScript izvršava tip konverziju. To znači da pokušava da konvertuje operande u isti tip pre nego što izvrši poređenje. U ovom slučaju, `null` i `undefined` smatraju se jednakima:

:::interactive_editor

```js
console.log(null == undefined); // true
```

:::

Međutim, kada se koristi operater stroge jednakosti (`===`), koji proverava i vrednost i tip bez obavljanja konverzije tipa, `null` i `undefined` nisu jednaki:

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

Ova razlika je važna da se zapamti prilikom pisanja uslovnih izjava ili obavljanja provera jednakosti u vašem kodu. Kada poredite `null` ili `undefined` sa drugim vrednostima koristeći operator jednakosti (`==`), ponašanje može biti neočekivano. Na primer:

:::interactive_editor

```js
console.log(null == 0);  // false
console.log(null == ''); // false
console.log(undefined == 0); // false
console.log(undefined == ''); // false
```

:::

Ovi poređivanja vraćaju `false` jer su `null` i `undefined` jednaki samo jedni drugima (i sebi) kada se koristi operator jednakosti. Ponašanje `null` u drugim poređivanjima je posebno komplikovano:

:::interactive_editor

```js
console.log(null > 0);  // false
console.log(null == 0); // false
console.log(null >= 0); // true
```

:::

`undefined`, s druge strane, uvek se konvertuje u `NaN` u numeričkim kontekstima, što uzrokuje da sva numerička poređivanja sa `undefined` vraćaju `false`:

:::interactive_editor

```js
console.log(undefined > 0);  // false
console.log(undefined < 0);  // false
console.log(undefined == 0); // false
```

:::

S obziма na ove nijanse, generalno se preporučuje korišćenje operatera stroge jednakosti prilikom poređenja vrednosti, posebno kada radite sa `null` i `undefined`. Ovaj pristup pomaže u izbegavanju neočekivane konverzije tipa (type coercion) i čini ponašanje vašeg koda predvidljivijim.

Zaključno, dok su `null` i `undefined` oba korišćena za predstavljanje odsustva vrednosti, oni se različito ponašaju u poređenjima. Razumevanje ovih razlika ključno je za pisanje čistog i bezgrešnog JavaScript koda.
