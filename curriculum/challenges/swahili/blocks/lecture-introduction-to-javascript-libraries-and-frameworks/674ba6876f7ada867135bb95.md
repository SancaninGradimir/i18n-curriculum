# --description--

U ranijim lekcijama naučili ste kako da radite sa uvozima i izvozima (imports and exports) u JavaScriptu. U ovoj lekciji, osvrnimoćemo se na to kako uvoziti i izvoziti komponente u React-u.

Izvoz (export) čini komponentu dostupnom drugim fajlovima. Uvoz (import) omogućava fajlu da koristi komponentu koja je negde drugde izveze. U React projektima, na ovaj način se komponente ponovo koriste i kombinuju za izgradnju korisničkih interfejsa.

U ovoj lekciji ćete naučiti kako da izvezete React komponente korišćenjem podrazumevanih izvoza (default exports) i imenovanih izvoza (named exports), kao i kako da ih uvezete tamo gde su potrebne.

U ovom primeru, imamo `Cat` komponentu koja pripada fajlu nazvanom `Cat.jsx`. Za ekstenziju fajla koristimo `.jsx` jer ovaj fajl radi uglavnom sa JSX-om.

Ova `Cat` komponenta vraća JSX oznaku (markup) sa naslovom i slikom mačke po imenu Mr Whiskers:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

Ako želimo da koristimo našu `Cat` komponentu u drugom fajlu, prvo moramo da je izvezemo ovako:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}

export default Cat;
```

Koristimo ključnu reč `default` jer će ovo biti podrazumevani izvoz (default export) iz modula. Takođe možete odlučiti da izvezete komponentu na istoj liniji kao definicija komponente, ovako:

```jsx
export default function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

Možete odlučiti da uvezete pomoćne komponente u druge roditeljske fajlove ili da ih uvezete u glavni (root) komponentni fajl. Za ovaj primer, `Cat` komponentu ćemo uvesti unutar glavnog komponentnog fajla.

Svaki React projekat ima top-level komponentu, obično nazvanu `App.jsx`:

```jsx
export default function App() {
  return (
    // return component here
  );
}
```

Ovaj fajl se obično nalazi u direktorijumu `src` vašeg projekta. Vi ćete saznati više o uobičajenim rasporedima projekata u budućoj lekciji.

Da bismo koristili `Cat` komponentu unutar glavne `App` komponente, moraćete da je uvezete ovako:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    // return component here
  );
}
```

Sada, možete vratiti (return) `Cat` komponentu unutar `App` komponente ovako:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    <Cat />
  );
}
```

Ovaj pristup dobro funkcioniše kada je fajl odgovoran za izvoz jedne glavne komponente, čime se sintaksa uvoza održava jednostavnom i lako čitljivom. Zapamtite, fajl može imati samo jedan podrazumevani izvoz (default export), što ga čini idealnim za fajl koji primarno sadrži jednu komponentu.

Sada kada znate kako da koristite podrazumevane izvoze, pogledajmo imenovane izvoze (named exports). Imenovani izvozi omogućavaju fajlu da deli više komponenti ili funkcija. Za razliku od podrazumevanih izvoza, oni moraju biti uvezeni koristeći tačno ime sa kojim su izvezeni (osim ako ih ne preimenujete korišćenjem ključne reči `as`).

Ovo je korisno kada fajl služi kao biblioteka komponenti. Na primer, evo fajla koji sadrži dve komponente: Cat i Dog:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

Koristimo ključnu reč `export` pojedinačno na svakoj komponenti. Da bismo ih koristili u drugom fajlu, uvozimo ih korišćenjem vitičastih zagrada:


```jsx
import { Cat, Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

Sintaksa `{ Cat, Dog }` govori JavaScriptu da uveze specifične imenovane izvoze iz fajla `Animals.jsx`. Imena unutar vitičastih zagrada moraju tačno odgovarati izvezenim imenima. Ako želite da izbegnete sukobe imena, ili ako preferirate drugačije ime u trenutnom fajlu, možete preimenovati komponentu tokom uvoza korišćenjem ključne reči `as`.

```jsx
import { Cat as Kitty } from "./Animals";

export default function App() {
  return <Kitty />;
}
```

Na kraju, fajl može imati jedan podrazumevani izvoz i više imenovane izvoze.

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

Prilikom uvoza mešovitih izvoza (mixed exports), podrazumevani izvoz dolazi prvi (bez vitičastih zagrada), praćen imenovanim izvozima (unutar vitičastih zagrada):

```jsx
// App.jsx
import Cat, { Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

U ovom primeru, `Cat` je podrazumevani izvoz, pa ne koristi vitičaste zagrade. `Dog` je imenovani izvoz, pa ih zahteva.

Razumevanjem kako funkcionišu podrazumevani i imenovani izvozi, možete organizovati React komponente u više fajlova i ponovo koristiti ih gde god je potrebno. Kako gradite veće aplikacije, biranje pravog stila izvoza pomoći će da vaš kod ostane čist i održiv, te olakša rad sa komponentama koje su kreirale druge osobe.
