# --description--

React pruža mnogo ugrađenih hookova koji vam omogućavaju da implementirate različite funkcije u vašim projektima. Oni uključuju `useState`, `useEffect`, `useContext`, i druge.

Ali ponekad ćete morati dodati funkcionalnost kojoj nijedan od ugrađenih hookova ne može da pomogne. Srećom, možete kreirati sopstvene prilagođene hookove u Reactu.

Prilagođeni hookovi nisu toliko komplikovani koliko izgledaju. To su samo ponovno upotrebljive funkcije koje vam omogućavaju da podelite logiku između više komponenti. To znači da je ponovna upotrebljivost još jedan razlog zašto biste želeli da napravite sopstveni hook.

Sa prilagođenim hook-om, možete izvući logiku iz bilo kojih komponenti koje ih koriste, kao što su dohvaćanje podataka (data fetching), upravljanje stanjem (state management), uključivanje/isključivanje (toggling), nuspojave poput proveravanja online ili offline statusa korisnika, i tako dalje.

Zatim možete uvesti kuka za korišćenje u bilo kom komponentu, tako da se možete fokusirati na renderovanje i prezentaciju unutar tih komponenti. To znači manje ponavljanja i manju duplikaciju, što znači manje mesta za izmene kada želite da napravite bilo kakve ažurmane.

Sada, hajde da pogledamo kako možete napraviti sopstveni prilagođeni hook.

U Reactu, svi ugrađeni hookovi počinju rečju `use`, pa vaš prilagođeni hook treba da prati istu konvenciju. Ime vašeg prilagođenog hooka takođe bi trebalo jasno da komunicira šta radi.

Dakle, ako vaš prilagođeni hook...

- dohvata podatke, možete ga nazvati `useFetch`

- prebacuje nešto na i isključeno, možete to nazvati `useToggle`

- ili ako implementira *debouncing*, `useDebounce` je dobar naziv

Pretpostavimo da želiš da napraviš prilagođeni hook za dodavanje debaunsinaga svojoj aplikaciji.

Debounsing je programski tehniku koja ograničava koliko često funkcija radi. Radi on čekanjem dok korisnik prestane sa obavljanjem akcije tokom određene vremenske periode pre izvršavanja funkcije. Na primer, u okviru za pretragu, umesto slanja poziva API za svaki kucani znak, debounsing čeka dok korisnik pauzira kucanje na, recimo, 500 milisekundi.

Da biste kreirali prilagođeni hak za debaunsinog efekta, prvo morate kreirati fajl `useDebounce.jsx` ili `useDebounce.js`. Konvencionalno, fajlovi za bilo koji prilagođeni hak koje kreirate čuvaju se u folderu `hooks`.

Možete koristiti neke ugrađene hook-ove unutar sopstvenog prilagođenog hook-a. Za debounsing, potrebni su vam hook-ovi `useState` i `useEffect`, pa ih uvezite na vrh vašeg fajla:

```js
import { useState, useEffect } from "react";
```

Zatim, kreirajte funkciju `useDebounce` koja uzima `value` i `delay` kao parametre. `value` je resurs na koji želite da sačekate, a `delay` je vremenski period na koji želite da sačekate. Pošto želite da sačekate određeni vremenski period, funkcije `setTimeout` i `clearTimeout` bi bile korisne:

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```

Stanje ``debouncedValue`` čuva i vraća odloženu vrednost, koja se ažurira tek nakon navedenog vremenskog ograničenja.

`useEffect` je gde se magija zaista dešava. Ako se setite iz prethodne lekcije, sve što postoji van React ciklusa renderovanja, kao što je postavljanje i brisanje tajmera, predstavlja nuspojavnost (side effect), i trebalo bi da koristite ``useEffect`` kuku za njihovo rukovanje.

Unutar ovog `useEffect` haka, koristite `setTimeout` da postavite `debouncedValue`. Zatim vraćate funkciju za čišćenje koja koristi `clearTimeout` da obriše prethodni tajmer svaki put kada se `value` ili `delay` promeni, ili kada komponenta nestane.

Evo `FootballerSearch` komponente koja koristi `useDebounce` kuku za odlaganje pretrage za 1 sekundu nakon što korisnik prestane da kuca:

```jsx
import { useState, useEffect } from "react";
import { useDebounce } from "./hooks/useDebounce";
import footballers from "./footballers";

const FootballerSearch = () => {
  const [query, setQuery] = useState("");
  const debouncedQuery = useDebounce(query, 1000); // Start searching 1 second after the user stops typing

  useEffect(() => {
    if (debouncedQuery) {
      const results = footballers.filter((footballer) =>
        footballer.toLowerCase().includes(debouncedQuery.toLowerCase()),
      );
      console.log("Search results:", results);
    } else {
      console.log("Search results: []");
    }
  }, [debouncedQuery]);

  return (
    <>
      <h1 style={{ textAlign: "center" }}>Footballer Search App</h1>
      <div style={{ textAlign: "center" }}>
        <input
          style={{ padding: "0.5rem", width: "30%" }}
          type="text"
          value={query}
          onChange={(e) => setQuery(e.target.value)}
          placeholder="Search for a footballer..."
        />
      </div>
    </>
  );
};

export default FootballerSearch;
```

Kao što vidite, varijabla `debouncedQuery` je ono što inicijalizuje kuku `useDebounce` sa stanjem upita (što korisnik unosi), i odlaganje za 1.000 milisekundi, ili 1 sekundu. Sam pretraga se obrađuje unutar kuke `useEffect`, a rezultati pretrage se loguju u konzolu.
