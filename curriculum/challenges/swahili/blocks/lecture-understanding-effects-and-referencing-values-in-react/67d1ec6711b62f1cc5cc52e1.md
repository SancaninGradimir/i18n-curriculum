# --description--

U React-u, efekat je sve što se dešava izvan procesa renderovanja komponente. To jest bilo šta što React ne obrađuje direktno kao deo renderovanja korisničkog interfejsa (UI).

Česti primeri uključuju dohvaćanje podataka, ažuriranje naslova kartice pretraživača, čitanje ili pisanje u lokalni skladište pregledača, dobijanje lokacije korisnika i još mnogo toga. Ove operacije komuniciraju sa spoljnim svetom i poznate su kao sporedni efekti (side effects).

React pruža `useEffect` kuku kako biste mogli da upravljate tim sporednim efektima. `useEffect` vam omogućava da pokrenete funkciju nakon što se komponenta renderuje ili ažurira.

Pogledajmo kako radi `useEffect` kuka i zašto je neophodna za moderno React razvijanje.

Da biste koristili `useEffect` kuku, prvo morate da je uvezete:

```js
import { useEffect } from "react";
```

Zatim je koristite kao funkciju, ovako:

```js
useEffect(() => {
  // Your side effect logic (usually a function) here
}, [dependencies]);
```

Funkcija efekta se izvršava nakon renderovanja komponente, dok argument opcionih `dependencies` kontroliše kada će se efekat pokrenuti.

Napomena da `dependencies` može biti niz "reaktivnih vrednosti" (stanje, propovi, funkcije, varijable i slično), prazan niz ili potpuno izostavljen. Evo kako sve te opcije kontrolišu rad `useEffect`-a:

- Ako je `dependencies` niz koji uključuje jednu ili više reaktivnih vrednosti, efekat će se pokrenuti svaki put kada one promene.

- Ako je `dependencies` prazan niz, `useEffect` se izvršava samo jednom kada komponenta prvi put renderuje.

- Ako izostavite `dependencies`, efekat se izvršava svaki put kada komponenta renderuje ili ažurira.

Na primer, u ovoj aplikaciji `Counter`, ne prosleđujemo argument `dependencies`, pa se efekat pokreće kada komponenta renderuje i svaki put kada se ažurira:

```jsx
import { useState, useEffect } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log("Component renders");
  });

  return (
    <div
      style={{
        display: "flex",
        alignItems: "center",
        flexDirection: "column",
      }}
    >
      <h2>{count}</h2>
      <div>
        <button onClick={() => setCount(count + 1)}>Increase</button>
        <button onClick={() => setCount(count - 1)}>Decrease</button>
      </div>
    </div>
  );
};

export default Counter;
```

Ali ako prosledimo prazan niz kao zavisnost, efekat se izvršava samo pri prvom renderovanju:

```js
useEffect(() => {
  console.log('Component renders');
}, []);
```

Ako prosledite stanje `count` kao zavisnost, efekat se pokreće kada komponenta prvi put renderuje i kada se `count` promeni:

```js
useEffect(() => {
  document.title = `The current count is ${count}`;
  console.log('component renders');
}, [count]);
```

Napomena da, ako efekat koji ste postavili traje izvan životnog ciklusa renderovanja komponente, možda ćete želeti još jednu funkciju za "čišćenje" te funkcije nakon što komponenta renderuje ili ažurira.

Na primer, ako vaša funkcija efekta koristi `setInterval()`, postavlja slušalicu događaja poput `window.addEventListener()` ili se povezuje na server, biće vam potrebna funkcija za čišćenje da pokrene odgovarajuće `clearInterval()`, `window.removeEventListener()` i odkonekcionira se od servera.

Evo sintakse za vraćanje funkcije za čišćenje iz `useEffect` kuke:

```js
useEffect(() => {
  // Your side effect logic here
  return () => {
    // Cleanup logic here (optional)
  };
}, [dependencies]);
```

Na primer, ako dodate slušalicu događaja skrolovanja, možete je očistiti uklanjanjem u svojoj funkciji za čišćenje:

```js
useEffect(() => {
  const handleScroll = () => {
    // Handle scroll logic
  };
  window.addEventListener("scroll", handleScroll);

  return () => {
    window.removeEventListener("scroll", handleScroll);
  };
}, []);
```
