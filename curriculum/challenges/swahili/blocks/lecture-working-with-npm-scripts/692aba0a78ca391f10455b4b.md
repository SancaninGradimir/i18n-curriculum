# --description--

U modernom web razvoju, možete podeliti svoj kod na više fajlova nazvanih „module“ (moduli).

Možete zamisliti modul kao samostalni i ponovno upotrebljiv blok koda.

Uvozi (importovanje) modula u druge module omogućava vam da ponovo koristite njihovu funkcionalnost, a istovremeno zadržavate kod u zasebnim fajlovima, što je mnogo održivije, posebno za velike i složene web aplikacije.

U Node.js okruženju, svaki fajl se smatra modulom.

Node ima dva različita sistema za rad sa modulima: CommonJS i ES.

CommonJS je originalan i podrazumevani način rada sa paketima u Node.js-u. On učitava module sinhrono, što znači da mora da sačeka da se modul potpuno učita pre nego što počne da učitavati sledeći ili nastavi izvršavanje koda.

I dalje je široko korišćen, ali ga postepeno zamenjuje novija ES sintaksa.

CommonJS koristi funkciju `require()` za uvoz modula, uzimajući njegov relativni put kao argument.

Na primer, recimo da imamo modul nazvan `math.js`, gde definišemo osnovne matematičke operacije.

Ovde ćemo koristiti samo funkciju množenja kako bismo ovaj primer zadržali jednostavnim i fokusiranim:

```js
function multiply(a, b) {
  return a * b;
}
```

Kako možemo ponovo iskoristiti ovu funkciju izvan našeg trenutnog modula? Podrazumevano, funkcije i varijable definisane u Node.js modulu su **privatne** — drugi moduli ne mogu da im pristupe osim ako ih eksplicitno ne izvezemo (export). Postoji nekoliko načina za to.

Jedan uobičajen način je dodeljivanje vrednosti `module.exports`. Ovaj objekat predstavlja ono što će modul vratiti kada se uvozi negde drugde, dajući drugim modulima pristup funkcijama, varijablama ili objektima koje ste odlučili da izložite (expose).

U ovom primeru, dodeljujemo objekt sa svojstvom `multiply` na `module.exports`. Vrednost tog svojstva je referenca na funkciju `multiply` koju smo ranije definisali. Ovo će eksportovati funkciju, tako da možemo pozvati iz drugih modula ako nam zatreba.

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = {
  multiply: multiply
};
```

Recimo da pokušamo da uvezemo modul `math.js` u drugi modul. Pozivamo funkciju `require()` i prosleđujemo relativni put kao argument. Zatim, dodeljujemo objekt koji vraća `require()` promenljivoj `math`.

```javascript
const math = require('./math');
```

Pogledajmo vrednost `math` u konzoli štampanjem sa `console.log()`:

```javascript
console.log(math);
```

Ovo je izlaz. To je objekat sa svojstvom `multiply` i funkcijom kao vrednošću:

```javascript
{ multiply: [Function: multiply] }
```

Ako pozovemo ovu funkciju sa `math.multiply()`, prosleđujući brojeve `6` i `7` kao argumente, izlaz je `42`. Funkcija uspešno radi, a ipak se nalazi u drugom fajlu! Dakle, naš statement za uvoz je radio ispravno.

```javascript
const math = require('./math');

console.log(math.multiply(6, 7)); // 42
```

Ali postoje i drugi načini za eksportovanje funkcije, varijable ili objekta.

Na primer, ako vam treba da eksportujete samo jednu funkciju, varijablu ili objekt, možete ga direktno dodeliti kao vrednost `module.exports`:

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = multiply;
```

Ako morate da eksportujete više funkcija, jednostavno ih dodate u objekat `module.exports`, razdvojene zarezom.

Ovde vidite da eksportovani objekat ima dva svojstva: `subtract` i `multiply`. Njihove vrednosti su reference na odgovarajuće funkcije, tako da ćete imati pristup obe u bilo kom modulu koji uvozi `math.js`:

```javascript
function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = {
  subtract: subtract,
  multiply: multiply
};
```

Kada objekat `module.exports` ima više svojstava, imate opciju da izaberete da li želite da uvezete sve ili da uvozite svojstva pojedinačno.

U ovom primeru, uvozimo ceo objekat koji je eksportovan sa strane `math.js`, tako da ćemo imati pristup i `subtract`-u i `multiply`:

```javascript
const math = require('./math');

console.log(math.subtract(6, 7));  // -1
console.log(math.multiply(6, 7));  // 42
```

Ako želimo da uvezemo samo `multiply` umesto toga, moramo samo da napišemo njegovo ime unutar vitičastih zagrada, nakon ključne reči `const`. U osnovi, koristite sintaksu dodeljivanja destrukturiranja (destructuring assignment) kako biste dobili njegovu vrednost iz eksportovanog objekta. Zatim, možemo se na njega direktno pozvati po imenu u našem kodu:

```javascript
const { multiply } = require('./math');

console.log(multiply(6, 7));
```

Ovo radi i za više elemenata. Samo morate da ih razdvojite zarezom unutar vitičastih zagrada. Ovde uvozimo obe funkcije `subtract` i `multiply`:

```javascript
const { subtract, multiply } = require('./math');
```

Podrazumevano, Node.js će tretirati sledeće kao CommonJS module:

* Fajlovi sa ekstenzijom `.cjs`.
    
* Fajlovi sa ekstenzijom `.js` kada najbliži roditeljski fajl `package.json` sadrži top-level polje `"type"` sa vrednošću `"commonjs"`.
    
* Fajlovi sa ekstenzijom `.js` ili bez ekstenzije, kada najbliži roditeljski fajl `package.json` ne sadrži top-level polje `"type"`, ili nema `package.json` u bilo kom roditeljskom folderu.
    
* Fajlovi sa ekstenzijom koja nije `.mjs`, `.cjs`, `.json`, `.node`, ili `.js`.
    

Iako ES Module moduli postepeno zamenjuju CommonJS module, razumevanje njih je i dalje suštinsko za programere koji rade sa postojećim Node.js bazama koda i mnogim npm paketima.
