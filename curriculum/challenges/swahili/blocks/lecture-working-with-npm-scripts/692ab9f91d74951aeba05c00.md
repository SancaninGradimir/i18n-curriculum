# --description--

Sada ćete naučiti o npm skriptama.

Pre nego što uđemo u detalje, hajde da pregledamo neke važne koncepte:

* **npm** je alat za komandnu liniju koji možete koristiti za instaliranje i upravljanje JavaScript paketima.
    
* **Paket** je direktorijum koji grupiše povezani kod u logičkoj strukturi.
    
* **Skripta** je sekvenca instrukcija ili komandi napisana na skripting programskom jeziku.

**npm skripte** su prilagođeni komandi koje možete definisati u fajlu ``package.json`` vaše aplikacije.

Oni su korisni za automatizaciju zadataka i radnih tokova koje ćete ponovo koristiti tokom faza razvoja, testiranja i implementacije vašeg projekta.

Na primer, jedna upotreba za npm skripte je prilagođavanje `npm start` da pokrene vaše celokupno razvojno okruženje. Umesto ručnog pisanja svih komandi jednu po jednu svaki put kada vam je potrebno pokrenuti razvojno okruženje, možete samo pokrenuti `npm start` i ceo proces će biti izvršen.

Takođe možete koristiti npm skripte za automatizaciju linovanja celokupne vaše kod baze i za pokretanje paketa testova, među ostalim slučajevima upotrebe.

Oni su veoma moćni, pa hajde da pogledamo njihove osnove.

## Kako kreirati fajl package.json

Da biste mogli da kreirate i pokrenete prilagođene npm skripte, biće vam potreban `package.json` fajl.

Fajl ``package.json`` je esencijalan za Node.js projekte. Ovaj fajl sadrži metapodatke, zavisnosti i skripte projekta.

Kao podsetnik, da biste kreirali fajl `package.json`, morate pokrenuti komandu `npm init` u korenom direktorijumu vašeg projekta. Zatim možete ručno uneti ime paketa, verziju, opis i druge detalje, ili ostaviti te polja prazna da biste koristili podrazumevane vrednosti.

Nakon što završite ovaj proces, imaćete fajl ``package.json`` u korenom direktorijumu vašeg projekta.

Ako ste zadržali sve podrazumevane vrednosti, trebalo bi da vidite ove parove ključ-vrednost kada otvorite `package.json` u svom uređivaču koda:

```json
{
  "name": "<project_name>",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC",
  "description": ""
}
```

Obratite pažnju na ovu paru ključ-vrednost ovde:

```json
"scripts": {
  "test": "echo \"Error: no test specified\" && exit 1"
}
```

Ovo je mesto gde ćete navesti svoje npm skripte, pod ključem `scripts`.

Videćete jedan skript tamo podrazumevano, testni skript:

```json
"test": "echo \"Error: no test specified\" && exit 1"
```

Ovo je primer npm skripte definisane u `package.json`.

Evo osnovne sintakse za definisanje npm skripte:

```json
"<script_name>": "<commands>"
```

Levo od dvotačke, nalazimo ime skripte. Uobičajeni uključuju `run`, `start`, `test`, `build`, `dev`, i `lint`. Ova imena deluju kao "prečice" za svoje osnovne komande desno.

Desno od dvotačke, nalazimo same osnovne komande.

Na primer:

```json
"scripts": {
  "start": "node app.js"
}
```

To run the script, enter this in the command line:

```bash
npm run <script_name>
```

I tako možete pokrenuti bilo koje prilagođene skripte definisane u `package.json`.

Ponekad možete koristiti ovu skraćenu verziju bez reči `run` za izvršavanje skripti:

```bash
npm <script_name>
```

Ali ovo radi samo za specifične npm skripte nazvane `start`, `test`, `stop`, i `restart`.

Vraćajući se našem primeru, ako imamo ovo u `package.json`:

```json
"scripts": {
  "start": "node app.js"
}
```

Da biste pokrenuli npm skriptu, morate uneti `npm run start` u komandnu liniju. Kada to učinite, stvarna osnovna komanda koja se izvršava je `node app.js`.

Pošto ovaj npm skript ima skraćenu verziju, takođe biste mogli da koristite `npm start` i rezultat bi bio isti.

U ovom slučaju, samo zamenjujete jednostavnu komandu, `node app.js`, sa drugom, `npm start`. Ali zamislite da imate sekvencu kompleksnih komandi koje treba izvršiti serijski ili paralelno, i da možete samo koristiti `npm start` da ih sve pokrenete. To bi vam uštedelo mnogo vremena, zar ne? Tamo je gde moć npm skripti zaista blista.

Evo kratkog pregleda nekih od najčešće korišćenih npm skripti:

* `npm run`: koristi se za pokretanje prilagođenog npm skripata.
    
* `npm start`: koristi se za pokretanje glavne aplikacije ili razvojnog servera.
    
* `npm lint`: koristi se za pokretanje proveravača stila koda kako bi se proverilo da li vaš kod ispunjava standarde stila koda i najbolje prakse.
    
* `npm test`: koristi se za pokretanje testne suite projekta.
    
Postoji mnogo drugih npm skripata za različite svrhe, ali ovo su neki od najčešćih koje ćete videti.

Takođe, uobičajena konvencija je korišćenje dvotačke za grupisovanje povezanih skripti pod istim neprostorijem (namespace). Na primer, `start:server` i `start:ui`.

## Kako pokrenuti komande sekvencijalno

Ako trebate pokrenuti više komandi u sekvenci, imate dve opcije. Možete koristiti logički AND operator (`&&`) ili semikolon (`;`). Pogledajmo njihove razlike.

Logički AND operater izvršava sledeću komandu samo ako se prva uspešno izvrši.

Na primer:

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build && node server.js"
}
```

Primećujete da postoji logički AND operater u komandi `start`:

```json
"start": "npm run build && node server.js"
```

Ovde, `npm start` će pokušati da pokrene `npm run build` prvo, i samo ako ovaj skript uspešno radi, `node server.js` će biti izvršen.

Ovo je veoma uobičajen pristup jer obezbeđuje pouzdanost i sprečava gubljenje vremena i resursa na naknadne komande koje mogu da propadnu ili bacaju greške ako prethodna ne uspe.

Alternativno, možete koristiti semikolon operater za sekvencijalno izvršavanje komandi, bez obzira na to da li je prethodna komanda bila uspešna ili ne.

U našem primeru, ova jednostavna promena bi imala značajan uticaj na način na koji će komande da rade:

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build; node server.js"
}
```

Fokusirajte se ponovo na komandu ``start``:

```json
"start": "npm run build; node server.js"
```

Sada, pošto koristimo razmaknicu umesto operatera `&&`, komanda `node server.js` će uvek biti izvršena nakon `npm run build`, čak i ako `node server.js` nije uspeo.

```json
"start": "npm run build; node server.js"
```

## Kako pokrenuti komande konkurentno

Ako je potrebno pokrenuti komande istovremeno (paralelno), možete koristiti operater ``&``.

```json
"scripts": {
  "start:server": "node server/index.js",
  "start:client": "npm run build:watch",
  "dev": "npm run start:server & npm run start:client"
}
```

Međutim, trebalo bi da znate da ova opcija pokreće procese u pozadini.

To znači da, ako ih pokrenete u istoj terminal sesiji, možete videti njihov kombinovani izlaz, što može biti teško za razumevanje.

Druge opcije za istovremeno pokretanje komandi uključuju alate kao što su `concurrently` i `npm-run-all`.

## Kako proslediti argumente npm skriptama

Ako trebate proslediti argumente direktno komandi koja se izvršava unutar npm skripte, možete ih specificirati u komandi upisivanjem dva crtica (`--`) praćena argumentima.

`--` govori npm-u da je sve što sledi argumenti za osnovne komande koje će npm skript izvršiti.

Ovo je opšta sintaksa:

```bash
npm run <script_name> -- <argument_1> <argument_2> <argument_3>
```

Na primer, recimo da definiš ovu `npm start` skriptu:

```json
{
  "name": "project-example",
  "scripts": {
    "start": "node app.js"
  }
}
```

Kako biste specificirali port gde želite da pokrenete svoj server ako koristite ``npm start``?

Recimo da želimo da prosledimo port direktno kao argument linije komandi. Npisali biste ovo:

```bash
npm start -- 8000
```

U vašem kodu, možete pristupiti ovim argumentima u nizu `process.argv` koristeći indekse:

```bash
process.argv[0] # The path to the node executable.
process.argv[1] # The script file being executed.
process.argv[2] # The first argument (8000).
```

Stoga, mogli biste pristupiti i koristiti broj porta gde god vam je potrebno sa `process.argv[2]`. To je moć argumenta komandne linije kombinovanih sa npm skriptama.

Kao što vidite, npm skripte su neophodni alati za pojednostavljenje razvojnog radnog toka. Pomažu vam da automatizujete i pretvorite složene i ponovno upotrebljive sekvence komandi u jednostavne i dosledne prečice koje možete koristiti u svakodnevnom radu.
