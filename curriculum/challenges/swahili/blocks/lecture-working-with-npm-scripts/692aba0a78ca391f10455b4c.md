# --description--

Sada kada razumete sistem modula CommonJS, pogledajmo sistem modula ES.

ES moduli, ili ECMAScript moduli, su zvanični, standardizovani format modula za JavaScript definisan ECMAScript specifikacijom. Koriste `import` i `export` naredbe, a u Node.js-u se učitavaju asinhrono, što je efikasnije.

Node.js tretira fajlove kao ES module u sledećim slučajevima:

* Fajlovi sa ekstenzijom `.mjs`.
    
* I fajlovi sa ekstenzijom `.js` kada najbliži roditeljski fajl `package.json` sadrži polje `"type"` na najvišem nivou sa vrednošću `"module"`.
    

Ovde imamo istu funkciju `multiply` koju ste već videli, ali sada ćemo koristiti sistem ESM. Za ovo dodeljujemo ekstenziju `.mjs` oba modula.

Da biste izvezili funkciju `multiply` iz modula `math.mjs`, koristite ključnu reč `export` praćenu imenom funkcije, varijable ili objekta koji želite da izvezete unutar vitičastih zagrada:

```javascript
function multiply(a, b) {
  return a * b;
}

export { multiply };
```

Takođe možete izvesti funkciju direktno korišćenjem ključne reči `export` pre ključne reči `function`. Ovo je poznato kao imenovani izvoz (named export). Možete imati više imenovanih izvora po modulu:

```javascript
export function multiply(a, b) {
  return a * b;
}
```

Zatim, da biste uvezli ovaj modul, koristite ključnu reč `import` praćenu vitičastim zagradama. Primećujte da koristimo `import` umesto `require()`. To je jedna od ključnih razlika između CommonJS i ES modula.

Unutar vitičastih zagrada pišemo ime funkcije, varijable ili objekta koji želimo da uvezemo. U ovom slučaju, to je `multiply`. Zatim koristite ključnu reč `from` i relativni put do modula. Ovaj kod će uspešno raditi, a izlaz biće `42`.

```javascript
import { multiply } from './math.mjs';

console.log(multiply(6, 7)); // 42
```

U ES sistemu, možete postaviti jedan podrazumevani izvoz (default export) po modulu. Na ovaj način, ne morate koristiti vitičaste zagrade za njegov uvoz.

Evo primera gde postavljamo funkciju `multiply` kao podrazumevani izvoz za ceo modul `math.mjs`:

```javascript
export default function multiply(a, b) {
  return a * b;
}
```

Sada nam ne trebaju vitičaste zagrade, jer će funkcija biti uvezena i dodeljena podrazumevano:

```javascript
import multiply from './math.mjs';
```

Ako ne koristite ekstenzije fajlova ili fajl `package.json` da eksplicitno kažete Node.js-u koji tip sistema modula želite da koristite, on će pregledati izvorni kod kako bi potražio sintaksu ES modula. Ako se ta sintaksa pronađe, izvršiće kod kao ES modul; u suprotnom, izvršiće ga kao CommonJS modul.

Iako je CommonJS služio kao podrazumevani standard za Node.js u ranim danima, kako programeri sve više usvajaju ESM zbog njegove efikasnosti i kompatibilnosti sa modernim JavaScript-om, važno je znati njihove razlike i kako da koristite oba.
