# --description--

Mokovanje je proces zamene stvarnih podataka lažnim podacima koji simuliraju ponašanje stvarnih komponenti. Ovo omogućava testerima i programerima da testiraju specifične komponente u aplikaciji bez oslanjanja na eksterne zavisnosti.

Čest slučaj upotrebe za *mockovanje* podataka je kada se radi sa pozivima API-ja. Zamislite da imate `fetch` poziv kao ovaj u vašoj aplikaciji:

```js
export const postService = {
  async getPosts() {
    const response = await fetch("https://api.example.com/posts");
    return response.json();
  },
};
```

Ako treba da testirate funkciju ``postService``, umesto da pravite kontinuirane API pozive radi preuzimanja podataka, možete simulirati (mockovati) odgovor API-ja. Dohvatanje podataka sa API-ja zahteva vreme i zavisi od eksternih faktora kao što su dostupnost mreže i odgovori servera. Simuliranje (mockovanje) odgovora API-ja uklanja te eksterne faktore i omogućava testerima da rade sa predvidljivim, kontrolisanim skupom podataka.

Evo primera simuliranog odgovora API-ja:

```json
[
  {
    "id": 1,
    "title": "Understanding Async/Await in JavaScript",
    "content": "Async/Await makes asynchronous code look and behave like synchronous code...",
    "author": "Jane Doe",
    "created_at": "2025-04-10T14:32:00Z"
  },
  {
    "id": 2,
    "title": "10 Tips for Writing Clean Code",
    "content": "Clean code is not just about code formatting. It involves naming, architecture...",
    "author": "John Smith",
    "created_at": "2025-04-08T09:21:00Z"
  },
  {
    "id": 3,
    "title": "Exploring React 19 Features",
    "content": "React 19 comes with exciting features like new hooks and better performance...",
    "author": "Alex Lee",
    "created_at": "2025-04-07T18:47:00Z"
  }
]
```

Sada kada razumemo šta je *mocking*, uporedimo ga sa *stubbing*-om. Stubi su objekti koji vraćaju unapred definisane odgovore ili lažne podatke za očekivano ponašanje u aplikaciji. Na primer, možete stubovati ponašanje za konekciju sa bazom podataka u vašim testovima bez oslanjanja na stvarnu konekciju sa bazom podataka.

Evo primera stuba u Jestu:

```js
import { postService } from "./postService";
import { db } from "./db";

jest.mock("./db", () => ({
  db: {
    fetchPosts: jest.fn(),
  },
}));

test("should return stubbed posts", async () => {
  const fakePosts = [
    { id: 1, title: "Stubbed Post", content: "This is a stubbed post." },
  ];

  db.fetchPosts.mockResolvedValue(fakePosts);

  const posts = await postService.getPosts();

  expect(posts).toEqual(fakePosts);
});
```

`db.fetchPosts.mockResolvedValue(fakePosts);` je placeholder u ovom primeru.

Poslednja komponenta o kojoj razgovaramo su fakersi. Fakersi su pojednostavljene verzije stvarnih komponenti bez složenosti ili nuspojava stvarne stvari. Na primer, možete napraviti lažnu bazu podataka skladištenjem podataka u memoriji umesto interakcije sa stvarnom bazom podataka. Ovo će vam omogućiti da imitirate operacije baze podataka u memoriji, što će biti mnogo brže od rukovanja stvarnom bazom podataka. Još jedan uobičajen primer bi bilo korišćenje fakersa prilikom rada sa sistemima fajlova. Možete kreirati lažni sistem fajlova umesto stvarnog kada je u pitanju testiranje.

Kako nastavite da razvijate sve složenije softverske aplikacije, razmislite o korišćenju mock-ova, stubova ili lažnih objekata (fakes) gde je to prikladno u vašem testiranju.
