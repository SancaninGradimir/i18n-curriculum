# --description--

End-to-end testiranje, ili skraćeno E2E, testira scenarije iz stvarnog sveta sa perspektive korisnika. Primeri E2E testiranih scenarija uključuju testiranje procesa registracije i prijave ili testiranje naplate za e-commerce sajt.

Da bismo bolje razumeli kako funkcioniše end-to-end testiranje, pogledajmo primer iz stvarnog sveta iz freeCodeCamp baze koda. Ovaj primer koristi Playwright, popularan okvir za end-to-end testiranje razvijen od strane Microsofta.

Unutar freeCodeCamp baze koda, postoji direktorijum nazvan `e2e` koji sadrži testove za donacije, sertifikate, ispite i više. Unutar direktorijuma, postoji fajl `donate-page-donor.spec.ts` koji ima testove za autentifikovanu stranicu podržavca donatora.

Prvi korak je navigacija na stranicu za donacije kao demo korisnik koji je već podržavac:

```js
test.beforeEach(async ({ page }) => {
  execSync("node ./tools/scripts/seed/seed-demo-user --set-true isDonating");
  await page.goto("/donate");
});
```

Pošto postoji više testova u ovoj datoteci, hook `beforeEach` će se pokrenuti pre svakog testa.

Kada korisnik dođe na stranicu za donacije, videće poruku zahvalnosti sa predlozima kako dalje da podrže freeCodeCamp. Evo samo nekoliko testova za proveru nekog teksta pronađenog na stranici za donacije:

```js
test("should render the donate page correctly", async ({ page }) => {
  await expect(
    page.getByText("Thank you for your continued support")
  ).toBeVisible();

  await expect(
    page.getByText(
      "Your contributions are crucial in creating resources that empower millions of people to learn new skills and support their families."
    )
  ).toBeVisible();

  ...
});
```

Postoje takođe testovi za proveru da donator ima link/veza podržavača u traci menija, kao i specijalizovana stilizovana ivica oko njihovog avatara kako bi ukazala da su podržavođač:

```js
test("The menu should have a supporters link", async ({ page }) => {
  const menuButton = page.getByTestId("header-menu-button");
  const menu = page.getByTestId("header-menu");

  await expect(menuButton).toBeVisible();
  await menuButton.click();

  await expect(menu).toBeVisible();

  await expect(page.getByRole("link", { name: "Supporters" })).toBeVisible();
});

test("The Avatar should have a special border for donors", async ({ page }) => {
  const container = page.locator(".avatar-container");
  await expect(container).toHaveClass("avatar-container gold-border");
});
```

Imanjem ovih tipova detaljnih testova, možete povećati pokrivenost testovima za svoju aplikaciju iznad samo unit testova. End-to-end testovi takođe pomažu da se osigura da vaša aplikacija radi ispravno i da je predvidiva za korisnike. Primeri drugih alata za end-to-end testiranje uključuju Cypress, Selenium i Puppeteer.

Iako je end-to-end testiranje važno, skupo je jer je vremenski zahtevno za postavljanje, dizajniranje i održavanje. E2E testovi su odlični za kritične korisničke tokove, dok su unit testovi odlični za testiranje malih jedinica u aplikaciji.
