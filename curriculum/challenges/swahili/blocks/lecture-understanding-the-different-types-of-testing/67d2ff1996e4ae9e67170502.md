# --description--

Kada dizajnirate svoje aplikacije, često ćete imati niz malih funkcija koje su odgovorne za jednu stvar. Ovo je poznato kao "princip jedne odgovornosti". Kada imate niz malih funkcija, najbolje je testirati te funkcije kako biste osigurali da sve radi onako kako se očekuje. U ovoj lekciji, pogledaćemo kako kreirati jednostavni test (unit test) koristeći popularni Jest testni okvir.

U ovom primeru, kreiraćemo funkciju koja je odgovorna za vraćanje novom formatiranog niza:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

Ova funkcija ``getFormattedWord`` ima parametar nazvan ``str`` i prvo će proveriti da li je ``str`` prazan. Ako jeste, vraća se prazan niz znakova. U suprotnom, vraća se novi niz znakova gde je prvi slovo kapitalizovano. Eksportujemo ovu funkciju kako bismo je mogli koristiti u test fajlu.

U zasebnoj datoteci `getFormattedWord.test.js`, možemo napisati neke testove kako bismo proverili da li funkcija radi ono što je namenjeno.

Na vrhu fajla `getFormattedWord.test.js`, moramo prvo uvesti funkciju na ovaj način:

```js
import { getFormattedWord } from "./getFormattedWord.js";
```

Zatim moramo instalirati Jest paket koristeći `npm i jest`. Zatim možemo dodati test da proverimo da li je reč sastavljena od malih slova, `hello`, ispravno formatirana u `Hello`, koja počinje velikim `H`:

```js
test('capitalizes the first letter of a word', () => {
  expect(getFormattedWord('hello')).toBe('Hello');
});
```

Funkcija `expect` se koristi za testiranje vrednosti. Kombinovana je sa *matcher*-om, koji je funkcija koja proverava da li vrednost radi kako se očekuje. U ovom slučaju *matcher* je `toBe()`. Jest ima raznolike *matchere* koji vam pomažu da testirate za istinitost (truthiness), nizove (strings), brojeve i još mnogo toga.

Jedan način za pokretanje vaših testova je dodavanje `npm` skripte u vaš fajl ``package.json``. Evo primera skripte za okvir Jest:

```js
"scripts": {
  "test": "jest"
},
```

Zatim samo pokrenite ``npm run test`` u vašem terminalu da pokrenete testove. Ako prođu vaši testovi, trebalo bi da vidite nešto u terminalu ovako:

```sh
PASS ./getFormattedWord.test.js
  ✓ capitalizes the first letter of a word (1 ms)

Test Suites: 1 passed, 1 total
Tests:       1 passed, 1 total
Snapshots:   0 total
Time:        0.103 s, estimated 1 s
Ran all test suites.
```

Da bismo videli kakav je neuspešan test, možemo namerno da pokvarimo test ažuriranjem funkcije na sledeće:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return "This is incorrect";
}
```

Sada kada pokrenete komandu ``npm run test``, biće poruka o grešci jer je test očekivao drugačiji rezultat:

```sh

  ✕ capitalizes the first letter of a word (1 ms)
```

Sada možemo da ažuriramo našu funkciju nazad na originalnu ovde:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

Dodajte još jedan test da proverite slučajeve kada nije dostavljen argument za `getFormattedWord`:

```js
test("returns an empty string when no argument is provided", () => {
  expect(getFormattedWord()).toBe("");
});
```

Postoje drugi testovi koje možete dodati kako biste stvari učinili robusnijim, ali ovi prvi par testova je dobar uvod u unit testiranje.

Kada je u pitanju testiranje JavaScript aplikacija, neki uobičajeni testni okviri uključuju Jest, Mocha i Vitest.

Jedinično testiranje je važno jer će vam pomoći da uhvatite više grešaka u vašim programima i osigura da sve radi kako se očekuje. Takođe može služiti kao oblik dokumentacije za vašu aplikaciju jer je namenjeno da predstavlja očekivano ponašanje vašeg koda.
