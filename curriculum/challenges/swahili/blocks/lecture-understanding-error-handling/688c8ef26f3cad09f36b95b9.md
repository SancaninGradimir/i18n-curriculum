# --description--

Debagovanje je neophodna veština za svakog Python programera. Razumevanje osnovnih tehnika može vam pomoći da efikasno identifikujete i popravite probleme.

Debugovanje je proces identifikovanja i rešavanja grešaka ili bugova u vašem kodu. Uključuje pregled koda, razumevanje protoka, i korišćenje alata za lociranje izvora problema.

U ovoj lekciji, pregledaćemo uobičajene tehnike debagovanja koje možete koristiti u vašem sledećem Python projektu.

## Korišćenje funkcije `print()` i f-stringova

Prvo, korišćenje funkcije `print()` i f-stringova na različitim mestima u vašem kodu može vam pomoći da razumete tok i stanje promenljivih. Na primer:

```py
def add(a, b):
    result = a + b
    print(f'Adding {a} and {b} gives {result}')
    return result
```

Prikazivanjem vrednosti `a`, `b`, i `result`, možete proveriti da li funkcija radi kako je očekivano.

## Interaktivno debagovanje sa modulom `pdb`

Zatim, možete koristiti ugrađeni Python modul `pdb` za interaktivno debagovanje:

```py
import pdb

def divide(a, b):
    pdb.set_trace()
    return a / b

print(divide(10, 2))
```

Postavljanjem praćenja sa funkcijom `set_trace()`, možete proći kroz kod, inspektovati varijable i razumeti ponašanje programa.

Ako pokrenete kod iznad, videćete neki izlaz koji pokazuje lokaciju fajla koji pokrećete, liniju gde ste pozvali funkciju `set_trace()` i kod odmah nakon nje, i interaktivni `pdb` prompt:

```py
> /Users/fcc/Desktop/debugging.py(5)divide()
-> return a / b
(Pdb)
```

Ako unesete ``help`` u prompte, videćete listu komandi koje možete koristiti:

```py

a      cl         debug    help      ll        quit     s        unt
```

Zatim možete koristiti komande za debagovanje svog koda.

Na primer, ako želite da vidite tip elemenata u celom kodu u tom trenutku, možete koristiti komandu `whatis`:

```py
(Pdb) whatis a
<class 'int'>
(Pdb) whatis divide
Function divide
```

Kao što možete da vidite, kada pokrenete `.set_trace()`, tip parametra `a` je ceo broj, i `divide` je funkcija.

Zatim, da biste nastavili izvršavanje vašeg koda, možete koristiti komandu `continue`, ili jedan od njenih aliasa, `cont` ili `c`:

```py
(Pdb) continue
5.0
```

## Alati za debagovanje u IDE-u

Mnogi integrisani razvojni okruženja (IDE-ovi) nude napredne alate za debagovanje, kao što su tačke prekida, izvršavanje korak po korak i inspekcija promenljivih.

## Korišćenje VS Code debagera

Ako koristite VS Code, možete postaviti tačke za prekid u svom kodu i pokrenuti debagera da biste pauzirali izvršavanje na tim mestima. Evo kako da debagujete istu funkciju ``divide``:

**Korak 1: Postavite svoj kod**

Kreiraj fajl nazvan `main.py` sa sledećim sadržajem:

```py
def divide(a, b):
    result = a / b
    return result

print(divide(10, 2))
print(divide(15, 3))
```

**Korak 2: Postavljanje tačke prekida**

1. Kliknite u žleb (leva margina) pored linije 2 (`result = a / b`) da biste postavili tačku prekida

2. Pojaviće se crvena tačka, što ukazuje na to da je postavljen prekidač.

**Korak 3: Početi sa debagovanjem**

1. Pritisnite `F5` ili idite na Pokreni > Početak debagovanja

2. Odaberite "Python Fajl" kada vas pita

3. Debager će pauzirati izvršavanje na vašoj tački prekida

**Korak 4: Pregledajte varijable**

- Pređite mišem preko promenljivih kako biste videli njihove trenutne vrednosti

- Koristite panel Varijabli na levoj strani da biste videli sve lokalne varijable

- Koristite konzolu za debagovanje na dnu da biste evaluirali izraze

**Korak 5: Prolazak kroz kod**

Koristite alat za debagovanje da:

- **Nastavi (F5)**: Nastavite izvršavanje do sledeće tačke za prekid

- **Pređi preko (F10)**: Izvrši trenutni red i pređi na sledeći

- **Uđi u (F11)**: Ulazak u pozive funkcija

- **Izlazak (Shift+F11)**: Napusti trenutnu funkciju

Alati za debagovanje IDE pružaju vizuelni interfejs za pregled stanja vašeg programa, što olakšava identifikaciju i popravljanje problema u poređenju sa korišćenjem samo izjava `print`.

Savladavanjem ovih osnovnih tehnika debagovanja - korišćenjem `print()` naredbi, `pdb` modula i alata za IDE - možete efikasno identifikovati i rešiti probleme u vašem Python kodu. Svaka tehnika ima svoje mesto: `print()` naredbe za brze provere, `pdb` za interaktivno istraživanje, i IDE debagere za vizuelnu inspekciju.
