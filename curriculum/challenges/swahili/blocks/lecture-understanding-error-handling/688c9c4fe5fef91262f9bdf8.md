# --description--

U Pythonu, naredba `raise` je moćan alat koji vam omogućava da ručno pokrenete izuzetke u vašem kodu. Daje vam kontrolu nad time kada i kako se greške generišu, omogućavajući vam kreiranje prilagođenih uslova grešaka i sprovođenje specifičnog ponašanja programa.

Naredba `raise` se koristi za eksplicitno bacanje izuzetka u bilo kojoj tački vašeg programa, omogućavajući vam da signalizirate da je nastojan uslov greške ili da nisu ispunjeni određeni zahtevi.

Naredba `raise` u Pythonu može se koristiti na nekoliko načina za pokretanje izuzetaka. U svojoj najosnovnijoj formi, možete baciti ugrađene izuzetke ili kreirati prilagođene poruke greške. Evo jednostavnog primera:

```
```py
def check_age(age):
    if age < 0:
        raise ValueError('Age cannot be negative')
    return age

try:
    check_age(-5)
except ValueError as e:
    print(f'Error: {e}') # Error: Age cannot be negative
```
```

Ovde možete videti da je `raise` ključna reč koja pokreće izuzetak.

U ovom primeru, bacamo `ValueError` sa prilagođenom porukom kada se pruži nevažeća starost.

Naredba `raise` takođe se može koristiti za ponovno bacanje trenutnog izuzetka, što je posebno korisno u obradi izuzetaka:

```
```py
def process_data(data):
    try:
        result = int(data)
        return result * 2
    except ValueError:
        print('Logging: Invalid data received')
        raise  # Re-raises the same ValueError

try:
    process_data('abc')
except ValueError:
    print('Handled at higher level')
```
```

Ovde ključna reč `raise` (bez argumenata), ponovo baca trenutni izuzetak koji se obrađuje.

Ovo vam omogućava da snimite ili izvršite čišćenje dok istovremeno propagate grešku ka vrhu steka poziva (call stack).

Možete kreirati i baciti prilagođene izuzetke definišući sopstvene klase izuzetaka:

```
```py
class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f'Insufficient funds: ${balance} available, ${amount} requested')

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount

try:
    new_balance = withdraw(100, 150)
except InsufficientFundsError as e:
    print(f'Transaction failed: {e}')
```
```

Ovde možete videti da prilagođene klase izuzetaka nasleđuju od `Exception` ili njegovih podklasa. 

Vi ćete saznati više o klasama i nasleđivanju u budućim lekcijama. Za sada, znajte da je ovo način za kreiranje sopstvenih izuzetaka sa prilagođenom logikom.

Naredba `raise` takođe se može koristiti sa ključnom reči `from` za lančano bacanje izuzetaka (chaining exceptions), pokazujući odnos između različitih grešaka:

```
```py
def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

config = parse_config('config.txt')
```
```

Ovde možete videti da `raise ... from None` potiskuje originalni kontekst izuzetka:

```
```bash
Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 7, in parse_config
    raise ValueError('Configuration file is missing') from None
ValueError: Configuration file is missing
```
```

A `raise ... from e` lančano povezuje novi izuzetak sa originalnim, čuvajući trag greške.

```
```bash
Traceback (most recent call last):
  File "main.py", line 5, in parse_config
    return int(data)
           ^^^^^^^^^
ValueError: invalid literal for int() with base 10: ''

The above exception was the direct cause of the following exception:

Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 9, in parse_config
    raise ValueError('Invalid configuration format') from e
ValueError: Invalid configuration format
```
```

Takođe možete baciti izuzetke uslovno koristeći `assert` naredbe, koje su u suštini skraćeni oblik za `raise` sa `AssertionError`:

```
```py
def calculate_square_root(number):
    assert number >= 0, 'Cannot calculate square root of negative number'
    return number ** 0.5

try:
    result = calculate_square_root(-4)
except AssertionError as e:
    print(f'Assertion failed: {e}')
```
```

Naredba `raise` je esencijalna za kreiranje robusnih aplikacija gde morate sprovesti poslovna pravila, validirati ulaz i pružiti smislene poruke o greškama. Strategičkom korišćenjem `raise`, možete učiniti svoj kod predvidljivijim i lakšim za debagovanje, istovremeno dajući korisnicima jasan povratni odgovor o tome šta je pošlo naopako.
