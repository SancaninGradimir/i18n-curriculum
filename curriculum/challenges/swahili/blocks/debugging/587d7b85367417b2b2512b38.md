# --description--

Programi koji koriste grane, tj. one koje rade različite stvari ako su ispunjeni određeni uslovi, zavise od deklaracija `if`, `else if` i `else` u JavaScriptu. Ti uslovi ponekad proveravaju da li je rezultat jednak određenoj vrednosti.

Ova logika se razgovara (na engleskom, barem) kao "ako x jednako y, onda..." što se lako može prevesti kodom koji koristi `=`, ili operater dodeljivanja. Ovo uzrokuje neočekivani tok kontrole u vašem programu.

Kao što je diskutovano u prethodnim izazovima, operater dodeljivanja (`=`) u JavaScriptu dodeljuje vrednost promenljivoj. Operatori `==` i `===` proveravaju jednakost (trostruki operater `===` proverava strogu jednakost, tj. da su i vrednosti i tipovi jednaki).

Kod ispod dodeljuje `x` vrednost 2, što se procenjuje kao `true`. Gotovo svaka vrednost sama po sebi u JavaScriptu se procenjuje kao `true`, osim onih poznatih kao "falsy" vrednosti: `false`, `0`, `""` (prazan niz znakova), `NaN`, `undefined`, i `null`.

```js
let x = 1;
let y = 2;
if (x = y) {

} else {

}
```

U ovom primeru, deo koda unutar deklaracije `if` će se izvršiti za bilo koju vrednost `y`, osim ako je `y` falsy. Deo `else`, koji očekujemo da se ovde izvrši, zapravo neće raditi.

# --instructions--

Ispravite uslov da program izvrši ispravnu granu, a odgovarajuća vrednost treba da bude dodeljena promenljivoj `result`.

# --hints--

Vaš kod bi trebalo da ispravi uslov kako bi proveravao jednakost, umesto korišćenja dodeljivanja.

```js

```

```js
assert(__helpers.removeJSComments(code).match(/x\s*?===?\s*?y/g));
```
