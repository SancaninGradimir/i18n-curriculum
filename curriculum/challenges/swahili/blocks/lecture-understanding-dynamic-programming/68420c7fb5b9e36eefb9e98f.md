# --description--
  
Dinamičko programiranje je algoritamska tehnika koja rešava složene probleme razbijanjem ih na jednostavnije podprobleme i skladištenjem rezultata kako bi se izbegle višakstanke proračune. Ovaj pristup transformiše probleme koji bi inače zahtevali eksponencijalno vreme u one koji se mogu rešiti u polinomialnom vremenu.

## Osnovni principi dinamičkog programiranja

Dinamičko programiranje funkcioniše kada su prisutni dva ključna uslova u problemu.

- **Preklapajući podproblemi**: Isti manji problemi se pojavljuju više puta prilikom rešavanja većeg problema. Umesto ponovnog izračunavanja ovih podproblema, skladištimo njihova rešenja.
    
- **Optimalna struktura**: Optimalno rešenje problema sadrži optimalna rešenja njegovih podproblema. To znači da možemo izgraditi najbolje rešenje kombinovanjem najboljih rešenja manjih delova.

Pogledajmo ove koncepte koristeći klasičan problem „penjanja stepenica“.

## Problem sa naivnom rekursijom

Razmotrite problem penjanja stepenicama: penjete se stepenicama sa `n` stepenica i možete da popnete 1 ili 2 stepena odjednom. Koliko različitih načina postoji da dođete na vrh?

```python
def climb_stairs_recursive(n):
    """Recursive approach"""
    if n <= 2:
        return n  # Base cases: 1 way for 1 step, 2 ways for 2 steps
    # To reach step n, we can come from step (n-1) or step (n-2)
    return climb_stairs_recursive(n-1) + climb_stairs_recursive(n-2)
```

Ova implementacija ima eksponencijalnu vremensku složenost zbog masivnih redundantnih proračuna. Kada se izračunava `climb_stairs(5)`, ovo je šta se dešava:

- `climb_stairs(5)` poziva `climb_stairs(4)` i `climb_stairs(3)`
    
- `climb_stairs(4)` poziva `climb_stairs(3)` i `climb_stairs(2)`
    
- Sada se `climb_stairs(3)` izračunava **dva puta**
    
- `climb_stairs(3)` poziva `climb_stairs(2)` i `climb_stairs(1)`
    
- `climb_stairs(2)` se računa ukupno **3 puta**

Za samo `n=5`, radimo 9 poziva funkcija kada nam je potrebno samo 5 jedinstvenih kalkulacija. Kako raste `n`, ova redundancija eksplodira eksponencijalno - `climb_stairs(30)` bi zahtevao nekoliko miliona poziva funkcija. Vremenska složenost postaje `O(2^n)`, što ga čini neefikasnim i nepraktičnim za veće vrednosti `n`.

## Rešenja dinamičkog programiranja

Dinamičko programiranje eliminiše ovo višakobracunanje kroz dva glavna pristupa:

### Memorizacija (Top-Down pristup)

Memoizacija skladišti rezultate skupih funkcionih poziva i vraća keširani rezultat kada se ponovo pojave isti ulazi:

```python
def climb_stairs_memo(n, memo={}):
    """Dynamic programming with memoization"""
    # Check if we've already calculated this value
    if n in memo:
        return memo[n]  # Return cached result - O(1) lookup!
    
    # Base cases
    if n <= 2:
        return n
    
    # Calculate once and store in memo for future use
    memo[n] = climb_stairs_memo(n-1, memo) + climb_stairs_memo(n-2, memo)
    return memo[n]
```

Memoizacija je mnogo efikasnija jer je svaka jedinstvena vrednost od `1` do `n` izračunana tačno jednom. Kada nam zatreba `climb_stairs(3)` ponovo, umesto da ga ponovno izračunavamo (što bi pokrenulo više rekurzivnih poziva), jednostavno ga tražimo u našem memorijskom rečniku za `O(1)` vreme.

Pratićemo izvršavanje `climb_stairs(5)` top-down pristupom da bismo videli kako memorizacija eliminše suvišan posao:

```md
Call: climb_stairs_memo(5)
  memo = {} (empty)
  
  Call: climb_stairs_memo(4) 
    memo = {} (empty)
    
    Call: climb_stairs_memo(3)
      memo = {} (empty)
      
      Call: climb_stairs_memo(2) → returns 2 (base case)
      Call: climb_stairs_memo(1) → returns 1 (base case)
      
      Result: 2 + 1 = 3
      memo = {3: 3} (stored!)
    
    Call: climb_stairs_memo(2) → returns 2 (base case)
    
    Result: 3 + 2 = 5
    memo = {3: 3, 4: 5} (stored!)
  
  Call: climb_stairs_memo(3) → returns 3 (FROM MEMO - no recursion!)
  
  Result: 5 + 3 = 8
  memo = {3: 3, 4: 5, 5: 8}
```

**Poređenje učinkovitosti**

- **Naivna rekurzija**: Izvršava 9 poziva funkcija sa ponovnim izračunavanjem

- **Memoizacija**: Vrši samo 5 jedinstvenih proračuna, a zatim ponovo koristi sačuvane rezultate

- **Vremenska složenost**: Smanjena je sa `O(2^n)` na `O(n)` jer vršimo samo `n` jedinstvenih proračuna

- **Prostorna složenost**: Za skladište memoizacije i stog poziva (`O(n)`)

- **Stvarni uticaj**: Pada sa miliona poziva funkcija na oko 30 jedinstvenih proračuna podproblema (`climb_stairs(30)`)

### Tabulacija (Pristup od dna ka vrhu)

Tabulacija gradi rešenje od osnova, popunjavanjem tabele rešenjima za podprobleme:

```python
def climb_stairs_tabulation(n):
    """Dynamic programming with tabulation"""
    if n <= 2:
        return n
    
    # Create array to store results for all steps from 0 to n
    dp = [0] * (n + 1)
    dp[1] = 1  # 1 way to reach step 1
    dp[2] = 2  # 2 ways to reach step 2
    
    # Build up the solution iteratively
    for i in range(3, n + 1):
        # Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]
```

Tabulacija potpuno eliminiše rekurziju tako što gradi rešenje iterativno, počevši od najmanjih podproblema sve do cilja.

Pogledajmo pristup od donjeg nivoa ka višim nivoima u akciji da bismo videli kako sistematski gradimo rešenje. Evo iterativne konstrukcije za `climb_stairs(5)`:

```md
Initial state:
dp = [0, 1, 2, 0, 0, 0]
     [0, 1, 2, 3, 4, 5] ← indices (step numbers)

Step by step construction:

i = 3:
  dp[3] = dp[2] + dp[1] = 2 + 1 = 3
  dp = [0, 1, 2, 3, 0, 0]
  
i = 4:
  dp[4] = dp[3] + dp[2] = 3 + 2 = 5
  dp = [0, 1, 2, 3, 5, 0]
  
i = 5:
  dp[5] = dp[4] + dp[3] = 5 + 3 = 8
  dp = [0, 1, 2, 3, 5, 8]

Final result: dp[5] = 8
```

**Ključne prednosti tabulisacije**

- **Nema režije rekurzije**: Za razliku od memorizacije, nema steka poziva rekurzije.
    
- **Predvidivo izvršavanje**: Izračunavamo vrednosti u unapred određenom redosledu (1, 2, 3, 4, 5...).
    
- **Prijateljski nastrojen za keš**: Sekvencijalni pristup nizovima optimizuje korišćenje memorije.
    
- **Lako za optimizaciju**: Može smanjiti složenost prostora na `O(1)` jer nam trebaju samo poslednje dve vrednosti.

```python
def climb_stairs_optimized(n):
    if n <= 2:
        return n
    
    prev2, prev1 = 1, 2  # Only store last two values
    for i in range(3, n + 1):
        current = prev1 + prev2
        prev2, prev1 = prev1, current
    return prev1
```

**Poređenje učinkovitosti**

- **Naivni rekurzivni**: 9 poziva funkcija za `n=5`, eksponencijalni rast.
    
- **Tabu(lacija)**: 3 jednostavna sabiranja za `n=5`, linearni rast.
    
- **Vremenska složenost**: `O(n)` umesto `O(2^n)`.
    
- **Prostorna složenost**: `O(n)` za niz, ili `O(1)` sa optimizacijom.
    
- **Predvidljiva performanse**: Nema rizika od preljeva steka (stack overflow) za velike ulaze.

Obe pristupa smanjuju složenost vremena sa eksponencijalne `O(2^n)` do linearnog `O(n)` , dramatično poboljšanje koje pravi razliku između rešavanja problema u milisekundama naspram čekanja godinama za veće ulaze.

## Primene u stvarnom svetu

Dinamičko programiranje ima široke primene u računarstvu i dalje:

- **Optimizacija rute**: GPS sistemi koriste algoritme dinamičkog programiranja za pronalaženje najkraćih putanja između lokacija.
    
- **Obrada teksta**: Proverivači pravopisa i funkcije automatskog dovršavanja često se oslanjaju na dinamičko programiranje za izračunavanje udaljenosti uređivanja između reči.
    
- **Finansijsko modelovanje**: Investicione strategije i optimizacija portfolija često koriste tehnike dinamičkog programiranja.
    
- **Alokacija resursa**: Problem ruksaka i njegove varijante pojavljuju se u raspoređivanju, budžetiranju i upravljanju resursima.

## Praktičan primer: Problem razmene novca

Problem sa promenom novca je klasičan programski izazov koji, kada se reši pomoću dinamičkog programiranja, demonstrira oba ključna principa DP-a: optimalna podstruktura i preklapajući podproblemi.

Problem sa promenom novca pita: "Koji je minimalan broj novčića potreban za dostižanje ciljanog iznosa?"

Evo jednog rešenja koje koristi dinamičko programiranje:

```python
def min_coins(amount, coins):
    """Find minimum number of coins needed to make the given amount"""
    # Initialize dp array with "infinity" - represents impossible to make
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0  # Base case: 0 coins needed for amount 0
    
    # For each amount from 1 to target amount
    for i in range(1, amount + 1):
        # Try each coin denomination
        for coin in coins:
            if coin <= i:  # Can only use coin if it doesn't exceed current amount
                # Update minimum: current minimum vs (coins for remaining amount + 1)
                dp[i] = min(dp[i], dp[i - coin] + 1)
    
    # Return result if possible, -1 if impossible
    return dp[amount] if dp[amount] != float('inf') else -1

# Example usage:
# coins = [1, 3, 4], amount = 6
# dp[6] = min(dp[5]+1, dp[3]+1, dp[2]+1) = min(3+1, 1+1, 2+1) = 2
# Result: 2 coins (3 + 3)
```

Evo kako algoritam za promenu novca dinamičkog programiranja radi korak po korak za `coins = [1, 3, 4]`, `amount = 6`:

```md
Initial state:
dp = [0, ∞, ∞, ∞, ∞, ∞, ∞]
     [0, 1, 2, 3, 4, 5, 6] ← amounts

Building up the solution:

For amount = 1:
  Try coin 1: dp[1] = min(∞, dp[0] + 1) = min(∞, 0 + 1) = 1
  dp = [0, 1, ∞, ∞, ∞, ∞, ∞]

For amount = 2:
  Try coin 1: dp[2] = min(∞, dp[1] + 1) = min(∞, 1 + 1) = 2
  dp = [0, 1, 2, ∞, ∞, ∞, ∞]

For amount = 3:
  Try coin 1: dp[3] = min(∞, dp[2] + 1) = min(∞, 2 + 1) = 3
  Try coin 3: dp[3] = min(3, dp[0] + 1) = min(3, 0 + 1) = 1
  dp = [0, 1, 2, 1, ∞, ∞, ∞]

For amount = 4:
  Try coin 1: dp[4] = min(∞, dp[3] + 1) = min(∞, 1 + 1) = 2
  Try coin 3: dp[4] = min(2, dp[1] + 1) = min(2, 1 + 1) = 2
  Try coin 4: dp[4] = min(2, dp[0] + 1) = min(2, 0 + 1) = 1
  dp = [0, 1, 2, 1, 1, ∞, ∞]

For amount = 5:
  Try coin 1: dp[5] = min(∞, dp[4] + 1) = min(∞, 1 + 1) = 2
  Try coin 3: dp[5] = min(2, dp[2] + 1) = min(2, 2 + 1) = 2
  Try coin 4: dp[5] = min(2, dp[1] + 1) = min(2, 1 + 1) = 2
  dp = [0, 1, 2, 1, 1, 2, ∞]

For amount = 6:
  Try coin 1: dp[6] = min(∞, dp[5] + 1) = min(∞, 2 + 1) = 3
  Try coin 3: dp[6] = min(3, dp[3] + 1) = min(3, 1 + 1) = 2
  Try coin 4: dp[6] = min(2, dp[2] + 1) = min(2, 2 + 1) = 2
  dp = [0, 1, 2, 1, 1, 2, 2]

Final result: dp[6] = 2 (achieved with coins 3 + 3)
```

Ovo rešenje demonstrira oba ključna principa dinamičkog programiranja. Ima preklapajuće podprobleme jer pronalaženje minimalnog broja novčića za iznos od 6 zahteva poznavanje rešenja za iznose 5, 3 i 2. Isti podproblemi se pojavljuju prilikom računanja drugih iznosa. Ima optimalnu podstrukturu jer optimalno rešenje za bilo koji iznos uključuje optimalna rešenja za manje iznose. Ako znamo da je minimalan broj novčića za iznos od 3 jedan, onda je jedan način da se napravi iznos od 6 korišćenje tog rešenja plus još jednog novčića vrednosti 3.

Bez DP-a, morali bismo da probamo svaku moguću kombinaciju novčića – eksponencijalan broj mogućnosti. Sa DP-om, gradimo rešenje sistematski:

- **Vremenska složenost**: `O(amount × number of coins)` umesto eksponencijalne.
    
- **Prostorna složenost**: `O(amount)` za niz `dp`.
    
- **Nema višak posla**: Svaki podproblem (pronalazak minimalnih novčića za svaku iznos) rešava se tačno jednom.
    
- **Ponovo upotrebljivi rezultati**: Kada znamo minimalne novčiće za iznos 3, koristimo ovo znanje za sve veće iznose koji mogu od toga da profitiraju.

## Kada koristiti dinamičko programiranje

Dinamičko programiranje je efikasno kada:

- Problem se može razložiti na preklapajući podprobleme.
    
- Problem pokazuje optimalnu podstrukturu.
    
- Naivno rekurzivno rešenje bi zahtevalo ponavljanje proračuna.
    
- Morate optimizovati za vremensku složenost na uštrb prostorne složenosti.

Uobičajeni obrasci dinamičkog programiranja uključuju probleme optimizacije (pronalaženje minimalnih/maksimalnih vrednosti), brojiljske probleme (broj načina na koji se nešto može postići) i probleme odlučivanja koje se mogu razbiti na manje odluke.

Dinamičko programiranje transformiše složene probleme u upravljive probleme sistematskim skladištenjem i ponovnim korišćenjem rešenja podproblema. Razumevanje ove tehnike otvara vrata za efikasno rešavanje širokog spektra računarskih izazova.
