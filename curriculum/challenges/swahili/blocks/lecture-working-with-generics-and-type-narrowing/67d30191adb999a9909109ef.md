# --description--

Zapravo ste videli generički tip u prethodnoj lekciji: `Array<string>`. Ali obično ćete koristiti generičke tipove u funkcijama. Zapravo, mogu se smatrati posebnim parametrom koji pružate funkciji kako biste kontrolisali ponašanje definicije tipa funkcije.

Vratimo se našem prethodnom primeru funkcije za dobijanje nasumične vrednosti iz niza:

```js
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Naša funkcija prima niz stringova i vraća string. Ali šta ako bismo želeli da prosledimo niz brojeva i vratimo broj?

Mogli biste koristiti tip unije:

```js
const getRandomValue = (array: (string|number)[]): (string|number) => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

Ali sa ovim pristupom, iako smo prosledili niz koji sadrži samo brojeve, TypeScript utvrđuje da varijabla `val` može biti broj ILI string.

Umesto toga, možete definisati generički tip za funkciju:

```js
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

Postoje nekoliko važnih komponenti u našem novom primeru. Prvo, ``<T>`` sintaksa govori TypeScript-u da definišete generički tip ``T`` za funkciju. ``T`` je uobičajeno ime za generičke tipove, ali ga zapravo možete nazvati kako god želite (unutar konvencija promenljivih JavaScript-a).

Zatim, kažemo TypeScriptu da je nizski parametar niz vrednosti koje odgovaraju generičkom tipu, i da je vraćena vrednost pojedinačan element tog istog tipa. Ovo omogućava TypeScriptu da pravilno detektuje da će `val` biti broj, jer niz koji smo prosledili sastoji se isključivo od brojeva.

Ali šta je sa funkcijama koje ne kontrolišete? Pogledajmo metod `document.querySelector()`:

```js
const email = document.querySelector("#email");
console.log(email.value);
```

Zato što upitujemo sa selektorom ID-a, TypeScript ne zna kakav tip elementa upitujemo. Znate da je to nekog vrsti `Element`, ali naš pokušaj pristupa svojstvu `value` će baciti grešku jer interfejs elementa na najvišem nivou nema to svojstvo.

Srećom, možemo proslediti argument tipa funkciji poziva:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Primećujete kako ponovo koristimo sintaksu sa uglastim zagradama (`<HTMLInputElement>`), ali ovoga puta u pozivu funkcije. Ovo govori TypeScriptu da će element koji očekujemo pronaći biti element tipa `input`, i više ne dobijamo grešku na svojstvu 'value' jer elementi za unos zaista imaju to svojstvo. Naravno, `querySelector` može vratiti `null` ako se element ne pronađe, i TypeScript će ovde baciti grešku kompajlera jer mi taj slučaj ne obrađujemo.

Ali ćeš o tome naučiti u sledećoj lekciji.
