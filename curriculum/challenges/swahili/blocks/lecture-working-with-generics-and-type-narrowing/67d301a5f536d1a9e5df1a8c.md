# --description--

Biće vremena kada imate vrednost sa širokim tipom i morate je suziti na specifičniji tip. Na primer, možda morate osigurati da objekat odgovara interfejsu koji ste definisali. Ili da je string unutar specifične liste vrednosti. Postoji prilično mnogo načina za postizanje ovoga.

Prvi je sužavanje po istinitosti (truthiness). Razmislite o našem primeru iz prošlog časa:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Dobijamo grešku kompajlera pokušavajući da pristupimo svojstvu `value` od `email`, jer bi `email` moglo biti `null`. Međutim, možemo koristiti uslovnu izjavu da potvrdimo da je `email` istinito pre pristupa svojstvu:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

U ovom ažuriranom primeru, jer `null` nije istinit vrednost, TypeScript je u stanju da zaključi da `email` MORA biti element `input` unutar uslovnog bloka. Tako više ne baca grešku kompajlera.

Provere istinitosti mogu takođe da rade i u suprotnom smeru:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

Ovim pristupom, bacamo grešku ako `email` je lažan. `null` je lažna vrednost. Bacanje greške prekida logičko izvršavanje ovog koda, što znači da kada dođemo do poziva `console.log()`, TypeScript zna da `email` ne može biti `null`.

Opcionalno lančanje je takođe oblik sužavljanja tipa, pod istom pretpostavkom da pristup svojstvu ne može da se desi ako je vrednost `email` jednaka `null`.

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

Ali šta je sa drugim tipovima? Pa, možete uskomiti tipove i pomoću operatera `typeof`. Pogledajmo primer varijable koju smo naznačili da bi mogla biti string ILI broj:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
console.log(myVal / 10)
```

U ovom primeru, vidimo grešku kompajlera jer ne možemo izvršiti aritmetiku na vrednosti tipa string. Ali možemo koristiti uslov da proverimo varijablu `typeof` `myVal`:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

Pošto smo koristili ključnu reč ``typeof``, TypeScript sada zna da ``myVal`` mora biti broj, i možemo bezbedno izvršiti aritmetiku sa njim.

Ali šta je sa složenijim tipovima objekata? Ako objekat u pitanju slučajno potiče iz klase, zapravo možete koristiti ključnu reč `instanceof` da suzujete tip. Vraćajući se našem primeru `querySelector()`:

```js
const email = document.querySelector("#email");
```

Umesto da prosleđujemo generički tip i kažemo TypeScriptu šta je element, možemo koristiti ``instanceof`` za sužavanje tipa i pisanje sigurnijeg koda:

```js
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

Ovaj pristup može delovati isto kao i naš prethodni, ali `instanceof` je runtime validacija - što znači da, čak i ako smo pogrešno razumeli tip u TypeScriptu, naš JavaScript kod će i dalje potvrditi da `email` je element `input`.

Zatim, pogledajmo primer u kojem dohvatamo objekat `User` sa API-ja i pokušavamo da ispišemo informacije:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json();
printAge(myUser);
```

Dobićemo grešku kompajlera pokušavajući da prosledimo ``myUser`` u funkciju zato što, iako znamo da API vraća ispravan objekat, TypeScript to ne radi. I metoda ``.json()`` ne prima generički tip.

Laki način za rešavanje ovog problema bio bi da se izvrši kastovanje tipa:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

Ali svaki put kada kasteš tip, suštinski oslabđuješ sposobnost TypeScript-a da uhvati potencijalne greške. Zato umesto kastovanja tipa, možeš napisati proveru tipa:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

Tip povratne vrednosti ovde je ključna komponenta ove definicije funkcije. Sintaksa `user is User` ukazuje da naša funkcija vraća booleanski vrednost, što kada `true` znači da vrednost `user` zadovoljava interfejs `User`. Zatim izvršavamo neke osnovne provere kako bismo osigurali da struktura objekta `user` odgovara – napomenuti je korišćenje sužavanja na istinitost (`!!user`) i sužavanje `typeof`. Moramo ovo uraditi jer `typeof null` vraća `"object"`:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
if (isValidUser(myUser)) {
    printAge(myUser);
}
```

Sada, ako spojimo svu našu logiku zajedno, više ne dobijamo greške kompajlera i možemo uspešno izgraditi svoj kod.

Sužavanje tipa je moćna karakteristika koja vam pomaže da pišete sigurniji i manje podložan greškama kod - ali zapamtite da tipovi TypeScripta nisu potpuno kruti, pa izbegavajte prakse poput pretvaranja tipa vrednosti bez sužavanja tog tipa.
