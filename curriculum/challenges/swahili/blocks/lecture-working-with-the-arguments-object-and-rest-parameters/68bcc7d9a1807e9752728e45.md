# --interactive--

U prethodnoj lekciji ste naučili kako da radite sa objektom `arguments`, koji je objekat sličan nizu i koji sadrži vrednosti argumenata prosleđenih funkciji.

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Iako je ovo validan način za pristup i rad sa promenljivim skupom argumenata iz funkcije, moderne JavaScript aplikacije će umesto toga koristiti sintaksu rest parametara.

Evo ažuriranog primera koji koristi sintaksu parametra 'rest' umesto objekta `arguments`:

:::interactive_editor

```js
function logArgs(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Ovaj ažurirani primer više ne referencira objekat `arguments` direktno. Umesto toga, poslednji parametar definicije funkcije ima tri tačke ispred sebe. To uzrokuje da se ovaj rest parametar postavi unutar objekta `Array`. Možete nazvati ovaj rest parametar kako god želite. Samo se uverite da je poslednji parametar u definiciji funkcije, kao što je ovo:

```js
function exampleFunction(a, b, ...restOfArgs) {
  // some code here
}

function anotherFunction(x, y, ...theArgs) {
  // some code here
}
```

Postoje još nekoliko ograničenja kada se radi sa sintaksom parametra ostataka. Jedno ograničenje je da definicije funkcija mogu imati samo jedan parametar ostataka. Zbog toga bi sledeći primer ovde bio smatran nevažećim.

```js
// Won't work

function badFunction(...args, ...moreArgs) {
  // some code here
}
```

Sledeće ograničenje je da se završni zarezi ne mogu koristiti nakon parametra ostataka: `function exampleFunction(a, b, ...restOfArgs, )`.

Još jedno ograničenje je da rest parametar ne može imati podrazumevanu vrednost. Inače, `SyntaxError` će biti izbačeno.

```js
function badFunction(...args = [1,2]){
  // some code here
}
```

Pa, koje su neke razlike između objekta `arguments` i parametara `rest`?

Jedna ključna razlika je u tome što ``arguments`` objekat nije pravi niz, pa ne podržava metode kao što su ``includes``, ``pop`` i ``push``. Ali rest parametar je instanca ``Array``. Zbog toga možete koristiti važeće ugrađene metode niza bez potrebe da ga prvo konvertujete u pravi niz.

:::interactive_editor

```js
function hasCat(...args) {
  return args.includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```

:::

## --text--

Koja je ključna razlika između objekta ``arguments`` i parametara tipa *rest*?
## --answers--

Mogu se koristiti samo ugrađene metode nizova poput `push` i `pop` na parametru ostataka, dok objekti `arguments` prihvataju sve ugrađene metode nizova.
### --feedback--

Pogledajte na kraju lekcije za odgovor.
---

You cannot use built in array methods like `push` and `pop` on the rest parameter but those methods can be used on the `arguments` object.

### --feedback--

Pogledajte na kraju lekcije za odgovor.
---

The `arguments` object is an `Array` instance while the rest parameter is array-like.
