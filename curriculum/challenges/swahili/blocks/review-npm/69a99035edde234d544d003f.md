## Šta je npm?

- **Definicija**: npm je online registar za softver otvorenog koda, koji funkcioniše kao "aplikacioni prodavnica" (app store) za kod.
- **Zvanično značenje**: npm zvanično ne znači "Node Package Manager"; to je rekurzivna bakronimika skraćenica za "npm is not an acronym".
- **Tri povezana dela**:

- **Sajt (npmjs.com)**: Za pretraživanje paketa u registru.
- **Registrar**: Velika javna baza podataka koja skladišti sve pakete.
- **CLI (Command Line Interface)**: Alat instaliran sa Node.js-om za upravljanje zavisnostima projekta.

- **Alternative**: Iako se npm koristi najčešće/najviše, popularne alternative uključuju **Yarn**, **PNPM**, i **Bun**.

## Fajl package.json

- **Svrha**: Konfigurački fajl koji skladišti metapodatke, zavisnosti i skripte za Node.js projekat.
- **Ključna polja**:

  - `name` i `version`: Identifikuju projekat.
  - `main`: Specifikuje fajl tačku ulaska (entry point), često je to `index.js`.
  - `license`: Definiše pravne termine (podrazumevano je `ISC`).
  - `dependencies`: Paketi potrebni za pokretanje aplikacije u produkciji.
  - `devDependencies`: Alati potrebni samo tokom razvoja, kao što su testni okviri (testing frameworks).

- **Kreiranje**:

  - `npm init`: Pokreće interaktivni proces podešavanja.
  - `npm init -y`: Instalira `package.json` sa podrazumevanim vrednostima odmah.

## Semantičko verziranje (SemVer)

- **Format**: `MAJOR.MINOR.PATCH`.
- **Pravila**:

- **MAJOR**: Promene koje mogu da utiču na postojeći kod.
- **MINOR**: Nove funkcionalnosti koje su kompatibilne sa starijim verzijama.
- **PATCH**: Male popravke grešaka i poboljšanja.

- **Simboli opsega**:

  - `^` (Caret): Omogućava manje i popravke verzije (npr., `^1.2.3` omogućava `1.x.x`).
  - `~` (Tilde): Omogućava samo popravke verzije (npr., `~1.2.3` omogućava `1.2.x`).
  - `*` (Asterisk): Poklapa se sa bilo kojom verzijom; koristi se primarno za testiranje.

## Upravljanje zavisnostima

- **Instalacija**:

  - `npm install <name>`: Dodaje proizvodnu zavisnost.
  - `npm install <name> -D`: Dodaje razvojnu zavisnost.
  - `npm install <name>@<version>`: Instalira određenu verziju.

- **Moduli i fajlovi zaključavanja Node-a**:

- `node_modules/`: Sadrži stvarni kod instaliranih paketa.
- `package-lock.json`: Zaključava tačne verzije svih zavisnosti i "zavisnosti zavisnosti" kako bi se osigurala konzistentnost u svim okruženjima.

- **Uklanjanje**: `npm uninstall <name>` uklanja paket i ažurira `package.json`.
- **Održavanje**:

  - `npm outdated`: Identifikuje koji paketi imaju dostupnije novije verzije.
  - `npm update`: Ažurira pakete unutar dozvoljenog raspona SemVer.

## npm Skripte

- **Funkcija**: Kreiranje prilagođenih komandi definisanih u sekciji `scripts` od `package.json` za automatizaciju zadataka.
- **Izvršavanje**: Pokreće se putem `npm run <script_name>`.
- Postoje skraćeni oblici (shorthands) za `start`, `test`, `stop` i `restart` (npr. `npm start`).
- **Sekvencijalno izvršavanje**:
- `&&`: Pokreće drugu komandu samo ako prva uspe.
- `;`: Pokreće komande jednu za drugom, bez obzira na uspeh.
- **Konkurentno izvršavanje**: Koristite operator `&` za pokretanje procesa u pozadini (background).
- **Argumenti**: Prosleđujte argumente skriptama koristeći separator `--` (npr. `npm start -- 8000`).

## Objavljivanje u npm Registri

- **Pristup registru**: Baza podataka javnih i privatnih paketa.
- **Koraci za publikovanje**:

1. Kreirajte besplatni nalog na ``npmjs.com``.
2. Prijavite se u terminalu koristeći ``npm login``.
3. Uverite se da su jedinstveni ``name`` i važeći ``version`` u ``package.json``.
4. Kreirajte ``README.md`` i ``.npmignore`` (radi isključivanja osetljivih fajlova).
5. Pokrenite ``npm publish`` (ili ``npm publish --access public`` za pakete sa opsegom).

- **Opsezi**: Koriste se za izbegavanje sukoba imena objavljivanjem pod `@username/package-name`.

## Sistemi modula u Node.js

### CommonJS (CJS)

- **Podrazumevani sistem**: Originalni način na koji Node.js rukuje modulima; učitava se sinhrono.
- **Sintaksa**:
- **Uvoz (Import)**: `const module = require('./path')`.
- **Izvoz (Export)**: `module.exports = { functionName }`.
- **Identifikacija**: Koristi se za fajlove `.cjs` ili fajlove `.js` kada nije specificiran `type` u `package.json`.

### ES Moduli (ESM)

- **Standardizovani format**: Moderni JavaScript standard; učitava se asinhrono radi efikasnosti.
- **Sintaksa**:

- **Izvoz**: `export { name }` ili `export default function`.
- **Uvoz**: `import { name } from './path.mjs'`.
- **Identifikacija**: Koristi se za fajlove `.mjs` ili fajlove `.js` kada `package.json` sadrži `"type": "module"`.
