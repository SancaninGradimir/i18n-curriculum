# --interactive--

U prethodnoj lekciji ste naučili kako da radite sa funkcijama, koje su ponovno upotrebljivi delovi koda koji pomažu da vaš kod bude modularniji, lakši za održavanje i efikasniji. Svi prethodni primeri koristili su standardnu sintaksu funkcija, kao što je ovo:

```js
function greetings(name) {
  console.log("Hello, " + name + "!");
}
```

Ali drugi način za pisanje funkcija u JavaScriptu je kreiranje izraza sa streličnom funkcijom. Evo kako možete da refaktorišete prethodni primer da koristite sintaksu strelične funkcije umesto toga:

```js
const greetings = (name) => {
  console.log("Hello, " + name + "!");
};
```

U ovom revidiranom primeru, kreiramo varijablu `const` nazvanu `greetings` i dodeljujemo joj anonimnu funkciju. Većina sintakse će vam biti poznata, osim nedostajućeg ključne reči `function` i dodavanja strelice (`=>`) između parametra `name` i tela funkcije. Ako vaša lista parametara sadrži samo jedan parametar, onda možete ukloniti zagrade ovako:

```js
const greetings = name => {
  console.log("Hello, " + name + "!");
};
```

Ako vaša strelična funkcija nema parametre, onda morate koristiti zagrade na ovaj način:

```js
const greetings = () => {
  console.log("Hello");
};
```

Kada ste prvi put učili o funkcijama, morali ste da umotate telo funkcije u vitičaste zagrade. Ali ako vaše telo funkcije sadrži samo jednu liniju koda, možete ukloniti vitičaste zagrade na ovaj način:

```js
const greetings = name => console.log("Hello, " + name + "!");
```

Važno je napomenuti da uklanjanje zagradi i vitičastih zagrada za standardnu funkcionalsku sintaksu neće raditi. Doći ćete do grešaka ako pokušate nešto slično ovome:

```js
// This will produce syntax errors 
function greetings name console.log("Hello, " + name + "!");
```

Ovi tipovi funkcija u jednoj liniji rade samo ako koristite sintaksu strelične funkcije. Drugi ključni koncept je naredba `return`. Evo primera korišćenja sintakse strelične funkcije za izračunavanje površine:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  const area = width * height;
  return area;
};

console.log(calculateArea(5, 3)); // 15
```

:::

Kreiramo varijablu unutar funkcije nazvane `area` i zatim vraćamo tu varijablu. Ali mogli bismo malo da očistimo naš kod i vratimo samu kalkulaciju:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  return width * height;
}; 

console.log(calculateArea(5, 3)); // 15
```

:::

Ako pokušate da uklonite kvastaste zagrade i postavite izračunavanje na istu liniju, onda ćete dobiti poruku: `Uncaught SyntaxError: Unexpected token 'return'`.

```js
const calculateArea = (width, height) => return width * height;
```

Razlog zašto dobijate ovu grešku jeste zato što morate ukloniti izjavu ``return``. Kada uklonite tu izjavu ``return``, greška će nestati i funkcija će i dalje implicitno vratiti kalkulaciju.

```js
const calculateArea = (width, height) => width * height;
```

Pa, kada bi trebalo da koristiš sintaksu strelične funkcije? Pa, zavisi. Mnogi programeri je dosledno koriste u svojim ličnim projektima. Međutim, kada radiš u timu, izbor obično zavisi od toga da li postojeći kod koristi obične funkcije ili strelične funkcije. U budućim lekcijama, pokrićemo kada koristiti strelične funkcije i kada ih izbegavati.
