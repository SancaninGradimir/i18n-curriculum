# --description--

Da li vam se ikada desilo da vaš Express aplikacija pukne bez jasne poruke o grešci? Debugovanje i logovanje vam mogu pomoći da vidite šta se tačno dešava iza kulisa, tako da možete pronaći i popraviti probleme brže.

U ovoj lekciji, istraživaćemo kako Express rukuje debagovanjem i logiranjem—i kako ga možete koristiti za poboljšanje vašeg radnog procesa razvoja.

## Ugrađeno debagovanje sa Expressom

Express već uključuje alat za debagovanje nazvan `debug`. Ne morate ništa instalirati da biste ga koristili.

Da biste uključili debagovanje, sve što treba da uradite je da postavite varijablu okruženja ``DEBUG`` kada pokrenete svoju aplikaciju.

Evo primera:

```js
DEBUG=express:* node index.js
```

Ova komanda omogućava sve poruke za debagovanje u prostoru imena `express:*`. Početi ćete da vidite izlaz sličan:

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

Ove poruke vam pomažu da pratite dispečiranje rute, pozive middleware-a i još mnogo toga. To je odličan način da razumete tok zahteva u vašoj aplikaciji.

Želite li da budete specifičniji? Možete suziti prostor imena:

```js
DEBUG=express:router node index.js
```

Ovo će prikazivati samo logove povezane sa ruteračkom logikom Expressa.

Na Windows-u, sintaksa je malo drugačija za PowerShell:

```js
$env:DEBUG = "express:*"; node index.js
```

A za CMD, izgleda ovako:

```js
set DEBUG=express:* && node index.js
```

Ista ideja, samo na drugačiji način za postavljanje promenljivih okruženja.

Sada ćemo pogledati prijavljivanje u Express-u.

Za logovanje HTTP zahteva, mnogi programeri koriste middleware kao što je `morgan`.

Evo kako ga koristiti:

```bash
npm install morgan
```

Then add it to your app:

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

Sada će svaki dolazni zahtev biti logovan u terminal sa detaljima kao što su metoda, URL, statusni kod i vreme odgovora.

A zašto je ovo važno?

Debagovanje vam daje uvid u to kako Express obrađuje zahteve, middlewere i rutiranje bez zagađivanja vaših produkcionih logova.

Logovanje vam daje zapis o tome šta se dešava tokom zahteva, što je ključno za debagovanje grešaka, razumevanje saobraćaja i rano uočavanje problema.

Zajedno, daju vam kontrolu i jasnoću u vašem procesu razvoja.
