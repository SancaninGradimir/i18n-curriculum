# --interactive--

Atribut `aria-controls` se koristi za kreiranje programskog odnosa između elementa koji kontroliše prisustvo drugog elementa na stranici i elementa koji ga kontroliše. Ovaj odnos može pomoći korisnicima čitača ekrana da bolje razumeju kako funkcioniše kontrola. Uobičajene upotrebe uključuju skup kartica (tabs) koji kontrolišu vidljivost različitih sekcija sadržaja, ili dugme koje menja vidljivost menija.

Pogledajmo primer kako ovo funkcioniše. U ovom primeru kartičnog interfejsa, imamo element ``div`` sa setom od tri dugmeta:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" id="tab1" aria-controls="section1" aria-selected="true">
    Tab 1
  </button>
  <button role="tab" id="tab2" aria-controls="section2" aria-selected="false">
    Tab 2
  </button>
  <button role="tab" id="tab3" aria-controls="section3" aria-selected="false">
    Tab 3
  </button>
</div>
```

```css
[role="tablist"] {
  display: flex;
  border-bottom: 2px solid #ccc;
  margin-bottom: 1em;
  gap: 4px;
}

[role="tab"] {
  background: #f5f5f5;
  border: 1px solid #ccc;

}

  background: #e8f0fe;
  outline: none;
}

[role="tab"][aria-selected="true"] {
  background: #fff;
  border-color: #ccc;
  border-bottom: 2px solid #fff;
  color: #000;
  font-weight: 600;
  position: relative;
  top: 1px; 
}

[role="tab"]:focus-visible {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

```

:::

`div` koristi `role="tablist"` da pokaže da služi kao kontejnerski element za grupu kartica.

Svaki taster predstavlja karticu i ima atribut ``role`` postavljen na ``tab``. U većini interfejsa sa karticama, prvi panel kartice je podrazumevano vidljiv, pa prvi taster kartice ima atribut ``aria-selected`` postavljen na ``true`` kako bi ukazao da je povezani deo sadržaja trenutno vidljiv. Atribut ``aria-controls`` se koristi za povezivanje svakog tastera sa sekcijom sadržaja koju kontroliše.

Evo tri sekcije sadržaja koje odgovaraju karticama:

```js
const tabs = document.querySelectorAll('[role="tab"]');
const tabList = document.querySelector('[role="tablist"]');

tabs.forEach(tab => {
  tab.addEventListener('click', () => {
    activateTab(tab);
  });

  tab.addEventListener('keydown', (e) => {
    const key = e.key;
    const currentIndex = Array.from(tabs).indexOf(tab);
    let newIndex = null;

    if (key === 'ArrowRight') {
      newIndex = (currentIndex + 1) % tabs.length;
    } else if (key === 'ArrowLeft') {
      newIndex = (currentIndex - 1 + tabs.length) % tabs.length;
    } else if (key === 'Home') {
      newIndex = 0;
    } else if (key === 'End') {
      newIndex = tabs.length - 1;
    }

    if (newIndex !== null) {
      tabs[newIndex].focus();
      activateTab(tabs[newIndex]);
    }
  });
});

function activateTab(tab) {
  const tabPanels = document.querySelectorAll('[role="tabpanel"]');

  tabs.forEach(t => {
    t.setAttribute('aria-selected', 'false');
    t.setAttribute('tabindex', '-1');
  });

  tabPanels.forEach(panel => panel.hidden = true);

  tab.setAttribute('aria-selected', 'true');
  tab.removeAttribute('tabindex');

  const panelId = tab.getAttribute('aria-controls');
  const panel = document.getElementById(panelId);
  panel.hidden = false;
  tab.focus();
}

```

:::

Svaki deo sadržaja ima atribut ``role`` postavljen na ``tabpanel`` i atribut ``aria-labelledby`` koji ukazuje na odgovarajući karticu kako bi se panel dao pristupačno ime. Atribut ``hidden`` se koristi za skrivanje delova sadržaja koji trenutno nisu izabrani. ID svakog dela odgovara vrednosti atributa ``aria-controls`` na odgovarajućem dugmetu. ID ``section1`` odgovara atributu ``aria-controls`` na prvom dugmetu, ``section2`` odgovara atributu ``aria-controls`` na drugom dugmetu, a ``section3`` odgovara atributu ``aria-controls`` na trećem dugmetu. Na ovaj način se uspostavlja veza između kartica i njihovih delova sadržaja.

Da biste prebacivali između različitih sekcija, moraćete da koristite JavaScript za ažuriranje atributa `hidden` na sekciji i atributa `aria-selected` na dugmetu, u zavisnosti od toga koja je sekcija trenutno vidljiva. Samo jedna sekcija može biti vidljiva u bilo kojem trenutku, i ne bi trebalo da ima atribut `hidden`, a `aria-selected` treba da bude podešen na `true` na njenom dugmetu. Preostale skrivene sekcije bi sve trebalo da imaju atribut `hidden`, a `aria-selected` treba da bude podešen na `false` na njihovim dugmetima.

Tabovi predstavljaju uobičajenu upotrebu za atribut `aria-controls`, ali se mogu koristiti i u drugim scenarijima gde jedan element kontroliše vidljivost ili ponašanje drugog elementa. Drugi primeri uključuju akordione, padajuće menije i modale.

Sljedeći put kada dizajnirate korisnički interfejs koji uključuje kontrolisanje vidljivosti elemenata, razmislite o korišćenju atributa `aria-controls` za uspostavljanje odnosa između kontrolišućeg elementa i kontrolisanog elementa.
