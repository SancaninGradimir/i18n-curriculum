# --interactive--

Semantički elementi kontrole forme kao što su `input`, `select`, `textarea`, `button` i `fieldset` imaju ugrađene stanja koja se prenose asistivnim tehnologijama.

Na primer, možete koristiti atribut `disabled` da onemogućite dugme ili atribut `checked` da pokaže da je čekboks označen.

Ali ako kreirate prilagođeni kontrolni element, morate koristiti ARIA atribute da prenesete stanje kontrole asistivnim tehnologijama.

U ovoj lekciji, razgovaraćemo o nekoliko uobičajenih ARIA stanja koja možete koristiti na prilagođenim kontrolnim elementima.

Prvi ARIA status koji ćemo razmotriti je `aria-selected`. Ovaj status se koristi za indikaciju da je element izabran. Ovaj status možete koristiti na prilagođenim kontrolama kao što su kartični interfejs, listbox ili mreža.

Evo primera kako možete koristiti `aria-selected` na prilagođenom kontrolnom tabu:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" aria-selected="true">Tab 1</button>
  <button role="tab" aria-selected="false">Tab 2</button>
  <button role="tab" aria-selected="false">Tab 3</button>
</div>
<script src="index.js"></script>
```

```css
[role="tablist"] {
  display: flex;
  border-bottom: 2px solid #ddd;
  gap: 0.25rem;
  font-family: system-ui, sans-serif;
}

[role="tab"] {
  appearance: none;
  border: none;
  background: none;
  padding: 0.5rem 1rem;
  cursor: pointer;
  font-size: 1rem;
  color: #444;
  border-radius: 4px 4px 0 0;
  transition: background-color 0.2s, color 0.2s;
}

[role="tab"]:hover {
  background-color: #f3f3f3;
}

[role="tab"][aria-selected="true"] {
  background-color: #fff;
  color: #0078d4;
  border: 2px solid #0078d4;
  border-bottom: 2px solid #fff; 
  font-weight: 600;
  position: relative;
  z-index: 1;
}

[role="tab"]:focus {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

```

```js
document.addEventListener("click", (event) => {
  const clickedTab = event.target.closest('[role="tab"]');
  if (!clickedTab) return;

  const tablist = clickedTab.closest('[role="tablist"]');
  const tabs = tablist.querySelectorAll('[role="tab"]');

  tabs.forEach((tab) => {
    const isSelected = tab === clickedTab;
    tab.setAttribute("aria-selected", isSelected);
    tab.tabIndex = isSelected ? 0 : -1;
  });
});

document.addEventListener("keydown", (event) => {
  const activeTab = document.activeElement;
  if (activeTab.getAttribute("role") !== "tab") return;

  const tablist = activeTab.closest('[role="tablist"]');
  const tabs = Array.from(tablist.querySelectorAll('[role="tab"]'));
  const index = tabs.indexOf(activeTab);

  let newIndex = index;
  if (event.key === "ArrowRight") newIndex = (index + 1) % tabs.length;
  if (event.key === "ArrowLeft") newIndex = (index - 1 + tabs.length) % tabs.length;

  if (newIndex !== index) {
    tabs[newIndex].focus();
    tabs[newIndex].click();
  }
});

```

:::

Tabovi se koriste za prikaz više panela sadržaja u ograničenom prostoru. Stanje `aria-selected` se koristi za indikaciju kog je taba trenutno odabrano.

Kada korisnik izabere karticu, stanje `aria-selected` izabrane kartice se postavlja na `true`, i stanje `aria-selected` ostalih kartica se postavlja na `false`.

Drugi uobičajen ARIA status je `aria-disabled`. Ovaj status se koristi za indikovanje da je element onemogućen samo za osobe koje koriste asistivne tehnologije, kao što su čitači ekrana. Važno je napomenuti da `aria-disabled` zapravo ne onemogućava element. Na vama, programeru, je da ga učinite tako da izgleda i radi kao onemogućen element. Ovaj atribut se takođe često koristi na izvornim HTML elementima umesto atributa `disabled`. Koji od njih izaberete zavisiće od konteksta u kojem se dugme koristi.

Evo primera kako možete koristiti `aria-disabled` na prilagođenom dugmetu za uređivanje:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="button" tabindex="-1" aria-disabled="true">Edit</div>
```

```css
[role="button"] {
  display: inline-block;
  background-color: #0078d4;
  color: #fff;
  padding: 0.5rem 1rem;
  border-radius: 6px;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  text-align: center;
  cursor: pointer;
  user-select: none;
  transition: background-color 0.2s, transform 0.1s, opacity 0.2s;
}

[role="button"]:not([aria-disabled="true"]):hover {
  background-color: #005fa3;
}

[role="button"]:not([aria-disabled="true"]):focus {
  outline: 2px solid #005fa3;
  outline-offset: 2px;
}

[role="button"]:not([aria-disabled="true"]):active {
  transform: scale(0.97);
}

[role="button"][aria-disabled="true"] {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
  background-color: #b0b0b0;
  color: #f2f2f2;
}
```

:::

Atribut ``aria-disabled`` se koristi da kaže korisnicima čitača ekrana da je dugme za uređivanje onemogućeno i da ne može biti interakciono. Ponovo, ono zapravo ne onemogućava dugme. Kada koristite ``aria-disabled``, moraćete da primenite stilizovanje i JavaScript kako biste učinili kontrolu koja izgleda i ponaša se kao onemogućeno dugme.

U većini slučajeva, verovatno ćete koristiti nativni element dugmeta, ali postoje slučajevi kada možda trebate koristiti prilagođenu kontrolu. Stoga je esencijalno znati kako preneti stanje kontrole asistivnim tehnologijama.

Sledeće ARIA stanje koje ćemo razmotriti je `aria-haspopup`. Ovo stanje se koristi za indikovanje da će interaktivni element pokrenuti popap element kada se aktivira. Možete koristiti atribut `aria-haspopup` samo kada popap ima jednu od sledećih uloga: `menu`, `listbox`, `tree`, `grid`, ili `dialog`. Vrednost `aria-haspopup` mora biti ili jedna od ovih uloga ili `true`, što podrazumeva ulogu `menu`.

Evo primera menija za uređivanje fajlova koji koristi `aria-haspopup`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button id="menubutton" aria-haspopup="menu" aria-controls="filemenu" aria-expanded="false">File</button>
<ul id="filemenu" role="menu" aria-labelledby="menubutton" hidden>
  <li role="menuitem" tabindex="-1">Open</li>
  <li role="menuitem" tabindex="-1">New</li>
  <li role="menuitem" tabindex="-1">Save</li>
  <li role="menuitem" tabindex="-1">Delete</li>
</ul>
```

:::

Stanje `aria-haspopup` se koristi za ukazivanje na to da će dugme menija `File` otvoriti popap-meni kada se aktivira. Korisnici čitača ekrana mogu čuti ove dodatne informacije kada navigiraju do dugmeta.

Biće vam potrebno da koristite JavaScript za prikazivanje i skrivanje popap menija, kao i za implementaciju odgovarajuće podrške tastaturi za interakciju sa menijem. Takođe, imajte na umu da uloga ARIA `menu` se odnosi na vrlo specifičan tip menija. Generalno se odnosi na listu akcija koje korisnik može aktivirati, slično meniju na desktop aplikaciji. Ne uključuje uobičajenije upotrebe onoga što obično nazivamo "menijima", kao što su navigacioni meniji. Realno govoreći, većina "menija" koje kreirate na vebu neće biti ARIA meniji i nećete koristiti `aria-haspopup` sa njima.

Sledeći ARIA status koji ćemo razraditi je `aria-required`. Atribut `aria-required` se koristi za ukazivanje na to da je potrebno popuniti polje pre slanja obrasca.

Ovo je primer korišćenja atributa `aria-required` za prilagođeni kontrolni element forme.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div id="name-label">Full Name*</div>
<div role="textbox" contenteditable aria-labelledby="name-label" aria-required="true" id="name"></div>
```

```css
#name-label {
  font-family: system-ui, sans-serif;
  font-size: 0.95rem;
  font-weight: 600;
  margin-bottom: 0.25rem;
  color: #333;
}

[role="textbox"] {
  display: block;
  width: 100%;
  min-height: 2rem;
  padding: 0.5rem 0.75rem;
  border: 1.5px solid #ccc;
  border-radius: 4px;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  color: #222;
  background-color: #fff;
  line-height: 1.4;
  transition: border-color 0.2s, box-shadow 0.2s;
}

[role="textbox"]:hover {
  border-color: #999;
}

[role="textbox"]:focus {
  outline: none;
  border-color: #0078d4;
  box-shadow: 0 0 0 3px rgba(0, 120, 212, 0.25);
}

[role="textbox"]:empty::before {
  content: attr(data-placeholder);
  color: #aaa;
  pointer-events: none;
}

[role="textbox"][aria-required="true"] {
  border-left: 3px solid #e81123;
  padding-left: calc(0.75rem - 3px);
}
```

:::

Moramo da koristimo atribut `contenteditable` kako bi korisnici mogli da unesu svoj unos. Takođe koristimo atribut `aria-required`, postavljen na `true`, da bismo ukazali da je ova prilagođena kontrola forme obavezna.

Da biste učinili da kontrola forme izgleda kao normalna kontrola forme, morali biste dodati CSS. Takođe bi bilo potrebno dodati JavaScript da sprečite slanje forme bez sadržaja.

Ako oznaka već sadrži reč `required`, onda biste trebali izostaviti atribut `aria-required`. Ovo osigurava da čitači ekrana najave reč samo jednom.

U većini slučajeva, verovatno ćete koristiti ugrađene `label` i `form` elemente sa atributom `required`. Ali ako morate kreirati prilagođeni kontrolni element forme, važno je dodati atribut `aria-required` kada je potrebno.

Dodatno, atribut `aria-required` takođe se može koristiti na izvornim (native) formularnim unosima, kao što su elementi `input`, `textarea` i `select`. Ovo se često preferira nad izvornim (native) atributom `required`, jer može imati potencijalne probleme sa upotrebljivošću i pristupačnošću, posebno sa podrazumevanom obradom grešaka koju pruža pregledač. Na kraju, moraćete da testirate kako biste utvrdili koji je atribut najbolji za vašu situaciju.

Poslednji ARIA status koji ćemo razraditi je `aria-checked`. Ovaj atribut se koristi za indikovanje da li je element u stanju "proveren" (checked). Najčešće se koristi prilikom kreiranja prilagođenih polja za čekiranje, dugmadi za radio opcije, prekidača i lista sa izborom.

Evo primera kako možete koristiti `aria-checked` na prilagođeni kontrolni okvir za čekiranje:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="checkbox" aria-checked="true" tabindex="0">Checkbox</div>
```

```css
[role="checkbox"] {
  display: inline-flex;
  align-items: center;
  gap: 0.5rem;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  cursor: pointer;
  user-select: none;
  color: #222;
}

[role="checkbox"]::before {
  content: "";
  display: inline-block;
  width: 1rem;
  height: 1rem;
  border: 2px solid #666;
  border-radius: 4px;
  background-color: #fff;
  transition: all 0.2s ease;
  box-sizing: border-box;
}

[role="checkbox"]:hover::before {
  border-color: #0078d4;
}

[role="checkbox"]:focus::before {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

[role="checkbox"][aria-checked="true"]::before {
  background-color: #0078d4;
  border-color: #0078d4;
  background-image: url("data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' width='14' height='14' viewBox='0 0 14 14'><path fill='white' d='M5.2 10.4L2 7.2l1.1-1.1 2.1 2.1L10.9 2.5 12 3.6z'/></svg>");
  background-repeat: no-repeat;
  background-position: center;
}

[role="checkbox"][aria-disabled="true"] {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
}

```

:::

Nativni elementi za čekboks imaju ugrađeno stanje `checked` koje se prenosi asistivnim tehnologijama. Ali ako kreirate prilagođeni kontrolni element za čekboks, moraćete da koristite atribut `aria-checked` kako biste ukazali na njegovo stanje.

Kada korisnik interaguje sa prilagođenom kontrolom za čekboks, moraće da koristite stanje `aria-checked` kako biste odrazili novo stanje čekboksa. Kada je čekboks označen, atribut `aria-checked` se postavlja na `true`. Kada nije označen, postavlja se na `false`.

Nativni elementi obično imaju bolju podršku i ugrađene funkcije pristupačnosti.

Međutim, ako morate kreirati prilagođene kontrole, korišćenje ARIA atributa je esencijalno da efikasno prenesete stanje ovih kontrola asistivnim tehnologijama.

Kao uvek, testirajte svoj rad kako biste osigurali da su ARIA atributi pravilno primenjeni i da prilagođena kontrola funkcioniše na način koji je i pristupačan i jednostavan za korišćenje.
