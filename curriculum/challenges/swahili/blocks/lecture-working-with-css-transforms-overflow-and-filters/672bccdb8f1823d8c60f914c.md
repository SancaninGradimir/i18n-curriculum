# --interactive--

CSS reset je stilski list koji uklanja sve ili neke od podrazumevanog formatiranja koje pregledači veb stranica primenjuju na HTML elemente. Na primer, možda ste primetili da određeni elementi, kao što su pasusi i naslovi, već imaju margine podrazumevano čak i ako ih eksplicitno ne definišete u svojim prilagođenim stilskim listovima.

Takođe ćete ovo videti sa raznim svojstvima u širokom opsegu HTML elemenata. Različiti pretraživači takođe mogu primeniti različite podrazumevane stilove na HTML elemente. Podrazumevani stilovi u Google Chromu možda neće biti tačno isti kao u Mozilla Firefoxu ili Microsoft Edge-u. Ovo može dovesti do nekonzistentnih stilova između pretraživača, što biste trebali izbegavati koliko god je moguće.

Da bi se ovo rešilo, CSS reset uklanja podrazumevane stilove. Uklanjanjem svih ili nekih od podrazumevanih stilova, možete dobiti doslednu osnovu za dizajn i minimalizovati potencijalne nekonzistentnosti između pregledača i uređaja. Uklanjanje podrazumevanih stilova takođe može olakšati proces stilizovanja jer ćete videti samo stilove koje ste eksplicitno primenili u svojim prilagođenim listovima stilova. Postoje dva glavna pristupa CSS resetima: možete ili definisati prilagođene CSS resetove ili koristiti CSS resetove treće strane.

Prilagođeni CSS resetovi su stilovi listovi koje kreirate od nule kako bi odgovarali potrebi vašeg projekta. Na ovaj način možete kontrolisati specifične stilove koji će biti resetovani, uz mnogo prostora za fleksibilnost. Međutim, morate uložiti i vreme za razvoj i održavanje tih stilskih listova. Evo primera veoma uobičajene CSS pravila za resetovanje margine i paddinga svih HTML elemenata:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>Example Heading</h1>
<p>This is a paragraph.</p>
```

```css
* {
  margin: 0;
  padding: 0;
}
```

:::

Obično je napisano na vrhu CSS listića za stilove (stylesheet). Asterisk selektor je wildkart selektor koji odgovara svim HTML elementima, tako da će imati podrazumevanu marginu od nule i podrazumevano punjenje (padding) od nule sa svih četiri strane. Ovo će vam dati početnu tačku, a zatim ih možete prilagoditi u specifičnijim CSS selektorima niže u listiću za stilove.

Možete koristiti ovaj pristup za selekciju bilo kog HTML elementa i resetovanje njegovih podrazumevanih stilova. Samo kreirajte prilagođeni fajl sa stilovima (stylesheet) i koristite odgovarajuće CSS selektore da poklapate elemente i postavite stilove.

Ali ovo može biti proces koji zahteva mnogo vremena. Ako želite da uštedite vreme, možete koristiti i eksterni CSS reset. Ove stilove listove su već predizgrađeni, tako da ih samo možete preuzeti i direktno dodati u svoj projekat. Odličan primer trećeg CSS reseta je `Normalize.css`. Ovaj stilski list normalizuje stilove za širok spektar HTML elemenata, istovremeno zadržavajući neke korisne podrazumevane stilove, posebno one koji su važni za pristupačnost. Takođe ispravlja uobičajene greške i nekonzistentnosti stila.

Druga opcija je `sanitize.css`. Ovo je CSS biblioteka koju možete koristiti da biste osigurali da će podrazumevani stilovi biti dosledni u svim glavnim modernim pregledačima. Ova biblioteka se razvija uz `Normalize.css`, tako da evoluiraju zajedno. Takođe ima pojedinačne stilove (stylesheets) koje možete preuzeti za specifične svrhe, kao što su normalizovanje obrazaca i tipografija.

Postoji mnogo dostupnih opcija, ali bi trebalo da izaberete one koje najbolje odgovaraju potrebama vašeg projekta. Takođe možete kombinovati oba pristupa koristeći eksterne CSS resetove sa prilagođenim resetovima.

Kada radite sa CSS resetima, važno je da uzmete u obzir i pristupačnost. Vaša veb aplikacija bi trebalo da bude dostupna svima. Stoga, ne biste trebali da resetujete stilove koji bi mogli biti korisni za čitače ekrana ili druge asistivne tehnologije. Takođe biste trebali da razmotrite uticaj koji ovi dodatni stilovi listova mogu imati na performanse vaše aplikacije, jer moraju biti preuzeti pre nego što se primene prilagođeni stili.

Uklanjanjem podrazumevanih stilova, CSS resetovi vam pružaju praznu početnu tačku za implementaciju vašeg dizajna. Ovo rezultira uniformijim i konzistentnijim korisničkim iskustvom na svim pregledačima i uređajima.
